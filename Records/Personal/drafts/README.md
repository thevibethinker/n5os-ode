# Records/Personal/drafts/

**Purpose**: Staging area for ad hoc generated outputs awaiting review, editing, and action.

---

## Overview

This directory stores outputs generated by the system (emails, documents, etc.) that require:
- Human review before sending/publishing
- Potential editing or refinement
- Action to be taken (send, publish, archive)

**Key Principle**: Drafts are temporary. After action is taken (sent, published), files should be archived or deleted.

---

## Directory Structure

```
drafts/
├── emails/      - Email drafts (intros, correspondence, etc.)
├── documents/   - Document drafts (proposals, summaries, reports)
├── other/       - Miscellaneous generated content
└── README.md    - This file
```

---

## Workflow

### 1. Generation
System generates output and saves to appropriate subdirectory:
```
Generated email → drafts/emails/YYYY-MM-DD-description.md
Generated doc   → drafts/documents/YYYY-MM-DD-description.md
```

### 2. Review
- Review content for accuracy
- Edit as needed
- Verify against style guide (if applicable)

### 3. Action
- **Email**: Send via client or use_app_gmail
- **Document**: Publish, share, or distribute
- **Other**: Take appropriate action

### 4. Cleanup
After action taken:
- **Delete**: If no need for record
- **Archive**: If needed for reference
  - Personal correspondence → External archive
  - Important docs → `Documents/` (if permanent reference needed)

---

## File Naming Convention

Use date-prefixed descriptive names:
```
Good:
- 2025-10-17-jabari-ben-intro.md
- 2025-10-17-quarterly-update-draft.md
- 2025-10-17-proposal-client-xyz.md

Bad:
- email.md
- draft.md
- document.md
```

---

## Retention Policy

- **Active drafts**: Keep until action taken
- **Sent/published**: Archive or delete within 24 hours
- **Stale drafts**: Review drafts older than 7 days
  - Revise and send
  - Archive
  - Delete

---

## Style Guide Integration

Generated outputs may reference style guides:
- Style guides stored in: `file 'N5/style_guides/'`
- Exemplars stored in: `file 'N5/exemplars/'`
- Validation via: `python3 N5/scripts/style_guide_manager.py validate`

When output meets standards, consider adding as exemplar:
```bash
python3 N5/scripts/style_guide_manager.py add-exemplar \
  --output-type [type] \
  --file drafts/emails/[filename].md \
  --name [descriptive-name]
```

---

## Best Practices

### DO:
✅ Review drafts promptly (within 24 hours)
✅ Use descriptive file names
✅ Take action (send/publish/archive) within 48 hours
✅ Delete or archive after action taken
✅ Add excellent outputs as exemplars

### DON'T:
❌ Let drafts accumulate indefinitely
❌ Use generic file names
❌ Store sent emails here long-term
❌ Skip review before sending

## Commands

### Review pending drafts
```bash
# List all drafts
ls -lh Records/Personal/drafts/emails/
ls -lh Records/Personal/drafts/documents/

# Find old drafts (>7 days)
find Records/Personal/drafts/ -type f -mtime +7
```

### After sending/publishing
```bash
# Delete draft
rm Records/Personal/drafts/emails/2025-10-17-name.md

# Or archive if needed
# (move to external storage system)
```
