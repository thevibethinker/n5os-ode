# Semi-Stable Knowledge

**Purpose:** Information that changes regularly (monthly/quarterly) but needs to be current for proposals, pitches, and strategic decisions.

**vs. Stable:** Stable = historical facts (founding date, mission). Semi-stable = current state (metrics, pipeline, team).

**vs. Evolving:** Evolving = granular facts that accumulate. Semi-stable = aggregated current state.

---

## What Lives Here

### Business Metrics (`current_metrics.md`)
- User/customer counts
- Placement rates
- Revenue (if disclosed)
- Growth rates
- Updated: Monthly or as significant changes occur

### Product State (`product_current.md`)
- Current features/capabilities
- Recent launches
- Roadmap items in development
- Known limitations
- Updated: As product evolves

### Team (`team_current.md`)
- Current team composition
- Recent hires
- Roles/responsibilities
- Advisors/board members
- Updated: As team changes

### Positioning (`positioning_current.md`)
- Current messaging
- Active value propositions
- Competition positioning
- Recent pivots or refinements
- Updated: As strategy evolves

---

## Usage

**For Proposals/Pitches:**
- Pull current numbers from `current_metrics.md`
- Describe capabilities from `product_current.md`

**For Meeting Prep:**
- Review relevant semi-stable context before meetings
- Update after meetings if new information learned

**Maintenance:**
- Update files as information changes (don't wait for perfect data)
- Include date of last update in each file
- Mark items as [ESTIMATED] or [AS OF DATE] when precision matters
