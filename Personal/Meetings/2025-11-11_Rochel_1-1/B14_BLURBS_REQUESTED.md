## 2. Context-Specific Recommendation Templates

### Purpose
Make it trivial for recommenders to generate high-quality, relevant recommendations for specific opportunities.

### Format
**ChatGPT Prompt Structure:**
```
You are writing a recommendation for Rochel Polter, a product designer.

Context about Rochel:
[One-pager content]

The role/opportunity:
[Job description or opportunity details]

Please write a recommendation that:
1. Addresses specific requirements of this opportunity
2. Uses concrete examples of Rochel's work
3. Highlights relevant unique capabilities
4. Is authentic and specific, not generic praise

Ask me any clarifying questions first, then generate the recommendation.
```

### Templates Needed
1. **LinkedIn Recommendation** (visible to public)
2. **Email Introduction** (to specific person)
3. **Reference Letter** (formal application)
4. **Warm Intro to Mutual Contact** (networking)

### Customization Variables
- Target role type (IC design, leadership, PM hybrid)
- Company stage (startup, growth, enterprise)
- Key requirements (technical, leadership, strategic)
- Relationship context (how they know Rochel)

### When to Create
After one-pager complete and tested with initial pitch sessions.

## 4. "Why I Did That" Content (Optional but Strategic)

### Purpose
Build thought leadership and online presence through principle-extraction content.

### Format Options
- Personal blog posts (Medium, Substack)
- LinkedIn articles
- Newsletter (if going deep)
- Career Span company blog (with Vrijen co-authoring)

### Content Framework
**Title template:** "Why I [Design Decision]: [Principle]"
- Example: "Why I Removed the Progress Bar: Trust Over Metrics"

**Structure:**
1. **The decision:** What did you do?
2. **The context:** What were you solving?
3. **The intuition:** What led you to this approach?
4. **The principle:** What transferable insight emerged?
5. **The learning:** How does this apply more broadly?

### Pilot Topics
Based on meeting:
- Designing for vulnerability in AI tools
- Making complicated experiences feel simple
- Balancing constraints with vision
- Bringing coherence to chaos

### Collaboration Model with Vrijen
- Rochel writes from design perspective
- Vrijen adds product/founder perspective
- Cross-pollinate audiences
- Positions both as thought leaders in AI UX

### When to Create
After foundation complete. This is enhancement, not requirement.

## Quality Standards

### All deliverables must pass these tests:

**Specificity Test:**
Could another product designer use this same language? If yes, not specific enough.

**Evidence Test:**
Can every claim be backed with concrete example? If no, remove or add evidence.

**Authenticity Test:**
Does this feel comfortable to say out loud? If no, keep refining.

**Memory Test:**
Would someone remember this 3 days later? If no, not distinctive enough.

**Action Test:**
Does this help someone understand how to help you or hire you? If no, add clarity.
