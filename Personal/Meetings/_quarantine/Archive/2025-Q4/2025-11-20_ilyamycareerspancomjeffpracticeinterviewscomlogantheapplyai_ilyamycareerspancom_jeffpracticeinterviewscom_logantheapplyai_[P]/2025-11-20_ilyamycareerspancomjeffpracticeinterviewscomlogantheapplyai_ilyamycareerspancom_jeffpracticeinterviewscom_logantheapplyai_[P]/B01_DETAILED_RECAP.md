# Detailed Meeting Recap: Practice Interviews & Careerspan Partnership Demo

## Overview
This was a technical partnership demonstration between Careerspan (Ilya), Practice Interviews (Jeff Chen), and Apply AI (Logan). The meeting centered on establishing admin capabilities and user engagement mechanics for integrating Practice Interviews as a value-add feature for Careerspan's platform.

## Meeting Flow

### Opening & Relationship Building (First 5 minutes)
The call began with personal rapport-building around Boston geography and regional dialects. Jeff and Logan discussed Boston's geographical pockets and how neighborhoods determine accent presence, with discussion touching on Cambridge, Kennedy mid-Atlantic accents, and neighborhood-specific linguistic patterns. This established a collegial tone before transitioning to business.

### Core Demo: Admin Platform Walk-Through
Jeff guided the team through the Practice Interviews admin dashboard and user management capabilities:

**Admin Access Setup:**
- Demonstrated login process at app.practiceinterviews.com
- Made all three participants (Ilya, Jeff, Logan) admins so each can independently add users without being blocked
- Showed that admin accounts include free access to the platform

**User Management Interface:**
- Add users feature operates on email-only basis (one-click process)
- Can edit user details (e.g., first names) post-addition if needed
- Admin dashboard shows user list and management controls at bottom of interface

### Data & Analytics Dashboard
Jeff walked through the user performance analytics that admins can access:

**User Profile Metrics:**
- Total number of questions attempted by user
- Rolling monthly average scores
- Interview Academy completion percentage (indicates learning mindset; rolling 4-week average)
- Question type breakdown (behavioral vs. hypothetical performance)
- Answer history with video playback capability
- Individual feedback on responses with comparison to "ideal answers"
- Scoring system where 75+ is categorized as "excellent" performance

**Key Analytics:**
- Jeff noted the current hypothetical average is artificially high (score of 100) due to a prompt generation error; typical score range is around 50
- Platform tracks question attempt patterns to show engagement levels and peak usage days
- Primary filter: actual tool usage + Interview Academy adoption

### Strategic Direction: Quality-Based Access Model
A critical strategic shift emerged during the conversation:

**Initial Approach:** Open access to all Careerspan users
**Refined Approach:** Tier access based on user engagement quality
- Gateway: Users must complete 3+ career stories before unlocking Practice Interviews access
- Rationale: "Action speaks louder than words" — filtering for motivated, signal-rich users
- Additional benefit: Creates exclusivity factor ("partnership only available to our most engaged users")

Jeff emphasized that quality over quantity matters: people who take action are motivated to learn and improve, similar to how free throw practice translates to game performance. The team agreed this barrier makes the offering more valuable.

### User Activation Mechanics
- Manual user addition process is acceptable to start ("start simple, right?")
- Bulk upload possible but unnecessary to begin with
- All users added will see same interface and data once signed in

## Key Insights from Jeff
1. **Scale isn't the constraint:** SaaS product usage is self-selecting; not all users will engage regardless of numbers given access
2. **Data as feedback loop:** Tracking user behavior and engagement metrics will provide real-time feedback on partnership viability
3. **Modular feedback limitation:** Current feedback system is "streamlined" compared to full modular feedback system (Jeff acknowledged this limitation)

## Partnership Value Proposition
- Careerspan gets a practice interviewing tool to enhance career outcomes
- Practice Interviews gets qualified user access and engagement data
- Users get additional skill-building capability (interview practice)
- Potential monetization path: "Eventually monetize in some way" (future discussion)
