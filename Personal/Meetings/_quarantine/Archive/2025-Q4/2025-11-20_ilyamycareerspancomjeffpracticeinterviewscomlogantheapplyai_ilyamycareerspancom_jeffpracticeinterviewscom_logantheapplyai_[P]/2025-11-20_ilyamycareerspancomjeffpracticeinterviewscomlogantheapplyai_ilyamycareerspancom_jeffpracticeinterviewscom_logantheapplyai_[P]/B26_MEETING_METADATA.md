## Strategic Context

**Partnership Goal:** Integrate Practice Interviews as a value-add for engaged Careerspan users to drive retention, deepen engagement, and create attribution data linking user practice behavior to employment outcomes.

**Value Proposition for Careerspan Users:**
- Free, exclusive access to AI-powered interview practice
- Measurable progress tracking with feedback on storytelling and interview technique
- Removes friction from interview preparation — users don't need to seek external tools

**Value Proposition for Practice Interviews:**
- Access to high-signal Careerspan user cohort (vetted, engaged, 3+ stories completed)
- Data on user adoption and engagement patterns
- Path to monetization through attribution tracking (showing ROI of interview prep → offer success)

## Key Discussion Points

### Admin Dashboard Capabilities (Demonstrated)

- **User Management:** Simple email-based user addition; can bulk upload (manual process recommended for now)
- **Performance Metrics Visible to Admins:**
  - Total questions attempted
  - Average score on rolling monthly basis
  - Interview Academy completion (learning mindset signal)
  - Behavioral vs. hypothetical question breakdown
  - Answer history with feedback links
  - Video processing status

- **Scoring System:** 75+ = Excellent; 50ish = average; heavily weighted on how many questions users actually attempt (usage signal)

### Access Strategy: Quality-Based Tiering

- **Barrier:** Access restricted to users with 3+ completed story profiles
- **Rationale:** "Action speaks louder than words" — users who've invested effort in storytelling are more likely to engage with interview prep
- **Network Effect Analogy:** Like basketball free throws — practice leads to success in-game, which drives more practice

### User Acquisition & Attribution

- **Rollout Method:** Email to eligible users + auto-trigger after 3rd story completion
- **Admin Workflow:** Ilya to mock up user email; Logan to finalize copy and set up auto-trigger
- **Attribution Tracking:** Eventually connect user practice behavior → Apply AI employer interactions → job offer outcomes
- **Data Sharing:** Cohort-level performance data shareable; individual-level data subject to privacy/terms

## Decisions Made

1. **Go-live Approach:** Start with broad user onboarding (no initial cap on number of users); monitor adoption; optimize based on data
2. **Access Requirements:** Set barrier at 3+ completed stories to ensure high-signal users
3. **Admin Setup:** All three Careerspan team members (Ilya, Logan, Jeff) made admins to enable independent user additions
4. **Email Strategy:** Two-phase (initial announcement + auto-trigger after milestone); copy to be approved by Jeff
5. **Attribution:** Commitment to tracking cohort-level data and eventually tracing individual user journeys (Practice Interviews → Apply AI offers)
