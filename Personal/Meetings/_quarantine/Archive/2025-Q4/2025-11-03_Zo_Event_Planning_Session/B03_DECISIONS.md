# B03_DECISIONS

## Strategic Decisions

### Event Structure
**Decision: Maintain two-talk + networking format**
- Rationale: Allows Tiffany to establish narrative foundation and demonstrate product capability before Vrijen's follow-up
- Confirmed participants: Vrijen + Tiffany (dual presenter model)

### Messaging & Positioning
**Decision: Lead with market critique, not product features**
- Rationale: Differentiates event from standard "company pitch"; leverages Tiffany's authority as technical builder to critique consumer AI landscape authentically
- Narrative flow: Historical context (early internet) → current state analysis (why AI tools fail) → Zo as alternative
- Rejected: Generic "build your own software" positioning; instead emphasize capability/power gap

**Decision: Emphasize agenticity as primary differentiator over UI/aesthetics**
- Rationale: Lovable competes on flashy interfaces; Zo's advantage is autonomous agent self-direction within constrained rules
- Implication: Demo selection focuses on workflow automation, not visual appeal
- Demo priority: Zero-touch processing + productivity tracker (showing complexity achievable by non-technical user)

### Demo Strategy
**Decision: Feature autonomous workflows over personal website showcase**
- Rationale: Personal website demo risks positioning Zo as Lovable competitor (same flashy result, more effort)
- Exception: Website remains as accessible entry-point demo, BUT enhanced with:
  - Dynamic content integration capability
  - Possible real-time stage manipulation (text Zo to change content live)
- Primary focus: Vrijen's workflow examples demonstrating true differentiation

**Decision: Create consumer AI tools comparison slide (new deliverable)**
- Rationale: Provides visual framework for market critique; positions Zo in unique category (server-based personal compute)
- Scope: Single slide, high-level segmentation by architecture/capability
- Purpose: Sets up positioning for why existing tools are insufficient

### Talk Flow & Narrative Arc
**Decision: Structured counter-narrative between talks**
- Tiffany's role: Technical builder perspective—honest critique from inside engineering + visual demo
- Vrijen's role: Non-technical user journey—validation that Zo solves real problems for ordinary people
- Transition point: Zapier frustration as crowd anchor (universal pain point)
- Complexity gradient: Tiffany (accessible) → Vrijen (PhD-level examples) → unified value prop

### Engagement & Outreach
**Decision: Ben Lang community engagement via combined credibility**
- Rationale: Adding Tiffany's name alongside Vrijen in email increases likelihood of community engagement
- Assessment: High-quality, curated, affordable community—worth prioritization
