# B21_KEY_MOMENTS

## Connection Friction: Theme Selection UX
**Significance:** First-time user design observation

When Gabi reached the theme selection screen, they examined multiple theme options and asked clarifying questions about each one ("What is Intermezzo?" "What is the nicest one?"). This suggests:
- The theme names are not self-explanatory to new users
- Users want guidance on aesthetic choices but may not have immediate context
- Vrijen's personal recommendation (Espresso) was helpful in resolving decision paralysis

**UX Implication:** New users may benefit from theme descriptions or visual previews beyond names alone.

## Account Architecture Discussion: Strategic Thinking
**Significance:** Advanced user consideration emerges mid-onboarding

When Gabi asked about calendar connections, Vrijen pivoted to proposing a sophisticated architecture:
- Personal Zo + Blue Tulip Zo (separate instances)
- Personal GitHub + Blue Tulip GitHub (organizational separation)
- Secure bridge between instances

Rather than dismissing this as premature, Gabi engaged with the proposal. This suggests:
- Gabi is thinking strategically about business/personal separation from the start
- Early clarity on intended system architecture was valuable, not overwhelming
- Multi-instance workflows are a natural conversation topic even during onboarding

**Implication:** Power users benefit from architectural context early; this isn't a beginner-only conversation.
