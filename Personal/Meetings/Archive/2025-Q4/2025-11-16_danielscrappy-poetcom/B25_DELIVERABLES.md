#### 3. Contractor Work Terms & Templates

**Owner:** Brady  
**Deadline:** Before signing first paid contract (this month)  
**Scope:** Templated baseline materials for common engagement types

**Materials to Create:**

##### A. Scope Documentation Template
- **Purpose:** Define what will & won't be delivered upfront
- **Sections:**
  - Deliverables (specific, measurable)
  - Out-of-scope (explicitly list what's not included)
  - Timeline / milestones
  - Dependencies (client responsibilities)
  - Success criteria (how we know it's done)

##### B. Measurement / Assessment Rubrics
- **Purpose:** Define how success will be evaluated
- **For code reviews:** Code quality, architecture soundness, security considerations, performance implications
- **For consultations:** Clarity of recommendations, actionability, depth of analysis
- **For projects:** Functional completeness, code quality, documentation, timeline adherence
- **Can generate initial versions using ChatGPT**

##### C. Contract Templates (3-5 scenarios)
1. **Free/low-cost initial consultation → escalation to paid work**
   - Structure: 30-min free session, then if they want more, define engagement (project vs. hourly)
   - Rubric: Shows professionalism, filters non-serious inquiries
   
2. **Project-based fixed scope**
   - Fixed fee for defined deliverables
   - Payment terms (e.g., 50% upfront, 50% on completion)
   - Scope lock (change order process)
   
3. **Hourly retainer**
   - Monthly hours allocation
   - Minimum/maximum hours
   - Rate card
   - Use-it-or-lose-it policy
   
4. **Code review / assessment engagement**
   - Fixed fee for analysis (e.g., $500 for codebase review + written assessment)
   - Turnaround time (e.g., 1 week)
   - Deliverable: Written report + optional sync call
   
5. **Architecture/design consultation**
   - Hourly or daily rate
   - Session structure (e.g., 4-hour deep dive)
   - Deliverable: Design doc, architecture diagrams, recommendations

**Flexibility Rules to Document:**
- What CAN be negotiated: Price (negotiable within range), timeline (within limits), engagement structure (project vs. hourly)
- What CANNOT be negotiated: Quality standards (Brady reviews everything), delivery of untested AI output (Brady is responsible), scope creep protection (change orders required)

**Example non-negotiable rule:**
> "I maintain responsibility for all deliverables regardless of how I generate them. I read, test, and verify all code before delivery. I won't ship AI-generated code I haven't personally reviewed and understood."

**Resource:**
- ChatGPT can draft initial contract language; Brady customizes
- Reference: Brady mentioned he already spent time thinking about flexible engagement models pre-Careerspan
- Start with 2-3 templates; iterate after first real engagement

**Purpose:**
- Prevents scope creep (clearest protection for contractor)
- Shows professionalism to prospects (signals seriousness, experience)
- Establishes clear expectations (reduces misunderstandings)
- Demonstrates Brady "doesn't fuck around" (Vrijen's term)

**Success Criteria:**
- 2-3 core templates drafted and ready to deploy
- Each template has example filled-in version (as reference)
- Flexibility rules clearly documented
- Brady has used at least one template in first engagement

**Next Step After Creation:**
- Brady runs first template by Vrijen for feedback before use
- Collect feedback from first client and iterate
