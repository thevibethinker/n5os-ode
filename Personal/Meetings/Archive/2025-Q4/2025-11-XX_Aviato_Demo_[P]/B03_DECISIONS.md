# Decisions

## Confirmed Decisions

1. **API Test Account**  
   - Aviato will provide Vrijen with an API key and a test quota of 100 people + 100 company credits (no time limit).  
   - This will be used first for personal CRM and meeting‑intelligence experiments on Zo.

2. **Integration Sequence**  
   - Phase 1: Personal use case (Zo + n8n + Aviato) to validate integration path and data usefulness.  
   - Phase 2: If successful, scope a Careerspan integration for recruitment workflows (vesting‑based outreach, candidate discovery).

3. **Mutual Introductions**  
   - Vrijen will introduce Konrad and Austin to the Zo founders (Ben and Rob) once he has initial use cases running.

## Directional / Exploratory Calls

1. **Use Vesting and Movement Signals for Proactive Outreach**  
   - Conceptually agree that using vesting cliffs, role endings, and open‑to‑network flags to reach candidates at the right moment is powerful; implementation is deferred to Phase 2.

2. **Replace Generic Data Providers (e.g., Apollo) with Aviato**  
   - For personal and possibly Careerspan use, Aviato is the preferred enrichment source if pricing and data quality check out.

3. **Diff‑based Flat File Updates as a Future Option**  
   - For larger‑scale deployments, V acknowledges the value of receiving full extracts plus diffs; reserved for a later stage when he is ready to store and manage data locally.

## Open Decisions

- Final pricing and refresh cadence for any production‑grade Careerspan deployment.  
- Compliance/privacy posture for proactive outreach (particularly when using vesting and hidden open‑to‑work signals).  
- How tightly Aviato will be branded or exposed inside Careerspan vs. remaining a behind‑the‑scenes data provider.
