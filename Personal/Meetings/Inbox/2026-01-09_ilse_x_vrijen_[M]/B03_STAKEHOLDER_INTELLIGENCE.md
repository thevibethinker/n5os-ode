# B03: Stakeholder Intelligence

## Ilse (Co-founder, Technical Lead)

### Role & Function
- Technical co-founder at Careerspan
- Responsible for product development, architecture decisions
- Manages the technical implementation of all features

### Communication Style
- Direct and pragmatic
- Asks clarifying questions before acting
- Needs full context before proceeding ("I need to sit with this for a while")
- Uses humor to defuse tension ("Did. You have so many reasons to be down on yourself, but that's not one of them")

### Key Interests & Priorities
1. **Plan stability:** "I want to get a plan, and I want to get the plan figured out and write so it doesn't change"
2. **Avoiding wasted work:** "Before I accidentally build things that we don't need"
3. **Understanding strategic intent:** Repeatedly asks "what are we doing this for?" before technical execution
4. **Clean architecture:** Frustrated by technical debt from rapid pivots

### Positions Demonstrated
- **Pre-list as trust mechanism:** Supports the concept but flags exploitation risk
- **Recruiter pivot scope:** Originally proposed using recruiters for Marvin success stories, NOT as primary customer segment
- **Applicant agency:** Advocated that applicants must click "submit" themselves (not auto-submit)

### Leverage Points
- Responds well to clear strategic rationale before technical asks
- Values acknowledgment of technical constraints
- Appreciates when V takes accountability for shifting requirements

### Skepticism/Resistance
- Skeptical of AI-first pitch to recruiters: "They don't trust AI. Why should they trust our app?"
- Concerned about scope creep: Distinguishes between "get success stories with recruiters" vs "become a recruiter app"
- Flags when CSV pre-list risks being exploited vs. serving trust function

### Relationship Dynamic with V
- Collaborative and supportive but holds firm on technical constraints
- Provides emotional support while maintaining honesty ("You have so many reasons to be down on yourself, but that's not one of them")
- Comfortable pushing back on timeline asks ("I can probably get them done on Monday. I'm sorry that it takes so long")
