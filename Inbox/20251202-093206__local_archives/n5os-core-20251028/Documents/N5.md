# N5 Operating System

**Version:** 2.0  
**Purpose:** Personal operating system for AI-assisted productivity

---

## What is N5?

N5 is an operating system for your personal AI workspace. It provides:
- **Structure** for organizing knowledge, tasks, and workflows
- **Automation** via scripts, commands, and recipes
- **Intelligence** through architectural principles and patterns
- **Integration** with your tools and services

---

## Core Philosophy

**From Planning Prompt:**
- Think → Plan → Execute (spend 70% on thinking/planning)
- Simple over easy (few intertwined concepts)
- Code is free, thinking is expensive
- Maintenance over organization

## Key Concepts

### 1. **Commands** (Formal Operations)
Registered in `N5/config/commands.jsonl`:
- `conversation-end` - Formal conversation closure
- `git-check` - Pre-commit safety audit
- `lists-add` - Add items to action lists
- `knowledge-add` - Ingest to knowledge base
- `index-rebuild` - Rebuild search index

### 2. **Recipes** (User Workflows)
Markdown files in `Recipes/` directory:
- Invocable via `/` in chat
- YAML frontmatter with description/tags
- Can reference files, tools, other recipes
- Examples: "Close Conversation", "Git Check"

### 3. **Architectural Principles**
In `Knowledge/architectural/principles/`:
- P2: SSOT (Single Source of Truth)
- P5: Anti-Overwrite
- P15: Complete Before Claiming
- P24: Simulation Over Doing
- P32: Simple Over Easy
- P34: Secrets Management
- ...and 28 more

### 4. **Session State**
Each conversation tracks:
- Objective & success criteria
- Files being modified
- Architectural decisions
- Progress & blockers

### For Users

3. Explore `Recipes/` directory

## Core Principles

### P2: Single Source of Truth
One canonical location per piece of information

### P5: Anti-Overwrite  
Preview changes before executing

### P15: Complete Before Claiming
Verify 100% done before marking complete

### P24: Simulation Over Doing
Dry-run everything destructive

### P32: Simple Over Easy
Few intertwined concepts > familiar/convenient

### P34: Secrets Management
Centralized, encrypted credential storage

**See:** `Knowledge/architectural/principles/` for all 34 principles

## Common Workflows

### Close Conversation
```
/close-conversation
```
- Generates AAR
- Organizes files
- Checks git status
- Archives thread
- Cleans workspace

### Pre-Commit Check
```
python3 N5/scripts/n5_git_check.py
```
- Detects protected file changes
- Scans for credentials
- Flags large deletions
- Blocks unsafe commits

### Add to Knowledge
```
command knowledge-add
```
- Process records → knowledge
- Apply classification
- Update index
