## System Bulletins

**Automated changelog providing AI transparency into recent system evolution**

- **File:** ./N5/data/system_bulletins.jsonl
- **Window:** Rolling 10-day history
- **Updates:** Every 2 hours (scheduled task)
- **Auto-loaded:** In session state init with `--load-system` flag

**Purpose:** Track significant workspace changes to prevent context gaps. When AI encounters contradictions, irregularities, or unexpected behavior, bulletins provide explanation of recent changes.

**Troubleshooting Workflow:**
1. Encounter irregularity (missing file, changed behavior, unexpected structure)
2. Check system bulletins for recent changes
3. Find relevant bulletin entry explaining the change
4. Reference bulletin_id and conversation_id for full context

**Manual Access:**
```bash
# View all bulletins
cat N5/data/system_bulletins.jsonl | jq -c '{timestamp, significance, summary}'

# View high-priority only
cat N5/data/system_bulletins.jsonl | jq -c 'select(.significance=="high") | {timestamp, summary}'
```

**What's Tracked:**
- High: Core scripts, principles, prefs, schemas, commands, architecture
- Medium: Workflows, system docs, recipes
- Low/Filtered: Everything else
