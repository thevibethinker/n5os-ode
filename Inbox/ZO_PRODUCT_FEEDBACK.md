## Feature Request: Conversation Bookmarks

### Problem Statement
In long strategic conversations (20+ exchanges), critical information gets buried. Users need to reference specific decisions, specifications, or insights later. Current options (search, scroll, export) create friction.

### Proposed Solution
**In-conversation bookmark/anchor system:**

**Core functionality:**
1. Right-click any message → "Bookmark this"
2. Optional user-defined label (e.g., "Architecture decision", "Demo spec")
3. Bookmark sidebar showing all bookmarks in current conversation
4. Click bookmark to jump to that point
5. Bookmarks persist across sessions
6. Export includes bookmarks as section markers

### User Impact
- Reduces cognitive load (no need to remember "where did we discuss X?")
- Faster context retrieval during execution
- Enables cleaner conversation flow
- Conversations become navigable knowledge artifacts

### Current Workarounds
- Manual notes in separate document
- Searching (requires remembering exact phrasing)
- Exporting and using text editor find
- All introduce friction

### Priority: High
This addresses real workflow pain from deep usage. Difference between conversations as "chat history" vs "structured knowledge artifacts."
