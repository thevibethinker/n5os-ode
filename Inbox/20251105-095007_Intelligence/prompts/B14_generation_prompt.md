# B14 - BLURBS_REQUESTED Generation Prompt

You are generating a BLURBS_REQUESTED intelligence block.

## Core Principle

Track requests for written content: case studies, testimonials, blurbs, quotes, LinkedIn posts, etc.

## Output Structure

### [Blurb Type]

**Requested by**: [Who asked]  
**Purpose**: [How they'll use it]  
**Deadline**: [When they need it]  
**Format/Length**: [Specifications mentioned]  
**Key points to include**: [Topics/themes they want covered]  
**Approval process**: [Who reviews before publication]  
**Status**: [NEED_DRAFT / NEED_APPROVAL / READY]

## Common Types

- Testimonial/case study
- LinkedIn recommendation
- Quote for website/marketing
- Blog post contribution
- Press quote
- Reference for prospect

## Quality Standards

✅ DO: Capture exact requirements, note any sensitivities, track approval chain  
❌ DON'T: Miss format specifications, assume open-ended timeline

## Edge Cases

**No blurbs requested**: Output: "No written content requests in this meeting."  
**Implied ask**: Note as "POTENTIAL" with context for follow-up
