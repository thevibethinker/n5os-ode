# Resume:Decoded Template Sketch v3.5

## What's New in v3.5

**Core linguistic shift: Signal-based framing**

Old: "Doesn't have X" / "Gap in Y" / "No evidence of Z"  
New: "More signal needed on X" / "Probe: Y" / "Verify in interview: Z"

This reframes the output from "what's wrong" to "what do we know, what do we need to learn."

---

## Underlying Design Principles (Updated)

| Principle | Implementation |
|-----------|----------------|
| **Signal > Verdict** | Every claim tied to evidence type (story/resume/inferred) |
| **Bold = Story-verified** | Highest confidence signal |
| **Italic = No signal yet** | Flag for interview, not assumption of absence |
| **(Xy) = Temporal depth** | Years practicing, not just possession |
| **Open-ended language** | "Ask for..." / "Verify..." / "More signal needed..." |
| **No deficit framing** | Never "lacks," "missing," "gap without probe path" |

---

## Language Guide

| Instead of... | Use... |
|---------------|--------|
| "No direct evidence" | "More direct evidence needed" |
| "Gap in early-stage experience" | "Probe: early-stage velocity" |
| "Doesn't have Node.js" | "Node.js: resume signal only — ask for story" |
| "Weak on collaboration" | "Collaboration signal: resume-only — verify with peers" |
| "Risk: perfectionism" | "Watch for: perfectionism tendencies — ask for 'shipped imperfect' examples" |
| "Not aligned on compensation" | "Verify: compensation expectations — signal incomplete" |

## Updated Info Layering System

| Layer | Encoding | Meaning |
|-------|----------|---------|
| **Bold text** | `**skill**` | Story-verified (strong signal) |
| Regular text | `skill` | Resume-backed (moderate signal) |
| *Italic text* | `*skill*` | No signal yet — verify in interview |
| `(Xy)` | `(4y)` | Years practicing = depth signal |
| `→` | arrow | Links question to signal gap |
| `█` bars | gradient | Importance (no rarity — data unavailable) |

---

## Key Changes from v3

1. **"What to Verify"** replaces "What to Probe" — action-oriented, not deficit-framed
2. **Two-column layout** — Dealbreakers (if any) + Areas to Verify
3. **Questions explicitly map** to signals with arrows
4. **"Not present"** instead of "gap" — signal language, not judgment
5. **"More signal needed"** — replaces "no evidence" everywhere
