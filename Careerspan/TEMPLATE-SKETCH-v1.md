# Resume:Decoded Template Sketch v1

Incorporating learnings + Alex's advisor feedback.

---

## Design Principles Applied

| Principle | Implementation |
|-----------|----------------|
| Risks build trust | Risks section BEFORE strengths |
| Show more with ranking | 5-10 spikes with gradient, not hard 3 cutoff |
| Relevance + rarity | Spikes sorted by importance × scarcity |
| Signal strength = sales | Prominent visualization, story-verified items get visual boost |
| Pair risks → questions | Interview Qs directly follow risks on Page 2 |
| Efficiency via hierarchy | Visual gradient does filtering, not omission |

## Key Layout Decisions

### 1. Spikes as Gradient Pills (Alex's Dictionary.com Pattern)

```
┌─────────────────────────────────────────────────────────────────┐
│  ▲ MLOps 0→1 ✓✓        ▲ Enterprise scale ✓✓    ▲ RAG/LLM ✓    │
│  ████████████          ████████████             ██████████      │
│  (darkest)             (dark)                   (medium)        │
│                                                                 │
│  ▼ Node.js gap         ▼ Early-stage pace                       │
│  ██████                █████                                    │
│  (medium-dark)         (medium)                                 │
└─────────────────────────────────────────────────────────────────┘
```

- **Sort by:** importance × rarity (not just JD relevance)
- **Show:** 5-10 items (not hard cutoff at 3)
- **Visual weight:** Darker bar = higher combined score
- **Evidence marker:** ✓✓ = story-verified, ✓ = resume-only

### 2. Risks Before Strengths

Traditional resume helper: Strengths first, risks buried.
This template: **Risks first.** Builds trust immediately.

### 3. Interview Questions Paired with Risks

Page 1 surfaces: "Early-stage pace is a risk"
Page 2 provides: "Here's exactly what to ask to probe it"

### 4. Signal Strength as Prominent Feature

Not hidden in footer. On Page 1 with explainer text. Sells Careerspan's differentiation.
