## File Organization Rules (Critical)

**N5 Naming Convention**: kebab-case
- ✅ GOOD: `voice.md`, `essential-links.json`, `follow-up-email-generator.md`
- ❌ BAD: `Companion [05] - Master Voice.txt`, `VoiceFile.md`

**File Localization Process**:
1. Identify external/foreign files in function
2. Create N5-native versions in proper location
3. Archive originals: `Documents/Archive/ingested/[date]/`
4. Update all references to point to N5 versions

**Storage Locations**:
- Commands: `N5/commands/`
- Prefs: `N5/prefs/` (communication, intellectual, etc.)
- Scripts: `N5/scripts/`
- Knowledge: `Knowledge/` (stable, context, etc.)

## The Process (5 Phases)

### Phase 1: Analysis & Deconstruction
- Read function completely
- Identify purpose, complexity, dependencies
- Extract inputs, outputs, processing logic
- Check N5 alignment (does similar exist? which category? prefs integration?)

### Phase 2: Design & Architecture
- Define command name (kebab-case)
- Choose implementation approach (prompt-only, script-based, hybrid)
- Design inputs/outputs
- Plan prefs integration
- Consider scheduling needs

### Phase 3: Implementation
- **FIRST**: Localize external files to N5-native format
- Create command file with proper structure
- Create script if needed
- Register in commands.jsonl
- Update documentation

### Phase 4: Validation & Testing
- Dry run with mock data
- Integration testing with N5 systems
- Real-world testing on actual use case
- Safety checks (no accidental overwrites, etc.)

### Phase 5: Refinement & Documentation
- Fix issues found in testing
- Define success metrics
- Document lessons learned
- Git commit with descriptive message
