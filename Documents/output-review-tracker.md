# Output Review Tracker

**Purpose:** Centralized tracking of generated outputs for quality review and training data collection

**Version:** 1.0.0  
**Created:** 2025-10-19

---

## Quick Start

### Add output for review
```bash
n5 review add path/to/file.md --title "Description"
```

### List all reviews
```bash
n5 review list
n5 review list --status pending
n5 review list --sentiment excellent
```

### Update status
```bash
n5 review status <output_id> approved --sentiment good
```

### Add comments
```bash
n5 review comment <output_id> --body "Great work!"
```

### Export for training
```bash
n5 review export --sentiment excellent --output training.json
```

---

## Workflow States

1. **pending** - Just added, awaiting review
2. **in_review** - Currently being reviewed
3. **approved** - Passed review, good quality
4. **issue** - Has problems, needs revision
5. **training** - Approved for training data
6. **archived** - No longer relevant

---

## Sentiment Levels

- **excellent** - Outstanding quality, exemplar
- **good** - Solid quality, usable
- **acceptable** - Meets minimum bar
- **issue** - Below standard, needs work

## Commands Reference

### n5 review

Show help and available subcommands.

```bash
n5 review --help
```

### n5 review add

Add an output for quality review. Auto-detects type (file/message/url) and captures full provenance.

```bash
n5 review add <reference> [options]

Options:
  --title "Title"              Custom title (auto-detected from filename if omitted)
  --type file|message|url      Output type (auto-detected if omitted)
  --conversation-id con_XXX    Conversation ID (auto-detected)
  --tags tag1,tag2            Comma-separated tags
  --notes "Context"           Additional context notes
  --dry-run                   Preview without saving
```

**Examples:**
```bash
n5 review add Documents/meeting-notes.md --title "Weekly sync notes"
n5 review add /tmp/generated-image.png --tags "dalle,landscape"
n5 review add https://example.com/doc --type url --notes "External reference"
```

### n5 review list

List all reviews with optional filters.

```bash
n5 review list [options]

Options:
  --status pending|in_review|approved|issue|training|archived
  --sentiment excellent|good|acceptable|issue
  --tags tag1,tag2
  --type file|message|image|video|transcript|url
```

**Examples:**
```bash
n5 review list
n5 review list --status pending
n5 review list --sentiment excellent --tags training
```

### n5 review show

Show full details for a specific review including provenance, quality scores, status history, and all threaded comments.

```bash
n5 review show <output_id>
```

**Example:**
```bash
n5 review show out_x7k9m2n4p8q1
```

### n5 review status

Update review status, sentiment, and quality scores. Supports workflow transitions.

```bash
n5 review status <output_id> <new_status> [options]

Options:
  --sentiment excellent|good|acceptable|issue
  --reviewer "Name"            Reviewer name
  --score dim=value           Quality score (can repeat)
  --note "Details"            Additional notes
  --dry-run                   Preview without saving
```

**Examples:**
```bash
n5 review status out_x7k9m2n4p8q1 in_review --sentiment good
n5 review status out_abc123 approved --reviewer V --score tone=9 --score clarity=8
n5 review status out_xyz789 archived --note "Superseded by v2"
```

### n5 review comment

Add threaded comment to a review. Supports up to 3 levels of threading.

```bash
n5 review comment <output_id> [options]

Options:
  --body "Comment text"       Comment body (required)
  --author "Name"             Comment author
  --context "Section ref"     Context excerpt or reference
  --parent cmt_XXX           Parent comment ID (for threading)
  --tags tag1,tag2           Comment tags
  --dry-run                  Preview without saving
```

**Examples:**
```bash
n5 review comment out_x7k9m2n4p8q1 --body "Great clarity in the introduction"
n5 review comment out_abc123 --body "Consider simplifying this section" --context "Lines 45-52" --tags tone
n5 review comment out_xyz789 --body "Agreed, good point" --parent cmt_parent123
```

### n5 review export

Export reviews to JSON for training data collection.

```bash
n5 review export [options]

Options:
  --sentiment excellent|good|acceptable|issue
  --status pending|in_review|approved|issue|training|archived
  --output /path/to/file.json  Output file (defaults to stdout)
  --dry-run                   Preview without saving
```

**Examples:**
```bash
n5 review export --sentiment excellent
n5 review export --status approved --output /tmp/training-data.json
n5 review export --sentiment good --status training
```

## Integration

### Automatic Detection

The system automatically detects:
- **Conversation ID** from environment
- **Output type** from reference (file extension, URL pattern)
- **Content hash** for change detection
- **Thread name** from conversation context

### Quality Dimensions

Available for `--score` parameter:
- **tone** - Voice and style consistency
- **clarity** - Understandability
- **accuracy** - Factual correctness
- **completeness** - Thoroughness
- **structure** - Organization and flow
- **custom** - Any custom dimension

### Tags

Common tag categories:
- **Content type:** `email`, `document`, `code`, `image`, `diagram`
- **Status:** `generated`, `revised`, `training`, `exemplar`
- **Domain:** `meeting`, `strategic`, `technical`, `communication`
- **Quality:** `excellent`, `needs-work`, `edge-case`

## Best Practices

1. **Review early** - Add outputs immediately after generation
2. **Tag thoroughly** - Use descriptive tags for future filtering
3. **Comment specifically** - Reference sections/lines when possible
4. **Track sentiment** - Use consistent criteria across reviews
5. **Export regularly** - Build training datasets incrementally

## System Architecture

The Output Review Tracker follows N5 architectural principles:

- **SSOT (P2):** JSONL files are the single source of truth
- **Portable (P1):** All data in `/Lists/` for easy backup/migration
- **Modular (P20):** CLI interface, separate storage, extensible schema
- **Verifiable (P18):** Content hashes detect file changes
- **Safe (P7, P19):** Dry-run support, error handling
