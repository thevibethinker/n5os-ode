## Directory Standards

All deliverables stored here should follow these conventions:

### Naming
```
[Name]_v[Version]_[Date]/
```
Example: `N5_Bootstrap_v1.0.0_20251018/`

### Structure
```
DeliverableName/
├── START_HERE.md          # Entry point for distributor
├── README.md              # Package overview
├── [MainFile].tar.gz      # Primary deliverable
├── [MainFile]_Extracted/  # Browsable before sending
└── [Supporting docs]
```

### Required Files
- **START_HERE.md** - Instructions for you (distributor)
- **README.md** - Overview for recipient
- **Main deliverable** - The actual file(s) to share
- **Extracted version** - For browsing before distribution

---

## Usage

### For You (Creating Deliverables)
1. Create new directory: `mkdir Deliverables/[Name]_v[Version]`
2. Add files and documentation
3. Create START_HERE.md with distribution instructions
4. Test if needed
5. Mark as ready for distribution

### For Distribution
1. Open deliverable directory
2. Read START_HERE.md
3. Share main file with recipient
4. Provide any necessary URLs/credentials
5. Recipient follows their instructions
