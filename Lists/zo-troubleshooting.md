## Feature Request: Conversation Bookmarks

**Date:** 2025-10-31 20:10 ET  
**Conversation:** con_yiJz5ECFH4KetJZd  
**Type:** Feature Request  
**Impact:** workaround-available  
**Tags:** feature-request, bookmarks, navigation, ux, power-user

### Problem
In long strategic conversations (20+ exchanges), critical information gets buried. Users need to reference specific decisions, specifications, or insights later without scrolling/searching through entire conversation.

### Proposed Solution
**Conversation bookmark/anchor system:**
- Right-click any message → "Bookmark this"
- Add optional label (e.g., "Architecture decision", "Requirements")
- Sidebar shows all bookmarks for quick navigation
- Click bookmark to jump to that point
- Persists across sessions

### Use Case
Power users in planning/design conversations need to quickly reference:
- Past decisions made earlier in conversation
- Specifications that emerged
- Key insights to revisit
- Critical requirements

**Example:** During demo planning conversation, needed to reference multiple decision points about structure, examples, messaging scattered across 30+ message thread.

### Current Workaround
- Manual note-taking in separate document
- Exporting full conversation and using text editor search
- Trying to remember approximate message position

### Value
- Reduces cognitive load (no need to remember "where did we discuss X?")
- Faster context retrieval during execution
- Enables conversations as structured knowledge artifacts
- Aligns with "cognitive liberation" thesis - externalize memory so you can stay present

### Context
Emerged from sustained power usage of Zo (N5 OS with 563 scripts, 2705 docs). This is a real workflow pain point after consistent deep usage.
