# Meta-Reflection Style Guide

**Block ID:** B91  
**Domain:** internal  
**Voice Profile:** `file 'N5/prefs/communication/voice.md'`  
**Auto-Approve Threshold:** 10 blocks

---

## Purpose

Reflection on the reflection practice itself. Thinking about thinking. Evaluating what's working in the reflection system, what needs adjustment, and how the practice is evolving. Meta-cognition about knowledge management, insight capture, and personal operating system.

---

## Structure

**Typical flow:**
- Observation: What I'm noticing about my reflection practice
- Evaluation: What's working / what's not
- Patterns: How practice is evolving over time
- Adjustments: Changes to make in the system
- Principles: Insights about effective reflection

**Frequency:** Monthly or when system needs adjustment (not daily—needs distance for perspective)

---

## Tone & Voice

**Core Characteristics:**
- **Self-aware:** Conscious of the practice itself
- **Iterative:** System is always evolving, never "done"
- **Pragmatic:** Focus on what actually helps (not theoretical ideals)
- **Honest:** Acknowledge what's not working
- **Experimental:** Willing to try new approaches
- **Principled:** Extract transferable insights about effective reflection

**Avoid:**
- Navel-gazing about navel-gazing (meta without purpose)
- Perfectionism about the system (done > perfect)
- Process for process sake (system serves thinking, not vice versa)
- Over-engineering reflection practice
- Forgetting the goal: better thinking and decisions

---

## Lexicon

**Meta Language:**
- Practice: The reflection system/habit
- System: The structure (blocks, style guides, processing)
- Process: How reflections move through the system
- Signal vs. noise: Valuable insights vs. unnecessary capture
- Friction: What makes reflection harder
- Flow: What makes reflection easier

**Evaluation Terms:**
- Working: Delivering value, sustainable
- Not working: Creating friction, being ignored
- Evolving: Changing naturally over time
- Stuck: Needs conscious adjustment

## Transformation Guidance

**Raw → Refined:**
- "I don't reflect enough" → "Noticing 3-day gaps in B50 blocks. Hypothesis: Friction is opening N5 separately. Test: Pin reflection template in daily workspace."
- "System feels complicated" → "Processing 8 reflection blocks takes 45 min. Signal: B50, B72, B73. Noise: B71 (redundant with B73). Adjustment: Merge market/strategic blocks."
- "Not sure this helps" → "Can't recall using any B60 blocks in last 30 days. Test: Does learning synthesis inform decisions? If not, pause B60 for month."

**Key Transforms:**
1. **Specific observations:** "Feels off" → Concrete data (frequency, time spent, blocks created)
2. **Hypothesis-driven:** "Should change X" → "Hypothesis: X causes Y. Test: Try Z for 2 weeks"
3. **Measure impact:** "This might help" → "Success metric: [measurable outcome]"
4. **Connect to purpose:** "System issue" → "How does this affect thinking quality or decision-making?"

## Examples

### Example 1: Evaluating Block Types

**Raw Input:**
```
Been using the reflection system for a month. Some blocks I use all the time 
(B50, B72, B73), others barely at all (B71, B91). Not sure if that means the 
system is wrong or I just don't need those blocks. Maybe I should simplify?
```

**Refined Output:**
```
**Observation: Usage patterns after 30 days**

Block usage frequency (Oct 1-31):
- B50 (Personal): 18 blocks (every 1.7 days avg)
- B72 (Product): 12 blocks (every 2.5 days)
- B73 (Strategic): 8 blocks (weekly)
- B60 (Learning): 4 blocks (weekly)
- B71 (Market): 2 blocks (bi-weekly)
- B82 (Memo): 1 block (used for investor update)
- B91 (Meta): 0 blocks (until now)
- B70, B81, B90: 0 blocks

**What's Working:**

1. **High-frequency blocks (B50, B72, B73):** These match my daily cognitive load
   - B50: Daily processing of emotions, decisions, observations
   - B72: Active product development creates daily product questions
   - B73: Weekly strategic thinking feels right (not too often, not too rare)

2. **Block separation:** Keeping personal (B50) separate from professional (B72/73) helps me see patterns in each domain

3. **Auto-approve threshold:** High thresholds (10 blocks) for internal reflections means I write freely without review friction

**What's Not Working:**

1. **B71 (Market Analysis) redundant with B73 (Strategic):** Market insights end up in strategic reflections anyway. The separation creates artificial friction—I hesitate about where to put competitive observations.

2. **B90 (Insight Compound) never used:** I compound insights naturally in B73 strategic blocks. Don't need separate meta-synthesis structure yet. Might be valuable at 6-month mark, but premature now.

3. **B70, B81, B82 (External blocks) low usage:** These are output formats, not reflection formats. I'm generating *input* (thinking) daily, but only occasionally producing *output* (posts, memos, blog). Usage is correct—these shouldn't be frequent.

4. **B91 (Meta) avoided:** Meta-reflection feels self-indulgent. But this exercise is useful—I just need lower frequency (monthly, not weekly).

**Patterns Over Time:**

- **Week 1-2:** Tried to use all block types "correctly"—forced B71 market reflections even when I had strategic thoughts
- **Week 3-4:** Naturally gravitated to B50/B72/B73, ignored others
- **Pattern:** System working when I stop thinking about system and just capture thinking

**Adjustments to Try:**

1. **Merge B71 → B73:** Eliminate market analysis as separate block. Strategic reflections can include market observations.
   - Update registry: B73 description includes "market trends and competitive landscape"
   - Archive B71 style guide (keep for reference but don't use)

2. **Archive B90 for now:** Insight compounding happening naturally. Revisit in 6 months when I have 100+ reflections and patterns are harder to see manually.

3. **Reduce B91 expectations:** Meta-reflection is monthly review, not regular practice. Don't force it.

4. **Accept low usage of B70, B81, B82:** These are output blocks, not input blocks. Current usage is correct.

**Success Metrics:**

After adjustments, I expect:
- **Less decision friction:** No more "Is this B71 or B73?" hesitation
- **Higher B73 usage:** Strategic reflections increase to 10-12/month (absorbing B71 topics)
- **Same total reflection volume:** Adjustment is reorganization, not reduction
- **System feeling lighter:** Fewer unused blocks = less guilt about "doing it wrong"

**Test Period:** Nov 1-30

**Check-in:** Dec 1 meta-reflection to evaluate if changes worked

**Underlying Principle:**

The reflection system should match natural thinking patterns, not impose structure. If a block type isn't being used, either:
1. It's redundant (merge with another block)
2. It's aspirational (archive until needed)
3. It's output, not input (low usage is correct)

Good system = invisible system. When I forget about the structure and just think, system is working.
```

### Example 2: Process Improvement

**Raw Input:**
```
The reflection processing workflow feels clunky. I write reflections in Zo chat, 
then manually copy to Records, then process them weekly. By the time I process 
them, I've forgotten context. Maybe I should process immediately?
```

**Refined Output:**
```
**Observation: Processing delay creates context loss**

**Current Workflow:**
1. Write reflection in Zo chat during the moment (context: high)
2. Reflection saved to Records/Temporary/
3. Weekly batch: Review 10-15 reflections, decide which go to Knowledge
4. Process into Knowledge with edits (context: low—forgot what I was thinking)

**Problem:**
By day 7, I've forgotten *why* I wrote the reflection. The raw content is there, but the **feeling** and **context** that made it important are gone. Result: Generic processing, lost nuance.

**Examples:**
- B72-008 (product decision): Raw reflection captured uncertainty about timeline vs. synthesis. By processing day, decision was made—couldn't remember the *felt* tension, only the outcome. Lost the valuable "how I think through uncertainty" insight.
- B50-015 (personal): Raw reflection about feeling behind on fundraising. Week later, felt better. Processed it as "just a bad day" but missed deeper pattern about my relationship with external validation.

**Hypothesis:**
**Immediate processing** (same day or next day) would preserve context and emotional truth. But might sacrifice synthesis across reflections (which requires time/distance).

**Two Types of Processing:**

| Type | Timing | Purpose | What's Preserved |
|------|--------|---------|------------------|
| **Capture** | Immediate (same day) | Preserve context, emotion, raw thinking | Feeling, uncertainty, process |
| **Synthesis** | Delayed (weekly) | Find patterns, connect dots | Through-lines, themes |

**Insight:**
I need BOTH. Current system only does synthesis (delayed). Missing immediate capture that locks in context.

**Adjusted Workflow:**

**Immediate (Same Day):**
1. Write reflection in Zo chat
2. Spend 2 min adding context while fresh:
   - "Why I'm thinking about this now: [trigger]"
   - "What I'm feeling: [emotion]"
   - "Open question: [uncertainty]"
3. Save to Records with context intact

**Weekly (Batch):**
1. Review reflections with context preserved
2. Identify patterns across multiple reflections
3. Move valuable insights to Knowledge
4. Create B90 insight compounds when patterns emerge

**Trade-off:**
- **Cost:** 2 extra minutes per reflection (10 reflections/week = 20 min/week)
- **Benefit:** Context preservation, emotional truth, better synthesis
- **Net:** Worth it if I actually use reflections for decisions

**Test:**
- **Period:** Nov 1-30
- **Method:** After writing reflection, immediately add "Context" section with trigger, feeling, question
- **Success metric:** In weekly review, can I recall why reflection mattered? (Yes/No for each)
- **Compare:** Oct average (context recalled: ~40%) vs. Nov target (>80%)

**Underlying Principle:**

**Insight has a half-life.** The context and emotional truth that make a reflection valuable decay rapidly. Capture them immediately, synthesize later.

This applies beyond reflection:
- User interviews: Write insight immediately after call, not at week-end
- Product decisions: Document thinking in the moment, not retrospectively
- Learning: Capture "aha" when it happens, not when you process notes

**Future Exploration:**
Could this work for other knowledge types? Meeting notes, research synthesis, decision logs?
```

## QA Checklist

Before finalizing:
- [ ] Focuses on the practice itself (not just using meta-reflection as another reflection type)
- [ ] Includes specific observations (data, frequency, patterns)
- [ ] Evaluates what's working AND what's not
- [ ] Proposes concrete adjustments (not vague "should reflect more")
- [ ] Defines success metrics for changes
- [ ] Extracts transferable principles (not just personal quirks)
- [ ] Honest about what's not working (no false optimization)
- [ ] Connected to purpose (better thinking/decisions, not perfect system)
- [ ] Actionable (leads to system changes or insights to apply)
