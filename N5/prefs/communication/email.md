## Detection Rules

**Authoritative source:** `file 'Lists/detection_rules.md'`

### Quick Reference

**Jobs/Tasks:**
- Keywords: task, job, follow-up, action
- Route to: `N5/lists/tasks.md`

**Attachments:**
- Types: pdf, docx, xlsx, jpg, png, mp4
- Route to: `N5/knowledge/attachments/{type}/`

**Articles/Newsletters:**
- Patterns: http/https URLs, "newsletter" in subject
- Route to: `Knowledge/articles/` and digest

**Other:**
- Default: `Knowledge/inbox/`

## Processing Workflow

### Step 1: Detect Category

Check email against detection rules:
1. Check for job/task keywords
2. Check for attachments
3. Check for article/newsletter patterns
4. Default to inbox

### Step 2: Extract Metadata

Capture:
- Sender
- Subject
- Date received
- Attachments (if any)
- Key entities (people, companies, topics)

### Step 3: Route & Store

Based on detection:
- **Tasks:** Add to tasks list via `command 'N5/commands/lists-add.md'`
- **Articles:** Save to Knowledge, add to tracker
- **Attachments:** Route by type, extract text if applicable
- **Other:** Store in inbox for manual review

### Step 4: Log

Write processing log:
- Email ID
- Detection rule matched
- Routing decision
- Actions taken
- Timestamp

### Step 5: Digest Update

If daily digest enabled:
- Append summary to today's digest
- Group by category
- Link to stored content
