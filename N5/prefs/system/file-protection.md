## File Classification System

Files are protected differently based on their role:

### HARD PROTECTION (Manual-Edit Only)
- `N5.md` — Core system index (hand-authored)
- `N5/prefs/prefs.md` — System preferences index (hand-authored)
- All module files in `N5/prefs/**/*.md` (hand-authored)

### MEDIUM PROTECTION (Requires Pre-Check)
- `Recipes/ (self-executing)` — Command registry (manually curated/validated)
- `N5/lists/*.jsonl` — User data lists (manually curated)
- `Knowledge/**/*.md` — Knowledge content (manually authored)
- `Lists/*.jsonl` — List data (manually curated)

### AUTO-GENERATED (Do Not Protect - Regeneratable)
- `N5/commands.md` — Generated by docgen
- `N5/commands/*.md` — Automatically DEPRECATED
- `N5/prefs.md` — Generated by index-update
- `N5/index.jsonl` — Database regenerated by index-update
- `Lists/*.md` — Generated views of JSONL data

## Overwrite Protection Workflow

For **HARD PROTECTION** files:

1. **Read first:** `read_file()` to verify current content
2. **Size check:** If file > 0 bytes, show preview and require explicit confirmation
3. **Git status:** If modified, require Git diff review
4. **Atomic backup:** Suggest backup before modification
5. **Verification:** After modification, verify content was preserved as intended

For **MEDIUM PROTECTION** files:

1. **Read first:** Verify current content
2. **Schema validation:** Check against appropriate schema in `N5/schemas/` or `Knowledge/schemas/`
3. **Dry-run:** Show proposed changes before applying
4. **Confirmation:** Require explicit approval
5. **Log:** Record change in appropriate audit log

---

## Recovery Protocol

For any overwrite incident:

1. **Document immediately** in timeline system:
   - Impact: high
   - Tags: incident, data-loss, overwrite
   - Include file path and timestamp

2. **Check Git history:**
   ```bash
   git log --oneline -10 -- [file]
   ```

3. **Restore from latest good commit:**
   ```bash
   git show [commit]:[file] > [file]
   ```

4. **Verify restoration:**
   - Check file content
   - Run validation if applicable
   - Confirm with user

5. **Add to N5 timeline** for tracking and pattern analysis
