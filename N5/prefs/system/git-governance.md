## Git Audit Command

Use `git-check` command regularly after adding new workflows or files to detect untracked important files.

```bash
# Via command registry
command 'N5/commands/git-check.md'
```

This will:
- Print exact shell commands to add missing files to Git
- No automatic changes are made
- Manual approval is required to add files

## Commit Guidelines

### When to Commit

Commit after:
- Creating or modifying preference modules
- Adding or updating commands in registry
- Updating schemas
- Creating or updating knowledge files
- Modifying list data
- System configuration changes

### Commit Message Format

Use conventional commit style:

```
<type>(<scope>): <subject>

[optional body]
```

**Types:**
- `feat` — New feature or capability
- `fix` — Bug fix or correction
- `docs` — Documentation updates
- `refactor` — Code/structure refactoring
- `chore` — Maintenance tasks
- `data` — Data updates (lists, knowledge)

**Examples:**
```
feat(prefs): modularize preferences into specialized files
data(knowledge): add Careerspan GTM pivot to timeline
fix(commands): correct lists-add schema validation
docs(readme): update knowledge ingestion standards
```
