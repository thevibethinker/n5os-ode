# Recipe Execution Guide

**Version:** 1.0  
**Date:** 2025-10-28  
**Principle:** Recipes are executable instructions, not documentation

---

## Core Understanding

**Recipes are instructions for AI execution, not just documentation.**

When a user invokes a recipe (e.g., `/close-conversation`), the AI:
1. Reads the recipe markdown file
2. Follows the instructions explicitly
3. Executes any bash commands specified
4. Reports results

**Recipes do NOT:**
- Auto-trigger scripts via some hidden registry
- Get parsed by special command processors
- Require registration in a command database

---

## Recipe Structure

### Required Sections

```markdown
---
description: |
  Brief summary of what this recipe does
tags:
  - category1
  - category2
---
# Recipe Name

[Brief introduction]

## What This Does

Clear description of the recipe's purpose and outcomes.

## Execution

**This is the critical section!**

Explicit bash commands or step-by-step instructions:

\`\`\`bash
python3 /home/workspace/N5/scripts/script_name.py \\
  --flag value \\
  --another-flag
\`\`\`

**Parameters:**
- `--flag`: Description
- `--another-flag`: Description

## When to Use

- Trigger condition 1
- Trigger condition 2

## What You Get

✅ Expected output 1  
✅ Expected output 2
```

## Execution Pattern

### Example: Script-Based Recipe

```markdown
## Execution

**Run the cleanup:**

\`\`\`bash
python3 /home/workspace/N5/scripts/cleanup_root.py \\
  --dry-run
\`\`\`

**Real execution:**
\`\`\`bash
python3 /home/workspace/N5/scripts/cleanup_root.py
\`\`\`
```

**AI behavior:** When recipe is invoked, AI runs these commands using `run_bash_command`.

### Example: Step-by-Step Recipe

```markdown
## Execution

**Instructions:**

1. Ask which meeting to analyze
2. Load `file 'N5/config/meeting-processing-blocks.json'`
3. Ask which blocks to apply
4. Run `python3 /home/workspace/N5/scripts/meeting_processor.py \\
     --meeting <path> \\
     --blocks B01,B05,B13`
```

**AI behavior:** AI follows steps sequentially, asking questions and running commands as specified.

## Commands.jsonl Migration

**Status:** DEPRECATED as of 2025-10-27

**Previous System:**
- `N5/config/commands.jsonl` - Registry of executable commands
- AI checked this file before operations
- Commands auto-executed via triggers

**Current System:**
- `N5/data/executables.db` - Index of recipe files (metadata only)
- Recipes contain explicit execution instructions
- AI reads recipe and executes directly

**Why Changed:**
- Simpler architecture (one less layer)
- More transparent (execution is visible in recipe)
- Easier maintenance (update recipe, not recipe + registry)
- Better for non-technical users (see exactly what will run)

  "category": "recipes"

## Best Practices

### ✅ DO

1. **Include explicit bash commands** in Execution section
2. **Show dry-run options** when available
3. **Document all parameters** with descriptions
4. **Use absolute paths** in commands
5. **Test commands** before committing recipe

### ❌ DON'T

1. **Rely on hidden registries** for execution
2. **Assume AI knows** what script to run
3. **Use relative paths** without context
4. **Skip parameter documentation**
5. **Create recipes without execution section**

## Troubleshooting

### Problem: Recipe not executing

**Symptoms:** AI reads recipe but doesn't run commands

**Causes:**
1. No Execution section in recipe
2. Commands not in bash code blocks
3. Vague instructions ("process the file")

**Fix:** Add explicit Execution section with bash commands

### Problem: Wrong script executed

**Symptoms:** AI runs different script than expected

**Causes:**
1. Recipe has outdated script path
2. Multiple recipes with similar names

**Fix:** Update recipe with correct absolute path
