# B31_STAKEHOLDER_RESEARCH

---
**Perspective:** Internal product/design team (Careerspan)

### Key Insights

1. Progressive-disclosure UX reduces user cognitive load — surfacing a short, actionable summary (cheat sheet) at the top level is valuable for time-pressed users (interviewers/applicants).
   - Implication: Add a 1–2 sentence summary or "top-line talking points" for each responsibility to improve on-the-fly usability.
   - Strategic value: Lowers barrier to using generated materials in live conversations and interviews.

2. Regenerate trade-offs are primarily compute-cost vs. user value; users tolerate 30s–4min latency if feedback (loading/toast) is clear.
   - Implication: Invest in UX that communicates progress and expected delay rather than trying to force instant responses.
   - Strategic value: Enables richer processing (narrative + fill-in-the-gaps) without poor UX surprises.

3. Internal alignment: team prefers shipping iterative functionality (no radar graph initially) over perfect visual fidelity.
   - Implication: Defer graph work; ship core functionality first.
   - Strategic value: Faster user feedback loop, reduced wasted effort.

4. Deal-breakers must be visible in canonical narrative after user proceeds (not stale Vibe Check content).
   - Implication: Incorporate deal-breaker answers into Narrative Prep copy / SWOT so facts persist and are discoverable.
