# B19: User Research & Customer Insights

## Objective

Extract user research findings, customer insights, behavioral patterns, and user experience considerations discussed during the meeting.

## What to Capture

**User Behaviors:**
- How users actually use products/services
- Observed patterns and workflows
- Pain points and frustrations
- Unexpected use cases

**Customer Needs:**
- What users are trying to accomplish
- Jobs-to-be-done insights
- Unmet needs identified
- Feature requests or desires

**User Feedback:**
- Direct quotes from users
- Sentiment and reactions
- Satisfaction levels
- Complaints or praise

**Personas & Segments:**
- User types identified
- Segment characteristics
- Behavioral differences
- Demographic insights

**UX Insights:**
- Usability issues
- Design considerations
- Interface feedback
- Accessibility needs

**Research Findings:**
- Study results shared
- Data/metrics discussed
- Key takeaways
- Surprising insights

**Customer Journey:**
- How users discover/adopt
- Activation patterns
- Retention insights
- Churn factors

## Output Format

### Key Insights
[2-3 sentence summary of most important user/customer insights]

### User Behaviors Observed

**Pattern: [Name]**
- **Description:** [What users do]
- **Context:** [When/why this happens]
- **Implications:** [What this means for product/business]

### User Needs & Pain Points

**Need: [Description]**
- **Current Solution:** [How they solve this today]
- **Gap:** [What's missing]
- **Opportunity:** [How to address]

**Pain Point: [Description]**
- **Impact:** [How serious/frequent]
- **Quote:** "[User feedback]" if available
- **Potential Solution:** [Ideas discussed]

### User Segments

**Segment: [Name]**
- **Characteristics:** [Who they are]
- **Behaviors:** [How they differ]
- **Needs:** [What they want]
- **Value:** [Why they matter]

### UX Findings
- **Issue:** [Problem observed]
  - Severity: High/Medium/Low
  - Frequency: [How often]
  - Recommendation: [How to fix]

### Research Data
**Study: [Name/Type]**
- **Method:** [How conducted]
- **Sample:** [Who/how many]
- **Key Findings:**
  1. [Finding 1]
  2. [Finding 2]
  3. [Finding 3]
- **Implications:** [What to do with this]

### Customer Quotes
> "[Verbatim quote]"
> — [User identifier/context]

[Why this quote matters]

## Generation Guidelines

1. Listen for user-centric language:
   - "Users told us..."
   - "We found that..."
   - "Customers are struggling with..."
   - "Research showed..."
   - "People want..."

2. Distinguish observation from interpretation:
   - What did users actually do/say?
   - What does it mean?
   - What should we do about it?

3. Capture specifics:
   - Concrete behaviors, not generalities
   - Actual quotes when shared
   - Quantitative data if mentioned
   - Segment differences

4. Note implications:
   - What does this mean for product?
   - How does this inform decisions?
   - What opportunities does it reveal?

5. Preserve nuance:
   - Conflicting findings
   - Edge cases
   - Surprising insights
   - Things that didn't work

## When to Generate

Generate B19 when meeting includes:
- User research findings
- Customer feedback discussions
- Usability testing results
- Customer interview insights
- Product feedback analysis
- User behavior patterns
- Persona/segment discussions

Skip for meetings without user/customer focus.

## Quality Checklist

Before finalizing, check:
- [ ] Key insights clearly stated
- [ ] User behaviors vs interpretations separated
- [ ] Direct quotes included when available
- [ ] Segments and patterns identified
- [ ] Implications for product/business noted
- [ ] Data/research properly attributed
- [ ] Actionable recommendations included
- [ ] Balanced (not cherry-picked)
