# Drop Brief: Jettison Command Implementation

**Mission:** Add `pulse jettison` subcommand that creates a connected-but-independent build from the current context, outputting a copy-pasteable launch prompt.

**Output:** Modified `Skills/pulse/scripts/pulse.py` with working `jettison` subcommand

---

## Context

Jettison is a **launch mode**, not a build type. It's for when you're in a conversation or build and hit a tangent — something worth pursuing but not worth derailing the current thread for.

Key insight: Jettisons ARE builds. They live in `N5/builds/j-<slug>/` and are self-contained. The difference is:
1. They have **lineage** — tracking where they came from
2. They're **manually launched** — V copies a launch prompt to a new thread
3. They **inherit context** — learnings from parent, the "moment" that triggered them

This Drop implements the command. D1.2 implements the folder scaffolding you'll call.

### 2. Type Auto-Detection

Implement `detect_build_type(task: str) -> str`:

```python
def detect_build_type(task: str) -> str:
    """Auto-detect build type from task description keywords."""
    task_lower = task.lower()
    
    # Code/debugging signals
    if any(kw in task_lower for kw in ['fix', 'bug', 'debug', 'error', 'broken', 'failing', 'crash', 'issue', 'refactor']):
        return 'code_build'
    
    # Research signals
    if any(kw in task_lower for kw in ['research', 'explore', 'investigate', 'analyze', 'study', 'understand', 'learn about']):
        return 'research'
    
    # Content signals
    if any(kw in task_lower for kw in ['draft', 'write', 'content', 'blog', 'post', 'email', 'document', 'copy']):
        return 'content'
    
    # Planning signals
    if any(kw in task_lower for kw in ['plan', 'design', 'architect', 'strategy', 'roadmap', 'spec', 'propose']):
        return 'planning'
    
    # Default
    return 'code_build'
```

### 3. Slug Generation

```python
def generate_jettison_slug(task: str) -> str:
    """Generate slug: j-<short-task>-<timestamp>"""
    import re
    from datetime import datetime
    
    # Extract key words (max 3-4)
    words = re.sub(r'[^a-z0-9\s]', '', task.lower()).split()
    # Filter out common words
    stopwords = {'the', 'a', 'an', 'to', 'for', 'of', 'and', 'in', 'on', 'is', 'it', 'this', 'that'}
    key_words = [w for w in words if w not in stopwords][:3]
    
    slug_base = '-'.join(key_words) if key_words else 'task'
    timestamp = datetime.now().strftime('%m%d-%H%M')
    
    return f"j-{slug_base}-{timestamp}"
```

### 4. Main Jettison Function

```python
async def cmd_jettison(args):
    """Create a jettison build."""
    task = args.task
    parent_slug = args.parent_slug
    parent_convo = args.parent_convo
    build_type = args.build_type or detect_build_type(task)
    moment = args.moment or f"Jettisoned to handle: {task}"
    
    # Generate slug
    slug = generate_jettison_slug(task)
    
    # Check if slug exists, append number if needed
    build_dir = BUILDS_DIR / slug
    counter = 2
    while build_dir.exists():
        slug = f"{slug.rsplit('-', 1)[0]}-{counter}"
        build_dir = BUILDS_DIR / slug
        counter += 1
    
    # Scaffold the build (calls D1.2's function)
    scaffold_jettison_build(
        slug=slug,
        task=task,
        build_type=build_type,
        parent_slug=parent_slug,
        parent_convo=parent_convo,
        moment=moment
    )
    
    # Generate drop brief
    brief_path = build_dir / "drops" / "D1.1-jettison-task.md"
    brief_content = generate_jettison_brief(slug, task, build_type, moment)
    brief_path.write_text(brief_content)
    
    # Output launch prompt

╚══════════════════════════════════════════════════════════════╝
```
