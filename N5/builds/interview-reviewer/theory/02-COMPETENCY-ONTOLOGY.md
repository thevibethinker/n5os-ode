# Competency Ontology

**Purpose:** Define the canonical vocabulary for mapping JD requirements → Interview questions → Candidate evaluation.

---

## How This Works

JDs use inconsistent language. Interviews probe inconsistently. We need a **shared vocabulary** to connect them.

```
JD says: "drive cross-functional initiatives"
         ↓ (synonym resolution)
Canonical: STAKEHOLDER_MANAGEMENT
         ↓ (question inference)
Interview: "Tell me about a time you had to align multiple teams..."
         ↓ (answer evaluation)
Rubric: Did they show the 6 points? Which competencies did they demonstrate?
```

---

## The Competency Clusters

We organize ~60 competencies into 8 clusters. Each cluster represents a major dimension of professional capability.

### Cluster 1: EXECUTION
*Getting things done. Bias to action. Delivering results.*

| ID | Canonical Name | Synonyms | Assessment Indicators |
|----|----------------|----------|----------------------|
| E01 | **Results Orientation** | outcome-driven, delivers results, goal-oriented, high performer | Quantified outcomes, met deadlines, exceeded targets |
| E02 | **Prioritization** | time management, focus, manages competing priorities | Triaged correctly, said no to low-value work, hit critical path |
| E03 | **Execution Speed** | bias to action, urgency, fast-paced, moves quickly | Shortened timelines, unblocked self, didn't wait for permission |
| E04 | **Attention to Detail** | thoroughness, accuracy, quality-focused, meticulous | Caught errors, QA'd own work, polished output |
| E05 | **Resourcefulness** | scrappy, figures it out, entrepreneurial, self-starter | Solved with constraints, found workarounds, didn't need hand-holding |
| E06 | **Follow-Through** | reliability, completes tasks, dependable, sees things through | Closed loops, didn't drop balls, status-updated proactively |
| E07 | **Process Improvement** | efficiency, optimization, streamlines, continuous improvement | Identified waste, automated manual work, documented for scale |

### Cluster 2: LEADERSHIP
*Influencing, directing, and developing others.*

| ID | Canonical Name | Synonyms | Assessment Indicators |
|----|----------------|----------|----------------------|
| L01 | **People Management** | manages team, direct reports, supervisor, people leader | Hired/fired decisions, performance reviews, career development |
| L02 | **Influence Without Authority** | cross-functional leadership, leads through influence, matrix leadership | Got buy-in without formal power, persuaded peers, built coalitions |
| L03 | **Vision Setting** | strategic direction, inspires, paints the picture | Articulated north star, motivated with purpose, created alignment |
| L04 | **Delegation** | empowers team, trusts others, appropriate handoff | Assigned stretch work, didn't micromanage, scaled through others |
| L05 | **Talent Development** | coaching, mentoring, grows people | Promoted reports, gave feedback, invested in growth |
| L06 | **Decision Making** | decisive, good judgment, makes the call | Made hard calls with incomplete info, owned consequences |
| L07 | **Accountability** | ownership, takes responsibility, doesn't blame | Said "I" not "we" for failures, course-corrected, learned publicly |

## Synonym Resolution Rules

When parsing JDs or interview questions, map to canonical competencies using these rules:

### Rule 1: Exact Match
If the JD/question uses the canonical name or a listed synonym, map directly.

### Rule 2: Phrase Patterns
Common phrases map to specific competencies:

| Phrase Pattern | Maps To |
|----------------|---------|
| "drive [X] initiatives" | L02 (Influence Without Authority) |
| "own [X] end-to-end" | E06 (Follow-Through), L07 (Accountability) |
| "partner with [teams]" | B02 (Stakeholder Management) |
| "build and lead team" | L01 (People Management) |
| "data-driven decisions" | T01 (Analytical Thinking) |
| "fast-paced environment" | E03 (Execution Speed), R03 (Resilience) |
| "ambiguous problems" | T06 (Judgment Under Uncertainty) |
| "executive stakeholders" | C03 (Presentation Skills), B02 (Stakeholder Management) |
| "roll up sleeves" | E05 (Resourcefulness) |
| "wear many hats" | E05 (Resourcefulness), T05 (Learning Agility) |

### Rule 3: Context Clues
Role level affects interpretation:

| Signal | Junior (IC1-2) | Mid (IC3-4) | Senior (IC5+/Mgr) |
|--------|----------------|-------------|-------------------|
| "leadership" | L02 (Influence) | L02 (Influence) | L01 (People Mgmt) |
| "strategic" | T02 (light) | T02 (medium) | T02 + L03 (Vision) |
| "ownership" | E06 (Follow-Through) | L07 (Accountability) | L07 + L06 (Decisions) |

### Rule 4: Cluster Affinity
If uncertain, assign to the most likely cluster based on role type:

| Role Type | Primary Clusters |
|-----------|------------------|
| Engineering | Execution, Thinking, Domain |
| Product | Thinking, Collaboration, Communication |
| Sales | Communication, Collaboration, Trajectory |
| Operations | Execution, Thinking, Character |
| Design | Thinking, Communication, Collaboration |
| Leadership | Leadership, Communication, Thinking |

## How the Ontology Gets Used

### Stage 2 (JD Analysis)
```
Input: "5+ years leading cross-functional product initiatives"
       ↓
Output: [L02, B02, E06] with weights [0.8, 0.7, 0.5]
```

### Stage 3 (Transcript Analysis)
```
Input: "Tell me about a time you had to get buy-in from skeptical stakeholders"
       ↓
Inferred competencies being tested: [L02, B02, C06]

Input: Candidate's answer
       ↓
Demonstrated competencies: [L02 ✓, B02 ✓, C06 ✗]
```

### Stage 4 (Gap Analysis)
```
JD requires: [L02, B02, D03, E03]
Interview tested: [L02, B02, T01]
Candidate demonstrated: [L02, B02]

Gaps:
- D03, E03: Required but not tested (interviewer gap)
- T01: Tested but not required (bonus or mismatch)
- L02, B02: Tested and demonstrated (coverage)
```

---

## Extensibility

This ontology is **not exhaustive**. New competencies can be added by:

1. Assigning to an existing cluster
2. Providing canonical name, synonyms, and indicators
3. Adding to synonym resolution rules

The system should gracefully handle unknown competencies by flagging them for human review.
