# Drop D1.3 - Post-Tick Hook and Writer

## Goal
Wire promotion execution as a post-`tick` stage with idempotent writes.

## Outputs
- Hook from meeting pipeline into promotion engine.
- Single writer path for semantic memory + graph.
- Idempotency key and retry/dead-letter behavior.

## Acceptance
1. Re-running the same meeting does not duplicate writes.
2. Write failures are recoverable and observable.
3. Manifest records promotion state transitions.
