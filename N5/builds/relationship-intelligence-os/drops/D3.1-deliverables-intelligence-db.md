# Drop D3.1 - Deliverables Intelligence DB

## Goal
Create a client-scoped deliverables database to track requests/promises and route reuse vs adapt vs net-new.

## Outputs
- SQLite schema for clients, deliverables, artifacts, similarity links.
- Extractor for deliverable promises from meetings/conversations.
- Similarity router with threshold-based recommendation.

## Acceptance
1. Deliverables can be linked to client/person/org context.
2. Similarity routing reduces duplicate deliverable creation.
3. Deliverable lifecycle states are enforceable.
