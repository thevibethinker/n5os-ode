# D8: FounderMaxxing Hotline — Mini Zo Hotline

## Objective

Build a dedicated version of the Zo Hotline specifically for FounderMaxxing members. This hotline:
- Knows each member by name
- Is contextually aware of the FounderMaxxing curriculum
- Can review what members have built and give advice
- Offers prompting guidance and implementation patterns
- Makes members feel like they have on-call AI co-building support

## Context

V already has a working Zo Hotline (Vibe Thinker Hotline). This is a lighter, scoped version for FounderMaxxing members specifically.

### Reference Architecture
- Check existing hotline: `file 'Skills/zo-hotline/SKILL.md'` (if exists)
- Check webhook service: look for `zo-hotline-webhook` in user services
- Study the existing implementation pattern

## Requirements

### Member Experience
- Call or text a dedicated number
- Hotline knows their name and what tier they're on
- Has context about recent sessions (what was built, what patterns were taught)
- Can review repos/implementations members share
- Gives prompting advice specific to their stack
- Feels personal, not generic

### Technical Implementation
- Dedicated phone number (via existing telephony provider)
- Webhook receiver that routes to Zo
- Member database: name, tier, phone, session history
- System prompt loaded with:
  - FounderMaxxing curriculum context
  - Member profile
  - Recent session topics/repos
  - V's teaching philosophy and meta OS concepts

### Integration with Intake (D6)
- When a member is approved and pays, they get added to the hotline member database
- Their profile is seeded with application form data (goals, current AI usage)

### Scope Control
- This is V1/MVP — don't over-engineer
- Start with: member recognition + curriculum context + basic co-building support
- Iterate based on actual member usage
