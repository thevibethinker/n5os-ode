# D1: Financial Model + Unit Economics

## Objective

Build a comprehensive financial model for FounderMaxxing. Create both a markdown summary AND a Google Sheet.

## Key Inputs

### Pricing Tiers (independent products)
- **Founding 15:** $100/month, capped at 15 members, locked forever
- **Standard:** $300/month, uncapped (but group sessions capped at 8 per group)
- **Zo-to-Zo:** $150/month, standalone AI mentorship, no live sessions

### Operational Constraints
- Group sessions: max 8 founders per group
- V's time: ~1.5 hrs per group session (1hr session + 30 min prep/followup)
- Sessions: weekly (4/month per group)
- Zo-to-Zo: near-zero marginal time (AI-delivered)

### Cost Side
- Zo subscription costs (for running Zoputer infrastructure)
- $100 Zo credits per new Founding/Standard member
- Stripe fees: 2.9% + $0.30 per transaction
- V's time opportunity cost: estimate $250-500/hr

## Deliverables

### 1. Three-tier revenue projections (independent)
For each tier, model: 5, 10, 15, 20, 30, 50 members

| Tier | 5 | 10 | 15 | 20 | 30 | 50 |
|------|---|----|----|----|----|-----|
| Founding 15 | | | (max) | n/a | n/a | n/a |
| Standard | | | | | | |
| Zo-to-Zo | | | | | | |

### 2. Combined scenario modeling
- Conservative: 15 Founding + 10 Standard + 5 Zo-to-Zo
- Moderate: 15 Founding + 25 Standard + 15 Zo-to-Zo
- Ambitious: 15 Founding + 50 Standard + 30 Zo-to-Zo

### 3. Time analysis
- Groups needed at each member count
- V's hours/month at each level
- Effective hourly rate at each level
- Capacity ceiling (when does V max out?)

### 4. Google Sheet
Create a Google Sheet in V's Drive with:
- Input assumptions tab (editable)
- Revenue projections tab
- Time analysis tab
- Charts: MRR growth, time allocation, effective rate
