# D6: Intake Form + Waitlist System

## Objective

Research, select, and implement the intake/waitlist mechanism for FounderMaxxing. The system needs to:
1. Capture applicant information (filter for quality)
2. Manage a waitlist (first 15 Founding spots → then Standard queue)
3. Provide V visibility into who's applied
4. Optionally gate access to payment links (apply first, then get invited to pay)

## Context

V said: "I don't want some scrubs signing up that I have to then refund." The first 15 Founding members will be distributed quietly through V's network. After that, there needs to be a filter.

### Application Filter Criteria (from V)
- Loyalty and engagement
- Intellect / genuine passion
- These people will form the nucleus of the community

## Research Phase

### Option 1: Fillout.com
- V mentioned Fillout — research their waitlist/application capabilities
- Can Fillout handle conditional logic (auto-accept vs waitlist)?
- Webhook support for notifications?
- Pricing?

### Option 2: Native Build (zo.space)
- Build directly on va.zo.space
- API route for submissions → stored in workspace + notification to V
- Admin dashboard route for V to review applications
- Accept/reject flow that sends payment link

### Option 3: Dedicated Waitlist Products
Research API-based waitlist products that offer utility beyond tracking names:
- LaunchList, Waitlist.me, Viral Loops, GetWaitlist
- Key criteria: API access, customizable fields, automation/webhooks, cheap
- What do they offer that a native build doesn't?

## Evaluation Criteria
- Cost (free/cheap preferred for MVP)
- Customization (needs to match FounderMaxxing brand)
- V's visibility (dashboard or notifications)
- Automation (auto-send payment link on approval?)
- Integration with Stripe payment links
- Data ownership (V should own the applicant data)

## Implementation

Based on research, implement the chosen solution:

**If native build:**
- Upgrade `/api/foundermaxxing-apply` to handle:
  - Store to workspace JSON + Airtable (if connected)
  - Email notification to V per application
  - Application status tracking (pending/approved/rejected/waitlisted)
- Build admin route at `/foundermaxxing-admin` (private) for V to:
  - See all applications
  - Approve/reject with one click
  - Approved → auto-sends payment link email

**If Fillout:**
- Set up the form on Fillout
- Configure webhooks to notify V
- Document how V manages the queue

**If third-party waitlist:**
- Set up account, configure fields
- Integrate API with zo.space if needed
- Document the flow
