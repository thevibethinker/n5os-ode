# D1.3: Pre-Task Staging Table

## Objective
Create a staging area for potential tasks captured from meetings, conversations, and other sources — before they're promoted to the official task registry.

## Context
V's workflow:
1. Meeting happens → Zo extracts potential action items
2. Items go to **staging** (not directly to tasks)
3. End of day: V reviews staged items with checkboxes
4. Confirmed items get **promoted** to task registry with proper metadata

This prevents task bloat and ensures V consciously commits to each task.

## Design

### Staged Task Fields
- id
- title (extracted)
- description (context)
- source_type: meeting, conversation, email, manual
- source_id: meeting_id, convo_id, etc.
- source_context: relevant quote or context
- suggested_domain, suggested_project (AI guesses)
- suggested_priority
- captured_at
- status: pending_review, promoted, dismissed
- promoted_task_id (if promoted, links to real task)

### Operations
1. **Capture**: Add item to staging from any source
2. **Review**: Generate markdown review file with checkboxes
3. **Promote**: Move staged item to task registry with full metadata
4. **Dismiss**: Mark as not-a-task, keep for audit trail

### Review File Format
```markdown
## Staged Tasks for Review - 2026-01-25

### From Meetings
- [ ] Follow up with Sarah about partnership (Meeting: Logan 1:1)
  - Suggested: Careerspan > Partnerships > Priority
  - [Promote] [Dismiss] [Edit]

### From Conversations  
- [ ] Draft the investor update memo (Convo: con_xyz)
  - Suggested: Careerspan > Fundraising > Strategic
  - [Promote] [Dismiss] [Edit]
```
