# D2.2: Morning Briefing Agent

## Objective
Create a scheduled agent that texts V every morning with the day's tasks and asks for commitment.

## Context
V's accountability system needs:
- Morning text with today's tasks (pulled from registry)
- Priority order: Strategic > External > Urgent > Age (LIFO within tier)
- Ask V to commit to the day's plan
- Capacity-aware: Don't overload (start with 5-7 tasks hypothesis)

## Design

### Morning Flow (8:00 AM ET)

1. **Pull today's tasks** from registry:
   - Due today or overdue
   - Ordered by priority buckets
   - Limited to capacity hypothesis (start: 6)

2. **Check calendar** for time blocks and meetings

3. **Generate briefing**:
   ```
   Morning, V. Here's your day:
   
   🔴 STRATEGIC
   1. Investor memo (due today)
   
   🟠 EXTERNAL  
   2. Reply to Sarah (2 days overdue)
   3. Logan follow-up
   
   🟡 URGENT
   4. Invoice review
   
   That's 4 tasks. You have 3 meetings (2.5 hrs blocked).
   Realistic capacity: 4-5 tasks.
   
   Reply with your plan or adjustments.
   ```

4. **Wait for response**:
   - V can reply with adjustments ("push #2 to tomorrow")
   - V can add tasks ("also need to do X")
   - V confirms or Zo assumes confirmed after 30 min

5. **Lock the day**: After confirmation, this becomes the accountability baseline

### Agent Configuration

```
RRULE: FREQ=DAILY;BYHOUR=8;BYMINUTE=0
Delivery: sms

Instruction:
[Agent prompt that executes the morning flow]
```
