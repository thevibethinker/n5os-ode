# D2.1: Thread Close Hooks for Task System

## Objective
Integrate the task system with the thread-close skill so that when an action conversation closes, the task registry is automatically updated.

## Context
V wants:
- When closing an action conversation, Zo checks what was accomplished
- Task status updates automatically (complete, partial, blocked)
- If task has a delivery step (e.g., "send email"), verify it happened
- Update latency tracking

## Dependencies
- D1.1: Task registry schema (need `update_task`, `complete_task`)
- D1.2: Action tagging (need `get_task_for_conversation`, `close_action_conversation`)

## Design

### On Thread Close (Action Conversation)

1. **Detect**: Is this an action conversation? (check D1.2's tagging)

2. **Assess**: What was accomplished?
   - Read conversation content
   - Compare against task's plan_json milestones
   - Check for delivery indicators (email sent, file created, etc.)

3. **Update**: 
   - If complete: `complete_task(task_id)`, record actual_minutes
   - If partial: Update plan_json with progress, set next milestone
   - If blocked: Mark blocked with reason

4. **Prompt V**: Before closing, show:
   ```
   Task: Draft investor memo
   Status: ✅ Memo drafted, ❌ Not sent yet
   
   Mark as: [Complete] [Partial - continue tomorrow] [Blocked]
   ```

5. **Latency**: Record completion timestamp for latency calculation

### Integration with thread-close Skill

Read the existing skill: `/home/workspace/Skills/thread-close/SKILL.md`

Add a hook that:
- Checks if conversation is action-tagged
- If yes, runs task assessment before normal close flow
- Updates task registry
- Includes task status in close summary
