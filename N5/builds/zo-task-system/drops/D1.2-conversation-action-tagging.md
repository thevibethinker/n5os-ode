# D1.2: Conversation Action Tagging System

## Objective
Create a system to tag conversations as "action conversations" — conversations where work is being done on a specific task.

## Context
V wants Zo to track when conversations are tied to tasks. An "action conversation" is one where:
- V is actively working on completing a task
- The conversation should link to the task registry
- When the conversation closes, task status should update

This is an **orthogonal characteristic** — any conversation can be tagged as action-related regardless of other attributes.

## Design

### Tagging Mechanism (Hybrid)
1. **Inference**: Detect action-like patterns in first message:
   - "Write the memo for X"
   - "Help me draft the email to Y"
   - "Let's work on the proposal"
   - Keywords: write, draft, build, create, send, complete, finish
   
2. **Confirmation**: After inference, confirm with V:
   - "This looks like an action conversation for [task]. Confirm? (y/task search/no)"
   
3. **Manual**: V can explicitly say "This is an action conversation for [task]"

### Storage
Create a lightweight tracking table or JSON file:
- conversation_id
- task_id (from task registry)
- tagged_at
- tag_method: inferred, confirmed, manual
- status: active, closed

### Integration Points
- On conversation start: check for action patterns, prompt for confirmation
- During conversation: can re-tag or link to different task
- On conversation close: trigger task status update (D2.1 will implement the close hook)
