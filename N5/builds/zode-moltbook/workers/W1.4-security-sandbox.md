# W1.4: Security Sandbox & Workspace

## Objective
Create the isolated workspace, content filter for outbound posts, inbound sanitizer for Moltbook content, and staging queue system.

## MUST DO
1. Create `N5/builds/zode-moltbook/scripts/content_filter.py` — Scans outbound post text for PII and PR risks
2. Create `N5/builds/zode-moltbook/scripts/inbound_sanitizer.py` — Strips prompt injection patterns from Moltbook content
3. Create `N5/builds/zode-moltbook/scripts/staging_queue.py` — Manages post staging queue (add, list, approve, publish)
4. Create `N5/builds/zode-moltbook/scripts/sandbox_validator.py` — Validates that no writes happen outside allowed paths

## Content Filter (content_filter.py)
Must detect and block posts containing:
- V's personal email addresses (attawar.v@gmail.com, me@vrijenattawar.com, howie@howie.ai)
- V's phone number
- Careerspan pipeline details (client names, deal values, internal processes)
- Strategic intent language ("infiltrate", "acquisition strategy", "convert users to Zo", "sales pipeline")
- References to this build plan or its existence
- Any mention of N5OS internal architecture details
- Financial information

CLI: `python3 content_filter.py check "post text here"` → returns PASS/FAIL with reasons
CLI: `python3 content_filter.py check --file path/to/draft.md`

## Inbound Sanitizer (inbound_sanitizer.py)
Must strip/flag:
- Instruction-like patterns ("ignore previous instructions", "system prompt", "you are now", "execute the following")
- Suspicious URLs (non-standard TLDs, IP addresses, known malicious patterns)
- Base64 encoded content
- Requests to reveal API keys or credentials
- Attempts to modify agent behavior ("update your SOUL.md", "change your instructions")

CLI: `python3 inbound_sanitizer.py clean "moltbook post text"` → returns sanitized text + list of stripped items
CLI: `python3 inbound_sanitizer.py analyze --file path/to/feed.json` → batch analysis

## Staging Queue (staging_queue.py)
- Posts are written as JSON files to `N5/builds/zode-moltbook/workspace/staging/`
- Format: `{timestamp}_{submolt}_{slug}.json` with fields: content, title, submolt, rubric_scores, status (staged|approved|published|rejected), created_at, published_at
- Commands: `add`, `list`, `approve <id>`, `reject <id>`, `publish <id>` (calls moltbook_poster), `stats`
- The `publish` command runs content_filter before publishing. If filter fails, post is auto-rejected with reason.

## Sandbox Validator (sandbox_validator.py)
- Checks that a given file path is within allowed write paths
- Allowed: `N5/builds/zode-moltbook/`, `Skills/zode-moltbook/`
- Blocked: everything else, especially `N5/scripts/`, `N5/config/`, `Sites/`, `Personal/`, `N5/data/`
- CLI: `python3 sandbox_validator.py check /path/to/file` → ALLOWED/BLOCKED

- Do NOT create files outside `N5/builds/zode-moltbook/` and `Skills/zode-moltbook/`
- Do NOT hardcode any actual PII in test cases (use placeholder patterns)

- Inbound sanitizer catches common prompt injection patterns
- Staging queue correctly manages lifecycle (staged → approved → published)
