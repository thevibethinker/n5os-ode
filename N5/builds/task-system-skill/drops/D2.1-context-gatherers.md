# D2.1: Context Gatherers (Replace Regex with LLM Context)

## Objective
Create `context.py` — the key script that gathers context for LLM reasoning. This REPLACES the regex-based inference in action_tagger.py, close_hooks.py, and b05_parser.py.

## Context
**The core insight:** Instead of regex patterns trying to infer:
- "Is this an action conversation?"
- "Did the task complete?"
- "What's the next step?"

We gather context and return it for the AI (Zo) to reason about semantically.

## Tasks

1. **Create `scripts/context.py`** with these commands:

### `context.py action-check --convo-id <id>`
Returns context for AI to determine if conversation is action-oriented:
```json
{
  "convo_id": "con_xxx",
  "session_state": {
    "focus": "Draft investor memo for Series A",
    "type": "planning",
    "artifacts_created": ["memo-draft.md"]
  },
  "first_message_preview": "Let's work on the investor memo...",
  "existing_tasks": [
    {"id": 5, "title": "Draft investor memo", "status": "pending"}
  ]
}
```
AI then reasons: "This conversation is working on task #5."

### `context.py completion-check --convo-id <id> --task-id <id>`
Returns context for AI to assess task completion:
```json
{
  "task": {
    "id": 5,
    "title": "Draft investor memo",
    "description": "Write memo for potential investors",
    "milestones": ["Draft content", "Review with team", "Send"]
  },
  "conversation_summary": "Drafted a 2-page memo covering...",
  "artifacts_created": ["investor-memo-v1.md"],
  "delivery_indicators": {
    "files_created": ["investor-memo-v1.md"],
    "emails_sent": false,
    "explicit_completion_statement": false
  }
}
```
AI then reasons: "Draft completed, but not yet sent. Status: partial."

### `context.py next-step --task-id <id>`
Returns context for AI to suggest next step:
```json
{
  "task": {
    "id": 5,
    "title": "Draft investor memo",
    "status": "in_progress",
    "completed_milestones": ["Draft content"],
    "remaining_milestones": ["Review with team", "Send"]
  },
  "recent_activity": "Drafted memo on Jan 25",
  "related_tasks": [],
  "time_since_last_activity": "2 hours"
}
```
AI then reasons: "Next step is to review with team."

### `context.py extract-tasks --file <path>`
Returns context for AI to extract tasks from meeting B05 block:
```json
{
  "source_file": "/path/to/B05_ACTION_ITEMS.md",
  "source_type": "meeting",
  "meeting_id": "2025-01-25_Logan_Call",
  "raw_content": "[full B05 content]",
  "attendees": ["V", "Logan Currie"],
  "existing_tasks_mentioned": []
}
```
AI then extracts: structured task objects with title, first_step, priority, etc.

2. **Helper functions** (internal, not CLI):
   - `load_session_state(convo_id)` — read SESSION_STATE.md
   - `get_conversation_summary(convo_id)` — summarize conversation content
   - `scan_for_deliveries(convo_id)` — find files created, messages sent
   - `get_recent_task_activity(task_id)` — task_events history

3. **NO INFERENCE** — These functions gather and format context. The AI does all reasoning.
