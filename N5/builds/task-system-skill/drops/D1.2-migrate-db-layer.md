# D1.2: Migrate Database Layer

## Objective
Extract and consolidate the database operations from scattered `N5/task_system/` files into clean, focused modules in the Skill.

## Context
Current state:
- `N5/task_system/task_registry.py` — Task CRUD + domains/projects
- `N5/task_system/staging.py` — Staging table operations
- `N5/task_system/action_conversations.db` — Separate DB for conversation tagging

We want:
- Single `Skills/task-system/data/tasks.db` with all tables
- Clean Python module with ONLY database operations (no regex, no inference)
- Separate files for task ops, staging ops, conversation tracking ops

## Tasks

1. **Create unified schema** that combines:
   - tasks, domains, projects, task_events (from task_registry)
   - staged_tasks (from staging)
   - action_conversations (from action_tagger)

2. **Create `scripts/db.py`** — Internal module (not CLI), pure DB operations:
   ```python
   # Task operations
   def create_task(title, domain_id, priority_bucket, ...) -> int
   def get_task(task_id) -> dict
   def update_task(task_id, **fields) -> bool
   def complete_task(task_id, actual_minutes=None) -> bool
   def list_tasks(status=None, domain=None, due_date=None) -> list
   
   # Domain/Project operations
   def get_or_create_domain(name) -> int
   def get_or_create_project(name, domain_id, project_type) -> int
   
   # Staging operations  
   def stage_task(source_type, source_id, raw_text, ...) -> int
   def get_staged_tasks(status='pending_review') -> list
   def promote_staged_task(staged_id, task_data) -> int
   def dismiss_staged_task(staged_id, reason) -> bool
   
   # Action conversation operations
   def tag_action_conversation(convo_id, task_id, method) -> bool
   def get_task_for_conversation(convo_id) -> int|None
   def close_action_conversation(convo_id) -> bool
   
   # Analytics
   def calculate_latency_stats(task_id=None) -> dict
   def get_completion_rate(days=7) -> float
   ```

3. **Migrate existing data:**
   - Copy `N5/task_system/tasks.db` to `Skills/task-system/data/tasks.db`
   - Merge `action_conversations` table into main DB
   - Merge `staging` table into main DB

4. **NO INFERENCE LOGIC** — This module is pure CRUD. No:
   - Pattern matching
   - Priority inference
   - Completion assessment
   - Next step calculation

## Key Principle
**Database layer = dumb storage.** All intelligence comes from LLM reasoning.

## Output
- `Skills/task-system/scripts/db.py` with all DB operations
- `Skills/task-system/data/tasks.db` with unified schema
- Old N5/task_system/ DB files can be deprecated
