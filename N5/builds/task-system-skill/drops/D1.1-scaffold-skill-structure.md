# D1.1: Scaffold Skill Structure

## Objective
Create the canonical Skill directory structure for task-system at `Skills/task-system/`.

## Context
We built the task system scattered in `N5/task_system/`. This needs to become a proper Skill following the Agent Skills spec. This drop creates the empty structure; other drops will fill it.

## Tasks

1. **Create directory structure:**
```
Skills/task-system/
├── SKILL.md              # Will be filled by D2.2
├── scripts/
│   ├── __init__.py
│   ├── task.py           # CLI entry point (stub)
│   ├── briefing.py       # CLI entry point (stub)
│   ├── stage.py          # CLI entry point (stub)
│   └── context.py        # CLI entry point (stub)

│   ├── reasoning.md      # How AI should reason about tasks
│   └── integration.md    # How this integrates with thread-close, meetings
└── data/
    └── .gitkeep          # DB will live here
```

2. **Create stub SKILL.md** with correct frontmatter:
```yaml
---
name: task-system
description: |
  ADHD-optimized task management system for Zo. Handles task registry,
  morning/evening briefings, action conversation tracking, and staged task review.
  Uses LLM reasoning for semantic understanding, Python for data operations.
compatibility: Created for Zo Computer
metadata:
  author: va.zo.computer
---
```

3. **Create stub scripts** with argparse structure but no implementation:

   - `context.py`: action-check, completion-check, next-step

4. **Document schema** in `references/schema.md`:

   - Document each table's purpose
   - Document relationships
