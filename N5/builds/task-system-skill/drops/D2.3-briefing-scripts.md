# D2.3: Morning & Evening Briefing Scripts

## Objective
Create `briefing.py` — CLI for generating morning and evening briefings. These are appropriate for Python because they're mostly data queries + formatting.

## Context
The existing `morning_briefing.py` and `evening_accountability.py` in `N5/task_system/` are mostly good — they query the DB and format output. We're keeping this pattern but:
1. Moving to Skills structure
2. Using the new unified DB from D1.2
3. Cleaning up any regex that snuck in

## Tasks

1. **Create `scripts/briefing.py`** with these commands:

### `briefing.py morning [--capacity N] [--format text|json]`
Generates morning briefing:
```
Morning, V. Here's your day (Monday, 1/27)

CAPACITY: 6 tasks (adjust? reply 'cap 4' or 'cap 8')

🔴 STRATEGIC
1. Draft thought leadership post on AI coaching

🟠 EXTERNAL  
2. Reply to Sarah re: partnership
3. Follow up with Logan on intro

🟡 URGENT
4. Submit proposal by EOD

⚪ NORMAL
5. Review meeting notes
6. Organize inbox

---
Adjust? Examples:
  • 'drop 5' — remove task
  • 'swap 1 and 3' — reorder
  • 'defer 4 to tomorrow'
  • 'confirm' — lock the day
```

Output includes:
- Today's tasks by priority bucket
- Capacity indicator
- Adjustment instructions

### `briefing.py evening [--format text|json]`
Generates evening accountability:
```
Evening check-in, V.

TODAY'S SCORE: 4/6 (67%)

✅ COMPLETED
• Reply to Sarah re: partnership (15 min)
• Submit proposal (45 min)
• Review meeting notes (20 min)
• Organize inbox (10 min)

⬜ INCOMPLETE
• Draft thought leadership post — Started, not finished
• Follow up with Logan — Not started

LATENCY CHECK:
• "Follow up with Logan" — 3 days since committed
  → Becoming external pressure

---
What happened with the incomplete tasks?
```

### `briefing.py staged [--format text|json]`
Shows staged tasks for review:
```
STAGED TASKS FOR REVIEW (3 items)

From: 2025-01-25_Logan_Call meeting
1. [PROMOTE?] "Send Logan the Careerspan deck"
   First step: Find deck in Drive, attach to email
   Priority: external | Est: 15 min
   
2. [PROMOTE?] "Research competitor pricing"
   First step: Google top 3 competitors
   Priority: normal | Est: 60 min

From: Email thread with Sarah
3. [PROMOTE?] "Schedule follow-up call with Sarah"
   First step: Check calendar for availability
   Priority: external | Est: 10 min

---
Reply: 'promote 1', 'dismiss 2 reason', 'promote all'
```

2. **Helper functions:**
   - `get_todays_tasks(capacity)` — Query with priority ordering
   - `get_day_results()` — Completed vs incomplete
   - `calculate_latency_warnings()` — Tasks getting stale
   - `format_task_line(task)` — Consistent formatting

3. **NO AI CALLS** — This is pure data + formatting. The scheduled agent that runs this can add voice/personality when texting.

## Key Difference from Old Code
- Uses new `db.py` from D1.2
- No regex inference for "what next" (that's in context.py for AI)
- Cleaner separation of concerns
