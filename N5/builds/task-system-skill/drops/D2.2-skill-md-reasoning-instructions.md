### Action Conversation Detection
**When to check:** At conversation start (after SESSION_STATE init)

**How to reason:**
1. Run `context.py action-check --convo-id <id>`
2. Look at the returned context
3. Ask yourself:
   - Does the focus/objective describe doing specific work?
   - Is there an existing task that matches?
   - Did V explicitly say "this is an action conversation"?
4. If YES: Tag with `task.py tag-conversation --convo-id <id> --task-id <id>`
5. If UNSURE: Ask V "Are you working on [task title]?"

**Signals of action conversation:**
- Focus mentions deliverables: "draft", "write", "send", "build", "fix"
- Matches an existing pending task
- V says "let's knock out" or "need to get this done"

**NOT action conversations:**
- Research/exploration
- Planning/strategy discussions
- General questions
- System maintenance

### Task Completion Assessment
**When to check:** At conversation close (thread-close integration)

**How to reason:**
1. Run `context.py completion-check --convo-id <id> --task-id <id>`
2. Compare task milestones against conversation outcomes
3. Look for delivery evidence:
   - Files created that match task intent
   - Emails sent (check artifacts)
   - Explicit "done" statements
4. Assess status:
   - **complete**: All milestones done OR clear delivery evidence
   - **partial**: Some progress, more work needed
   - **blocked**: Cannot proceed, external dependency

**Show V the assessment and ask for confirmation before updating.**

### Next Step Inference
**When to use:** Morning briefing, after partial completion, "what next?" queries

**How to reason:**
1. Run `context.py next-step --task-id <id>`
2. If milestones defined: Next milestone is next step
3. If no milestones: Reason about task type:
   - Email task → "Review draft, then send"
   - Memo task → "Polish draft, get feedback, finalize"
   - Build task → "Test current state, identify gaps"
4. Consider context:
   - How long since last activity?
   - Are there blockers?
   - What's V's current energy/capacity?

### Task Extraction from Meetings
**When to use:** Processing B05_ACTION_ITEMS blocks

**How to reason:**
1. Run `context.py extract-tasks --file <B05 path>`
2. For each action item, extract:
   - **title**: Clear, actionable task name
   - **first_step**: The immediate next action (2-5 min to start)
   - **priority_bucket**: strategic|external|urgent|normal
   - **domain**: Infer from meeting context
   - **owner**: V or someone else?
   - **estimated_minutes**: Reasonable estimate
3. Only extract tasks for V (not tasks for other attendees)
4. Stage tasks for review, don't auto-create

### Priority Buckets
| Bucket | When to use | Examples |
|--------|-------------|----------|
| strategic | Long-term value, V's goals | "Write thought leadership post" |
| external | Someone else waiting | "Reply to investor email" |
| urgent | Time-sensitive | "Submit proposal by Friday" |
| normal | Everything else | "Organize notes" |

### Integration Points

- **evening accountability**: Uses completion data from DB
