# D2.2: SMS Follow-Up Integration (Capstone)

## Context

You are wiring together the complete SMS follow-up system for the Vibe Thinker Hotline. After a substantive call, Zoseph offers to text the caller a follow-up. If they agree (or automatically at end-of-call for 2+ min calls), the system:

1. Generates personalized page content using Anthropic API
2. Creates a zo.space page at `/hotline/<slug>` using `/zo/ask`
3. Sends SMS to the caller's phone with the page link using `/zo/ask`
4. Logs everything to DuckDB

## MUST DO

### 1. Add sendFollowUp Tool to VAPI Assistant Config

In the `assistant-request` response, add to the `tools` array:

```typescript
{
  type: "function",
  function: {
    name: "sendFollowUp",
    description: "Send the caller a text message with a link to their personalized follow-up page. Call this when the caller agrees to receive a text follow-up, or offer it proactively at the end of a substantive conversation.",
    parameters: {
      type: "object",
      properties: {
        confirmed: {
          type: "boolean",
          description: "Whether the caller explicitly said yes to receiving a text follow-up"
        }
      },
      required: ["confirmed"]
    }
  }
}
```

### 2. Implement sendFollowUp Tool Handler

In the `tool-calls` handler, add a case for `sendFollowUp`:

```typescript
case "sendFollowUp": result = await sendFollowUp(params, callId); break;
```

The `sendFollowUp` function should:

1. **Get caller phone number** from the active call context (it's in the VAPI payload at `data.message.call.customer.number`)
2. **Store confirmation** in the `collectedEmails` Map (repurpose as `collectedFollowups` or use a new Map) — the actual page generation + SMS happens at end-of-call
3. **Return to Zoseph:** `{ success: true, message: "I'll send that text as soon as we hang up." }`

The actual heavy lifting (page generation + SMS) happens in the **end-of-call-report** handler, not during the call. This keeps the tool response fast.

### 3. End-of-Call Follow-Up Flow

In the `end-of-call-report` handler, after the existing caller profile update and tool usage persist:

**Trigger conditions** (either):
- `sendFollowUp` was called during the call (explicit consent), OR
- Call duration >= 120 seconds AND call was not a drop-off (ended_reason != "silence-timed-out" with duration < 60)

**Flow:**

```
1. Generate page content (Anthropic API — same pattern as email generation)
   - Input: summary, pathway, level, primaryInterest, callerName, transcript excerpt
   - Output: structured content for the page (summary, prompts, nextSteps)

2. Generate page component source
   - Use D2.1's template source as base
   - Replace sample data with real call data
   - Generate URL-safe slug

3. Create zo.space page via /zo/ask
   - POST to https://api.zo.computer/zo/ask
   - Auth: process.env.ZO_CLIENT_IDENTITY_TOKEN
   - Input: "Create a public zo.space page route at /hotline/<slug> with route_type 'page' and this code: <component_source>"
   - Extract the URL from the response

4. Send SMS via /zo/ask
   - POST to https://api.zo.computer/zo/ask
   - Input: "Send an SMS to <phone_number> with this message: 'Hey <name>, here's your Vibe Thinker Hotline follow-up: https://va.zo.space/hotline/<slug> — it's got everything we talked about plus a prompt you can paste right into Zo. Call back anytime. —Zoseph'"

5. Log to DuckDB
   - UPDATE calls SET followup_sent = TRUE, followup_url = '<url>', followup_sent_at = CURRENT_TIMESTAMP WHERE id = '<callId>'

6. Update caller profile
   - Store last_recommendations and last_next_steps (same as email flow)
```

### 4. Update System Prompt — Replace Email Collection with SMS Follow-Up

In `zoseph-system-prompt.md`, find the **Email Collection** section and replace it with:

```markdown
## Follow-Up

After a substantive conversation (2+ minutes), before winding down, offer once:
"Before you go — want me to text you a follow-up? It'll have everything we talked about, plus a ready-to-paste prompt you can try on Zo. I'll send it to this number."

If they say yes:
1. Call the sendFollowUp tool with confirmed=true
2. Confirm: "Done — you'll get that text in about a minute after we hang up."

If they decline, move on immediately. One ask only.

The follow-up is a personalized page built by Zo — which is itself a demo of what Zo can do. If they ask about it, lean into that: "The page you're about to get was built by Zo in real-time, based on our conversation. That's the kind of thing you can build too."
```
