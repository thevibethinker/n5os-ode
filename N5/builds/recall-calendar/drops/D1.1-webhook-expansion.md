# D1.1: Webhook Event Expansion

## Objective
Expand the webhook receiver to handle ALL Recall.ai bot events, not just recording completion.

## Context
- Existing webhook: `file 'Integrations/recall_ai/webhook_receiver.py'`
- Config: `file 'Integrations/recall_ai/config.py'`
- Webhook URL: https://recall-webhook-va.zocomputer.io/webhook/recall
- Signing secret available: `RECALL_WEBHOOK_SIGNING_SECRET`

## Events to Handle

**Bot Lifecycle:**
- `joining_call` → Log, optionally notify
- `in_waiting_room` → Log with timestamp
- `in_call_not_recording` → Warning state
- `in_call_recording` → Active recording started
- `call_ended` → Trigger deposit pipeline
- `fatal` → Alert V via SMS
- `done` → Final state, cleanup

**Recording States:**
- `recording_permission_allowed` → Log compliance
- `recording_permission_denied` → Alert + log
- `media_expired` → Archive warning

**Participant Events (if available):**
- Track join/leave for participant timeline

## Deliverables
1. Update `webhook_receiver.py` with handlers for all events
2. Add event-specific logging to DB schema
3. Add SMS alerting for fatal/permission_denied states
4. Implement signature verification using `RECALL_WEBHOOK_SIGNING_SECRET`

## Technical Notes
- Recall uses Svix for webhook delivery
- 15-second timeout on webhook responses - respond 200 immediately, process async
- Retries for 24 hours with backoff

## Success Criteria
- [ ] All bot status events logged to DB
- [ ] Signature verification working
- [ ] SMS alerts for fatal errors
- [ ] Async processing (no timeout issues)

- `Integrations/recall_ai/config.py` (if needed)
