# D1.1: Webhook Enhancement

## Objective

Enhance the existing Recall.ai webhook receiver to handle ALL bot lifecycle events (not just `bot.done`), add proper Svix signature verification, comprehensive event logging, and SMS alerting on failures.

## Context

- Existing webhook receiver: `file 'Integrations/recall_ai/webhook_receiver.py'`
- Config: `file 'Integrations/recall_ai/config.py'`
- Webhook URL: `https://recall-webhook-va.zocomputer.io/webhook/recall`
- Signing secret available in env: `RECALL_WEBHOOK_SIGING_SECRET` (note typo)

## Requirements

### 1. Svix Signature Verification
Recall uses Svix for webhooks. Implement proper signature verification:
```python
# Headers from Recall webhook:
# svix-id: msg_xxx
# svix-timestamp: 1234567890
# svix-signature: v1,base64signature

# Verification:
# 1. Build signed content: "{svix-id}.{svix-timestamp}.{body}"
# 2. HMAC-SHA256 with secret (base64 decoded after "whsec_" prefix)
# 3. Compare with signature
```

### 2. Handle All Bot Events
Add handlers for each event type:

| Event | Action |
|-------|--------|
| `bot.status_change` | Log to SQLite, update bot state |
| `bot.joining_call` | Log, track "joining" state |
| `bot.in_waiting_room` | Log, track "waiting_room" state |
| `bot.in_call_not_recording` | Log, may indicate permission issue |
| `bot.recording_permission_allowed` | Log success |
| `bot.recording_permission_denied` | Log, SMS alert |
| `bot.in_call_recording` | Log, track "recording" state |
| `bot.call_ended` | Log, track "call_ended" state |
| `bot.done` | Trigger deposit (existing), mark "done" |
| `bot.fatal` | Log error, SMS alert to V |

### 3. Event Logging Schema
Create/update SQLite table for event history:
```sql
CREATE TABLE IF NOT EXISTS bot_events (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    event_id TEXT UNIQUE,  -- svix-id for deduplication
    bot_id TEXT NOT NULL,
    event_type TEXT NOT NULL,
    sub_code TEXT,  -- bot sub-status codes
    timestamp DATETIME DEFAULT CURRENT_TIMESTAMP,
    payload JSON,
    processed BOOLEAN DEFAULT FALSE,
    error TEXT
);

CREATE INDEX idx_bot_events_bot_id ON bot_events(bot_id);
CREATE INDEX idx_bot_events_type ON bot_events(event_type);
```

### 4. SMS Alerting
On `bot.fatal` or `bot.recording_permission_denied`:
- Send SMS to V: "[RECALL] Bot {bot_id} failed: {reason}"
- Include meeting URL if available

### 5. Idempotency
- Use `svix-id` header to deduplicate events
- Skip processing if event_id already exists with processed=TRUE
