# Worker Brief: W2.1 - CRM Script Updates

## Objective

Update core CRM scripts to use the unified n5_core.db database and the `people` table instead of `individuals` or `profiles`.

## Scope

### Files to MODIFY
- `N5/scripts/crm_cli.py` — Main CRM CLI
- `N5/scripts/crm_lookup.py` — Fast lookup service
- `N5/scripts/crm_query_helper.py` — Query utilities
- `N5/scripts/crm_enrichment_worker.py` — Enrichment pipeline

### Files to READ (for reference)
- `N5/scripts/db_paths.py` — Import DB connection from here
- `N5/data/n5_core.db` — Verify queries work

### Must NOT Touch
- Deal scripts (W2.2)
- Meeting scripts (W2.3)
- Migration scripts (W1.1)

## MUST DO

1. **Update `crm_cli.py`:**
   - Change imports: `from db_paths import get_db_connection, N5_CORE_DB`
   - Change table references: `individuals` → `people`
   - Update column names if different (check schema)
   - Test all CLI commands work

2. **Update `crm_lookup.py`:**
   - Use `get_db_connection()` instead of direct sqlite3.connect
   - Query `people` table instead of `profiles` or `individuals`
   - Ensure `lookup_by_email()`, `lookup_by_name()` work correctly

3. **Update `crm_query_helper.py`:**
   - Update all SQL queries to use `people` table
   - Update JOIN queries to use new schema
   - Add helper for `deal_roles` junction queries

4. **Update `crm_enrichment_worker.py`:**
   - Read from `people` table
   - Write enrichment results back to `people` table
   - Update status tracking

## Table Mapping

| Old (crm.db) | Old (crm_v3.db) | New (n5_core.db) |
|--------------|-----------------|------------------|
| individuals | profiles | people |
| organizations | organizations | organizations |
| interactions | — | interactions |
| relationships | — | relationships |

## Key Query Changes

```sql
-- OLD (crm.db)
SELECT * FROM individuals WHERE email = ?

-- NEW (n5_core.db)
SELECT * FROM people WHERE email = ?

-- OLD: Get person's deals
-- (not possible - no link)

-- NEW: Get person's deals
SELECT d.* FROM deals d
JOIN deal_roles dr ON d.id = dr.deal_id
WHERE dr.person_id = ?
```
