# D0.1: Validator Wiring Fix

## Objective

Wire `pulse_code_validator.py` and `pulse_llm_filter.py` into Pulse's tick loop so deposits are actually validated before being marked complete.

**This is the critical gap from the prompt-to-skill build. Ship this FIRST.**

## Context

The validators exist but aren't called:
- `Skills/pulse/scripts/pulse_code_validator.py` — detects stubs, TODOs, empty functions
- `Skills/pulse/scripts/pulse_llm_filter.py` — LLM-based semantic validation

Currently, `pulse.py` tick loop auto-passes all deposits. We need to insert validation.

## Deliverables

1. **Update `Skills/pulse/scripts/pulse.py`** — Wire validators into tick loop
2. **Test script** — Verify validation rejects bad deposits

## Implementation

### 1. Find the deposit check in tick loop

In `pulse.py`, locate where deposits are checked (likely in `tick()` function). Look for:
```python
# Something like:
if deposit_exists and not filtered:
    mark_deposit_filtered(...)
```

### 2. Insert validation before marking filtered

```python
if deposit_exists and not filtered:
    # === NEW: Run mechanical validation ===
    try:
        code_result = subprocess.run([
            "python3", "Skills/pulse/scripts/pulse_code_validator.py",
            "check", slug, drop_id
        ], capture_output=True, text=True, timeout=60)
        
        if code_result.returncode == 0:
            code_data = json.loads(code_result.stdout)
            if code_data.get("critical_count", 0) > 0:
                mark_deposit_failed(slug, drop_id, "code_validation", code_data)
                # Log lesson
                subprocess.run([
                    "python3", "N5/scripts/lessons.py", "add",
                    "--category", "stub_code",
                    "--severity", "critical", 
                    "--summary", f"Drop {drop_id} failed code validation",
                    "--build-slug", slug,
                    "--drop-id", drop_id
                ])
                continue
    except Exception as e:
        print(f"[WARN] Code validator failed: {e}, proceeding anyway")
    
    # === NEW: Run LLM validation ===
    try:
        llm_result = subprocess.run([
            "python3", "Skills/pulse/scripts/pulse_llm_filter.py",
            "validate", slug, drop_id
        ], capture_output=True, text=True, timeout=120)
        
        if llm_result.returncode == 0:
            llm_data = json.loads(llm_result.stdout)
            if not llm_data.get("pass", False):
                mark_deposit_failed(slug, drop_id, "llm_validation", llm_data)
                subprocess.run([
                    "python3", "N5/scripts/lessons.py", "add",
                    "--category", "llm_rejection",
                    "--severity", "high",
                    "--summary", f"Drop {drop_id} failed LLM filter: {llm_data.get('reason', 'unknown')}",
                    "--build-slug", slug,
                    "--drop-id", drop_id
                ])
                continue
    except Exception as e:
        print(f"[WARN] LLM filter failed: {e}, proceeding anyway")
    
    # Mark as filtered (passed both)
    mark_deposit_filtered(slug, drop_id)
```

### 3. Add `mark_deposit_failed` function if missing

```python
def mark_deposit_failed(slug: str, drop_id: str, reason: str, details: dict):
    """Mark a deposit as failed validation."""
    build_dir = Path(f"N5/builds/{slug}")
    deposit_path = build_dir / "deposits" / f"{drop_id}.json"
    
    if deposit_path.exists():
        deposit = json.loads(deposit_path.read_text())
        deposit["validation_status"] = "failed"
        deposit["validation_reason"] = reason
        deposit["validation_details"] = details
        deposit_path.write_text(json.dumps(deposit, indent=2))
    
    # Update meta.json
    meta_path = build_dir / "meta.json"
    meta = json.loads(meta_path.read_text())
    meta["drops"][drop_id]["status"] = "failed"
    meta["drops"][drop_id]["failure_reason"] = reason
    meta_path.write_text(json.dumps(meta, indent=2))
    
    print(f"[FAIL] {drop_id} failed {reason}")
```
