# Drop Brief: Validate Pulse System

**Mission:** Independently validate that the Pulse build orchestration system works correctly.

**Output:** 
- Validation report at `N5/builds/pulse-validation/artifacts/validation-report.md`
- Deposit at `N5/builds/pulse-validation/deposits/D1.1.json`

---

## Context

The Pulse system was just built. It consists of:

1. **Core Skill** at `Skills/pulse/`
   - `scripts/pulse.py` — Main orchestrator (start, stop, tick, status, resume)
   - `scripts/sentinel.py` — Lightweight scheduled monitor
   - `references/` — Drop brief template, filter criteria, escalation protocol

2. **Interview Skill** at `Skills/pulse-interview/`
   - `SKILL.md` — Structured pre-build interview
   - `references/interview-questions.md` — Question bank
   - `references/decomposition-patterns.md` — Common patterns

3. **Control File** at `N5/config/pulse_control.json`
   - States: active, paused, stopped

4. **Terminology:**
   - Build = complete orchestrated work
   - Stream = parallel wave (Wave 1, Wave 2...)
   - Drop = individual worker (synonym: Worker)
   - Current = sequential chain within a Stream
   - Deposit = completion report from a Drop
