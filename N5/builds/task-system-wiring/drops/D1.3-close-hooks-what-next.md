# D1.3: Enhance Close Hooks with "What Next" Capability

## Objective
Upgrade `close_hooks.py` to recognize partial task completion and surface the next step, either as a prompt or by auto-creating the follow-up task.

## Context
Current close_hooks behavior:
- Assesses what was done in conversation
- Marks task as complete/partial/blocked
- That's it

New behavior:
- Assesses what was done
- If partial: identifies completed milestones AND remaining milestones
- **Surfaces "what next"** — the immediate next step
- Optionally creates follow-up task for remaining work

## Example Flow

**Task**: "Draft and send investor memo"
**Milestones**: 
1. Draft memo content
2. Review and polish
3. Send to investors

**Conversation work**: Drafted the memo, saved to file

**Close hooks assessment**:
```
Status: PARTIAL (1/3 milestones)

Completed:
  ✅ Draft memo content

Remaining:
  ⬜ Review and polish  
  ⬜ Send to investors

WHAT NEXT: Review and polish the memo
  → Create follow-up task? [yes/no]
  → Or mark original as "in_progress" with next step noted
```

## Task Patterns for Inference

When no milestones are defined, use these patterns:

| Task Pattern | Likely Next Steps |
|--------------|-------------------|
| "Draft X" | Review X → Send X |
| "Write X" | Edit X → Publish/Send X |
| "Research X" | Summarize findings → Share/Present |
| "Meet with X" | Send follow-up → Complete action items |
| "Review X" | Provide feedback → Finalize |
| "Plan X" | Execute first step |
| "Email X" | Await response → Follow up if needed |
