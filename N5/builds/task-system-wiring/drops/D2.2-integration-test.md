# D2.2: Integration Test - End to End Flow

## Objective
Validate the complete task system flow works end-to-end:
1. Meeting → B05 decomposition → Staging
2. Conversation tagging → Task tracking → Close hooks → What next

## Test Scenarios

### Test 1: Meeting to Staging Flow

**Setup:**
1. Find or create a meeting folder in [R] state
2. Ensure it has a B05_ACTION_ITEMS.md file

**Test:**
```bash
# Run the meeting staging bridge
python3 /home/workspace/N5/task_system/meeting_staging_bridge.py --meeting <folder>

# Verify items staged
sqlite3 /home/workspace/N5/task_system/staging.db \
  "SELECT * FROM pre_task_staging WHERE source_type='meeting' ORDER BY created_at DESC LIMIT 5;"
```

**Expected:**
- Action items appear in staging with decomposition metadata
- Each item has: title, priority_bucket, domain_name, first_step, milestones

### Test 2: Action Conversation Tagging

**Setup:**
1. Create a test task in the registry

```python

task_id = create_task(
    title="Test: Draft project proposal",
    priority_bucket="normal",
    domain_id=1,  # Use existing domain
    estimated_minutes=30,
    plan_json='{"milestones": ["Draft outline", "Write content", "Review", "Send"]}'

```

**Test:**
```python
from N5.task_system.action_tagger import tag_conversation

    conversation_id="con_TEST123",
    task_id=task_id,
    tag_method="explicit"
)
print(result)
```

**Expected:**
- Conversation tagged in action_conversations.db
- Task status updated to "in_progress"
