# D1.1: Upgrade B05 Action Items to Decompose at Source

## Objective
Modify the B05 Action Items meeting block generation to decompose tasks at extraction time, so staged items arrive ready for the task registry without duplicate analysis.

## Context
Currently:
1. Meeting ingestion extracts action items as raw text
2. Staging system receives raw items
3. Decomposition would happen later (wasteful)

New flow:
1. Meeting ingestion extracts AND decomposes action items in one LLM pass
2. Each action item comes out with:
   - First actionable step (2-minute rule)
   - Suggested domain/project
   - Priority bucket inference
   - Time estimate
   - Milestone structure if multi-step

## Current B05 Format
Check: `/home/workspace/N5/config/canonical_blocks.yaml` for B05 spec
Check: `/home/workspace/N5/builds/meeting-intel-gen-vrijen-miami/B05_ACTION_ITEMS.md` for example

Current format is simple checklist:
```markdown
- [ ] **V**: Follow up with Sarah about partnership
- [ ] **Investor**: Consider the proposal...
```

## New B05 Format
Each action item should include decomposition metadata:

```markdown
## Action Items

### 1. Follow up with Sarah about partnership
- **Owner**: V
- **Priority**: external (commitment to external party)
- **First Step**: Open email to Sarah, write 2-sentence check-in
- **Estimated**: 5 min
- **Domain**: Careerspan
- **Project**: Partnerships (infer or null)
- **Milestones**: 
  1. Draft email
  2. Send email
  3. Await response

### 2. Review investor memo
- **Owner**: V
- **Priority**: strategic
- **First Step**: Open memo draft, read first section
- **Estimated**: 15 min
- **Domain**: Careerspan
- **Project**: Fundraising
- **Milestones**:
  1. Read memo
  2. Add comments
  3. Send feedback
```
