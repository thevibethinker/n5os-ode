## Context

Wave 3 of rice-capabilities. Zo2Zo enables communication between Zo Computer instances (parent↔child). The parent can teach skills, delegate work, and escalate decisions. Child instances can report back and ask for help.

This builds on the existing va↔zoputer teaching protocol but makes it generic, configurable infrastructure. Trust is verified through a registry (loaded from security.yaml). The parent link protocol provides structured escalation.

For Layer 1, this creates the framework. Actual /zo/ask API calls happen when wired by rice-integration.

## Scenarios

S1: Trust registry validates trusted instance
  Given: security.yaml has trusted_instances: ["va.zo.computer"]
  When: `registry.verify_trust("va.zo.computer")` is called
  Then: Returns `{"trusted": true, "level": "full", "instance": "va.zo.computer"}`
  Verify: `python3 -c "from Zoffice.capabilities.zo2zo.trust.registry import verify_trust; r = verify_trust('va.zo.computer'); assert r['trusted'] == True; print('PASS')"`

S2: Trust registry rejects unknown instance
  Given: security.yaml trusted_instances does not include "evil.zo.computer"
  When: `registry.verify_trust("evil.zo.computer")` is called
  Then: Returns `{"trusted": false, "level": "untrusted", "instance": "evil.zo.computer"}`
  Verify: `python3 -c "from Zoffice.capabilities.zo2zo.trust.registry import verify_trust; r = verify_trust('evil.zo.computer'); assert r['trusted'] == False; print('PASS')"`

S3: Parent link packages escalation context
  Given: A decision that needs parent escalation with summary and options
  When: `parent_link.prepare_escalation(summary="Should we accept this client?", context={"revenue": "50k"}, options=["accept", "decline"])` is called
  Then: Returns an EscalationRequest dict with formatted message, decision_id, and parent_instance from config
  Verify: `python3 -c "from Zoffice.capabilities.zo2zo.protocols.parent_link import prepare_escalation; r = prepare_escalation(summary='Test decision', context={'key': 'val'}, options=['a', 'b']); assert 'message' in r; assert 'parent_instance' in r or r.get('parent_instance') is None; print('PASS')"`

S4: Skill receiver validates bundle format
  Given: A skill bundle dict with name, files (dict of path→content), and metadata
  When: `skill_receiver.validate_bundle({"name": "test-skill", "files": {"SKILL.md": "# Test"}, "version": "1.0"})` is called
  Then: Returns `{"valid": true, "issues": []}`
  Verify: `python3 -c "from Zoffice.capabilities.zo2zo.protocols.skill_receiver import validate_bundle; r = validate_bundle({'name': 'test-skill', 'files': {'SKILL.md': '# Test'}, 'version': '1.0'}); assert r['valid'] == True; print('PASS')"`

## Requirements

### trust/registry.py
- `verify_trust(instance_id, security_config_path=None) -> TrustResult` — checks against security.yaml
- `list_trusted() -> list[str]` — returns all trusted instances
- `add_trusted(instance_id) -> bool` — would add to trust list (returns False, logs warning — modifying security.yaml requires human)
- TrustResult: trusted (bool), level (full|limited|untrusted), instance (str)

### protocols/parent_link.py
- `prepare_escalation(summary, context=None, options=None, recommendation=None) -> EscalationRequest`
- EscalationRequest: message (formatted str), decision_id (uuid), parent_instance (from config, can be None), context, options
- `parse_response(response_text) -> dict` — extracts resolution from parent response
- Reads parent instance from zo2zo/config.yaml
- If no parent configured, returns EscalationRequest with parent_instance=None (escalation falls through to human)

### protocols/skill_receiver.py
- `validate_bundle(bundle) -> ValidationResult` — checks bundle has required fields (name, files, version)
- `install_bundle(bundle, skills_dir="Skills/") -> InstallResult` — writes files to target dir (dry-run by default)
- InstallResult: installed (bool), path (str), dry_run (bool), issues (list)
- Required bundle fields: name (str), files (dict of relative_path→content), version (str)
- Optional: description, metadata

### config.yaml
- `parent_instance`: parent Zo instance handle (null for root offices)
- `escalation_timeout_seconds`: how long to wait for parent response (default: 300)
- `skill_install_dir`: where to install received skills (default: Skills/)
- `auto_install_skills`: false (require human approval for skill installs in Layer 1)
