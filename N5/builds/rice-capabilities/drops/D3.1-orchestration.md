## Context

Wave 3 of rice-capabilities. Orchestration is the "daily schedule and workflow clipboard." It manages what the office does on a recurring basis and provides a framework for multi-step workflows.

Dispatchers are YAML-defined scheduled routines. The `dispatcher.py` reads these definitions and can generate Zo scheduled agent configurations from them. For Layer 1, the dispatcher framework exists but actual Zo agents are created by rice-integration.

The workflow engine is a skeleton state machine ‚Äî it provides the pattern for intake‚Üíclassify‚Üíroute‚Üíprocess‚Üírespond‚Üíclose flows. Not fully wired in Layer 1.

  Given: morning.yaml defines 3 tasks (check_pending_decisions, review_overnight_inbound, generate_daily_briefing)

S2: Dispatcher generates Zo agent config
  Given: A loaded dispatcher definition
  When: `dispatcher.to_agent_config(dispatcher_def)` is called
  Then: Returns a dict with rrule, instruction (text), and delivery_method suitable for Zo create_agent
  Verify: `python3 -c "from Zoffice.capabilities.orchestration.scheduler.dispatcher import load_dispatcher, to_agent_config; d = load_dispatcher('morning'); cfg = to_agent_config(d); assert 'rrule' in cfg; assert 'instruction' in cfg; print('PASS')"`

S3: Workflow engine advances through states
  Given: A workflow with states: [intake, classify, route, process, respond, close]
  When: `workflow.advance("intake", "item_received")` is called
  Then: Returns "classify" (the next state)
  Verify: `python3 -c "from Zoffice.capabilities.orchestration.workflows.base import Workflow; w = Workflow.default(); assert w.advance('intake', 'item_received') == 'classify'; print('PASS')"`

## Requirements

### dispatcher.py
- `load_dispatcher(name, scheduler_dir=None) -> DispatcherDef` ‚Äî loads YAML from scheduler/ dir
- `list_dispatchers(scheduler_dir=None) -> list[str]` ‚Äî lists available YAML files
- `to_agent_config(dispatcher_def) -> dict` ‚Äî converts to Zo agent format (rrule + instruction)
- `validate_dispatcher(dispatcher_def) -> list[str]` ‚Äî returns list of issues (empty = valid)
- DispatcherDef: name, schedule (cron), tasks (list of {name, description, employee}), timezone

### Dispatcher YAML format
```yaml
name: morning-dispatcher
schedule: "0 8 * * *"
timezone: "America/New_York"
employee: chief-of-staff
tasks:
  - name: check_pending_decisions
    description: "Review and action pending items in the decision queue"
  - name: review_overnight_inbound
    description: "Check ingestion queue for items received overnight"
  - name: generate_daily_briefing
    description: "Summarize today's schedule, pending items, and priorities"
```

### workflows/base.py
- Class `Workflow` with: states (list), transitions (dict), advance(current_state, event) -> next_state
- `Workflow.default()` ‚Äî creates the standard intake‚Üíclassify‚Üíroute‚Üíprocess‚Üírespond‚Üíclose workflow
- Transitions defined as dict: `{"intake": {"item_received": "classify"}, "classify": {"classified": "route"}, ...}`
- `get_state_info(state) -> dict` ‚Äî returns description and valid events for a state

### config.yaml
- `scheduler_dir`: path to scheduler YAML files
- `default_timezone`: office timezone
- `workflow_defaults`: default states and transitions
