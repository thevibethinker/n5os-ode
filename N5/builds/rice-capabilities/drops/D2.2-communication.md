# Drop Brief: Communication — Dispatch & Approval

**Mission:** Build the Communication capability — the outbound system that dispatches messages through the right channel, enforces autonomy-based approval gates, provides markdown templates, and rate-limits to prevent duplicates.

## Context

Wave 2 of rice-capabilities. Communication is the outbound counterpart to Ingestion (D2.1). When an employee wants to send a message, it flows: employee request → approval gate (checks autonomy.yaml) → template rendering → channel dispatch → audit log.

Communication depends on Security (D1.1) for audit logging and Memory (D1.2) for conversation logging. It reads `Zoffice/config/autonomy.yaml` for decision thresholds.

This is Layer 1 — dispatch doesn't actually call Zo send_sms/send_email tools. It prepares the message and returns a dispatch decision (send/queue/escalate). The actual sending is wired in by rice-integration.

## Scenarios

S1: Approval gate allows auto_act for high confidence
  Given: autonomy.yaml thresholds (auto_act: 0.9) and action="respond_to_inquiry" (not in always_escalate)
  When: `approval.check_approval(action="respond_to_inquiry", confidence=0.95, employee="receptionist")` is called
  Then: Returns `{"decision": "auto_act", "reason": "Confidence 0.95 >= auto_act threshold 0.9"}`
  Verify: `python3 -c "from Zoffice.capabilities.communication.channels.approval import check_approval; r = check_approval(action='respond_to_inquiry', confidence=0.95, employee='receptionist'); assert r['decision'] == 'auto_act'; print('PASS')"`

S2: Approval gate escalates for always_escalate actions
  Given: autonomy.yaml has "send_email" in always_escalate list
  When: `approval.check_approval(action="send_email", confidence=0.99, employee="receptionist")` is called
  Then: Returns `{"decision": "escalate_to_human", "reason": "Action 'send_email' is in always_escalate list"}` regardless of confidence
  Verify: `python3 -c "from Zoffice.capabilities.communication.channels.approval import check_approval; r = check_approval(action='send_email', confidence=0.99, employee='receptionist'); assert r['decision'] == 'escalate_to_human'; print('PASS')"`

S3: Template rendering with variables
  Given: Template `acknowledgment.md` with `{{name}}` and `{{topic}}` placeholders
  When: `dispatch.render_template("acknowledgment", {"name": "Jane", "topic": "your inquiry"})` is called
  Then: Returns rendered markdown with "Jane" and "your inquiry" substituted
  Verify: `python3 -c "from Zoffice.capabilities.communication.channels.dispatch import render_template; r = render_template('acknowledgment', {'name': 'Jane', 'topic': 'your inquiry'}); assert 'Jane' in r; assert 'your inquiry' in r; print('PASS')"`

S4: Rate limiter blocks duplicate message
  Given: A message was sent to "jane@example.com" with content hash X within the last 5 minutes
  When: `rate_limiter.check_rate_limit(recipient="jane@example.com", content_hash=X)` is called
  Then: Returns `{"allowed": false, "reason": "Duplicate message within rate limit window"}`
  Verify: `python3 -c "from Zoffice.capabilities.communication.channels.rate_limiter import check_rate_limit, record_send; import hashlib; h = hashlib.sha256(b'test message').hexdigest(); record_send('jane@example.com', h); r = check_rate_limit('jane@example.com', h); assert r['allowed'] == False; print('PASS')"`

S5: Dispatch returns structured send request (not actually sending)

  When: `dispatch.prepare_send(recipient="jane@example.com", content="Hello Jane", channel="email", employee="receptionist")` is called
  Then: Returns a SendRequest dict with: recipient, content, channel, employee, template_used (if any), approval_decision, ready_to_send (bool)
  Verify: `python3 -c "from Zoffice.capabilities.communication.channels.dispatch import prepare_send; r = prepare_send(recipient='jane@example.com', content='Hello', channel='email', employee='receptionist'); assert 'recipient' in r; assert 'channel' in r; assert 'ready_to_send' in r; print('PASS')"`

## Requirements

### dispatch.py
- `prepare_send(recipient, content, channel, employee, confidence=0.9, template=None, template_vars=None) -> SendRequest`
- SendRequest is a dict: recipient, content, channel, employee, template_used, approval_decision, ready_to_send, rate_limit_result, audit_id
- Flow: check approval → render template (if provided) → check rate limit → audit log → return
- `render_template(template_name, variables) -> str` — loads from templates/ dir, substitutes `{{var}}`
- `list_templates() -> list[str]` — lists available templates
- Does NOT actually send anything — returns the prepared request for rice-integration to wire

### approval.py
- `check_approval(action, confidence, employee, autonomy_config_path=None) -> ApprovalResult`
- ApprovalResult dict: decision (auto_act|act_and_notify|escalate_to_parent|escalate_to_human), reason (str)
- Reads autonomy.yaml for thresholds and always_escalate/never_escalate lists
- always_escalate → always returns escalate_to_human regardless of confidence
- never_escalate → always returns auto_act regardless of confidence
- Otherwise: compare confidence against thresholds

### rate_limiter.py
- `check_rate_limit(recipient, content_hash, window_seconds=300) -> dict`
- Returns: allowed (bool), reason (str)
- `record_send(recipient, content_hash) -> None` — records a send event
- In-memory storage (dict) for v1 — not persistent across restarts
- Deduplication: same recipient + same content_hash within window = blocked

### Templates (markdown files)
- `base.md`: documents the template format ({{variable}} substitution)
- `acknowledgment.md`: "We received your message, {{name}}. Regarding {{topic}}, we'll get back to you shortly."
- `escalation-notice.md`: "I'm connecting you with {{employee_or_human}} who can better assist with {{topic}}."
- `follow-up.md`: "Following up on our conversation about {{topic}}. {{content}}"

### config.yaml
- `default_channel`: email
- `rate_limit_window_seconds`: 300
- `template_dir`: path to templates
- `channels`: list of available channels with preference order
