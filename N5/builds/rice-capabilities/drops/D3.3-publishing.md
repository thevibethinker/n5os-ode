## Scenarios

S1: Content pipeline enforces review gate
  Given: A content item in "draft" state
  When: `content_pipeline.advance(item_id, "submit_for_review")` is called, then `content_pipeline.advance(item_id, "publish")` is called without approval
  Then: The second call returns an error — item must go through "approved" state before "published"
  Verify: `python3 -c "from Zoffice.capabilities.publishing.pipelines.content_pipeline import ContentPipeline; p = ContentPipeline(); iid = p.create_item('Test content', 'test-page'); p.advance(iid, 'submit_for_review'); try: p.advance(iid, 'publish'); assert False, 'Should have failed'; except ValueError: print('PASS')"`

S2: Content pipeline full lifecycle
  Given: A new content item
  When: create → submit_for_review → approve → publish
  Then: Item reaches "published" state
  Verify: `python3 -c "from Zoffice.capabilities.publishing.pipelines.content_pipeline import ContentPipeline; p = ContentPipeline(); iid = p.create_item('Test', 'page'); p.advance(iid, 'submit_for_review'); p.advance(iid, 'approve'); p.advance(iid, 'publish'); assert p.get_item(iid)['state'] == 'published'; print('PASS')"`

S3: zo.space helper generates page config
  Given: A page title and content
  When: `zo_space.prepare_page(path="/about", title="About Us", content="<h1>About</h1>", public=True)` is called
  Then: Returns a dict with path, route_type="page", code (TSX string), and public=True
  Verify: `python3 -c "from Zoffice.capabilities.publishing.pipelines.zo_space import prepare_page; r = prepare_page(path='/about', title='About Us', content='About content', public=True); assert r['path'] == '/about'; assert r['route_type'] == 'page'; print('PASS')"`

## Requirements

### content_pipeline.py
- Class `ContentPipeline` with in-memory state tracking
- States: draft, review, approved, published, archived
- `create_item(content, slug, author=None) -> str` — returns item ID
- `advance(item_id, event) -> str` — returns new state or raises ValueError
- `get_item(item_id) -> dict` — returns item with current state and history
- `list_items(state=None) -> list[dict]`
- Events: submit_for_review, approve, reject, publish, archive
- Review gate: draft→published requires going through review→approved
- Rejection: review→draft (back to drafting)
- Audit logging for every state transition

### zo_space.py
- `prepare_page(path, title, content, public=False, style="default") -> dict` — generates page config
- Returns dict with: path, route_type, code (TSX component string), public
- `prepare_api_route(path, handler_description) -> dict` — generates API route config
- Simple TSX templates — not complex React apps

### config.yaml
- `review_required`: true (default) — whether review gate is enforced
- `auto_publish_threshold`: 0.95 — confidence above which auto-publish is allowed (if review_required is false)
- `default_visibility`: "private"
