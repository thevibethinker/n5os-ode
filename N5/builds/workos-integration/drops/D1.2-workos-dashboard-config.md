# Drop Brief: WorkOS dashboard config (prod redirect + invite-only)

**Mission:** Configure WorkOS AuthKit in the dashboard for Where’s V with production redirect URI and invite-only access.

**Why manual:** Requires logged-in browser interaction and picking the correct production public URL.

**Output (artifacts):**
- Deposit only: `N5/builds/workos-integration/deposits/D1.2.json` describing what you configured.

---

## Context

V’s requirements:
- Use **WorkOS production** environment
- Redirect URI should be the **production Zo public domain** + `/callback`
- Where’s V must be private: the whole site requires login
- Users cannot self-register: only invited / pre-authorized users

WorkOS UI hints (from screenshots):
- Redirect URIs (Developer → Redirects)
- AuthKit Sign-up feature toggle
- Users → Invitations (invite user modal)

---

## Tasks

1) Determine the production public URL for Where’s V once published on Zo.
   - This must be the URL users will access publicly.

2) In WorkOS dashboard:
   - Set **Redirect URI** to: `<PRODUCTION_PUBLIC_URL>/callback`
   - Set **App homepage URL** to: `<PRODUCTION_PUBLIC_URL>/`
   - Set **Sign-in endpoint** to: `<PRODUCTION_PUBLIC_URL>/login`
   - Set **Logout redirect** to: `<PRODUCTION_PUBLIC_URL>/login`

3) Configure AuthKit to be invite-only:
   - Disable self sign-up (or enable “invite-only” flow per WorkOS docs)

4) Seed initial users:
   - Invite the first set of allowed emails (V will provide)
