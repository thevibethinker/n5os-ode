## Requirements

### 1. New Test Types

Add these test types to `pulse_integration_test.py`:

#### `broadcast_injection`
Verifies Forward Broadcast works:
```python
{
    "type": "broadcast_injection",
    "name": "Broadcasts injected into briefs",
    "config": {
        "deposit_with_broadcast": {
            "drop_id": "TEST1",
            "status": "complete",
            "broadcast": "Test broadcast message"
        },
        "expect_in_brief": "## Broadcasts from Prior Drops"
    }
}
```

Test logic:
1. Create a mock deposit with `broadcast` field
2. Call `collect_broadcasts()` — verify it finds the broadcast
3. Call `inject_broadcasts()` on a sample brief — verify section appears

#### `first_wins`
Verifies Hypothesis Racing works:
```python
{
    "type": "first_wins",
    "name": "First-wins supersedes other drops",
    "config": {
        "confirmed_drop": "D1.2",
        "expect_superseded": ["D1.1", "D1.3"]
    }
}
```

Test logic:
1. Create mock meta with `first_wins: true`
2. Create deposit for D1.2 with `verdict: "confirmed"`
3. Call `check_first_wins()` — verify D1.1 and D1.3 are marked `superseded`

#### `task_pool_claim`
Verifies Dynamic Task Claiming works:
```python
{
    "type": "task_pool_claim",
    "name": "Atomic task claiming",
    "config": {
        "pool_size": 5,
        "expected_claim": true
    }
}
```

Test logic:
1. Create mock meta with `task_pool.tasks`
2. Call `claim_task()` — verify a task is claimed
3. Call `claim_task()` again — verify different task claimed
4. Verify claimed tasks have `status: "claimed"` and `claimed_by` set
