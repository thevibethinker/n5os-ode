# Drop Brief: Hypothesis Racing Pattern

**Mission:** Add `first_wins` mode for builds where multiple Drops test competing hypotheses — when one confirms, others are superseded.

**Output:** Modified `pulse.py` with first-wins logic in tick handler.

---

## Context

This is part of a Pulse self-upgrade build. The hypothesis racing pattern is useful for debugging scenarios where you want to test multiple theories in parallel and stop as soon as one is confirmed.

**Use case example:**
- D1.1: "Theory: Rate limit is server-side"
- D1.2: "Theory: Rate limit is client-side timeout"  
- D1.3: "Theory: Rate limit is DNS cache"

When D1.2 deposits with `verdict: "confirmed"`, D1.1 and D1.3 are marked `superseded` and skipped.

## Requirements

### 1. Meta.json Schema Extension

Add optional `first_wins` field at build level:

```json
{
  "slug": "debug-rate-limit",
  "build_type": "hypothesis_race",
  "first_wins": true,
  "hypothesis_group": ["D1.1", "D1.2", "D1.3"],
  ...
}
```

- `first_wins: true` — Enable hypothesis racing
- `hypothesis_group` — Optional list of Drop IDs in the race (defaults to all Drops in active Wave if not specified)

### 2. Deposit Verdict Field

Racing Drops should include `verdict` in their deposit:

```json
{
  "drop_id": "D1.2",
  "status": "complete",
  "verdict": "confirmed",
  "summary": "Rate limit is client-side timeout — reproduced by increasing timeout to 60s"
}
```

Valid verdicts:
- `"confirmed"` — Hypothesis validated, stop the race
- `"rejected"` — Hypothesis disproven, continue racing
- `"inconclusive"` — Couldn't determine, continue racing

### 3. Tick Logic for First-Wins

In `tick()`, after processing deposits:

```python
def check_first_wins(slug: str, meta: dict) -> bool:
    """Check if a hypothesis was confirmed and supersede others."""
    if not meta.get("first_wins"):
        return False
    
    hypothesis_group = meta.get("hypothesis_group") or list(meta.get("drops", {}).keys())
    
    # Find confirmed hypothesis
    winner = None
    for drop_id in hypothesis_group:
        deposit = get_deposit(slug, drop_id)
        if deposit and deposit.get("verdict") == "confirmed":
            winner = drop_id
            break
    
    if not winner:
        return False
    
    # Supersede others
    drops = meta.get("drops", {})
    for drop_id in hypothesis_group:
        if drop_id == winner:
            continue
        if drops[drop_id].get("status") in ("pending", "running"):
            drops[drop_id]["status"] = "superseded"
            drops[drop_id]["superseded_by"] = winner
            print(f"[FIRST_WINS] {drop_id} superseded by {winner}")
    
    save_meta(slug, meta)
    return True
```

### 4. Status Handling

Add `superseded` as a valid Drop status:
- Superseded Drops are NOT spawned (skip in spawn logic)
- Superseded Drops are NOT blocking (Wave can advance)
- Superseded Drops appear in STATUS.md with reason

### 5. STATUS.md Updates

In `update_status_md()`, add section for superseded:

### Superseded (2)
| Drop | Superseded By | Reason |
|------|---------------|--------|
| D1.1 | D1.2 | First-wins: D1.2 confirmed hypothesis |

### 6. Edge Cases

- **All reject:** If all hypotheses are rejected (no confirmed), build continues normally — no winner means no superseding
- **Multiple confirm:** First confirmed wins (process deposits in order received)
- **Race condition:** If two confirm "simultaneously", the one processed first wins

## Success Criteria

- [ ] `first_wins: true` in meta.json enables racing mode
- [ ] Deposits with `verdict: "confirmed"` trigger superseding
- [ ] Superseded Drops are skipped (not spawned)
- [ ] Superseded Drops don't block Wave advancement
- [ ] STATUS.md shows superseded Drops with reason
- [ ] Backward compatible — builds without `first_wins` work unchanged

## Anti-Patterns (Avoid)

- Don't add timeout for races — if all hypotheses are slow, that's fine
- Don't auto-kill running Drops — just mark pending ones as superseded (running ones finish naturally, their deposits are ignored)
- Don't require `hypothesis_group` — default to all Drops in Wave
