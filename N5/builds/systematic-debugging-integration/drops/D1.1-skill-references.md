## MUST DO

1. **Create `references/root-cause-tracing.md`:**
   - Title: Root Cause Tracing
   - Purpose: Trace bugs backward through call stack to find original trigger
   - Content:
     - When to use (error deep in call stack, bad value surfacing)
     - The backward tracing technique step-by-step
     - Example walkthrough with pseudocode
     - Common mistakes (stopping too early, assuming instead of checking)
   - Length: ~200-300 words, focused and practical

2. **Create `references/defense-in-depth.md`:**
   - Title: Defense in Depth
   - Purpose: Add validation at multiple layers after finding root cause
   - Content:
     - Why single-layer fixes are fragile
     - The multi-layer validation pattern
     - Example: input validation + business logic check + database constraint
     - When NOT to use (over-engineering simple cases)
   - Length: ~200-300 words

3. **Create `references/condition-based-waiting.md`:**
   - Title: Condition-Based Waiting
   - Purpose: Replace arbitrary timeouts with condition polling
   - Content:
     - Why hardcoded timeouts fail (too short = flaky, too long = slow)
     - Condition polling pattern (poll for expected state, timeout as fallback)
     - Code examples (bash, Python patterns)
     - Anti-patterns (sleep in loops without backoff)
   - Length: ~200-300 words

4. **Add frontmatter to each file:**
