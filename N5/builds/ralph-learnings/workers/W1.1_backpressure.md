# W1.1: Backpressure Validation Script

## Context

You are implementing a key concept from the "Ralph Wiggum" AI coding technique: **backpressure**. This is automated validation that rejects invalid work before human review.

Backpressure = tests + lints + type checks + custom validators that run automatically and gate progress.

## MECE Declaration

**SCOPE:** Create unified backpressure validation script and config

**MUST DO:**
1. Create `N5/scripts/backpressure.py` with CLI interface
2. Accept build slug OR file/directory path as input
3. Run configurable validators:
   - pytest (if tests exist)
   - ruff or flake8 (Python lint)
   - mypy (if configured)
   - Custom scripts (defined in config)
4. Return structured JSON result:
   ```json
   {
     "status": "PASS|WARN|FAIL",
     "validators": [
       {"name": "pytest", "status": "PASS", "details": "12 passed"},
       {"name": "ruff", "status": "WARN", "details": "3 warnings"}
     ],
     "summary": "Overall assessment"
   }
   ```
5. Exit codes: 0=PASS, 1=WARN, 2=FAIL
6. Support per-build config via `backpressure.yaml` in build folder
7. Create default config at `N5/config/backpressure_rules.yaml`
8. Include `--help` with clear usage
9. Graceful degradation (skip missing validators with warning)

**MUST NOT DO:**
- Modify any files (read-only validation)
- Block execution if optional validators missing
- Touch existing validation.py or mece_validator.py

## Config Schema

```yaml
# N5/config/backpressure_rules.yaml
defaults:
  pytest:
    enabled: true
    args: ["-q", "--tb=short"]
    fail_on: error  # error|warning|never
  ruff:
    enabled: true
    args: ["check", "."]
    fail_on: error
  mypy:
    enabled: false  # opt-in
    args: ["--ignore-missing-imports"]
    fail_on: warning
  custom: []  # list of {name, command, fail_on}
```
