# W1.2: Struggle Detection Script

## Context

From Ralph methodology: detect when an agent is "spinning" — making attempts without progress. This is critical for knowing when to intervene or restart with fresh context.

## MECE Declaration

**SCOPE:** Create struggle detection script and pattern config

**MUST DO:**
1. Create `N5/scripts/struggle_detector.py` with CLI interface
2. Detect these struggle patterns:
   - **Circular errors:** Same error appears 3+ times in DEBUG_LOG.jsonl
   - **Stalled progress:** SESSION_STATE.md shows same progress % for N updates
   - **Revert cycles:** Git log shows apply-revert-apply pattern
   - **Repeated hypotheses:** Same hypothesis tried multiple times
3. Return structured result:
   ```json
   {
     "status": "HEALTHY|STRUGGLING|STUCK",
     "patterns_detected": [
       {"type": "circular_error", "count": 4, "evidence": "..."}
     ],
     "recommendation": "Continue|Intervene|Fresh start"
   }
   ```
4. Configurable thresholds in `N5/config/struggle_patterns.yaml`
5. Graceful degradation if DEBUG_LOG doesn't exist
6. Accept --convo-id OR --build-slug as input
7. Include `--help` with clear usage

**MUST NOT DO:**
- Take corrective action (detection only, no modifications)
- Require DEBUG_LOG to exist (must handle missing gracefully)
- Modify debug_logger.py or session_state_manager.py

## Config Schema

```yaml
# N5/config/struggle_patterns.yaml
thresholds:
  circular_error:
    min_repetitions: 3
    window_minutes: 30
    severity: STRUGGLING
  stalled_progress:
    unchanged_updates: 5
    severity: STUCK
  revert_cycles:
    min_cycles: 2
    severity: STRUGGLING
  repeated_hypothesis:
    min_repetitions: 3
    severity: STRUGGLING

severity_escalation:
  HEALTHY: 0 patterns
  STRUGGLING: 1-2 patterns
  STUCK: 3+ patterns OR any STUCK-severity pattern
```
