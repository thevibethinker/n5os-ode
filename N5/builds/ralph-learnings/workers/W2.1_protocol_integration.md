## Protocol Integration Points

### Orchestrator Protocol Additions

Add after "Worker Completion" section:

```markdown
## Backpressure Gate

After each worker completes:

1. Run: `python3 N5/scripts/backpressure.py <slug>`
2. Interpret results:
   - **PASS**: Proceed to next worker or merge
   - **WARN**: Review warnings, proceed if acceptable
   - **FAIL**: Worker must address issues before proceeding

If FAIL persists after worker retry:
- Check struggle detector
- Consider loop runner for automated retry
- Escalate to V if stuck

## Struggle Detection

Monitor during long-running workers:

1. Run: `python3 N5/scripts/struggle_detector.py --build-slug <slug>`
2. Interpret results:
   - **HEALTHY**: Continue
   - **STRUGGLING**: Increase monitoring, consider intervention
   - **STUCK**: Pause, fresh context restart, or escalate

## Loop Runner (For Stuck Workers)

When a worker is stuck but the task is well-specified:

1. Create focused prompt from worker brief
2. Run: `python3 N5/scripts/loop_runner.py --prompt <file> --build-slug <slug> --execute`
3. Monitor iterations via dashboard
4. Stop when backpressure passes or struggle detected
```

## Lessons from Wave 1

From W1.3: All scripts use `--json` flag for machine-readable output. When integrating:

- Check for `"stub": true` in output — indicates fallback mode
- Dashboard should detect and surface stub usage

From W1.2: Struggle detector reads DEBUG_LOG.jsonl and SESSION_STATE.md formats. The stalled_progress detection uses heuristics, not git history tracking.
