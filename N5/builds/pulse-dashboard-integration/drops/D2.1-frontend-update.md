# Drop Brief: Update React components for Pulse terminology

**Mission:** Update dashboard UI to use Pulse terminology (Drops/Streams instead of Workers/Waves).

**Output:** Modified React components in `Sites/build-tracker/src/`

---

## Context

The Build Tracker dashboard currently uses "Workers" and "Waves" terminology from the old orchestration system. The new Pulse system uses "Drops" and "Streams". We need to update the UI to reflect this.

## Requirements

1. **Terminology changes:**
   - "Workers" → "Drops"
   - "Waves" → "Streams"
   - "Worker X.Y" → "Drop X.Y"

2. **Add format badge:**
   - If `build.format === "legacy"` → show a small "Legacy" badge/chip
   - If `build.format === "pulse"` → no badge needed (it's the default now)

3. **Update progress display:**
   - Show "Stream X/Y" instead of "Wave X/Y"
   - Show drop status breakdown: "2 complete, 1 running, 3 pending"

4. **Files to modify:**
   - `src/components/BuildCard.tsx` — Main build card display
   - `src/pages/BuildsPage.tsx` — If it has terminology hardcoded

5. **Keep styling consistent** with existing design
