# Drop Brief: Create pulse_dashboard_sync.py

**Mission:** Create a script that scans Pulse builds and generates dashboard-compatible JSON.

**Output:** `/home/workspace/Skills/pulse/scripts/pulse_dashboard_sync.py`

---

## Context

The Build Tracker dashboard (`Sites/build-tracker`) needs to display Pulse builds. Currently it reads from `data/builds.json` but the old script (`build_status.py`) was deleted. We need a new script that understands the Pulse format (`meta.json` with `drops` key).

## Requirements

1. **Scan for Pulse builds:**
   - Look for `N5/builds/*/meta.json` files
   - If file has `drops` key → Pulse format
   - If directory has `workers/` folder → Legacy format
   - Skip directories without either

2. **Generate unified JSON schema:**
   ```json
   {
     "generated_at": "2026-01-24T17:30:00+00:00",
     "builds": [
       {
         "slug": "build-name",
         "title": "Build Title",
         "status": "pending|active|complete|failed",
         "format": "pulse|legacy",
         "streams": { "current": 1, "total": 2 },
         "drops": {
           "complete": 2,
           "running": 1,
           "pending": 3,
           "dead": 0,
           "failed": 0,
           "total": 6
         },
         "progress_pct": 33,
         "created_at": "2026-01-24T10:00:00+00:00",
         "last_activity": "2026-01-24T12:00:00+00:00",
         "path": "N5/builds/build-name"
       }
     ]
   }
   ```

3. **CLI interface:**
   ```bash
   python3 pulse_dashboard_sync.py                    # Default output to Sites/build-tracker/data/builds.json
   python3 pulse_dashboard_sync.py --output /tmp/out.json  # Custom output
   python3 pulse_dashboard_sync.py --help
   ```

4. **Sort builds:** Active first, then by last_activity descending

5. **Graceful handling:**
   - Missing fields → use sensible defaults
   - Malformed JSON → skip with warning to stderr
   - No builds → empty array (not error)

## Success Criteria

- [ ] Script runs without errors
- [ ] `--help` shows usage
- [ ] Output JSON is valid
- [ ] Pulse builds show correct drop counts
- [ ] Legacy builds marked with `format: "legacy"`
