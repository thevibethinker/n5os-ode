# D2.3: Prompt Lint CLI

## Objective
Create a CLI tool that scans Drop briefs, persona definitions, and prompts for vague language that reduces effectiveness — enforcing Watts' "100x leverage" principle.

## Context
Watts emphasized: "Vibe coding the prompt was not good enough." Every word in a prompt has 100x leverage when running long agentic loops. Vague language like "try to" or "if possible" compounds into poor outcomes.

## Requirements

### 1. Create Prompt Lint Script
Location: `N5/scripts/prompt_lint.py`

### 2. Lint Rules

**High Severity (must fix):**
| Pattern | Problem | Suggestion |
|---------|---------|------------|
| "try to X" | Permits failure | "X" or "X, or escalate if blocked" |
| "if possible" | Permits skipping | "X" or "X unless [specific condition]" |
| "maybe", "perhaps" | Ambiguous intent | Remove or make definitive |
| "etc", "and so on" | Incomplete spec | List all items explicitly |
| "as needed" | Undefined trigger | Specify when needed |
| "improve X" | No measurable target | "improve X by Y%" or "improve X until Z" |
| "optimize" (alone) | No success criteria | "optimize until [metric] reaches [target]" |

**Medium Severity (should fix):**
| Pattern | Problem | Suggestion |
|---------|---------|------------|
| Passive voice | Unclear actor | Use active voice |
| "should" vs "must" | Weak requirement | Use "must" for requirements |
| "good", "better" | Subjective | Define measurable criteria |
| "soon", "quickly" | Vague timing | Specify timeframe |
| "some", "few", "many" | Vague quantity | Use specific numbers |

**Low Severity (consider fixing):**
| Pattern | Problem | Suggestion |
|---------|---------|------------|
| > 50 words per sentence | Hard to parse | Split into shorter sentences |
| No success criteria section | Missing exit condition | Add "## Success Criteria" |
| No constraints section | Unbounded scope | Add "## Constraints" |

### 3. CLI Interface
```bash
# Lint a specific file
python3 prompt_lint.py file path/to/brief.md

# Lint all Drop briefs in a build
python3 prompt_lint.py build <slug>

# Lint persona definitions
python3 prompt_lint.py personas

# Lint with auto-fix suggestions
python3 prompt_lint.py file path/to/brief.md --suggest

# Output formats
python3 prompt_lint.py file path/to/brief.md --format json
python3 prompt_lint.py file path/to/brief.md --format table  # default
```

### 4. Output Format
```
PROMPT LINT: drops/D2.1.md
==========================

🔴 HIGH SEVERITY (2 issues)

Line 23: "Try to optimize the database queries"
  Problem: "try to" permits failure
  Fix: "Optimize the database queries" or "Optimize the database queries, escalate if blocked after 3 attempts"

Line 45: "Improve performance if possible"
  Problem: "if possible" permits skipping
  Fix: "Improve performance by at least 20%" or remove if not required

🟡 MEDIUM SEVERITY (1 issue)

Line 12: "The file should be created..."
  Problem: Passive voice, unclear actor
  Fix: "Create the file at..."

📊 SUMMARY
  High: 2 | Medium: 1 | Low: 0
  Lint Score: 65/100 (needs attention)
  
Recommendation: Fix high-severity issues before spawning Drop
```

### 5. Lint Score Calculation
```
score = 100 - (high * 15) - (medium * 5) - (low * 1)
```

Thresholds:
- 90+: Ready to use
- 70-89: Acceptable, consider fixes
- 50-69: Needs attention
- <50: Rewrite recommended
