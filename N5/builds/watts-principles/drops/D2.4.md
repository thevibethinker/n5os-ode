# D2.4: Plateau Detector CLI

## Objective
Create a CLI tool that detects when Drops are "spinning" — making repeated similar attempts without meaningful progress — so the system can suggest tooling changes rather than just retrying.

## Context
Watts' insight: When stuck, the solution isn't more attempts — it's changing the tooling. "Giving AI the guidance & importantly the tools to identify real optimization opportunities from the program trace was extremely helpful."

Plateau signals:
- Repeated similar approaches
- Metrics not improving
- Confidence scores declining
- Same errors recurring

## Requirements

### 1. Create Plateau Detector Script
Location: `N5/scripts/plateau_detector.py`

### 2. Detection Signals

**From Deposits:**
- Multiple deposits from same Drop with similar content (semantic similarity > 0.8)
- Confidence/quality scores declining or flat across attempts
- "Tried X but failed" patterns repeating

**From Logs:**
- Same error messages recurring
- Similar tool calls with same failures
- Retry patterns without variation

**From Timing:**
- Drop running > 15 min without deposit
- Multiple spawns of same Drop in short window

### 3. CLI Interface
```bash
# Check specific Drop for plateau
python3 plateau_detector.py drop <build_slug> <drop_id>

# Monitor active build for plateaus
python3 plateau_detector.py build <slug>

# Real-time monitoring (for Sentinel)
python3 plateau_detector.py watch <slug> --interval 60

# Analyze historical build for plateau patterns
python3 plateau_detector.py analyze <slug>
```
