# D1.1: Learning Schema v2

## Objective
Enhance the learnings system with confidence scoring, decay/expiration, and status tracking to prevent contamination from stale or disputed learnings.

## Context
Current `SYSTEM_LEARNINGS.json` is append-only with no quality controls. Watts' key failure was AI concluding something was "impossible" and that belief propagating unchallenged. We need learnings to have:
- Confidence scores (increase on validation, decrease on age)
- Expiration/decay (unvalidated learnings eventually expire)
- Status field (active/disputed/invalidated/expired)

## Requirements

### 1. Update SYSTEM_LEARNINGS.json Schema
Location: `N5/learnings/SYSTEM_LEARNINGS.json`

New fields per learning:
```json
{
  "text": "string",
  "source": "manual|sms|build|inference",
  "origin_build": "string|null",
  "added_at": "ISO8601",
  "tags": ["string"],
  
  // NEW FIELDS
  "confidence": 0.7,           // 0.0-1.0, default 0.7
  "validated_count": 0,        // times this learning was confirmed
  "last_validated": null,      // ISO8601 or null
  "decay_days": 30,            // days until confidence starts decaying
  "expires_at": null,          // ISO8601, null = never expires
  "status": "active",          // active|disputed|invalidated|expired
  "disputed_by": null,         // reference to contradicting source
  "dispute_reason": null       // explanation of dispute
}
```

### 2. Update pulse_learnings.py
Location: `Skills/pulse/scripts/pulse_learnings.py`

Modify functions:
- `add_learning()`: Accept new optional params (confidence, decay_days), set defaults
- `load_system_learnings()`: Handle both old and new schema (backward compatible)
- Add `validate_learning(index)`: Increment validated_count, update last_validated, boost confidence
- Add `dispute_learning(index, reason, source)`: Set status=disputed, record reason
- Add `invalidate_learning(index)`: Set status=invalidated
- Add `expire_stale_learnings()`: Check decay, update status to expired for stale items

### 3. Update LESSONS_SCHEMA.md
Location: `N5/learnings/LESSONS_SCHEMA.md`

Document the new schema with examples.

### 4. Migrate Existing Learnings
Write migration that:
- Reads current SYSTEM_LEARNINGS.json
- Adds default values for new fields to existing learnings
- Preserves all existing data
- Creates backup before migration

## Success Criteria
- [ ] Schema documented in LESSONS_SCHEMA.md
- [ ] pulse_learnings.py updated with new functions
- [ ] Existing learnings migrated with defaults
- [ ] `pulse_learnings.py validate <slug> <index>` works
- [ ] `pulse_learnings.py dispute <slug> <index> "reason"` works
- [ ] `pulse_learnings.py expire-stale` works
- [ ] Backward compatible with old schema entries

## Constraints
- Do NOT break existing pulse_learnings.py functionality
- Preserve all existing learnings data
- Default confidence: 0.7
- Default decay_days: 30
- Default status: "active"
