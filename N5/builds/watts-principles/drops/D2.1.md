# D2.1: Impossibility Audit CLI

## Objective
Create a CLI tool that scans deposits, learnings, and build artifacts for ceiling beliefs, flagging them for human review before they contaminate the system.

## Context
Uses the impossibility_detector.py module from D1.2 to scan N5 artifacts. This is the operational tool that V and the system will use to catch contamination.

## Requirements

### 1. Create Impossibility Audit Script
Location: `N5/scripts/impossibility_audit.py`

### 2. Scan Targets
The tool should scan:
- `N5/learnings/SYSTEM_LEARNINGS.json` — all learning texts
- `N5/builds/*/deposits/*.json` — all deposit conclusions
- `N5/builds/*/BUILD_LESSONS.json` — build-specific learnings
- Optional: arbitrary file or directory

### 3. CLI Interface
```bash
# Scan all learnings
python3 impossibility_audit.py learnings

# Scan specific build deposits
python3 impossibility_audit.py build <slug>

# Scan all builds
python3 impossibility_audit.py builds

# Scan arbitrary file
python3 impossibility_audit.py file <path>

# Full system scan
python3 impossibility_audit.py all

# Output formats
python3 impossibility_audit.py all --format json
python3 impossibility_audit.py all --format table  # default
python3 impossibility_audit.py all --format markdown
```

### 5. Integration Hook
Add function that Pulse can call:
```python
def audit_before_build(slug: str) -> dict:
    """Run audit and return blocking issues."""
    results = scan_learnings()
    high_severity = [r for r in results if r["severity"] == "high"]
    return {
        "can_proceed": len(high_severity) == 0,
        "blocking_issues": high_severity,
        "warnings": [r for r in results if r["severity"] == "medium"]
    }
```
