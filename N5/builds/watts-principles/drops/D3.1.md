```python
def pre_build_safety_checks(slug: str) -> dict:
    """Run all safety checks before starting build."""
    results = {
        "can_proceed": True,
        "blockers": [],
        "warnings": []
    }
    
    # 1. Impossibility Audit on learnings
    from impossibility_audit import audit_before_build
    audit = audit_before_build(slug)
    if not audit["can_proceed"]:
        results["blockers"].extend(audit["blocking_issues"])
        results["can_proceed"] = False
    results["warnings"].extend(audit.get("warnings", []))
    
    # 2. Lint all Drop briefs
    from prompt_lint import lint_before_spawn
    for brief in get_drop_briefs(slug):
        lint = lint_before_spawn(brief)
        if not lint["can_spawn"]:
            results["blockers"].append(f"{brief}: Lint score {lint['score']}")
            results["can_proceed"] = False
        if lint["warnings"]:
            results["warnings"].extend(lint["warnings"])
    
    return results
```

### 4. Create Prompt Engineering Principles Doc
Location: `N5/prefs/principles/prompt-engineering.md`

Document:
```markdown
# Prompt Engineering Principles

## Source
Derived from Jarrod Watts' approach to AI orchestration
(X post: how to beat Anthropic's kernel challenge with zero kernel knowledge)

## Core Principle: 100x Leverage
Every word in a prompt has 100x leverage when running agentic loops.
"Vibe coding the prompt was not good enough."

## Anti-Patterns (Never Use)

| Pattern | Problem | Instead |
|---------|---------|---------|
| "try to X" | Permits failure | "X" or "X, or escalate" |
| "if possible" | Permits skipping | "X unless [condition]" |
| "maybe/perhaps" | Ambiguous | Be definitive |
| "improve X" | No target | "improve X by Y%" |
| etc. | Incomplete | List all items |

## Required Sections in Drop Briefs
1. **Objective** — Single sentence, what success looks like
2. **Context** — Why this matters, what came before
3. **Requirements** — Numbered, specific, measurable
4. **Success Criteria** — Checklist of verifiable outcomes
5. **Constraints** — What NOT to do, boundaries

## Contamination Prevention
- Never document "impossible" conclusions without human review
- Learnings expire after 30 days if not validated
- Contradictions flagged, not silently overwritten
- Fresh context available when inherited beliefs may be wrong

## Plateau Response
When stuck, change tooling — not just retry:
1. Add instrumentation/logging
2. Try different approach (algorithm, library, method)
3. Expand search space
4. Reframe the problem
```
