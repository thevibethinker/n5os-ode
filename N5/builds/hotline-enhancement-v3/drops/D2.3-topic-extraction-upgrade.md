# D2.3: Topic Extraction Upgrade

## Mission

Replace the hardcoded `"general_advisory"` topic in the webhook's `logCall` function with LLM-based topic classification. When a call ends, use the transcript to extract actual topics discussed and store them in the `topics_discussed` field.

## Context

- Currently in hotline-webhook.ts `logCall()` function: `const topics = "general_advisory";` (hardcoded)
- Transcripts available at `data.message.artifact.transcript` in the end-of-call-report payload
- call_analyzer.py already has `analyze_transcript_with_zo()` that does LLM topic extraction via /zo/ask — but that's Python post-hoc analysis
- For real-time: use Zo API from the webhook (TypeScript) to classify topics at call end
- Topic taxonomy should match what call_analyzer uses: calendar_automation, meeting_intelligence, email_management, getting_started, troubleshooting, use_cases, concepts, escalation, etc.
- Must not block the end-of-call response — classify async (fire and forget, update DB after)

## Affected Files

- Modify: `Skills/zo-hotline/scripts/hotline-webhook.ts` (logCall function + async topic extraction)

## Success Criteria

- [ ] logCall extracts topics from transcript using Zo API
- [ ] Topics stored as comma-separated list in topics_discussed field
- [ ] Classification runs async — doesn't delay the webhook response
- [ ] Falls back to "general_advisory" if transcript missing or API fails
- [ ] Topic taxonomy is consistent (defined list, not freeform)
