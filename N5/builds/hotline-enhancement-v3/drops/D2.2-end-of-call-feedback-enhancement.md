# D2.2: End-of-Call Feedback Enhancement

## Mission

Add an optional end-of-call feedback collection flow to Zoseph's system prompt and a new `collectFeedback` tool to the webhook. At conversation end, Zoseph asks if the caller wants to share their name and a quick satisfaction signal. This is entirely optional — no pressure.

## Context

- Webhook at `Skills/zo-hotline/scripts/hotline-webhook.ts` handles tool-calls and end-of-call-report
- System prompt at `Skills/zo-hotline/prompts/zoseph-system-prompt.md` controls Zoseph behavior
- DuckDB has `calls` table — need a new `feedback` table for collected data
- Feedback fields: caller_name (optional), satisfaction (1-5 or thumbs up/down), comment (optional)
- Zoseph should ask once at end, accept "no" gracefully, never push
- Voice-friendly: "Before you go — want to leave your name so I can personalize next time? And on a scale of helpful to not helpful, how'd I do?"

## Affected Files

- Modify: `Skills/zo-hotline/scripts/hotline-webhook.ts` (add collectFeedback tool + DB table)
- Modify: `Skills/zo-hotline/prompts/zoseph-system-prompt.md` (add feedback section)

## Success Criteria

- [ ] New `collectFeedback` tool added to webhook with name/satisfaction/comment params
- [ ] New `feedback` table created in DuckDB on startup
- [ ] System prompt updated with feedback collection guidance (optional, one ask, accept no)
- [ ] Feedback linked to call_id when available
- [ ] Tool returns graceful response whether caller provides info or declines
