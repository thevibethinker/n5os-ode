# D2.4: Pulse Integration for Hotline Builds

## Mission

Create a hotline-specific Pulse build template so future hotline improvement cycles can be orchestrated through Pulse. This means a reusable init pattern that pre-populates standard drops for the hotline improvement workflow (analysis review, prompt tuning, tool changes, testing, deploy).

## Context

- Pulse-CC at `Skills/pulse/scripts/pulse_cc.py` handles build orchestration in Claude Code
- Full Pulse at `Skills/pulse/scripts/pulse.py` handles automated /zo/ask builds
- Hotline improvements follow a repeatable pattern: analyze calls → identify improvement → implement → test → deploy → verify
- The template should be a simple script that wraps pulse_cc.py init + brief commands
- Store in `Skills/zo-hotline/scripts/` alongside existing hotline scripts

## Affected Files

- Creates: `Skills/zo-hotline/scripts/hotline_build_init.py` (template init script)
- Modifies: `Skills/zo-hotline/SKILL.md` (document the Pulse workflow)

## Success Criteria

- [ ] Script creates a new hotline build with standard drops pre-populated
- [ ] Standard template includes: analyze, identify improvements, implement changes, test, deploy
- [ ] SKILL.md updated with section on how to run a hotline improvement cycle via Pulse
- [ ] Script takes a description argument to customize the build title
