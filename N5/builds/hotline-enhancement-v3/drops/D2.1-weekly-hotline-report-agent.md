# D2.1: Weekly Hotline Report Agent

## Mission

Create a scheduled agent that runs weekly, executes both call_analyzer.py and dropoff_analyzer.py, and emails V a consolidated hotline performance report.

## Context

- call_analyzer.py exists at `Skills/zo-hotline/scripts/call_analyzer.py` with `analyze`, `patterns`, `report` subcommands
- dropoff_analyzer.py exists at `Skills/zo-hotline/scripts/dropoff_analyzer.py` with `analyze`, `report` subcommands
- DuckDB at `Datasets/zo-hotline-calls/data.duckdb` has `calls` and `escalations` tables
- Pattern analysis requires ZO_CLIENT_IDENTITY_TOKEN (available in env)
- Should run Sunday evenings to cover the full week
- Fits into existing dispatcher pattern but runs weekly, so standalone agent is appropriate
- Delivery: email (report is too long for SMS)

## Affected Files

- Creates: scheduled agent (via create_agent)
- No new scripts needed — leverages existing analyzers

## Success Criteria

- [ ] Scheduled agent created, runs weekly (Sunday evening ET)
- [ ] Agent executes call_analyzer.py report + dropoff_analyzer.py report
- [ ] Combined report emailed to V with call volume, duration distribution, drop-off analysis, and pattern highlights
- [ ] Agent handles edge case of zero calls gracefully
