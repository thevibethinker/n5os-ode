# D1.1: Skill Extractor Script + Close Integration

## Objective

Create `skill_extractor.py` that transforms BUILD_LESSONS.json entries and thread learnings into skill candidates, with semantic deduplication against existing Skills/.

## Context

SkillRL research shows agents improve by distilling experiences into reusable skills. We already capture experiences via:
- `BUILD_LESSONS.json` (Pulse builds)
- Learnings field in thread-close output
- DEBUG_LOG.jsonl (failure patterns)

This Drop bridges capture → skill candidates.

## Deliverables

### 1. `Skills/pulse/scripts/skill_extractor.py`

CLI tool with these commands:

```bash
# Extract from a completed build
python3 Skills/pulse/scripts/skill_extractor.py extract --build <slug>

# Extract from a thread (reads thread-close output)
python3 Skills/pulse/scripts/skill_extractor.py extract --convo <convo_id>

# Check similarity against existing skills
python3 Skills/pulse/scripts/skill_extractor.py check-similarity "lesson text"

# Review pending candidates
python3 Skills/pulse/scripts/skill_extractor.py list-candidates

# Promote a candidate to a skill
python3 Skills/pulse/scripts/skill_extractor.py promote <candidate_id> --name <skill-name>
```

### 2. Extraction Logic

For each lesson/learning:

1. **Classify scope** via LLM (`/zo/ask`):
   - `universal` — applies across all work (reasoning patterns, meta-cognitive)
   - `domain` — applies to a domain (debugging, writing, research)
   - `task-specific` — applies to specific task type

2. **Check semantic similarity** against existing Skills/:
   - Use embeddings comparison (can call N5 brain or simple approach)
   - If similarity > 0.85 to existing skill → flag as "refinement candidate"
   - If similarity < 0.85 → flag as "new skill candidate"

3. **Extract skill template** via LLM:
   ```yaml
   name: suggested-skill-name
   scope: universal|domain|task-specific
   domain: (if domain-scoped)
   trigger: "When to activate this skill"
   procedure: "What to do"
   anti-patterns: "What NOT to do"
   source_lesson: "Original lesson text"
   confidence: 0.0-1.0
   provenance: build-slug or convo_id
   ```

4. **Write to review queue**: `N5/review/skills/YYYY-MM-DD_candidates.md`

### 3. Integration Points

**In `Skills/build-close/scripts/close.py`:**
After aggregating BUILD_LESSONS, call:
```python
from Skills.pulse.scripts.skill_extractor import extract_from_build
extract_from_build(slug)
```

**In `Skills/thread-close/scripts/close.py`:**
For Tier 2+ closes with learnings, call:
```python
from Skills.pulse.scripts.skill_extractor import extract_from_thread
extract_from_thread(convo_id, learnings)
```

## Technical Constraints

- Use `/zo/ask` for LLM classification (not inline model calls)
- Keep similarity check lightweight (don't load all skills into memory)
- Candidates should be human-readable markdown, not just JSON
- Integrate without breaking existing close flows (additive, not replacing)
