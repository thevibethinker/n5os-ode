# D1.2: Evolution Agent + SKILL.md Frontmatter Enhancement

## Objective

Create a weekly scheduled agent that analyzes skill health (usage, failures, staleness) and proposes refinements or deprecations. Also enhance SKILL.md frontmatter spec with SkillRL-inspired fields.

## Context

SkillRL's key insight: skills must co-evolve with the agent. Static skills become stale. This Drop implements the "recursive evolution" layer — periodic analysis that keeps the skill library healthy.

## Deliverables

### 1. Enhanced SKILL.md Frontmatter Spec

Update the canonical skill spec to include:

```yaml
---
name: skill-name
description: |
  What this skill does and when to use it.
# NEW FIELDS (SkillRL-inspired):
scope: universal|domain|task-specific  # Required for new skills
domain: debugging|writing|research|building|...  # If scope=domain
provenance: con_XXX|build-slug  # Where this skill originated
created_from: lesson|pattern|manual  # How it was created
usage_count: 0  # Auto-incremented when skill is activated
last_used: null  # ISO timestamp, auto-updated
last_refined: 2026-02-11  # When skill was last updated
refinement_history:
  - date: 2026-02-11
    source: build:skillrl-integration
    change: Initial creation
---
```

**Document this in:** `Documents/System/Skills-Spec.md` (create or update)

### 2. Usage Tracking Script

`Skills/pulse/scripts/skill_usage.py`:

```bash
# Record skill activation (called by Zo when skill is used)
python3 Skills/pulse/scripts/skill_usage.py record <skill-name>

# Get usage stats
python3 Skills/pulse/scripts/skill_usage.py stats [--stale-days 30]

# List skills by usage
python3 Skills/pulse/scripts/skill_usage.py list --sort usage|last_used|created
```

**Integration:** Add a note to SKILL.md spec that Zo should call `skill_usage.py record` when activating a skill. This is a convention, not enforced.

### 3. Evolution Analysis Script

`Skills/pulse/scripts/skill_evolution.py`:

```bash
# Full evolution analysis
python3 Skills/pulse/scripts/skill_evolution.py analyze

# Check specific skill health
python3 Skills/pulse/scripts/skill_evolution.py check <skill-name>

# Generate weekly digest
python3 Skills/pulse/scripts/skill_evolution.py digest --email
```

**Analysis logic:**

1. **Staleness detection:**
   - Skills with `last_used` > 60 days ago → "stale candidate"
   - Skills with `usage_count` = 0 and `created` > 30 days ago → "unused candidate"

2. **Failure pattern detection:**
   - Scan `DEBUG_LOG.jsonl` for patterns (3+ similar failures)
   - Cross-reference with existing skills
   - If failures occur despite skill existing → "skill may be incomplete"
   - If failures have no matching skill → "skill gap detected"

3. **Refinement suggestions:**
   - Parse recent BUILD_LESSONS.json entries
   - Check if any relate to existing skills
   - Propose refinements with source attribution

4. **Output format:**
   ```markdown
   # Skill Evolution Digest: 2026-02-11
   
   ## Stale Skills (2)
   - `foo-skill`: Last used 90 days ago, 3 total uses
   - `bar-skill`: Never used since creation (45 days)
   
   ## Skill Gaps (1)
   - Pattern: "API rate limiting" appears 5x in DEBUG_LOG
   - No matching skill found
   - Suggested: Create `api-rate-limit-handling` skill
   
   ## Refinement Candidates (1)
   - `systematic-debugging`: Recent lesson suggests adding "circular pattern" checkpoint
   - Source: build:skillrl-integration
   
   ## Healthy Skills (top 5 by usage)
   - `thread-close`: 47 uses, last used today
   - `pulse`: 23 uses, last used yesterday
   ...
   ```

### 4. Weekly Evolution Agent

**Scheduled agent spec:**

```
Name: ⇱ Skill Evolution Digest
RRULE: FREQ=WEEKLY;BYDAY=SU;BYHOUR=10
Delivery: email

Instruction:
Run the skill evolution analysis and send V a digest.

1. Run: python3 Skills/pulse/scripts/skill_evolution.py analyze
2. Read the output and compose an email digest.

EMAIL FORMAT:
Subject: [Skills] Weekly Evolution Digest

Body:
## Skill Library Health

**Total skills:** X | **Active (used in 30d):** Y | **Stale:** Z

### Attention Needed

[List stale skills, skill gaps, refinement candidates]

### Pending Review

[Link to N5/review/skills/ if candidates exist]

### Top Performers

[Top 5 skills by usage this month]

---

Reply with:
- `deprecate <skill>` — Archive a stale skill
- `refine <skill>` — I'll draft a refinement
- `create <name>` — I'll draft a new skill for a gap
```

**Note:** Mark with ⇱ (cornerstone) since this is infrastructure.

### 5. Deprecation Flow

When V approves deprecation:

1. Move skill to `Skills/_archived/<skill-name>/`
2. Update any references (search for skill name in rules, personas, AGENTS.md)
3. Log to refinement_history

Script: `skill_evolution.py deprecate <skill-name> --reason "..."`

## Technical Constraints

- Don't break existing skills — frontmatter additions are optional for existing skills
- Usage tracking is convention-based (not enforced) — relies on Zo calling the script
- Evolution agent should be idempotent (safe to run multiple times)
- Keep digest concise — V reviews weekly, don't overwhelm
