## Context
The decomposer skill currently extracts overview/profile/experience but **NOT** the critical skill assessments. The `scores_complete.json` files were created manually in separate conversations, causing schema drift between Hardik and Sridhar outputs.

**2b. Add extraction function:**

```python
async def extract_skills_assessment(session: aiohttp.ClientSession, doc_content: str) -> list:
    """Extract all skill assessments using LLM."""
    prompt = EXTRACTION_PROMPTS["skills_assessment"].format(doc=doc_content)
    
    print("  Extracting skills_assessment...")
    response = await call_llm(session, prompt)
    
    # Clean JSON response
    cleaned = response.strip()
    if cleaned.startswith("```json"):
        cleaned = cleaned[7:]
    elif cleaned.startswith("```"):
        cleaned = cleaned[3:]
    if cleaned.endswith("```"):
        cleaned = cleaned[:-3]
    
    try:
        data = json.loads(cleaned)
        print(f"    Extracted {len(data)} skills")
        return data
    except json.JSONDecodeError as e:
        print(f"  WARNING: JSON parse error: {e}")
        return [{"_error": str(e), "_raw": cleaned[:500]}]
```

**2d. Add schema validation:**

```python
import jsonschema

def validate_scores(scores: list, schema_path: str) -> bool:
    """Validate scores against canonical schema."""
    with open(schema_path) as f:
        schema = json.load(f)
    try:
        jsonschema.validate(scores, schema)
        return True
    except jsonschema.ValidationError as e:
        print(f"  Schema validation failed: {e.message}")
        return False
```
