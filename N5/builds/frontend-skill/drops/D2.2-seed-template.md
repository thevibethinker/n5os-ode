# D2.2: Seed Template + Capture Instructions

## Context

We need:
1. At least one working template to prove the system works
2. Instructions in SKILL.md for how to capture new templates from existing pages

## MUST DO

### Part 1: Seed Template

1. **Create `Skills/frontend/templates/minimal-saas.yaml`** — A clean SaaS landing page template

   Base it on good SaaS landing page patterns:
   - Clean, modern aesthetic
   - NOT the purple gradient AI slop
   - Good typography hierarchy
   - Restrained color palette
   - Professional but not boring

   Example direction:
   ```yaml
   name: minimal-saas
   description: |
     Clean, professional SaaS landing page. Works for B2B tools,
     developer products, productivity apps. Emphasizes clarity
     and trust over flashiness.
   source: Synthesized from Linear, Vercel, Stripe landing page patterns
   created: 2026-02-06
   
   palette:
     primary: "#0A0A0A"      # Near-black
     secondary: "#525252"    # Gray
     accent: "#2563EB"       # Blue (restrained)
     background: "#FFFFFF"   # White
     surface: "#FAFAFA"      # Off-white
     text: "#0A0A0A"         # Near-black
     text_muted: "#737373"   # Gray
   
   typography:
     heading_font: "Inter"
     body_font: "Inter"
     scale: normal
     heading_weight: 600
   
   layout:
     max_width: standard
     spacing: balanced
     border_radius: subtle
   
   sections:
     - type: hero
       variant: centered
     - type: logos
       variant: simple
     - type: features
       variant: grid-3
     - type: testimonials
       variant: single-featured
     - type: cta
       variant: centered
   
   style_notes: |
     - Relies on typography and spacing, not color
     - Subtle shadows (shadow-sm) for depth
     - No gradients — solid colors only
     - Icons: Lucide, stroke-width 1.5
     - Buttons: solid fill, subtle hover state
     - Links: underline on hover, not color change
     - Overall feel: "grown-up startup"
   ```

### Part 2: Capture Instructions

2. **Update `Skills/frontend/SKILL.md`** — Add a "Template Capture" section:

   The section should explain:
   
   a. **When to capture**: After generating a page the user loves
   
   b. **How to analyze**: Look at the page and extract:
      - Color palette (use browser devtools or eyedropper)
      - Typography (fonts, weights, scale)
      - Layout patterns (max-width, spacing, border-radius)
      - Section structure
      - The intangible "feel" (goes in style_notes)
   
   c. **How to create the template**:
      - Copy `templates/_TEMPLATE.yaml` as starting point
      - Fill in all fields based on analysis
      - Write detailed style_notes capturing nuance
      - Save as `templates/<name>.yaml`
   
   d. **Template naming conventions**:
      - Use descriptive slugs: `minimal-saas`, `bold-agency`, `warm-nonprofit`
      - Name should convey the aesthetic

## MUST NOT DO
