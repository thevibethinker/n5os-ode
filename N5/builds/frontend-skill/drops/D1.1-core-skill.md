# D1.1: Core Frontend Design SKILL.md

## Context

We're building a frontend design skill that generates high-quality landing pages. This is inspired by Anthropic's `frontend-design` skill which Theo (YouTube creator) praised for making Opus 4.5 dramatically better at design.

**Key insight from research:** The skill's power comes from:
1. Anti-slop guardrails (avoiding purple gradients, generic AI aesthetics)
2. Multi-variation generation (create 5, pick best)
3. Clear platform-specific output instructions

## MUST DO

1. **Create `Skills/frontend/SKILL.md`** with proper Agent Skills frontmatter:
   - `name: frontend` (must match directory)
   - `description`: Clear description of what the skill does
   - `compatibility: Created for Zo Computer`
   - `metadata.author: va.zo.computer`

2. **Input Spectrum Section** — The skill accepts varying degrees of input:
   - **Vibes only**: "a modern SaaS landing page that feels trustworthy"
   - **Colors/mood**: Hex values, mood words, reference images
   - **Content provided**: Headlines, copy, features list, CTAs
   - **Template specified**: Use a captured template from `templates/`

3. **Generation Workflow Section**:
   - If input is sparse → ask clarifying questions OR make tasteful defaults
   - Always ask for target platform (or detect from context)
   - When `--variants N` is requested, generate N distinct directions
   - Apply anti-slop guardrails (reference `references/anti-patterns.md`)

4. **Platform-Specific Output Sections** — Three platforms:

   **Zo.space** (React/Tailwind/Hono):
   - Use `update_space_route(path, route_type="page", code, public=True)`
   - Component must be default export React function
   - Use Tailwind 4 classes
   - Can import from `lucide-react` for icons
   - Reference any uploaded assets via their asset_path

   **Standalone HTML**:
   - Single HTML file with Tailwind CDN (`<script src="https://cdn.tailwindcss.com">`)
   - Inline all styles, no external dependencies
   - Should work when opened directly in browser

   **Next.js** (App Router):
   - Export default async function component
   - Use `"use client"` directive if needed for interactivity
   - Tailwind classes, standard Next.js patterns

5. **Anti-Slop Section** — Reference the anti-patterns file:
   ```markdown
   Before generating, review `references/anti-patterns.md` and ensure your design avoids:
   - Generic purple/blue gradients
   - Overly symmetrical layouts
   - Stock illustration aesthetic
   - Excessive animations
   - Poor typography hierarchy
   ```

6. **Template Application Section** — When a template is provided:
   - Load the template YAML from `templates/<name>.yaml`
   - Apply palette, typography, layout values
   - Use `style_notes` for aesthetic guidance
   - Content still comes from user input

## MUST NOT DO

- Do NOT create any templates yet (that's Wave 2)
- Do NOT create the anti-patterns.md file (that's D1.2)
- Do NOT add complex scripting (keep it as a prompt/instructions skill)
- Do NOT hardcode specific design opinions beyond anti-slop — the skill should be flexible
