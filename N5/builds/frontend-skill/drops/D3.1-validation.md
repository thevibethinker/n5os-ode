# D3.1: Validation Tests

## Context

The skill and template system are built. Now we validate that everything works together.

**This is a MANUAL drop** because validation requires judgment and V may want to see the outputs.

## MUST DO

Run these validation tests and document results:

### Test 1: Vibe-Only Generation (Zo.space)

**Input:** "Create a landing page for a consulting firm that helps startups with go-to-market strategy. Should feel trustworthy and premium but not stuffy."

**Expected:** 
- Skill should ask clarifying questions OR make tasteful defaults
- Output should be valid Zo.space React/Tailwind code
- Design should avoid anti-patterns (no purple gradients, etc.)

**How to test:**
1. Invoke the skill with the input above
2. Review the generated code
3. Optionally deploy to va.zo.space to see it live

### Test 2: Template-Based Generation (Standalone HTML)

**Input:** "Create a landing page using the minimal-saas template. Content: A project management tool called 'Cadence' that helps remote teams stay synchronized. Target: standalone HTML."

**Expected:**
- Should load and apply minimal-saas.yaml template
- Colors, typography, layout should match template
- Output should be single HTML file with Tailwind CDN
- Should work when opened in browser

**How to test:**
1. Invoke skill with template specified
2. Save output as .html file
3. Open in browser and verify appearance

### Test 3: Template Capture

**Input:** After Test 1 or 2 produces a design you like, ask to capture it as a template.

**Expected:**
- Skill should analyze the generated page
- Extract palette, typography, layout
- Produce valid YAML following _TEMPLATE.yaml schema
- style_notes should capture the nuance

**How to test:**
1. Generate a page you like
2. Ask skill to capture it as a template
3. Verify output YAML is valid and complete

### Test 4: Multi-Variant Generation

**Input:** "Create 3 variants of a landing page for a meditation app called 'Stillness'. Each should have a distinct aesthetic direction."

**Expected:**
- 3 meaningfully different designs
- Not just color swaps — different layouts, typography, feels
- All avoid anti-patterns

**How to test:**
1. Invoke skill with --variants 3 (or however it's specified)
2. Compare the 3 outputs
3. Verify they're genuinely different

## Validation Rubric

For each test, score:
- **Functional**: Does the output work? (code runs, HTML renders)
- **Aesthetic**: Does it avoid AI slop? Would a human designer approve?
- **Faithful**: Does it follow the skill's instructions / template?

## MUST NOT DO

- Do NOT modify any skill files based on test results (just document)
- Do NOT skip any tests

## Expected Output

A detailed validation report documenting:
1. Each test run with actual input/output
2. Screenshots or descriptions of results
3. Pass/fail for each rubric dimension
4. Any issues discovered (for future improvement)
