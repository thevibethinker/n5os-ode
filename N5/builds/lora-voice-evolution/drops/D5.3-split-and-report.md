# D5.3: Split Train/Holdout Sets and Generate Dataset Report

## Objective

Split the formatted training data into train and holdout sets. Generate a comprehensive dataset report for V's review before Phase B (training).

## Input

- `N5/builds/lora-voice-evolution/artifacts/training_formatted.jsonl` (from D5.2)
- `N5/builds/lora-voice-evolution/artifacts/dataset_summary.md` (from D5.1)

## Train/Holdout Split

- **Holdout set:** 10% of examples, minimum 10, maximum 30
- **Training set:** Everything else
- **Stratified split:** Maintain platform distribution proportions in both sets
  - If 60% LinkedIn, 20% X, 20% email in full set → same ratios in both train and holdout
- **Random seed:** Use seed=42 for reproducibility

## Output Files

1. **Training set:** `Datasets/lora-voice-training/train.jsonl`
   - Create the `Datasets/lora-voice-training/` directory if it doesn't exist
   - Also create a `datapackage.json` for Zo Datasets compatibility

2. **Holdout set:** `Datasets/lora-voice-training/holdout.jsonl`

3. **Dataset report:** `N5/builds/lora-voice-evolution/artifacts/phase_a_report.md`
