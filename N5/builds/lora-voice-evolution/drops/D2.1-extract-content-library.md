# D2.1: Extract Content Library Originals

## Objective

Extract writing samples authored by V from the content library. This includes social posts, memos, and personal pieces — NOT saved articles or links written by others.

## Data Sources

1. **Social posts** (37 files): `/home/workspace/Knowledge/content-library/social-posts/`
   - Includes LinkedIn subfolder (32 files) and top-level posts (5 files)
   - Markdown files with YAML frontmatter
   - **IMPORTANT:** The LinkedIn subfolder posts overlap with D1.1 LinkedIn extraction — tag these as `source: content_library` so D5.1 can deduplicate

2. **Memos** (1 file): `/home/workspace/Knowledge/content-library/memos/`
   - V's authored memos

3. **Personal** (2 files): `/home/workspace/Knowledge/content-library/personal/`
   - Personal writing samples

4. **Inspiration** (3 files): `/home/workspace/Knowledge/content-library/inspiration/`
   - Check if these are V-authored or saved from others. Only include V-authored content.

## Filtering Criteria

For each file:
1. Read the YAML frontmatter
2. **Include** if:
   - `author` contains "V" or "Attawar" or is absent (default to V-authored)
   - `type` is "social-post", "memo", "personal", or similar
   - The body content is ≥50 characters
3. **Exclude** if:
   - `type` is "article" or "link" (saved from others)
   - `author` is clearly someone else
   - The file is a template or empty
