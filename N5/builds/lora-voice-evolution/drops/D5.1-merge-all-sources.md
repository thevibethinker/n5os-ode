# D5.1: Merge All Sources into Unified Dataset

## Objective

Merge scored examples from all four streams into a single unified dataset. Deduplicate across sources, apply a global quality ranking, and produce a clean dataset ready for formatting.

## Input Files

- `N5/builds/lora-voice-evolution/artifacts/linkedin_scored.jsonl` (D1.3)
- `N5/builds/lora-voice-evolution/artifacts/content_library_scored.jsonl` (D2.2)
- `N5/builds/lora-voice-evolution/artifacts/x_email_scored.jsonl` (D3.3)
- `N5/builds/lora-voice-evolution/artifacts/voice_library_scored.jsonl` (D4.2)

## Deduplication Strategy

LinkedIn content appears in multiple sources (DuckDB shares, content library, voice library chunks). Deduplication rules:

1. **Exact match:** If first 100 chars of `text` match (normalized: lowercase, strip whitespace), keep the version with:
   - Higher `voice_score` first
   - Richer `metadata` second
   - Content library version preferred over DuckDB version (cleaner formatting)

2. **Chunk overlap:** Voice library paragraph chunks may be subsets of full LinkedIn posts. Keep BOTH:
   - The full post (teaches full-post generation)
   - The chunk (teaches paragraph-level voice patterns)
   - Tag the chunk with `is_chunk: true` so the formatter can handle it differently

3. **Cross-platform:** Same content posted on both LinkedIn and X is NOT a duplicate — keep both with their respective platform tags.

## Quality Ranking

After deduplication, rank all examples by a composite score:

```
composite = voice_score * platform_weight * length_bonus
```

Where:
- `platform_weight`: linkedin=1.0, content_library=1.1, x=0.9, email=0.8, voice_library=1.0
- `length_bonus`: 1.0 for ≥200 chars, 0.9 for 100-199, 0.8 for 50-99

This ensures LinkedIn and content library examples are prioritized (per V's request).

## Output

Write to: `N5/builds/lora-voice-evolution/artifacts/unified_dataset.jsonl`

Each line:


Sort by `composite_score` descending.
