# D5.2: Format as Instruction-Following JSONL

## Objective

Convert the unified dataset into the instruction-following format required for LoRA fine-tuning. Each example becomes a conversation turn that teaches the model "given this prompt, produce this output in V's voice."

## Input

- `N5/builds/lora-voice-evolution/artifacts/unified_dataset.jsonl` (from D5.1)

## Output Format

Together.ai expects ChatML/ShareGPT format. Each training example:

```json
{
  "messages": [
    {
      "role": "system",
      "content": "You are V, a career coach and tech entrepreneur. Write in V's authentic voice: direct, specific, warm through substance, assertive without hedging. Platform: {platform}."
    },
    {
      "role": "user",
      "content": "{instruction}"
    },
    {
      "role": "assistant",
      "content": "{V's actual writing}"
    }
  ]
}
```

## Instruction Generation

For each example, generate a realistic instruction that would produce the given output. Use `/zo/ask` to generate instructions.

**Prompt for instruction generation:**
```
Given this piece of writing by V, generate a realistic instruction/prompt that someone would give an AI to produce this output. The instruction should be specific enough to produce this type of content but generic enough that it's a useful training signal.

Writing (platform: {platform}, type: {content_type}):
"""
{text}
"""

Respond with ONLY the instruction text. No quotes, no explanation. Example formats:
- "Write a LinkedIn post about [topic]"
- "Draft a follow-up email about [topic]"
- "Tweet about [observation]"
- "Write a comment responding to [topic]"
```

**Important:** Do NOT use `output_format` for this — we want natural text instructions, not structured JSON.

## Platform-Specific System Prompts

Vary the system prompt by platform:

- **LinkedIn:** "You are V, a career coach and tech entrepreneur. Write a LinkedIn post in V's voice: authoritative, warm, evidence-based. No profanity. Developed arguments (500+ chars typical)."
- **X/Twitter:** "You are V (@thevibethinker). Write a tweet/thread in V's voice: punchy, sharp, contrarian edge. Profanity OK when natural. Keep it tight."
- **Email:** "You are V, founder of Careerspan. Write an email in V's voice: direct, specific, professional but human. No corporate speak, no hedging."
- **General:** "You are V. Write in V's authentic voice: direct, warm through substance, assertive. Specific over vague."

## For Chunks (is_chunk: true)

Chunks are paragraph-level, not full pieces. Use a different instruction style:
- "Write an opening paragraph for a LinkedIn post about [topic]"
- "Write the core argument paragraph for a post about [topic]"

## Output

Write to: `N5/builds/lora-voice-evolution/artifacts/training_formatted.jsonl`

One conversation per line in the messages format above.

## Requirements

1. Process all `include_in_training: true` entries from unified dataset.
2. Generate instructions via `/zo/ask` (batch 20 concurrent).
3. Validate each output: messages array has exactly 3 entries (system, user, assistant).
4. Handle API errors gracefully.
5. Log examples where instruction generation failed.
