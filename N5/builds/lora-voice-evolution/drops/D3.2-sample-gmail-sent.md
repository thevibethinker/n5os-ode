# D3.2: Sample and Extract Sent Gmail Emails

## Objective

Sample V's sent Gmail emails to capture email-style voice data. Use random sampling to get a representative spread, then filter for substance.

## Approach

Use the Gmail API via `use_app_gmail` tools. The goal is a random sample of ~100 sent emails that represent V's writing across different contexts.

### Step 1: Search Sent Emails

Search for V's sent emails from the past 2 years:
- Use `gmail-search-email` with query: `in:sent after:2024/01/01`
- Get up to 200 results (we'll sample down from there)

### Step 2: Random Sample

From the results, take a random sample of ~100 emails. This averages out formality levels and topics.

### Step 3: Extract Content

For each sampled email:
- Use `gmail-get-email` to get the full content
- Extract the body text (strip signatures, quoted replies, forwarded content)
- **CRITICAL:** Anonymize recipient names and email addresses. Replace with generic placeholders like "[Recipient]", "[Company]"
- Remove any PII: phone numbers, addresses, account numbers

### Step 4: Filter for Substance

**Include** emails where:
- V's original text is ≥100 characters (not just "sounds good")
- Contains at least one complete paragraph
- Is not an auto-reply or calendar response

**Exclude:**
- One-liners under 100 chars
- Forwarded content without V's commentary
- Auto-generated emails (calendar invites, notifications)
- Emails that are purely logistics ("meeting at 3pm")

## Output

Write to: `N5/builds/lora-voice-evolution/artifacts/gmail_sent_raw.jsonl`

Each line:
```json
{
  "id": "gmail_sent_001",
  "source": "gmail_sent",
  "platform": "email",
  "content_type": "email",
  "text": "Anonymized email body text",
  "char_count": 350,
  "timestamp": "2025-06-15",
  "metadata": {
    "subject_anonymized": "Re: [Topic] Discussion",
    "is_reply": true,
    "original_length": 400,
    "anonymization_applied": true
  }
}
```

## Privacy Requirements

1. **NEVER** store recipient email addresses or real names in the output.
2. Replace all names with "[Name]" or "[Recipient]".
3. Replace company names ONLY if they appear in a sensitive context. Company names in professional context (e.g., "Careerspan") are OK to keep.
4. Replace phone numbers, addresses, account numbers.
5. Keep the subject line but anonymize if it contains names.
