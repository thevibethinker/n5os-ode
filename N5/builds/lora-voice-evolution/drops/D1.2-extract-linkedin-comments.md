# D1.2: Extract LinkedIn Comments (Substantive Only)

## Objective

Extract V's substantive LinkedIn comments. These are high-quality voice data — shorter than posts but still authentic V writing. Filter aggressively for substance.

## Data Sources

1. **DuckDB comments table** (689 comments, avg 137 chars): `/home/workspace/Datasets/linkedin-full-pre-jan-10/data.duckdb`
   - Table: `comments`
   - Key column: `message`
   - Average length: 137 chars — many are short reactions ("Great post!" etc.)

2. **Voice-library-v2 corpus** (261 comments): `/home/workspace/N5/builds/voice-library-v2/linkedin_corpus.jsonl`
   - Filter for `type == "comment"`
   - These may overlap with DuckDB comments

## Filtering Criteria

**Include** comments that:
- Are ≥100 characters (substantive enough to show voice)
- Contain at least one complete sentence (not just emoji or tags)
- Express an opinion, share an insight, or make an argument

**Exclude** comments that:
- Are pure reactions: "Great post!", "Love this!", "Congrats!", "Agreed!", "+1"
- Are just @mentions or tags
- Are shorter than 100 characters
- Are purely logistical ("Sent you a DM", "Check your email")

Use a Python script with simple heuristics — no need for LLM classification at this stage. Check for:
- Length ≥ 100 chars
- Contains at least one period or exclamation (indicates complete thought)
- Does NOT match common reaction patterns (regex: `^(great|love|congrats|amazing|awesome|nice|agreed|this)[!\s.]*$` case insensitive)

## Output

Write to: `N5/builds/lora-voice-evolution/artifacts/linkedin_comments_raw.jsonl`

Each line:
```json
{
  "id": "linkedin_comment_001",
  "source": "duckdb_comments|voice_library_v2",
  "platform": "linkedin",
  "content_type": "comment",
  "text": "Full comment text",
  "char_count": 250,
  "timestamp": null,
  "metadata": {}
}
```

## Requirements

1. Deduplicate across sources (text similarity on first 80 chars).
2. Apply filtering criteria strictly — better to have 100 high-quality comments than 400 mediocre ones.
3. Log stats: total per source, filtered out (with reason breakdown), final count.
4. Print summary to stdout.
