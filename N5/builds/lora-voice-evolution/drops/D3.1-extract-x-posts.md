# D3.1: Extract X/Twitter Posts

## Objective

Extract V's X/Twitter posts from the existing dataset. These provide punchy, personality-forward voice data.

## Data Source

**DuckDB X history** (516 tweets, avg 136 chars): `/home/workspace/x-history-pre-jan-8/data.duckdb`
- Table: `tweets`
- Key column: `full_text`
- Also check: `note_tweets` table (for long-form Twitter Notes, if any exist)

## Filtering Criteria

**Include:**
- Original tweets with ≥50 characters of original content
- Quote tweets where V added commentary ≥50 characters
- Thread tweets (connected by reply chain to own tweets)

**Exclude:**
- Pure retweets (text starts with "RT @")
- Tweets that are just URLs with no commentary
- Tweets shorter than 50 chars
- Tweets that are purely @mentions ("@someone yes" type replies)

## Thread Detection

If the tweets table has reply metadata (in_reply_to_status_id or similar), attempt to reconstruct threads:
- Group tweets that reply to V's own tweets
- Concatenate thread tweets into a single entry with `content_type: "thread"`
- Keep individual tweets as well for shorter-form training

Check the table schema first: `DESCRIBE tweets`
