# D1.3: Curate and Score LinkedIn Examples

## Objective

Merge LinkedIn posts and comments, then score each example for voice quality using the `/zo/ask` API. The LLM judges how well each example represents V's authentic writing voice.

## Input Files

- `N5/builds/lora-voice-evolution/artifacts/linkedin_posts_raw.jsonl` (from D1.1)
- `N5/builds/lora-voice-evolution/artifacts/linkedin_comments_raw.jsonl` (from D1.2)

## Scoring Process

For each example, call `/zo/ask` with this prompt:

```
You are evaluating a writing sample for inclusion in a voice fine-tuning dataset. The writer (V) has these voice characteristics:
- Direct, specific, warmth-through-substance communicator
- Assertive without hedging
- Earns authority through evidence and specificity
- Dry, observational humor
- Professional but human; no corporate speak

Rate this writing sample on a 1-5 scale:
5 = Strongly characteristic of a unique, authentic voice. Rich with personality markers, specific language choices, distinctive rhythm.
4 = Clearly shows personal voice. Good training signal.
3 = Decent but somewhat generic. Some voice markers present.
2 = Mostly generic or formulaic. Weak training signal.
1 = No voice signal. Could be written by anyone.

Writing sample:
"""
{text}
"""

Respond with ONLY a JSON object: {"score": N, "reason": "brief explanation"}
```

Use `output_format` for structured response:
```json
{"type": "object", "properties": {"score": {"type": "integer"}, "reason": {"type": "string"}}, "required": ["score", "reason"]}
```

**Batch processing:** Process in batches of 20 concurrent requests. Rate limit to avoid overwhelming the API.

## Output

Write to: `N5/builds/lora-voice-evolution/artifacts/linkedin_scored.jsonl`

Each line — original entry plus score:
```json
{
  "id": "linkedin_post_001",
  "source": "duckdb_shares",
  "platform": "linkedin",
  "content_type": "post",
  "text": "...",
  "char_count": 500,
  "voice_score": 4,
  "voice_reason": "Strong assertive tone, specific examples, no hedging",
  "include_in_training": true
}
```

Set `include_in_training: true` for scores ≥ 3.

## Requirements

1. Score ALL entries from both input files.
2. Use asyncio + aiohttp for concurrent API calls (max 20 concurrent).
3. Write progress to stdout every 50 examples.
4. Handle API failures gracefully — retry up to 3 times, then mark as score 0 with reason "api_error".
5. Log distribution of scores in summary.
