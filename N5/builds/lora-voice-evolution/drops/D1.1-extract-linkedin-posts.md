# D1.1: Extract LinkedIn Posts from Shares Table

## Objective

Extract all of V's original LinkedIn posts from the DuckDB dataset and the voice-library-v2 corpus. These are the **highest priority** training data — V's most intentional, voice-forward writing.

## Data Sources

1. **DuckDB shares table** (130 posts): `/home/workspace/Datasets/linkedin-full-pre-jan-10/data.duckdb`
   - Table: `shares`
   - Key columns: `shared_at`, `share_commentary`, `shared_url`, `media_url`, `visibility`
   - Query: `SELECT * FROM shares WHERE share_commentary IS NOT NULL AND LENGTH(share_commentary) > 50`

2. **Voice-library-v2 corpus** (385 entries, mostly comments): `/home/workspace/N5/builds/voice-library-v2/linkedin_corpus.jsonl`
   - Format: JSONL with keys: `id`, `type`, `text`, `timestamp`, `context_url`, `char_count`
   - Filter for `type == "post"` or entries without type that look like posts (longer form)

3. **Content library LinkedIn posts** (32 files): `/home/workspace/Knowledge/content-library/social-posts/linkedin/`
   - Markdown files with YAML frontmatter
   - Extract the body content (skip frontmatter metadata)

4. **Paragraph chunks** (150 chunks): `/home/workspace/Knowledge/voice-library/paragraph-chunks.jsonl`
   - Keys: `id`, `text`, `post_type`, `position`, `char_count`, `source_post_id`
   - These are already segmented — useful for understanding post structure

## Output

Write a single JSONL file to: `N5/builds/lora-voice-evolution/artifacts/linkedin_posts_raw.jsonl`

Each line:
```json
{
  "id": "linkedin_post_001",
  "source": "duckdb_shares|voice_library_v2|content_library|paragraph_chunks",
  "platform": "linkedin",
  "content_type": "post",
  "text": "Full text of the post",
  "char_count": 500,
  "timestamp": "2025-12-18T22:02:41",
  "metadata": {
    "shared_url": "...",
    "has_media": true,
    "originality_score": null
  }
}
```

## Requirements

1. **Deduplicate** across sources — same post may appear in multiple places. Use text similarity (first 100 chars match) to dedupe.
2. **Filter out** posts shorter than 50 characters (reactions, simple shares without commentary).
3. **Preserve** the full original text — do NOT summarize or modify.
4. **Log** stats: total extracted per source, duplicates removed, final count.
5. Print summary to stdout.
