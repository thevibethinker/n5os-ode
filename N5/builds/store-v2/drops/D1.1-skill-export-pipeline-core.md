# D1.1: Skill Export Pipeline Core

## Context

V sells Zo-optimized skills on store-va.zocomputer.io. Currently, packaging skills for sale is manual. This drop creates an automated pipeline to sanitize, package, and prepare skills for the store.

## Scope

### Files You Own
- `Skills/skill-export/SKILL.md` (create)
- `Skills/skill-export/scripts/sanitize.py` (create)
- `Skills/skill-export/scripts/generate_listing.py` (create)
- `Skills/skill-export/scripts/package.py` (create)
- `Skills/skill-export/scripts/export.py` (create - main orchestrator)

### Must NOT Touch
- `Sites/store-va/*` (D1.2 owns store integration)
- Any existing skills (read-only for testing)

## MUST DO

1. **Create `Skills/skill-export/SKILL.md`** with:
   - Frontmatter: name, description
   - Usage instructions for the export command
   - Explanation of sanitization rules

2. **Create `sanitize.py`** that:
   - Takes a skill directory path as input
   - Creates a sanitized copy in a temp directory
   - Removes/replaces:
     - Absolute paths containing `/home/workspace` → relative paths
     - References to `N5/` internal systems → generic placeholders
     - API keys, tokens (patterns like `sk-`, `FILLOUT_SECRET_*`)
     - Personal identifiers (V's email, phone patterns)
     - Zo-specific webhook URLs → `<YOUR_WEBHOOK_URL>`
   - Returns path to sanitized directory

3. **Create `generate_listing.py`** that:
   - Reads SKILL.md frontmatter (name, description)
   - Extracts features from markdown body (look for ## Features or bullet lists)
   - Outputs JSON with: id, name, tagline, description, features, badges
   - Badges auto-detected: "zo-optimized" always, "claude-code" if adapted version exists

4. **Create `package.py`** that:
   - Takes sanitized directory
   - Creates `<skill>-zo.zip`
   - Optionally creates `<skill>-claude-code.zip` (flag: --claude-code)
   - For Claude Code version: adjust any Zo-specific patterns
   - Outputs to specified directory (default: current dir)

5. **Create `export.py`** (main orchestrator) that:
   - CLI: `python3 export.py <skill-slug> [--output-dir PATH] [--claude-code]`
   - Runs: sanitize → generate_listing → package
   - Prints summary of created files
   - Outputs listing JSON to stdout (for D1.2 to consume)

## MUST NOT DO

- Do NOT modify the store site (D1.2's scope)
- Do NOT create Stripe products (already exist)
- Do NOT generate thumbnails (Phase 2)
