# D1.2 — Adapter & Template: Signal Bar + Cross-Section Consistency

## Objective

Fix two bugs in the Resume:Decoded adapter and template:

### Bug 1: Signal percentages don't total 100%

**Root cause:** `adapter.py` lines 946-947 only read `story_verified_pct` and `resume_only_pct` from `scores_complete.json`, completely ignoring `inferred_pct`. The template only renders two bars.

**Data from decomposer:**
```json
"signal_strength": {
  "story_verified_pct": 76.7,
  "resume_only_pct": 20.9,
  "inferred_pct": 2.3
}
```

**Fix required:**
1. `adapter.py` must read all three categories and pass them to template
2. `template.html` must render a third "Inferred" bar (use a distinct color — perhaps a medium purple between the screen-verified dark purple and resume-only light purple)
3. Ensure the three bars visually sum to 100% (use percentage widths)
4. Label should say "Inferred" to match the recruiter feedback about surfacing inferred exposure

### Bug 2: "End-to-end solutions" appears as both Strong Fit AND a question to probe

**Root cause:** Two independent LLM calls (`generate_tradeoffs_llm` and `generate_interview_questions_llm`) produce overlapping topics with no cross-referencing or deduplication.

- Tradeoffs (→ "Your Priorities → Fit") marks "Deliver end-to-end solutions" as **Strong fit**
- Questions (→ "Questions That Matter") includes "End-to-End Solution Delivery" as an area to **probe**

This is contradictory — if something is a strong fit with screen-verified evidence, you shouldn't be asking the interviewer to probe it. The section header "Key areas to probe" implies uncertainty.

**Fix required:**
1. After both LLM calls complete, deduplicate: if a topic appears in tradeoffs as "Strong fit", it should NOT appear in questions unless the question is specifically about depth/growth (not verification)
2. Alternatively: restructure the questions section to distinguish "Confirm" questions (for strong fits — validate depth) from "Investigate" questions (for gaps/weak areas). The current label system partially does this (`Confirms:` vs `Verifies:`) but the section header undercuts it
3. Change section subtitle from "Key areas to probe in the interview" to "Key areas to explore in the interview" — probe implies suspicion, explore implies curiosity

**Additionally, consider:**
- The `evidence_questions` prompt tells the LLM to "probe strengths deeper" — this is fine for questions about HOW they did something, but the topic overlap with tradeoffs creates cognitive dissonance for the reader
- A simple dedup: filter out any question whose `topic` fuzzy-matches a tradeoff `need` that has verdict "Strong fit"
