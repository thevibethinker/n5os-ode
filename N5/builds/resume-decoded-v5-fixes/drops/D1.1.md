# D1.1 — Decomposer Signal Categorization

## Objective

Ensure the decomposer skill (`Skills/careerspan-decomposer/`) produces signal strength percentages that sum to 100% with all three categories properly populated.

## Bug

The decomposer's `scores_complete.json` output has three signal categories:
- `story_verified_pct`: 76.7
- `resume_only_pct`: 20.9
- `inferred_pct`: 2.3

These total 99.9% (rounding), which is acceptable. **The decomposer itself is NOT the bug source for the 96% display issue** — that's in the adapter (Drop D1.2).

However, the decomposer needs review for:

1. **Inferred signal category quality**: The `inferred_pct` of 2.3% suggests very few skills are being classified as "inferred". Review whether the decomposer's skill classification logic properly identifies inferred skills (skills not explicitly stated but reasonably implied by context — e.g., "Python not mentioned but worked at X fintech infra-heavy company, highly likely Python exposure").

2. **Per-skill signal classification**: Each skill in `scores_complete.json` should have a `signal_type` field (`story_verified`, `resume_only`, or `inferred`) that downstream consumers can use. Verify this exists and is accurate.

3. **Percentage calculation robustness**: Ensure percentages always sum to exactly 100% (not 99.9% due to floating point). Round to whole numbers that sum correctly.

## Files to Examine

- `Skills/careerspan-decomposer/scripts/decompose.py` — main decomposer script, especially the scoring/signal classification logic
- `Skills/careerspan-decomposer/scripts/chunk_processor.py` — chunked processing mode

- `Careerspan/meta-resumes/inbox/hardik-flowfuse/hard_skills.yaml` — skill data with signal sources

## Success Criteria

- [ ] Signal percentages sum to exactly 100%
- [ ] Each skill has an explicit `signal_type` field
- [ ] Inferred skills are identified where contextually appropriate (not just explicit mentions)
- [ ] Scoring prompt instructs LLM to classify signal source per skill
- [ ] No regression in existing decomposer functionality
