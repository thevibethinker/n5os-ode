# D2.1 Deposit: GraphRAG Entity Extraction + Storage

## Summary
Added entity extraction and knowledge graph storage to enable "show me everything connected to X" queries.

## Files Created

### `N5/cognition/entity_extractor.py`
LLM-based entity extraction using /zo/ask API:
- `extract_entities(text)` → Returns (entities, relationships)
- Uses structured JSON output format
- Supports entity types: PERSON, CONCEPT, ORG, BELIEF, TOOL, EVENT
- Relationship types: BELIEVES, KNOWS, WORKS_WITH, USES, MENTIONS, RELATED_TO
- Built-in caching to avoid re-extraction
- Graceful timeout/error handling

### `N5/cognition/graph_store.py`
Graph CRUD and traversal:
- `add_entity(name, type, context)` — Upserts, increments mention_count
- `add_relationship(from, to, type, context)` — Creates both entities if needed
- `get_entity(name_or_id)` — Lookup by name or ID
- `get_connections(entity, depth)` — BFS traversal
- `search_entities(query, type)` — Fuzzy name search
- `get_stats()` — Graph statistics

## Files Modified

### `N5/cognition/n5_memory_client.py`
- Added `search_graph(query, entity_type, limit)` — Entity search
- Added `get_graph_connections(entity_name, depth)` — Graph traversal
- Added `hybrid_search(query, limit, graph_boost)` — Vector + graph combined
- Added GraphStore import

### `N5/scripts/brain_migrate_v2.py`
- Added graph schema creation (entities, relationships tables)

## Schema Created (brain.db only)

```sql
CREATE TABLE entities (
    id TEXT PRIMARY KEY,
    name TEXT NOT NULL,
    type TEXT NOT NULL,
    canonical_name TEXT,
    first_seen_at TIMESTAMP,
    last_seen_at TIMESTAMP,
    mention_count INTEGER DEFAULT 1,
    source_block_id TEXT,
    metadata JSON
);

CREATE TABLE relationships (
    id TEXT PRIMARY KEY,
    from_entity_id TEXT NOT NULL,
    to_entity_id TEXT NOT NULL,
    relation_type TEXT NOT NULL,
    confidence REAL DEFAULT 1.0,
    context TEXT,
    source_block_id TEXT,
    extracted_at TIMESTAMP
);

-- Indexes
idx_entities_name, idx_entities_type
idx_relationships_from, idx_relationships_to, idx_relationships_type
```
