# D1.2 Deposit: Temporal Tracking + Schema Migration

## Summary
Added temporal tracking to N5 Brain enabling "what did V believe at time X" queries and content evolution tracking.

## Files Created

### `N5/scripts/brain_migrate_v2.py`
Migration script that:
- Creates automatic backups before migration
- Adds temporal columns to blocks table (indexed_at, valid_from, valid_until, supersedes_block_id)
- Adds temporal columns to resources table (first_indexed_at, version)
- Creates graph schema (entities, relationships tables)
- Idempotent (safe to run multiple times)
- Supports --dry-run mode

## Files Modified

### `N5/cognition/n5_memory_client.py`
- Updated `index_file()` to populate temporal fields:
  - `indexed_at` = current time
  - `first_indexed_at` = preserved from first indexing
  - `version` = auto-incremented
  - `supersedes_block_id` = links to previous version's blocks
- Added `search_temporal(query, before=None, after=None, limit=10)` method
- Added `get_belief_history(topic, limit=10)` method

## Schema Changes

Applied to BOTH brain.db and vectors_v2.db:
```sql
-- blocks table
indexed_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
valid_from TIMESTAMP
valid_until TIMESTAMP
supersedes_block_id TEXT

-- resources table
first_indexed_at TIMESTAMP
version INTEGER DEFAULT 1
```
