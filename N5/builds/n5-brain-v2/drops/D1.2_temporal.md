# Drop D1.2: Temporal Tracking + Schema Migration

## Objective
Add temporal tracking to the N5 Brain so we can query "what did V believe at time X" and track content evolution.

## Scope

### Files to Create/Modify
- `N5/scripts/brain_migrate_v2.py` — NEW: Migration script for schema changes
- `N5/cognition/n5_memory_client.py` — Add temporal query methods, update index_file()

### Schema Changes (BOTH brain.db AND vectors_v2.db)
```sql
-- Add to blocks table
ALTER TABLE blocks ADD COLUMN indexed_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP;
ALTER TABLE blocks ADD COLUMN valid_from TIMESTAMP;
ALTER TABLE blocks ADD COLUMN valid_until TIMESTAMP;
ALTER TABLE blocks ADD COLUMN supersedes_block_id TEXT;

-- Add to resources table  
ALTER TABLE resources ADD COLUMN first_indexed_at TIMESTAMP;
ALTER TABLE resources ADD COLUMN version INTEGER DEFAULT 1;
```

### Must NOT Touch
- Reranker logic (that's D1.1's job)
- Entity/graph tables (that's D2.1's job)
- Files outside N5/cognition/ and N5/scripts/

## Requirements

### MUST DO
1. Create `brain_migrate_v2.py` with:
   - Automatic backup before migration
   - Schema migration for BOTH databases
   - Idempotent (safe to run multiple times)
   - Dry-run mode
2. Run migration on brain.db and vectors_v2.db
3. Update `index_file()` to populate:
   - `indexed_at` = now
   - `first_indexed_at` = now (if new resource) or preserve existing
   - `version` = increment if resource exists
   - `supersedes_block_id` = link to previous version's blocks
4. Add `search_temporal(query, before=None, after=None)` method
5. Add `get_belief_history(topic, limit=10)` method — shows how content evolved
6. Update `brain_sync_check.py` to verify temporal columns exist

### MUST NOT DO
- Change reranker logic
- Add graph/entity tables (that's next phase)
- Delete any existing data
