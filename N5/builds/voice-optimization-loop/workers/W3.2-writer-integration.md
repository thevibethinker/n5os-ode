## Learnings from W2.1

**Sample lesson format Writer will retrieve:**

```json
{
  "id": "lesson_action_first",
  "content_type": "cold_email",
  "lesson": "Open with immediate concrete action rather than tentative introductions",
  "anti_pattern": "Opening with 'I wanted to reach out because'",
  "positive_pattern": "Open with '[Name] — just [action]'",
  "confidence": "high",
  "global_candidate": false
}
```

**Memory storage format (from W3.1):**

```markdown
Voice preference (cold_email): Open with immediate concrete action rather than tentative introductions
Avoid: Opening with 'I wanted to reach out because'
Prefer: Open with '[Name] — just [action]'
```

**Sample primitives extracted:**

- `[Name] — just [action]` (opener)
- `Worth a [duration] call?` (closer)

**Key insight:** Lessons have both anti-patterns (what to avoid) and positive patterns (what to do). Writer should apply both as constraints.

## Requirements

### 1. Lesson Retrieval Script

Create `file N5/scripts/retrieve_voice_lessons.py`:

```markdown
Usage:
  python3 N5/scripts/retrieve_voice_lessons.py \
    --content-type "cold_email" \
    [--include-global] \
    [--limit 10] \
    [--format text|json]
```

**Retrieval logic:**

1. Query semantic memory for type = "voice_preference"
2. Filter by content_type tag (exact match)
3. If --include-global, also include lessons tagged "global"
4. Sort by relevance/recency (prefer recent learnings)
5. Return top N lessons

**Output format (text, default):**

```markdown
=== Voice Preferences for cold_email (3 lessons) ===

1. Lead with Action, Not Intent
   Avoid: "I wanted to...", "I'm reaching out to..."
   Prefer: Specific action that demonstrates engagement
   (learned: 2026-01-18)

2. ...
```

**Output format (JSON):**

```json
{
  "content_type": "cold_email",
  "lessons": [
    {
      "id": "lesson_001",
      "lesson": "...",
      "anti_pattern": "...",
      "positive_pattern": "...",
      "learned_at": "2026-01-18"
    }
  ],
  "global_lessons": [...]
}
```

### 2. Update Writer Prompt

Find the Writer prompt (likely `Prompts/Generate With Voice.prompt.md` or similar) and add a step:

**Before drafting, add:**

```markdown
## Voice Lessons Retrieval

Before generating content, retrieve V's voice lessons for this content type:

1. Identify the content type (cold_email, linkedin_post, memo, etc.)
2. Run: `python3 N5/scripts/retrieve_voice_lessons.py --content-type "{type}" --include-global`
3. Apply retrieved lessons to the draft:
   - Avoid listed anti-patterns
   - Use preferred patterns
   - These lessons reflect V's actual preferences from past corrections

If no lessons exist for this content type, proceed with general voice guidance.
```
