# Worker Brief: Diff + Lesson Extraction

**Your Mission:** Build the analysis layer that takes before/after pairs and extracts actionable lessons.

**Output:** Diff analyzer + lesson extractor scripts

---

## Context

This is Wave 2 of the voice-optimization-loop build. Wave 1 (W1.1) created the capture infrastructure:
- `feedback_pairs` table in `N5/data/voice_library.db`
- `N5/scripts/voice_feedback_capture.py` for storing pairs

Your job is to build the analysis layer that processes captured pairs.

## Learnings from W1.1

**Query pattern for unanalyzed pairs:**
```sql
SELECT * FROM feedback_pairs WHERE analyzed = 0 ORDER BY created_at
```

**Content type note:** The `content_type` field is free-text. Consider using a canonical list to prevent drift (e.g., `cold_email` vs `cold-email`). For now, normalize input to snake_case.

**Schema reference:**
```sql
feedback_pairs (
  id INTEGER PRIMARY KEY,
  original TEXT NOT NULL,
  improved TEXT NOT NULL,
  content_type TEXT,
  content_hash TEXT UNIQUE,
  conversation_id TEXT,
  analyzed INTEGER DEFAULT 0,
  created_at TEXT
)
```

## Requirements

### 1. Diff Analyzer

Create `N5/scripts/voice_diff_analyzer.py`:

```
Usage:
  python3 N5/scripts/voice_diff_analyzer.py analyze \
    --original "text" --improved "text" [--output json|text]
  
  python3 N5/scripts/voice_diff_analyzer.py analyze-pair --id 5
```

**Diff categories to detect:**
- **Word choice:** Different vocabulary, register
- **Sentence structure:** Length, complexity, rhythm
- **Tone:** Formal ↔ casual, warm ↔ direct
- **Directness:** Hedging removed, passive → active
- **Specificity:** Vague → concrete, generic → particular
- **Length:** Trimmed, expanded
- **Opening/closing:** Hook changes, CTA changes
- **Structure:** Reordering, section changes

**Output format (JSON):**
```json
{
  "categories": {
    "word_choice": ["replaced 'reach out' → 'recorded a video'", ...],
    "tone": ["shifted formal → conversational"],
    "directness": ["removed 'I wanted to' hedging"],
    ...
  },
  "significant_changes": [...],  // top 3-5 most notable
  "original_length": 150,
  "improved_length": 120
}
```

**Implementation approach:**
- Use LLM (via /zo/ask or direct API) for semantic analysis
- Prompt should be structured to categorize changes
- Include both specific examples and overall patterns

### 2. Lesson Extractor

Create `N5/scripts/voice_lesson_extractor.py`:

```
Usage:
  python3 N5/scripts/voice_lesson_extractor.py extract \
    --diff-json '{"categories": ...}' \
    --content-type "cold_email" \
    [--context "Partnership outreach"]
  
  python3 N5/scripts/voice_lesson_extractor.py extract-from-pair --id 5
```

**Lesson format:**
```json
{
  "lessons": [
    {
      "id": "lesson_001",
      "content_type": "cold_email",
      "lesson": "When writing cold emails, V prefers leading with a specific action taken (e.g., 'I recorded a video') rather than generic intent ('I wanted to reach out').",
      "anti_pattern": "Opening with 'I wanted to...' or similar hedging",
      "positive_pattern": "Open with concrete action or artifact",
      "confidence": "high",
      "global_candidate": false
    }
  ],
  "candidate_primitives": [
    {
      "text": "recorded a video using your product",
      "type": "phrase",
      "function": "Establishes credibility through action",
      "source": "v_edit"
    }
  ]
}
```

**Lesson extraction prompt should:**
- Be specific and actionable
- Include both what to avoid (anti_pattern) and what to do (positive_pattern)
- Flag lessons that might apply globally
- Extract any reusable phrases/moves V introduced

### 3. Combined Pipeline Script

Create `N5/scripts/voice_analyze_pair.py`:

```
Usage:
  python3 N5/scripts/voice_analyze_pair.py --id 5
```

Runs diff → extract → outputs combined analysis.
