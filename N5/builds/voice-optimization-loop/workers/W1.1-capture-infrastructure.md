# Worker Brief: Capture Infrastructure

**Your Mission:** Build the foundation for storing before/after pairs when V improves Zo's drafts.

**Output:** Database schema + capture script

---

## Context

When V improves a draft Zo generated, we want to capture that pair for learning. This worker builds the storage layer — the database table and the script to write pairs to it.

**Existing infrastructure:**
- Voice library DB: `N5/data/voice_library.db` (SQLite, already has 419 primitives)
- Memory client: `N5/cognition/n5_memory_client.py`

**Design decision (locked):**
- Store pairs permanently in SQLite
- Trigger is implicit (V provides improved version → assume learning intent)

---

## Requirements

### 1. Database Schema

Add `feedback_pairs` table to `N5/data/voice_library.db`:

```sql
CREATE TABLE IF NOT EXISTS feedback_pairs (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    original_text TEXT NOT NULL,
    improved_text TEXT NOT NULL,
    content_type TEXT NOT NULL,  -- e.g., 'cold_email', 'linkedin_post', 'memo'
    context TEXT,                 -- optional: what was the ask, who was the audience
    conversation_id TEXT,         -- for provenance
    created_at TEXT DEFAULT (datetime('now')),
    analyzed INTEGER DEFAULT 0,   -- 0 = not yet analyzed, 1 = lessons extracted
    content_hash TEXT UNIQUE      -- hash of original+improved to prevent duplicates
);

CREATE INDEX IF NOT EXISTS idx_feedback_pairs_content_type ON feedback_pairs(content_type);
CREATE INDEX IF NOT EXISTS idx_feedback_pairs_analyzed ON feedback_pairs(analyzed);
```

### 2. Capture Script

Create `N5/scripts/voice_feedback_capture.py`:

```
Usage:
  python3 N5/scripts/voice_feedback_capture.py capture \
    --original "Zo's version" \
    --improved "V's version" \
    --content-type "cold_email" \
    [--context "Partnership outreach to Julien"] \
    [--conversation-id "con_xxx"]

  python3 N5/scripts/voice_feedback_capture.py list \
    [--content-type "cold_email"] \
    [--unanalyzed-only]

  python3 N5/scripts/voice_feedback_capture.py mark-analyzed --id 5
```

**Capture behavior:**
- Compute content_hash = hash(original + improved) for dedup
- If duplicate exists, skip and return existing ID
- Otherwise insert and return new ID
- Print confirmation: "✓ Captured feedback pair #{id} ({content_type})"

**List behavior:**
- Show recent pairs with truncated previews
- Filter by content_type or analyzed status
