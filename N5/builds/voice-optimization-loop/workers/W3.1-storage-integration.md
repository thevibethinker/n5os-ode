# Worker Brief: Storage Integration

**Your Mission:** Store extracted lessons in semantic memory and voice library.

**Output:** Storage scripts + updated voice-lessons.md

---

## Context

This is Wave 3 of the voice-optimization-loop build. Previous waves created:
- W1.1: `feedback_pairs` table + capture script
- W1.2: 419 primitives synced to semantic memory
- W2.1: Diff analyzer + lesson extractor scripts

Your job is to store the extracted lessons and candidate primitives.

## Learnings from W2.1

**Recommended lesson schema (from W2.1):**
```sql
voice_lessons (
  id TEXT PRIMARY KEY,
  content_type TEXT NOT NULL,
  lesson TEXT NOT NULL,
  anti_pattern TEXT,
  positive_pattern TEXT,
  confidence TEXT DEFAULT 'medium',  -- high, medium, low
  global_candidate INTEGER DEFAULT 0,
  source_changes TEXT,  -- JSON array of specific changes that led to lesson
  source_pair_id INTEGER,
  content_hash TEXT UNIQUE,  -- for dedup
  created_at TEXT DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (source_pair_id) REFERENCES feedback_pairs(id)
)
```

**Recommended primitive schema (from W2.1):**
```sql
candidate_primitives (
  id TEXT PRIMARY KEY,
  text TEXT NOT NULL,
  type TEXT NOT NULL,  -- phrase, opener, closer, transition, structure
  function TEXT,
  source TEXT DEFAULT 'v_edit',
  source_pair_id INTEGER,
  approved INTEGER DEFAULT 0,  -- 0=pending, 1=approved, -1=rejected
  created_at TEXT DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (source_pair_id) REFERENCES feedback_pairs(id)
)
```

**Sample lesson format from W2.1 test:**
```json
{
  "id": "lesson_action_first",
  "content_type": "cold_email",
  "lesson": "Open with immediate concrete action rather than tentative introductions",
  "anti_pattern": "Opening with 'I wanted to reach out because'",
  "positive_pattern": "Open with '[Name] — just [action]'",
  "confidence": "high",
  "global_candidate": false,
  "source_changes": ["directness: Removed hedging", "opening_closing: Changed opener style"]
}
```

**Sample primitive format from W2.1 test:**
```json
{
  "text": "[Name] — just [action]",
  "type": "opener",
  "function": "Direct opener with immediate action"
}
```

**Dedup strategy:** Hash `lesson + anti_pattern + positive_pattern` to detect near-duplicates across pairs.

## Requirements

### 1. Lesson Storage to Semantic Memory

Create `N5/scripts/voice_lesson_store.py`:

```
Usage:
  python3 N5/scripts/voice_lesson_store.py store \
    --lesson-json '{"lessons": [...], "candidate_primitives": [...]}' \
    --pair-id 5

  python3 N5/scripts/voice_lesson_store.py list \
    [--content-type "cold_email"] \
    [--global-only]
```

**Memory storage format:**
For each lesson, store to semantic memory with:
- `type`: "voice_preference"
- `content`: The full lesson text (actionable, includes anti-pattern and positive-pattern)
- `tags`: [content_type, "voice_preference", "global" if applicable]
- `metadata`: { lesson_id, pair_id, content_type, confidence, created_at }

**Content format for memory:**
```
Voice preference ({content_type}): {lesson}
Avoid: {anti_pattern}
Prefer: {positive_pattern}
```

### 2. Primitive Storage to Voice Library

Update voice library with candidate primitives:

```python
# Add to primitives table
INSERT INTO primitives (text, type, function, tags, source, source_ref, approved)
VALUES (
    'recorded a video using your product',
    'phrase',
    'Establishes credibility through action',
    'cold_email,credibility',
    'v_edit',
    'pair_5',  -- reference to feedback_pairs.id
    0          -- not yet approved (needs HITL review)
);
```

**Note:** Primitives from V's edits are added with `approved = 0` for human review.
Create a simple review queue in `N5/review/voice/primitives-from-edits.md`.

### 3. Update voice-lessons.md

Append to `N5/prefs/communication/voice-lessons.md`:

```markdown
### Cold Email

#### Lesson: Lead with Action, Not Intent
**Learned:** 2026-01-18 (from pair #5)
**Pattern:** When writing cold emails, lead with a concrete action taken (e.g., "I recorded a video") rather than stating intent ("I wanted to reach out").
**Avoid:** "I wanted to...", "I'm reaching out to...", "I thought I'd..."
**Prefer:** Specific action + artifact that demonstrates engagement
```

### 4. Mark Pair as Analyzed

After storage, update the feedback_pair:

```sql
UPDATE feedback_pairs SET analyzed = 1 WHERE id = ?;
```

### 5. End-to-End Pipeline Update

Update `N5/scripts/voice_analyze_pair.py` (from W2.1) to include storage:

```
Usage:
  python3 N5/scripts/voice_analyze_pair.py --id 5 [--store]
```

With `--store`, it should:
1. Run diff analysis
2. Extract lessons
3. Store lessons to semantic memory
4. Add candidate primitives (unapproved)
5. Update voice-lessons.md
6. Mark pair as analyzed
