# Worker Brief: Primitives to Semantic Memory

**Your Mission:** Sync existing voice library primitives to semantic memory so they're retrievable contextually.

**Output:** Sync script + one-time migration

---

## Context

The voice library currently has 419 approved primitives in `N5/data/voice_library.db`. These are retrievable via `N5/scripts/retrieve_primitives.py`, but that's script-based lookup — not semantic.

We want primitives in semantic memory so:
1. They can be retrieved contextually (not just by tag)
2. Writer persona can pull relevant primitives based on content being written
3. Cross-conversation memory works

**Existing infrastructure:**
- Voice library DB: `N5/data/voice_library.db` (SQLite)
- Memory client: `N5/cognition/n5_memory_client.py`
- Current primitives table schema (check it — likely has: text, type, function, tags, source)

---

## Requirements

### 1. Understand Current Schema

First, check what's in the primitives table:

```bash
sqlite3 N5/data/voice_library.db ".schema primitives"
sqlite3 N5/data/voice_library.db "SELECT * FROM primitives LIMIT 3"
```

Document the schema in your completion report.

### 2. Create Sync Script

Create `N5/scripts/sync_primitives_to_memory.py`:

```
Usage:
  python3 N5/scripts/sync_primitives_to_memory.py sync [--dry-run]
  python3 N5/scripts/sync_primitives_to_memory.py status
```

**Sync behavior:**
1. Read all approved primitives from voice_library.db
2. For each primitive, store to semantic memory with:
   - `type`: "voice_primitive"
   - `content`: The primitive text + its function/usage
   - `tags`: Original tags + source type
   - `metadata`: { primitive_id, primitive_type, source }
3. Track which primitives have been synced (add `synced_to_memory` column or separate tracking table)
4. On subsequent runs, only sync new/updated primitives

**Memory content format:**
```
Voice primitive: "{primitive_text}"
Type: {analogy|phrase|pivot|etc}
Function: {what it accomplishes}
When to use: {conditions}
Tags: {comma-separated tags}
```

### 3. Add Auto-Sync Hook

Document how new primitives should trigger sync. Options:
- A. Manual: run sync script after adding primitives
- B. Trigger: modify primitive add script to call sync
- C. Scheduled: agent runs sync periodically

Recommend option A for now (manual), note for future automation.
