# D2.2: Verification Suite

## Objective

Create `Skills/build-promote/scripts/verify.py` — a post-promotion verification tool that validates a promoted skill is fully self-contained and functional.

## Context

After promoting a build to a skill (via promote.py), we need to verify nothing is broken. This script runs a comprehensive check suite — the same checks we did manually for the Zøde Moltbook promotion.

## Scope

### MUST DO
1. Create `Skills/build-promote/scripts/verify.py` with these CLI commands:
   - `check <skill-slug>` — Run all verification checks on `Skills/<skill-slug>/`
   - `check <skill-slug> --fix` — Auto-fix simple issues (clear pycache, etc.)
   - `--json` flag — Machine-readable output
   - `--verbose` flag — Show details for passing checks too

2. **Verification checks (in order):**

   **Check 1: Stale Path References**
   - Grep all `.py`, `.md`, `.json`, `.ts` files in `Skills/<slug>/` for references to `N5/builds/`
   - PASS: Zero matches
   - FAIL: List each stale reference with file:line
   - `--fix`: Not auto-fixable (needs semantic understanding)

   **Check 2: Script Health**
   - Run `python3 <script> --help` on every `.py` file in `Skills/<slug>/scripts/`
   - PASS: All scripts exit with code 0
   - FAIL: List scripts that fail with error output
   - `--fix`: Not auto-fixable

   **Check 3: Import Health**
   - Scan `.py` scripts for `sys.path.insert` or `sys.path.append` referencing build paths
   - Also check for relative imports that assume build directory structure
   - PASS: No build-path imports found
   - FAIL: List offending imports
   - `--fix`: Remove `sys.path` lines referencing build paths

   **Check 4: SKILL.md Completeness**
   - Verify `Skills/<slug>/SKILL.md` exists
   - Check that all scripts in `scripts/` are mentioned in SKILL.md
   - Check that frontmatter has required fields (`name`, `description`)
   - PASS: SKILL.md exists with all scripts documented
   - FAIL: List missing scripts or fields
   - `--fix`: Not auto-fixable (needs content generation)

   **Check 5: State Accessibility**
   - If `Skills/<slug>/state/` exists, verify at least one script references it
   - Try reading a file from state/ to confirm it's accessible
   - PASS: State directory is referenced and readable
   - SKIP: No state/ directory (acceptable for stateless skills)
   - FAIL: State exists but no script references it

   **Check 6: Protection Markers**
   - If `Skills/<slug>/state/` exists, verify `.n5protected` exists in it
   - PASS: Protection marker present
   - FAIL: State directory exists without protection
   - `--fix`: Create `.n5protected` with default protection metadata

   **Check 7: Build Residuals**
   - Check the source build folder (from meta.json `promoted_from` or by slug match)
   - Verify it only contains scaffolding: meta.json, PLAN.md, STATUS.md, BUILD.md, workers/, completions/, deposits/, .n5protected
   - PASS: Only scaffolding remains
   - WARN: Operational files still in build folder (not blocking but should clean up)
   - SKIP: No matching build folder found

   **Check 8: Bytecode Cleanup**
   - Check for `__pycache__/` directories in `Skills/<slug>/`
   - If found, check if `.pyc` files reference old paths (stale compiled code)
   - PASS: No pycache, or pycache is clean
   - FAIL: Stale bytecode found
   - `--fix`: Delete `__pycache__/` directories

3. **Output format:**
   ```
   ══════════════════════════════════════════
     Verification: Skills/<slug>/
   ══════════════════════════════════════════

   [✓] Stale Path References — PASS (0 references found)
   [✓] Script Health — PASS (12/12 scripts OK)
   [✓] Import Health — PASS (no build-path imports)
   [✓] SKILL.md Completeness — PASS (all 12 scripts documented)
   [✓] State Accessibility — PASS (state/ referenced and readable)
   [✓] Protection Markers — PASS (.n5protected present)
   [✓] Build Residuals — PASS (only scaffolding remains)
   [✓] Bytecode Cleanup — PASS (no stale bytecode)

   Result: 8/8 PASSED
   ```

4. **JSON output format (with --json):**
   ```json
   {
     "skill_slug": "example",
     "checks": [
       {"name": "stale_paths", "status": "pass", "details": "0 references found"},
       {"name": "script_health", "status": "fail", "details": "2/12 scripts failed", "failures": ["script_a.py", "script_b.py"]}
     ],
     "passed": 7,
     "failed": 1,
     "overall": "fail"
   }
   ```

### MUST NOT DO
- Do NOT modify skill files (except with `--fix` flag for specific auto-fixable issues)
- Do NOT use external packages — stdlib only
- Do NOT delete any skill files
- Do NOT run scripts with side effects (only `--help`)

### Expected Output
- `Skills/build-promote/scripts/verify.py` — Working CLI with `--help`
- Must handle: skills without state/, skills without SKILL.md (pre-promotion), skills with no scripts

## Technical Notes
- Use `subprocess.run` to execute scripts for health check (capture stdout/stderr, timeout 10s)
- Use `pathlib.glob` for finding files
- For stale path grep, use Python's built-in string search (no need for regex)
- Exit code: 0 if all pass, 1 if any fail
