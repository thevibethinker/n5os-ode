# D3.1: SKILL.md + End-to-End Test

## Objective

Write the build-promote SKILL.md and validate the entire pipeline end-to-end by running it on a real completed build.

## Context

Waves 1-2 produced the four scripts. This drop packages them into a proper skill with documentation and proves the pipeline works on real data.

## Scope

### MUST DO
1. Create `Skills/build-promote/SKILL.md` with:
   - Proper frontmatter (name: `build-promote`, description, compatibility, metadata)
   - Purpose: when and why to invoke this skill
   - Directory structure documentation (including the `state/` standard)
   - CLI reference for all 4 scripts with examples
   - Promotion workflow (step-by-step)
   - The `state/` skill standard documentation (what it means, why it exists)
   - Integration notes (how it composes with `pulse.py finalize`)

2. Run lineage analyzer on the full build corpus:
   ```bash
   python3 Skills/build-promote/scripts/lineage_analyzer.py scan
   ```
   Save output to `Skills/build-promote/state/lineage.json`

3. Pick ONE real completed build to promote end-to-end:
   - Choose something simple and self-contained (not a multi-build cluster)
   - Good candidates from the build list: `port-conflict-detector`, `agent-conflict-gate`, `task-system-skill`, or `systematic-debugging-integration`
   - Run: `python3 Skills/build-promote/scripts/promote.py run <slug> --dry-run` first
   - Then run the actual promotion (with `--name` flag if slug needs cleaning)
   - Then verify: `python3 Skills/build-promote/scripts/verify.py check <skill-slug>`

4. Fix any issues discovered during the E2E test

5. Write deposit JSON to `N5/builds/build-promote/deposits/D3.1.json`

### MUST NOT DO
- Do NOT promote builds that V hasn't approved
- Do NOT modify existing skills that are already working
- Do NOT pick an active build for the test (only `complete` or `finalized`)

### Expected Output
- `Skills/build-promote/SKILL.md` — Complete skill documentation
- `Skills/build-promote/state/lineage.json` — Full lineage analysis output
- One real build successfully promoted and verified
- Deposit JSON documenting the E2E results

## Notes
- This is a manual drop because the E2E test requires judgment about which build to pick and how to handle edge cases
- The SKILL.md should document the `state/` standard as a new convention for all skills going forward
- If the E2E test reveals issues in D2.1 or D2.2 scripts, fix them in place and note the fixes in the deposit
