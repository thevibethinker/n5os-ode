# Drop Brief: Pulse Engine — spawn_mode + build_mode

**Mission:** Update pulse.py and init_build.py to support `build_mode: "learning"` with manual spawn default, and add a `rush` command for overriding learning mode.

**Output:**
- `Skills/pulse/scripts/pulse.py` (UPDATE)
- `N5/scripts/init_build.py` (UPDATE)

---

## Context

The Learning-Engaged Build Mode needs two engine-level changes:

1. **Default spawn_mode to manual** when `build_mode == "learning"`. Currently, Drops auto-spawn via `/zo/ask` by default. In learning mode, pedagogical Drops should generate launcher files instead, so V can open them manually in new threads. Mechanical Drops (tagged in the Learning Landscape) can still auto-spawn.

2. **Build mode in meta.json** — init_build.py should set `build_mode: "learning"` by default. pulse.py should read this field and adjust spawn behavior.

3. **Rush command** — A new `pulse.py rush` command that overrides learning mode at various scopes (per-Drop, per-Wave, per-Build).

## Requirements

### pulse.py Changes

**1. Build mode detection:**

Add a helper function:
```python
def get_build_mode(meta: dict) -> str:
    """Get build mode from meta.json, defaulting to 'learning'."""
    return meta.get("build_mode", "learning")
```

**2. Spawn mode logic in tick():**

In the tick function, after `get_ready_drops()` returns ready Drops, check build mode before spawning:

```python
build_mode = get_build_mode(meta)

for drop_id in ready_drops:
    drop_info = meta["drops"][drop_id]
    
    # Determine effective spawn mode
    if build_mode == "rush":
        # Rush mode: everything auto-spawns
        effective_spawn = "auto"
    elif drop_info.get("spawn_mode"):
        # Explicit per-Drop override takes precedence
        effective_spawn = drop_info["spawn_mode"]
    elif drop_info.get("engagement_tag") == "mechanical":
        # Mechanical Drops auto-spawn even in learning mode
        effective_spawn = "auto"
    else:
        # Learning mode default: manual
        effective_spawn = "manual"
    
    if effective_spawn == "auto":
        # Existing auto-spawn logic
        spawn_drop(slug, drop_id, ...)
    else:
        # Generate launcher, notify V
        ensure_launcher(slug, drop_id)
        # Update status to "ready" (not "running")
        drop_info["status"] = "ready"
        drop_info["ready_at"] = datetime.now(timezone.utc).isoformat()
        print(f"[LEARNING] {drop_id} ready for manual launch. Launcher generated.")
```

Note: The existing `ensure_launcher()` function already creates launcher files. The key change is that instead of auto-spawning, we set status to "ready" and let V pick it up.

**3. Add "ready" status handling:**

The existing status flow is: pending → running → complete/failed. Add "ready" as an intermediate state for manual-launch Drops:

- `ready` Drops are NOT spawned automatically
- `ready` Drops appear in STATUS.md as "Ready for manual launch"
- V launches them using `pulse.py launch <slug> <drop_id>` (already exists)
- When V launches, status moves to `running`

In `get_ready_drops()`: exclude Drops with status "ready" (they're already surfaced, just not launched yet).

In `update_status_md()`: show "ready" Drops in a separate section.

**4. Add rush command:**

```python
rush_parser = subparsers.add_parser("rush", help="Override learning mode to auto-spawn")
rush_parser.add_argument("slug", help="Build slug")
rush_parser.add_argument("--drop", help="Auto-spawn specific Drop")
rush_parser.add_argument("--wave", help="Auto-spawn all Drops in a wave")
```

Implementation:
```python
def rush_mode(slug: str, drop_id: str = None, wave: str = None):
    """Override learning mode for specified scope."""
    meta = load_meta(slug)
    
    if not drop_id and not wave:
        # Entire build → rush mode
        meta["build_mode"] = "rush"
        save_meta(slug, meta)
        print(f"[RUSH] Build {slug} switched to rush mode. All Drops will auto-spawn.")
        return
    
    if drop_id:
        # Specific Drop
        if drop_id in meta.get("drops", {}):
            meta["drops"][drop_id]["spawn_mode"] = "auto"
            # If it's in "ready" state, move back to "pending" so next tick spawns it
            if meta["drops"][drop_id].get("status") == "ready":
                meta["drops"][drop_id]["status"] = "pending"
            save_meta(slug, meta)
            print(f"[RUSH] {drop_id} set to auto-spawn.")
        else:
            print(f"Error: Drop {drop_id} not found in {slug}")
            return
    
    if wave:
        # All Drops in a wave
        wave_drops = meta.get("waves", {}).get(wave, [])
        for did in wave_drops:
            if did in meta.get("drops", {}):
                meta["drops"][did]["spawn_mode"] = "auto"
                if meta["drops"][did].get("status") == "ready":
                    meta["drops"][did]["status"] = "pending"
        save_meta(slug, meta)
        print(f"[RUSH] Wave {wave}: {len(wave_drops)} Drops set to auto-spawn.")
    
    update_status_md(slug, meta)
```
