# Worker Brief: W1.1 — Physics Tuning

## Context

The mind map at `vrijenattawar.com/mind` has been migrated to a heterogeneous graph with:
- 20 **concept nodes** (gold diamonds) — should act as distributed hubs
- 164 **thought nodes** (colored circles) — connect TO concepts, not to each other

**Current problem:** All concept nodes cluster together in top-left instead of distributing across the canvas as hubs.

## Scope

  - Any force imports from `d3-force`

### Must NOT Touch

## Current Physics Code

```tsx
import { forceCollide, forceCenter, forceRadial } from "d3-force";

// In useEffect after graphRef is set:
graphRef.current.d3Force('charge').strength((node: any) => 
  node.type === 'concept' ? -800 : -150
);
graphRef.current.d3Force('collide', forceCollide((node: any) => 
  node.type === 'concept' ? 60 : 15
));
graphRef.current.d3Force('center', forceCenter(dimensions.width / 2, dimensions.height / 2).strength(0.05));
graphRef.current.d3Force('radial', forceRadial(200, dimensions.width / 2, dimensions.height / 2)
  .strength((node: any) => node.type === 'concept' ? 0.3 : 0)
);
```

## Goal

Make concept nodes spread out as **distributed hubs** across the canvas, with thought nodes clustering around their connected concepts.

## Approaches to Try

1. **Pre-computed initial positions** — Set `node.x` and `node.y` for concepts in a ring/grid pattern before simulation starts
2. **Stronger inter-concept repulsion** — Increase charge for concepts
3. **Custom many-body force** — Only apply between concept nodes
4. **Radial force with angle** — Distribute concepts at different angles around center
5. **Link distance by type** — Longer links between concept-thought, shorter within clusters
