# D2.2: Index Documents/ Gap + Fix Chunk Overlap

## Objective

Two tasks:
1. Index the unindexed 57% of `/home/workspace/Documents/` into the semantic memory
2. Fix the unused `overlap` parameter in `_chunk_content_markdown()` to enable overlapping context between chunks

## Context

### Indexing Gap
- `Personal/`: 97% indexed
- `Knowledge/`: 100% indexed  
- `Documents/`: only 43% indexed (161 of 378 .md files)
- `N5/prefs/`: 100% indexed

The Documents/ directory contains system docs, deliverables, and specifications that should be searchable.

### Chunk Overlap Bug
`_chunk_content_markdown()` in `n5_memory_client.py` accepts an `overlap` parameter (default 100) but never uses it. The caller `_chunk_content()` passes it:
```python
return self._chunk_content_markdown(content, max_chunk_size=chunk_size * 1.5,
                                     min_chunk_size=chunk_size // 5)
```
But note: it doesn't even pass the overlap parameter (relies on default). And inside `_chunk_content_markdown`, the `overlap` variable is never referenced in the body.

## Tasks

### Task A: Fix Chunk Overlap

### A1. Read `_chunk_content_markdown()` carefully

Find where `flush_chunk()` is called. Currently it just flushes and resets. To implement overlap:

When flushing, keep the last N characters (or lines) of the current chunk as the beginning of the next chunk.

### A2. Implement overlap

After `flush_chunk()` is called, carry over the last `overlap` characters worth of lines from the flushed chunk into the new chunk. A practical approach:

```python
# After flush, carry overlap lines into next chunk
if overlap > 0 and chunks:
    last_chunk_text = chunks[-1]['text']
    overlap_lines = []
    overlap_size = 0
    for line in reversed(last_chunk_text.split('\n')):
        if overlap_size + len(line) > overlap:
            break
        overlap_lines.insert(0, line)
        overlap_size += len(line)
    if overlap_lines:
        current_chunk_lines = overlap_lines.copy()
        current_chunk_size = overlap_size
```

**Be careful**: The overlap should not affect start_line tracking — the start_line should still reflect the actual start of new content, not the overlap prefix.

### A3. Also fix `_chunk_content()` to pass overlap through

```python
return self._chunk_content_markdown(content, max_chunk_size=chunk_size * 1.5,
                                     min_chunk_size=chunk_size // 5,
                                     overlap=overlap)
```

### Task B: Index Documents/ Gap

### B1. Find unindexed Documents/ files

```python
import sqlite3, os

db = '/home/workspace/N5/cognition/vectors_v2.db'
conn = sqlite3.connect(db)
cur = conn.cursor()
cur.execute('SELECT path FROM resources')
indexed = set(r[0] for r in cur.fetchall())
conn.close()

unindexed = []
for root, dirs, files in os.walk('/home/workspace/Documents'):
    dirs[:] = [d for d in dirs if d not in ('.git', 'node_modules', '__pycache__', 'Trash', 'Archive')]
    for f in files:
        if f.endswith('.md'):
            path = os.path.join(root, f)
            if path not in indexed:
                unindexed.append(path)

print(f"Unindexed .md files in Documents/: {len(unindexed)}")
```

### B2. Index them using the memory client

```python
import sys
sys.path.insert(0, '/home/workspace')
from N5.cognition.n5_memory_client import N5MemoryClient

client = N5MemoryClient()

for path in unindexed:
    try:
        if client.needs_indexing(path):
            with open(path, 'r', encoding='utf-8', errors='ignore') as f:
                content = f.read()
            if len(content.strip()) > 50:  # Skip near-empty files
                content_date = client.extract_content_date(path)
                client.index_file(path, content, content_date=content_date)
                print(f"Indexed: {path}")
    except Exception as e:
        print(f"Error indexing {path}: {e}")
```

**IMPORTANT**: This uses the OpenAI embedding API and will take time due to rate limiting. Expect ~0.3s per file. For ~217 files, that's about 1-2 minutes.
