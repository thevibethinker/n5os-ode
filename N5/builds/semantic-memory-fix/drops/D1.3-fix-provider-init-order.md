# D1.3: Fix Provider Init Order & Startup Waste

## Objective

Fix the provider initialization logic in `n5_memory_client.py` so it doesn't waste ~2 seconds loading the local SentenceTransformer model only to immediately discard it and switch to OpenAI.

## Context

Current `_init_provider()` flow:
1. Checks for OPENAI_API_KEY in env or secret file
2. If `provider == "openai"` and key exists → uses OpenAI ✅
3. If `provider == "local"` (default) → **loads SentenceTransformer model** (~2s) 
4. Then checks vectors_v2.db for stored vector dimensions
5. Finds 3072-dim vectors → **switches to OpenAI, discards local model** ❌

The problem: step 3 loads a heavyweight model that step 5 throws away. The auto-detection in step 4-5 should happen BEFORE step 3.

### Key File
- `N5/cognition/n5_memory_client.py` — `_init_provider()` method (starts around line ~148)

## Tasks

### 1. Read the current `_init_provider()` method carefully

Understand the full logic, including:
- How `self.provider` is set (default from env var `N5_EMBEDDING_PROVIDER`, defaults to "local")
- The OpenAI API key check from secret file
- The auto-detection from stored vector dimensions

### 2. Restructure `_init_provider()` 

New logic order:
1. Check for OpenAI API key (env + secret file) — same as now
2. If provider is explicitly "openai" and key exists → init OpenAI, return
3. **BEFORE loading local model**: auto-detect from stored vectors
   - If vectors_v2.db exists and has vectors with 3072 dims → set provider to "openai"
   - If key exists → init OpenAI, return
   - If no key → log warning, fall through to local
4. Only NOW load local SentenceTransformer (if still needed)

### 3. Test the fix

```python
import time
t0 = time.time()
from N5.cognition.n5_memory_client import N5MemoryClient
client = N5MemoryClient()
t1 = time.time()
print(f"Init time: {t1-t0:.2f}s")
print(f"Provider: {client.provider}")
print(f"Local model loaded: {client.local_model is not None}")
print(f"OpenAI client loaded: {client.openai_client is not None}")
```

**Expected**: Init time should drop by ~2s. `local_model` should be `None`. `provider` should be "openai".

## Important Constraints
- Do NOT change the behavior — only the order of operations
- The fallback to local model MUST still work if OpenAI key is missing
- The secret file check (`N5/config/secrets/openai.key`) must still work
- The auto-detect logic stays the same, just moves earlier
- Use `edit_file_llm` to make the change — do not rewrite the entire file
