# D3.1: Preprocessor Pipeline

## Objective
Create a multi-stage preprocessor pipeline for content ingestion, extending the existing `content_ingest.py` with YCB-style stages.

## Context
YCB has a clean preprocessor concept. N5 already has `content_ingest.py` with some normalization. We need to formalize this into a pipeline with clear stages.

## Pipeline Stages

```
Raw Content → Extract → Clean → Enrich → Store
```

### Stage 1: Extract
Convert source to plain text.
- HTML → use trafilatura
- PDF → extract text (pdfplumber or similar)
- Markdown → parse frontmatter + body
- Already exists in content_ingest.py - refactor into stage

### Stage 2: Clean
Remove boilerplate and normalize.
- Strip navigation, ads, footers
- Normalize whitespace
- Handle encoding issues
- Already exists in content_ingest.py - refactor

### Stage 3: Enrich (NEW)
Add metadata and derived data.
- Generate summary
- Extract entities (keywords, people, companies)
- Classify content type (article, social, profile, etc.)
- Create embeddings for semantic search
- **NEW**: Detect related content (semantic links)

### Stage 4: Store
Save to content library.
- Create item record
- Link to original source
- Update search indexes
- Already exists
