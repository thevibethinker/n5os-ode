# D4.1: Semantic Links Engine

## Objective
Implement auto-discovery of relationships between content entries using embeddings and similarity scoring.

## Context
YCB has "associations" between entries. We already have the `entry_links` table from D1.1. Now we need to populate it automatically.

## How It Works

1. **Generate embeddings** for all content items (use existing N5 embedding store)
2. **Compute similarity** between items using cosine similarity
3. **Filter** to only strong relationships (> threshold)
4. **Classify** link type (related, derived, similar_topic, etc.)
5. **Store** in `entry_links` table

class SemanticLinkEngine:
    def __init__(self, db_path, vector_store_path):
        self.db = db_path
        self.vector_store = vector_store_path
    
    def compute_links(self, item_id=None, threshold=0.75, max_links=5):
        """
        Compute semantic links for an item (or all items).
        
        Args:
            item_id: Specific item to process, or None for all
            threshold: Minimum similarity score (0.0-1.0)
            max_links: Max links to create per item
        """
        if item_id:
            return self._compute_for_item(item_id, threshold, max_links)
        else:
            return self._compute_for_all(threshold, max_links)
    
    def _compute_for_item(self, item_id, threshold, max_links):
        """Find similar items for a single item."""
        # Get embedding for this item
        embedding = self._get_embedding(item_id)
        
        # Query vector store for similar vectors
        similar = self._query_similar(embedding, k=max_links+10)  # +10 for filtering
        
        # Filter by threshold and dedupe
        links = []
        for sim_item_id, score in similar:
            if sim_item_id == item_id:
                continue
            if score < threshold:
                continue
            
            link_type = self._classify_link(item_id, sim_item_id, score)
            links.append({
                'source': item_id,
                'target': sim_item_id,
                'type': link_type,
                'strength': score
            })
        
        # Store top N
        return self._store_links(links[:max_links])

### CLI Commands

```bash
# Compute links for a specific item
n5 content links compute <item_id> [--threshold 0.75] [--max-links 5]

# Compute links for all items (batch)
n5 content links compute-all [--threshold 0.75] [--max-links 5]

# Show related items
n5 content links related <item_id> [--min-strength 0.7]
# Output:
# Related to: The Future of AI Hiring
# 
# [0.92] very_similar: AI in Recruitment (2024)
# [0.85] related: Machine Learning for HR
# [0.78] related: Algorithmic Bias in Hiring

# Show link graph (for visualization)
n5 content links graph <item_id> [--depth 2]
```
