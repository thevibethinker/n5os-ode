# D1.2: Question Tracking Schema + Extraction Logic

## Objective
Extend the Vapi DuckDB schema with tables for tracking caller questions, auto-generating prefab answers after repeated occurrences, and flagging weak/missing responses.

## Context
- Current DB: `Datasets/vapi-calls/data.duckdb` with a single `calls` table
- Current schema: id, phone_number, direction, started_at, ended_at, duration_seconds, summary, transcript, cost, raw_data
- Webhook at `Skills/vapi/scripts/webhook.ts` has an `insertCall()` function that fires on `end-of-call-report`
- The Zo API is available at `https://api.zo.computer/zo/ask` with auth via `process.env.ZO_CLIENT_IDENTITY_TOKEN`

## Schema Additions

Create these new tables in `Datasets/vapi-calls/data.duckdb`:

### call_questions
Extracted questions from each call transcript.
```sql
CREATE TABLE IF NOT EXISTS call_questions (
    id VARCHAR PRIMARY KEY,           -- UUID
    call_id VARCHAR NOT NULL,         -- FK to calls.id
    question_text VARCHAR NOT NULL,   -- The question as asked
    normalized_question VARCHAR,      -- LLM-normalized version (for dedup/counting)
    answer_text VARCHAR,              -- Frank's response (if captured)
    answer_quality VARCHAR,           -- 'strong', 'weak', 'missing', 'redirect'
    category VARCHAR,                 -- 'investment', 'product', 'technical', 'personal', 'pricing', 'other'
    extracted_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (call_id) REFERENCES calls(id) -- Note: DuckDB FK enforcement is advisory
);
CREATE INDEX IF NOT EXISTS idx_cq_call ON call_questions(call_id);
CREATE INDEX IF NOT EXISTS idx_cq_category ON call_questions(category);
```

### prefab_answers
Auto-generated answers for frequently asked questions.
```sql
CREATE TABLE IF NOT EXISTS prefab_answers (
    id VARCHAR PRIMARY KEY,
    normalized_question VARCHAR NOT NULL UNIQUE,
    answer_text VARCHAR NOT NULL,
    occurrence_count INTEGER DEFAULT 0,
    auto_generated BOOLEAN DEFAULT TRUE,
    manually_approved BOOLEAN DEFAULT FALSE,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

### question_stats (view)
```sql
CREATE OR REPLACE VIEW question_stats AS
SELECT 
    normalized_question,
    COUNT(*) as ask_count,
    COUNT(DISTINCT cq.call_id) as unique_calls,
    MODE(category) as primary_category,
    MODE(answer_quality) as typical_quality,
    MAX(extracted_at) as last_asked,
    pa.answer_text as prefab_answer
FROM call_questions cq
LEFT JOIN prefab_answers pa ON cq.normalized_question = pa.normalized_question
GROUP BY cq.normalized_question, pa.answer_text
ORDER BY ask_count DESC;
```

## Implementation: Question Extraction

### New function: `extractQuestions(callId, transcript)`

Add to `webhook.ts`, called from `insertCall()` after the call is saved:

1. Call `/zo/ask` with this prompt structure:
```
Given this phone call transcript, extract every question the CALLER asked (not the assistant).
For each question, provide:
- question_text: The exact question as spoken
- normalized_question: A clean, canonical version for dedup (e.g., "What makes Zo different?" and "How is Zo different from ChatGPT?" both normalize to "What differentiates Zo from other AI tools?")
- answer_quality: Did the assistant give a 'strong' answer, 'weak' answer, 'missing' (no answer), or 'redirect' (offered to book a call)?
- answer_text: What the assistant actually said in response (brief summary)
- category: One of: investment, product, technical, personal, pricing, other

Transcript:
{transcript}

Respond as JSON array.
```

2. Use `output_format` to get structured JSON back
3. Insert each extracted question into `call_questions`
4. After insertion, check if any `normalized_question` now has ≥3 occurrences — if so, trigger prefab answer generation

### New function: `generatePrefabAnswer(normalizedQuestion, examples)`

When a question reaches 3+ occurrences:
1. Gather the 3 best previous answers (where `answer_quality = 'strong'`)
2. Call `/zo/ask`: "Given these examples of how the assistant answered this question, generate a polished 2-3 sentence answer in V's voice style (third person, conversational, confident)."
3. Insert into `prefab_answers` with `auto_generated=true`
4. Log: "New prefab answer generated for: {question}"

### Prefab Answer Injection (prepare for D2.2)
Add a helper function `getPrefabAnswers()` that returns all prefab answers formatted as a prompt section:
```
FREQUENTLY ASKED QUESTIONS (use these answers):
Q: {normalized_question}
A: {answer_text}
...
```
This will be injected into Frank's system prompt in D2.2.
