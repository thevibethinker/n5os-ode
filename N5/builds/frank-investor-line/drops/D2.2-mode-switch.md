# D2.2: Persona Tuning + Investor vs. User Mode

## Objective
Update Frank's system prompt (in `webhook.ts`) to detect whether the caller is an investor or product user, store that context, and inject the revised `zo-101-briefing.md` plus prefab answers and voice primitives so the assistant speaks in third-person investor cadence.

## Context
- D1.1 refreshes the briefing and FAQ with investor tone
- D1.2 provides prefab answers and question tracking tables that should be referenced
- Frank currently says: "You are Assistant, V's assistant…" and doesn't differentiate callers or mention investors
- `process.env.ZO_CLIENT_IDENTITY_TOKEN` gives access to Zo API for generating prefab sections

## Requirements
1. **Mode detection prompt**
   - Add to system prompt: "If the caller identifies themselves as an investor or it is clear they're asking about fundraising, respond in investor mode. Otherwise, assume they are a user. Always ask: 'Is this investor or user mode?' in your first message. Store the chosen mode in call context for later recap." 
   - Store selection in call context (extend `raw_data` JSON or add a new DuckDB column `caller_mode VARCHAR` with values INVESTOR/USER/UNKNOWN)
2. **Third-person voice**
   - Frank should refer to V as "V" or "Vrijen" in investor mode, not "I"
   - System prompt should load the new briefing plus a new `FREQUENTLY ASKED QUESTIONS` section derived from `prefab_answers`
3. **Voice primitives injection**
   - Pull the `prefab_answers` table (normalized_question + answer_text)
   - Format as:
     ```
     Investor FAQ (auto answers):
     Q: {normalized_question}
     A: {answer_text}
     ```
   - Append to the policy segment in system prompt (after briefing) so Frank can reference them
4. **Context tracking**
   - After mode determination, include a short context note in `sendRecapEmail()` (D2.1) referencing whether the caller was investor vs user
   - Store the mode via migration: `ALTER TABLE calls ADD COLUMN IF NOT EXISTS caller_mode VARCHAR DEFAULT 'UNKNOWN';`

## Implementation Notes
- Use `getPrefabAnswers()` helper from D1.2 to assemble FAQ section
- The prompt's first message to caller could be: "Frank here — you reached the Zo investor line. Are you calling as an investor or as a Zo user?"
- Only after caller confirms investor mode should the assistant switch to investor-specific phrasing;
- In user mode, keep the tone practical and second-person friendly
- Document the mode detection logic in `Skills/vapi/assets/runbook-mode-switch.md`

## Success Criteria
- [ ] System prompt records whether caller is investor or user
- [ ] `calls.caller_mode` column tracks the result
- [ ] Prefab FAQ section is appended to the system prompt
- [ ] Frank now refers to V in third-person when investor mode is active
- [ ] Recap email (D2.1) shows caller mode in the details section
- [ ] Runbook describes how to add new investor-mode phrases
