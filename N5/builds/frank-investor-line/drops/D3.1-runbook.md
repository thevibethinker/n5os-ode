# D3.1: Integration Test + Runbook

## Objective
Prove that the new investor intelligence pipeline works end-to-end and produce a runbook describing how to operate, tune, and extend the system.

## Test Steps
1. **Simulate a call**
   - Use recorded transcript or create a synthetic call description
   - Ensure the transcript includes at least two investor-specific questions (pricing, differentiation)
2. **Run question extraction** (D1.2 logic)
   - Ensure `call_questions` rows are created with categories and answer_quality
3. **Ensure prefab answer generation triggers** when a question reaches 3 occurrences
4. **Run `sendRecapEmail()`** with updated D2.1 logic
   - Verify email includes sentiment, concerns, follow-ups, and question summary
5. **Verify mode switching** (D2.2) sets `caller_mode=INVESTOR` when caller indicates investor intent
6. **Check prefab FAQ injection** into system prompt (log current prompt from webhook startup)
7. **Export schema** and confirm migrations applied (calls.analysis JSON column, caller_mode column, new tables)

## Runbook Sections
1. **Overview** (goal, context)
2. **Voice Protocol** (caller's investor vs user mode, third-person phrasing, voice primitives list)
3. **Question Tracking Workflow** (how transcripts enter DuckDB, schema, prefab triggers)
4. **Prefab Answers Maintenance** (how to review, approve, or override auto-generated answers)
5. **Recap Oversight** (how to interpret sentiment, grade, follow-ups; how to rerun summary)
6. **Instrumentation** (where logs land, how to tail `/tmp/vapi-debug.log`, and how to rerun migrations)
7. **Troubleshooting** (common failure modes, e.g., prompt size too big, sentiment failing, Zo token missing)
8. **Future Work** (ideas for dashboards, reporting, hooking into investor pipeline)

## Outputs
- `N5/builds/frank-investor-line/artifacts/post-mortem.md`
- `N5/builds/frank-investor-line/artifacts/runbook.md`
- `Skills/vapi/assets/runbook-inventory.md` (summarized sections with references to docs)

- [ ] Runbook explains how to operate and tune the system
- [ ] Post-mortem captures test results and next steps
- [ ] Hand-off notes highlight any manual approvals needed
