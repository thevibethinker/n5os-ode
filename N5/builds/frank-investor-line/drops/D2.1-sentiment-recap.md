# D2.1: Sentiment + Insight Recap Emails

## Objective
Upgrade `sendRecapEmail()` in `webhook.ts` to produce intelligence-grade recap emails with sentiment analysis, investor insights, question tracking summary, and suggested follow-ups.

## Context
- Current `sendRecapEmail()` in `Skills/vapi/scripts/webhook.ts` sends a basic email with call details, summary, and transcript
- It uses `/zo/ask` to trigger `send_email_to_user`
- D1.2 will have added `call_questions` and `prefab_answers` tables by the time this runs
- The goal: V should open a recap email and immediately know (1) how the call went emotionally, (2) what the investor cared about, (3) what to do next

## Dependencies
- D1.2 must be complete (question extraction functions exist in webhook.ts)
- Read the D1.2 deposit to understand exact function signatures and table structure

## Implementation

### Enhanced `sendRecapEmail(data, transcript, summary)`

Replace the current email body construction with a two-step process:

**Step 1: LLM Analysis**
Call `/zo/ask` with:
```
Analyze this phone call transcript and provide a structured assessment:

Caller type: {investor/user/unknown — from call context if D2.2 mode switching exists, otherwise infer}
Transcript: {transcript}

Provide:
1. SENTIMENT: Overall caller sentiment (enthusiastic/positive/neutral/skeptical/negative) with a 1-sentence rationale
2. KEY_INTERESTS: What 2-3 topics did the caller care most about? (one line each)
3. CONCERNS: Any objections, hesitations, or unanswered concerns? (one line each, or "None detected")
4. SUGGESTED_FOLLOWUPS: 1-3 concrete next steps V should take (e.g., "Send the deck", "Book a follow-up for Thursday", "Share user metrics")
5. CALL_GRADE: A/B/C/D — how well did Frank handle this call? Brief rationale.
6. NOTABLE_QUOTES: 1-2 direct quotes from the caller that capture their main interest or concern

Respond as JSON.
```

Use `output_format` for structured response.

**Step 2: Build Rich Email**
Construct the email body (markdown) with these sections:

```markdown
## 📞 Call Intelligence Report

### Call Details
- **Direction:** {Inbound/Outbound}
- **Phone:** {number}
- **Date:** {date in ET}
- **Duration:** {duration}
- **Caller Mode:** {Investor / User / Unknown}

### Sentiment: {emoji} {sentiment_label}
{rationale}

### Call Grade: {grade}
{grade_rationale}

### Key Interests
{bulleted list}

### Concerns / Objections
{bulleted list or "None detected"}

### Questions Asked ({count})
{list from call_questions table for this call_id}
- ✅ Strong: {count}
- ⚠️ Weak: {count}  
- ❌ Missing: {count}
- 🔄 Redirected: {count}

### Suggested Follow-ups
{numbered list}

### Notable Quotes
> "{quote}"

### Full Transcript
<details>
<summary>Click to expand</summary>

{transcript}
</details>
```

Sentiment emoji mapping: enthusiastic → 🔥, positive → 😊, neutral → 😐, skeptical → 🤔, negative → 😟

**Step 3: Store Analysis**
After generating the analysis, store it in DuckDB. Add a new column to the `calls` table:
- Run migration: `ALTER TABLE calls ADD COLUMN IF NOT EXISTS analysis JSON;`
- Store the full LLM analysis JSON in this column for the call

### Error Handling
- If LLM analysis fails, fall back to the current basic email format
- Log the failure but don't block the email from sending
- If question extraction (D1.2) hasn't run yet for this call, show "Questions: Pending extraction"
