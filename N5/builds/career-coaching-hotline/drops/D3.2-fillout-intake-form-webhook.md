# D3.2: Fillout Intake Form + Webhook Pipeline

## Objective

Design the Fillout.com intake form and build the webhook pipeline that pre-processes caller submissions. When someone fills out the form before calling, their data gets stored in DuckDB so the hotline can greet them by name and reference their background.

## Context

V wants callers to optionally submit a Fillout form before calling. The form collects:
- Name
- Phone number (critical — used for caller matching)
- Resume (upload or paste)
- LinkedIn URL
- Current career stage (self-assessed)
- What they want help with

When someone calls from a phone number that matches a submission, the hotline greets them personally and references their pre-processed profile.

- Career stages: `N5/builds/career-coaching-hotline/artifacts/career-stages.md` (from D2.3)

### 2. Webhook Handler

Create: `Skills/career-coaching-hotline/scripts/intake-webhook.ts`

A Bun HTTP handler (can be a route in the main webhook server OR a separate endpoint) that:

1. Receives Fillout webhook POST
2. Validates the payload
3. Normalizes phone number to E.164 format
4. If resume uploaded: downloads and extracts text using `read_file` or pdf-parse
5. Pre-processes the submission using `/zo/ask`:
   - Extract key career signals from resume
   - Identify career stage
   - Generate a 2-3 sentence "caller brief" the hotline can reference
6. Stores everything in the `caller_profiles` DuckDB table
7. Sends V an SMS notification: "📝 New career hotline intake: [Name] — [stage] — wants help with [topic]"

### 3. DuckDB Schema for Caller Profiles

Add table to the career hotline database:

```sql
CREATE TABLE caller_profiles (
  id VARCHAR PRIMARY KEY,
  name VARCHAR NOT NULL,
  phone VARCHAR NOT NULL,     -- E.164 format
  email VARCHAR,
  linkedin_url VARCHAR,
  career_stage VARCHAR,       -- from form dropdown
  help_topic TEXT,            -- what they need help with
  resume_text TEXT,           -- extracted resume content
  caller_brief TEXT,          -- LLM-generated summary
  source VARCHAR,             -- how they heard about it
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  last_call_at TIMESTAMP,    -- updated when they call
  call_count INTEGER DEFAULT 0
);

CREATE INDEX idx_caller_phone ON caller_profiles(phone);
```

### 4. Phone Number Lookup Function

Create the `lookupCaller` function that the webhook server's tool handler will use:

```typescript
async function lookupCaller(phoneNumber: string): Promise<CallerProfile | null>
```

This normalizes the incoming phone number and queries the `caller_profiles` table.

## Integration Points

- The main webhook server (D3.1) imports `lookupCaller` to check incoming calls
- On `assistant-request`, if caller phone matches a profile:
  - Inject caller brief into system prompt
  - Change greeting to personalized version
  - Set mode to "Pre-briefed"
