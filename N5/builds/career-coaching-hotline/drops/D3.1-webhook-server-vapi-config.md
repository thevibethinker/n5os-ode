# D3.1: Webhook Server + VAPI Configuration

## Objective

Build the Bun/TypeScript webhook server that handles VAPI webhooks for the Career Coaching Hotline. This is the core runtime — equivalent to `Skills/zo-hotline/scripts/hotline-webhook.ts` but for career coaching.

## Context

The VibeThinker Hotline webhook server handles:
- `assistant-request` — returns assistant config (prompt, voice, tools, transcriber)
- `tool-calls` — executes tools (assessCallerLevel, explainConcept, etc.)
- `end-of-call-report` — logs calls, sends SMS notification to V

The Career Coaching Hotline needs the same architecture with career-specific tools and configuration.

## Inputs

- System prompt: `N5/builds/career-coaching-hotline/artifacts/career-coach-system-prompt.md`
- Tool specs: `N5/builds/career-coaching-hotline/artifacts/tool-specs.json`
- Concept map: `N5/builds/career-coaching-hotline/artifacts/concept-map.json`
- Knowledge base: `Knowledge/career-coaching-hotline/`
- VibeThinker webhook (reference): `Skills/zo-hotline/scripts/hotline-webhook.ts`

## Architecture Reference

Study `Skills/zo-hotline/scripts/hotline-webhook.ts` thoroughly. The career hotline webhook follows the same patterns:

### What carries over exactly:
- Server structure (Bun.serve)
- Webhook auth (X-Vapi-Secret header)
- Call logging to DuckDB
- SMS notification via `/zo/ask` relay
- Topic classification async
- Feedback collection
- End-of-call report handling

### What changes:
- **Port**: Different from 4243 (check PORT_REGISTRY for next available in 8420-8499 range)
- **System prompt**: Career coaching (not Zo Computer)
- **Tools**: Career-specific (assessCareerStage, getCareerRecommendations, explainCareerConcept, requestCareerSession, lookupCaller, collectFeedback)
- **Knowledge base path**: `Knowledge/career-coaching-hotline/`
- **Database path**: New DuckDB database for career hotline calls
- **Voice ID**: Read from `CAREER_HOTLINE_VOICE_ID` env var
- **First message**: Career coaching greeting in V's voice
- **Concept mapping**: Career topic aliases (from concept-map.json)
- **Caller lookup**: New tool that checks DuckDB for Fillout form submissions by phone number
- **Transcriber keywords**: Career-relevant (Careerspan, resume, LinkedIn, cover letter, etc.)

## Environment Variables

| Variable | Purpose |
|----------|---------|
| `CAREER_HOTLINE_VOICE_ID` | ElevenLabs voice ID (already set) |
| `VAPI_API_KEY` | Same VAPI key (shared) |
| `CAREER_HOTLINE_SECRET` | Webhook auth secret (V needs to create) |
| `CAREER_HOTLINE_PORT` | Server port (default from port registry) |
