# D4.1: Integration Testing + End-to-End Verification

## Objective

Verify all components work together: webhook server starts, tools respond correctly, database writes succeed, knowledge base resolves, and caller lookup functions properly.

## Tests to Run

### 1. Server Startup
- Webhook server starts without errors on assigned port
- System prompt loads correctly
- Knowledge base files are all readable
- Concept map resolves all aliases

### 2. Tool Handler Tests
- `assessCareerStage` returns valid career stage from sample answers
- `getCareerRecommendations` returns stage-appropriate recommendations
- `explainCareerConcept` resolves at least 10 different concept aliases
- `requestCareerSession` logs escalation to DuckDB and includes booking link
- `collectFeedback` stores feedback correctly
- `lookupCaller` returns null for unknown numbers, returns profile for known

### 3. Database Tests
- All tables exist in DuckDB
- Insert + query works for each table
- Phone number index works for lookups
- Caller profile CRUD operations work

### 4. Webhook Integration
- POST to /webhook with assistant-request type returns valid assistant config
- POST to /webhook with tool-calls type routes to correct handler
- POST to /webhook with end-of-call-report logs the call
- Invalid auth token returns 401

### 5. Intake Pipeline
- Fillout webhook payload processes correctly
- Phone number normalization works (various formats → E.164)
- Caller brief generation produces non-empty result
- Profile stored in DuckDB after intake
