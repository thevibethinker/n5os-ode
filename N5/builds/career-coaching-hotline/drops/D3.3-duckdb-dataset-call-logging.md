# D3.3: DuckDB Dataset + Call Logging Infrastructure

## Objective

Create the DuckDB dataset and logging infrastructure for the Career Coaching Hotline. This includes the database schema, dataset configuration, and the call analysis pipeline (ported from zo-hotline).

## Context

The VibeThinker Hotline uses `Datasets/zo-hotline-calls/data.duckdb` with tables for calls, escalations, feedback, daily_analysis, and caller_insights. The Career Coaching Hotline needs equivalent infrastructure.

## Reference

- Dataset: `Datasets/zo-hotline-calls/` (structure and schema)
- Analysis scripts: `Skills/zo-hotline/scripts/call_analysis_loop.py`, `dropoff_analyzer.py`, `call_analyzer.py`

### 2. Database Schema

Tables:

```sql
-- Calls log
CREATE TABLE calls (
  id VARCHAR PRIMARY KEY,
  started_at TIMESTAMP,
  ended_at TIMESTAMP,
  duration_seconds INTEGER,
  caller_phone VARCHAR,
  caller_profile_id VARCHAR,  -- FK to caller_profiles if matched
  topics_discussed TEXT,
  career_stage_assessed VARCHAR,
  escalation_requested BOOLEAN,
  raw_data JSON
);

-- Career session escalations (Careerspan bookings)
CREATE TABLE escalations (
  id VARCHAR PRIMARY KEY,
  call_id VARCHAR,
  name VARCHAR,
  contact VARCHAR,
  career_stage VARCHAR,
  reason TEXT,
  booking_link_sent BOOLEAN DEFAULT FALSE,
  created_at TIMESTAMP
);

-- Caller feedback
CREATE TABLE feedback (
  id VARCHAR PRIMARY KEY,
  call_id VARCHAR,
  caller_name VARCHAR,
  satisfaction INTEGER,
  comment TEXT,
  created_at TIMESTAMP
);

-- Daily analysis results
CREATE TABLE daily_analysis (
  id VARCHAR PRIMARY KEY,
  analysis_date DATE,
  period_start TIMESTAMP,
  period_end TIMESTAMP,
  total_calls INTEGER,
  substantive_calls INTEGER,
  dropoff_calls INTEGER,
  avg_duration DOUBLE,
  avg_satisfaction DOUBLE,
  career_stages_json JSON,    -- distribution of career stages
  topics_json JSON,           -- topic frequency
  careerspan_conversions INTEGER, -- escalations to Careerspan
  patterns_json JSON,
  improvements_json JSON,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- Caller profiles (from Fillout form intake)
CREATE TABLE caller_profiles (
  id VARCHAR PRIMARY KEY,
  name VARCHAR NOT NULL,
  phone VARCHAR NOT NULL,
  email VARCHAR,
  linkedin_url VARCHAR,
  career_stage VARCHAR,
  help_topic TEXT,
  resume_text TEXT,
  caller_brief TEXT,
  source VARCHAR,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  last_call_at TIMESTAMP,
  call_count INTEGER DEFAULT 0
);

CREATE INDEX idx_caller_phone ON caller_profiles(phone);

-- Caller insights (merged from calls + profiles)
CREATE TABLE caller_insights (
  id VARCHAR PRIMARY KEY,
  first_name VARCHAR,
  phone VARCHAR,
  topics_discussed TEXT,
  career_stage VARCHAR,
  total_calls INTEGER DEFAULT 0,
  avg_satisfaction DOUBLE,
  first_call_at TIMESTAMP,
  last_call_at TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```
