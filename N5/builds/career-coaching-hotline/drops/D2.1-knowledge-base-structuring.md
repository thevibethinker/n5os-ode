# D2.1: Knowledge Base Structuring

## Objective

Transform the 24 extracted source documents from D1.1 into a structured, voice-optimized knowledge base that the hotline's `explainConcept` tool can serve. Follow the pattern established by `Knowledge/zo-hotline/` but adapted for career coaching topics.

## Context

The VibeThinker Hotline uses a knowledge base at `Knowledge/zo-hotline/` with ~46 files organized by topic (levels, tactics, patterns, etc.). Each file is written in a voice-optimized format — concise, practical, structured for an LLM to summarize on a phone call.

This Drop creates the equivalent for career coaching at `Knowledge/career-coaching-hotline/`.

## Inputs

- All extracted markdown files in `N5/builds/career-coaching-hotline/artifacts/extracted/`
- The extraction manifest at `N5/builds/career-coaching-hotline/artifacts/extraction-manifest.json`

## Reference Architecture

Study the structure at `Knowledge/zo-hotline/` — specifically:
- How files are numbered by category (00-, 10-, 20-, etc.)
- How each file is written for voice delivery (concise, actionable)
- The concept mapping in `Skills/zo-hotline/scripts/hotline-webhook.ts` (the `conceptFiles` object)

### 2. Concept Mapping File

Create: `N5/builds/career-coaching-hotline/artifacts/concept-map.json`

A JSON object mapping concept aliases to file paths (same pattern as `conceptFiles` in the zo-hotline webhook). Include multiple aliases per concept (kebab-case, snake_case, natural language).

### 3. Writing Guidelines

Each knowledge file MUST follow these rules:
- **Voice-optimized**: Written as if being summarized on a phone call
- **Concise**: Each file should be 200-500 words (not the full PDF content)
- **Actionable**: End with specific things the caller can do
- **V's voice**: Use V's style — direct, practical, occasionally irreverent
- **Frontmatter**: Include `source_file`, `topics`, `provenance`

- [ ] Each file is voice-optimized (not raw PDF dumps)
- [ ] Concept map JSON covers all files with multiple aliases
