# D4.2: Value Prop Tree + Careerspan Funnel Logic (MANUAL — requires V review)

## Objective

Finalize the personalized Careerspan sell logic — the value prop tree that determines what to recommend based on who the caller is and what they need. This is the commercial heart of the hotline.

## Context

V's core insight: "Everyone needs to do the groundwork first." The hotline gives free career guidance, but the natural next step for deeper work is Careerspan. The sell should feel organic — like a coach saying "here's what you need, and here's where you can get help doing it."

The value prop tree personalizes the pitch:
- A student hears about internship coaching
- A career changer hears about industry transition guidance
- A laid-off professional hears about rapid job search strategy
- A senior exec hears about strategic positioning

## Inputs

- Career stages: `N5/builds/career-coaching-hotline/artifacts/career-stages.md` (from D2.3)
- Value prop tree draft: `N5/builds/career-coaching-hotline/artifacts/value-prop-tree.md` (from D2.3)
- System prompt: `N5/builds/career-coaching-hotline/artifacts/career-coach-system-prompt.md` (from D2.2)
- Working webhook server: `Skills/career-coaching-hotline/scripts/hotline-webhook.ts` (from D3.1)

## Deliverables

### 1. Finalized Value Prop Tree

Update: `N5/builds/career-coaching-hotline/artifacts/value-prop-tree.md`

For each career stage, define:
- **Opening hook**: The one sentence that makes them lean in
- **Careerspan pitch**: What specifically Careerspan does for their situation
- **Call to action**: The specific next step (booking link, form, etc.)
- **Transition language**: How V naturally moves from advice to recommendation
- **Objection handling**: Common pushback and V's response

### 2. System Prompt Update

Update the Careerspan positioning section in the system prompt to incorporate the finalized value prop tree. The prompt should guide the AI to:
- Match caller to career stage
- Use stage-appropriate language
- Transition naturally (not abruptly)
- Handle "is this a sales pitch?" gracefully

### 3. getCareerRecommendations Tool Update

Update the tool handler in the webhook server to return stage-appropriate Careerspan recommendations as part of its response.

## Success Criteria

- [ ] Value prop tree covers all career stages
- [ ] Each stage has specific, non-generic language
- [ ] System prompt updated with finalized positioning
- [ ] Transition from advice to recommendation feels natural
- [ ] Tool handler returns correct recommendations per stage
