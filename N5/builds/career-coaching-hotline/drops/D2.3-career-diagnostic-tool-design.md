# D2.3: Career Diagnostic Tool Design

## Objective

Design the Socratic diagnostic question framework and scoring system that the hotline uses to understand where a caller is in their career journey. This replaces Zoseph's "assessCallerLevel" with a career-specific diagnostic.

## Context

V uses diagnostic questions to understand where someone is in their career. The hotline should ask targeted questions that:
1. Quickly categorize the caller's career stage
2. Identify their specific pain points
3. Determine which knowledge base topics are most relevant
4. Inform the value prop tree (which Careerspan pitch fits)

## Inputs

- Extracted source material: `N5/builds/career-coaching-hotline/artifacts/extracted/`
- Zoseph's assessment system: `Knowledge/zo-hotline/60-assessment/questions.md` and `scoring.md`

## Reference

Study Zoseph's assessment:
- 4 questions → level 1-3 assessment
- Simple A/B/C/D format
- Quick enough for phone

The career diagnostic should be similarly concise but career-specific.

## Deliverables

### 1. Career Stage Framework

Create: `N5/builds/career-coaching-hotline/artifacts/career-stages.md`

Define 4-5 career stages based on V's coaching material:
- **Early Career** — students, recent grads, first job seekers
- **Transition** — switching industries, roles, or getting back in
- **Growth** — employed but want advancement, better role
- **Senior/Executive** — experienced, strategic career moves
- **Crisis** — laid off, urgent job search

### 2. Diagnostic Questions

Create: `N5/builds/career-coaching-hotline/artifacts/diagnostic-questions.md`

Design 3-4 Socratic questions that:
- Can be asked conversationally on a phone call
- Each has clear mapping to career stages
- Are open-ended enough for natural conversation but structured enough for tool scoring
- Follow Zoseph's voice discipline (one question at a time, max 2 options)

### 3. Tool Specifications

Create: `N5/builds/career-coaching-hotline/artifacts/tool-specs.json`

Define the VAPI tool schemas for:

1. **assessCareerStage** — Processes diagnostic answers → career stage + pain points
2. **getCareerRecommendations** — Stage-appropriate next steps + relevant knowledge base topics
3. **explainCareerConcept** — Pull career coaching concept from knowledge base
4. **requestCareerSession** — Escalate to Careerspan booking
5. **collectFeedback** — Same as Zoseph's
6. **lookupCaller** — Check if caller's phone number matches a Fillout submission

### 4. Value Prop Tree

Create: `N5/builds/career-coaching-hotline/artifacts/value-prop-tree.md`

Map career stages to Careerspan value propositions:
- For each stage, what specific Careerspan offering is most relevant?
- What's the natural transition from free advice → paid session?
- What language does V use to make this transition feel organic?

## Success Criteria

- [ ] Career stages are grounded in V's actual coaching categories
- [ ] Diagnostic questions work in voice conversation format
- [ ] Tool specs are valid JSON matching VAPI function calling format
- [ ] Value prop tree maps every stage to a Careerspan offering
- [ ] All artifacts written to specified paths
