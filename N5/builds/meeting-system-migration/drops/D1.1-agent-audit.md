# D1.1: Full Agent Audit

## Context

V has 35+ scheduled agents. Many have accumulated over time and may be:
- Redundant (doing overlapping work)
- Obsolete (disabled but not deleted)
- Inefficient (running too frequently)
- Consolidatable (multiple agents could be one)

**This is AUDIT ONLY - do not change anything.**

## Scope

### Files to Read
- Current agent list via `list_agents` tool
- Agent instructions (analyze each one)

### Responsibilities
- Audit all scheduled agents
- Categorize by domain (health, meetings, CRM, deals, events, system, etc.)
- Identify consolidation opportunities
- Flag disabled agents that should be deleted
- Flag agents with overlapping responsibilities
- Note frequency concerns (too often/too rare)

### Must NOT Touch
- Do NOT delete any agents
- Do NOT modify any agents
- Do NOT create any agents
- This is observation only

## MUST DO

1. Call `list_agents` to get the full list
2. For each agent, analyze:
   - Domain/category
   - Active vs disabled status
   - Frequency (RRULE)
   - What it does (instruction summary)
   - Dependencies on scripts/prompts
3. Group agents by domain
4. Identify:
   - **Consolidation candidates**: Agents that could be merged
   - **Deletion candidates**: Disabled agents with no value
   - **Frequency concerns**: Running too often or too rarely
   - **Dependency risks**: Relying on deprecated scripts/prompts
5. Write findings to deposit

## MUST NOT DO

- Do not modify any agents
- Do not execute any agent changes
- Do not delete anything
- Do not create new agents
