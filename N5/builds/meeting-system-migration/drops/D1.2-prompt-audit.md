# D1.2: Meeting Prompt Audit

## Context

The meeting-ingestion skill has been created and some prompts were already deleted. We need to identify ALL remaining meeting-related prompts and determine which should be deleted vs kept.

The skill (`Skills/meeting-ingestion/`) now handles:
- Pull (Drive ingestion)
- Process (manifest + blocks)
- Archive (to be added in D2.1)

## Scope

### Files to Search
- `/home/workspace/Prompts/` - All prompt files
- `/home/workspace/Prompts/Blocks/` - Block generation prompts
- `/home/workspace/N5/workflows/` - Workflow prompts

### Responsibilities
- Find all meeting-related prompts
- Categorize as: DELETE (replaced by skill), KEEP (still needed), ARCHIVE (historical value)
- Check if any active agents depend on prompts marked for deletion

### Must NOT Touch
- Do NOT delete any files
- Do NOT modify any files
- This is audit only

## MUST DO

1. Search for all meeting-related prompts:
   ```bash
   find /home/workspace/Prompts -iname '*meeting*' -type f
   find /home/workspace/Prompts/Blocks -name 'Generate_B*.prompt.md' -type f
   find /home/workspace/N5/workflows -iname '*meeting*' -type f
   ```

2. For each prompt found, determine:
   - Is it replaced by the skill? → DELETE
   - Is it still needed by active agents? → KEEP (for now)
   - Is it historical but valuable? → ARCHIVE
   - Is it a block generator (B01, B05, etc.)? → KEEP (used by processor.py)

3. Cross-reference with active agents:
   - MG-1 uses: Meeting Manifest Generation.prompt.md
   - MG-2 uses: Meeting Block Generation.prompt.md  
   - MG-4 uses: Meeting Warm Intro Generation.prompt.md
   - MG-6 uses: Meeting State Transition.prompt.md
   - These will be deleted AFTER agents are migrated (D3.1)

4. Check `Skills/meeting-ingestion/references/legacy_prompts.md` for what's already documented as deleted

## MUST NOT DO

- Do not delete any files
- Do not modify any files
- Do not move any files
