# D1.1: Audit Active Builds

## Objective

Check each of the 8 builds marked "active" in meta.json, determine their true status, and fix the metadata.

## Builds to Audit

1. `task-system-wiring`
2. `gamma-survey-dashboard`
3. `b05-backfill-extended`
4. `workos-integration`
5. `personal-site-v1`
6. `n5os-ode-v2`
7. `store-v2`
8. `meeting-ingestion-skill`

## For Each Build

1. Check if workers/drops directory has briefs
2. Check if completions/deposits directory has outputs
3. Read PLAN.md or BUILD.md to understand scope
4. Determine true status:
   - `complete` - all work done
   - `abandoned` - started but won't continue
   - `blocked` - waiting on something specific
   - `active` - genuinely in progress
5. Update meta.json with correct status

## Output

Deposit JSON with:
```json
{
  "builds_audited": 8,
  "verdicts": {
    "<build-name>": {
      "previous_status": "active",
      "true_status": "<status>",
      "reason": "<explanation>",
      "action_taken": "updated meta.json" | "no change needed"
    }
  },
  "summary": "<brief summary>"
}
```
