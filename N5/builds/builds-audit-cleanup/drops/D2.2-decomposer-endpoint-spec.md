# D2.2: Document Decomposer Endpoint Integration

## Objective

Document what the `careerspan-decomposer` skill needs to receive from Careerspan's API endpoint (push model instead of file-based pull).

## Context

- Current: Decomposer reads local files from `Careerspan/meta-resumes/inbox/<candidate>/`
- Future: Careerspan backend will push data to a Zo endpoint
- V clarified: "Careerspan is going to push to us. We're not going to pull from Careerspan."

## Deliverable

Create `Skills/careerspan-decomposer/references/ENDPOINT_SPEC.md` documenting:

1. **Expected Payload Schema** - What Careerspan should send
   - Candidate metadata
   - Raw brief content (text or structured)
   - JD content
   - Any flags/options

2. **Endpoint Requirements**
   - Where webhook should hit (Zo service registration)
   - Authentication approach
   - Response format

3. **Integration Path**
   - How to wire incoming webhook → decomposer script
   - Output location for processed YAML
   - Trigger for meta-resume-generator

## Note

This is `spawn_mode: manual` because V may want to provide input on the exact schema Careerspan will use.
