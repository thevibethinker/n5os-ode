# Drop 1.2: Dual-Sided Audit System

## Objective
Create an immutable, independently maintained audit trail on BOTH va.zo.computer and zoputer.zo.computer. Every communication between the two systems is logged with cryptographic verification to detect any discrepancies.

## Requirements

### Database Schema (SQLite)
Table: `consulting_communications`
```sql
CREATE TABLE consulting_communications (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    timestamp TEXT NOT NULL,  -- ISO 8601
    direction TEXT NOT NULL,  -- 'va_to_zoputer' | 'zoputer_to_va'
    message_type TEXT,        -- 'api_call' | 'email_relay' | 'export_bundle'
    content_hash TEXT NOT NULL,  -- SHA-256 of content
    metadata_json TEXT,       -- JSON with context
    checksum TEXT NOT NULL,   -- Hash of (timestamp + direction + content_hash)
    created_at TEXT DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX idx_timestamp ON consulting_communications(timestamp);
CREATE INDEX idx_direction ON consulting_communications(direction);
CREATE INDEX idx_content_hash ON consulting_communications(content_hash);
```

### va.zo.computer Implementation
- Location: `N5/data/consulting_audit.db`
- Logger: `N5/scripts/audit_logger.py log --direction va_to_zoputer --content "..." --type api_call`
- Automatic logging on all zoputer_client.py calls

### zoputer.zo.computer Implementation
- Location: `N5/data/consulting_audit.db`
- Logger: `N5/scripts/audit_logger.py log --direction zoputer_to_va --content "..." --type api_call`
- Automatic logging on all ingestion endpoint calls

### Verification Script
Build `N5/scripts/audit_verify.py` that:
1. Queries both databases for given time range
2. Compares content_hash for matching timestamps
3. Reports discrepancies (missing entries, hash mismatches)
4. Generates reconciliation report

### Sync Protocol
- Daily at 6 PM ET: Run verification
- On discrepancy: Alert V via SMS with details
- Monthly: Export audit trail to Google Drive for long-term storage

## Files to Create
- `N5/scripts/audit_logger.py` - CLI for logging
- `N5/scripts/audit_verify.py` - Cross-system verification
- `N5/scripts/audit_query.py` - Query interface for V
- `Documents/consulting/audit-schema.md` - Schema documentation

## Success Criteria
- [ ] Every API call between systems is logged within 1 second
- [ ] Content hashes match on both sides for 100% of communications
- [ ] Verification script runs in <10 seconds for daily check
- [ ] V can query any communication by timestamp in <30 seconds

## Notes
- This is NON-NEGOTIABLE. No communication escapes logging.
- Use WAL mode for SQLite to prevent corruption
- Consider compressing old records (>90 days) to save space
