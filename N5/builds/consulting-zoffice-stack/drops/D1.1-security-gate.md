# Drop 1.1: Security Gate Validator

## Objective
Create an event-triggered security validation system for all client-facing communications. This gate analyzes inbound emails for adversarial patterns, prompt injection, and social engineering before any processing occurs.

## Requirements

### Core Functionality
1. Build `/zo/ask` prompt template for security analysis
2. Create wrapper script (`Skills/security-gate/scripts/validate.py`) that:
   - Accepts email body + sender metadata as input
   - Calls /zo/ask with security audit prompt
   - Returns structured risk assessment
3. Define three risk levels:
   - **LOW**: Proceed with normal processing
   - **MEDIUM**: Hold for V review (15-min window)
   - **HIGH**: Quarantine + immediate SMS alert to V

### Adversarial Patterns to Detect
- Prompt injection markers: "ignore previous", "DAN", "sudo", "developer mode"
- Social engineering: urgency triggers, authority impersonation, excessive flattery
- Scope creep: requests for file listings, configs, personal information
- Jailbreak framing: roleplay requests, fictional scenarios presented as real
- Data exfiltration: asking for API keys, secrets, PII, operational details

### Output Schema
```json
{
  "risk_level": "low|medium|high",
  "flags": ["pattern1", "pattern2"],
  "recommended_action": "proceed|hold_for_review|quarantine",
  "rationale": "explanation string",
  "confidence": 0.0-1.0
}
```

### Test Suite
Create test file with 20+ examples:
- 5 known benign emails
- 5 prompt injection attempts
- 5 social engineering patterns
- 5 scope creep attempts

All tests must pass before drop is complete.

## Files to Create
- `Skills/security-gate/SKILL.md` - Usage documentation
- `Skills/security-gate/scripts/validate.py` - Main validation script
- `Skills/security-gate/tests/test_suite.py` - Test cases
- `Skills/security-gate/config/patterns.json` - Detectable patterns library

## Success Criteria
- [ ] Script runs in <5 seconds per email
- [ ] 100% detection rate on known adversarial patterns in test suite
- [ ] Zero false positives on benign professional inquiries
- [ ] Logs all decisions to audit system (Drop 1.2)
