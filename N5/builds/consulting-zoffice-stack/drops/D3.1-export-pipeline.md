# Drop 3.1: Daily Export Pipeline

## Objective
Build the automated daily export that bundles curated skills from va and ships them to zoputer via the API layer.

## Requirements

### Pipeline Flow

```
9:00 AM ET trigger
├── 1. Run content classifier (Drop 2.2)
│   └── Generate CONSULTING_MANIFEST.json
├── 2. Filter for Tier 0 content modified since last export
├── 3. Bundle each skill:
│   ├── Create tarball
│   ├── Generate metadata.json
│   └── Calculate checksums
├── 4. Upload via zoputer_client.py (Drop 1.3)
├── 5. Verify ingestion
├── 6. Update CONSULTING_MANIFEST.md
└── 7. Notify V via SMS
```

### Export Format

Each skill bundle:
```
content-classifier-v1.2.3-20260206.tar.gz
├── SKILL.md
├── scripts/
├── assets/
├── references/
└── metadata.json

metadata.json:
{
  "name": "content-classifier",
  "version": "1.2.3",
  "exported_from": "va.zo.computer",
  "exported_at": "2026-02-06T09:00:00Z",
  "git_sha": "abc123...",
  "files": [...],
  "checksums": {...},
  "notes": "Added tier detection"
}
```

### Scheduled Agent

Create agent `DAILY_EXPORT_LIBRARIAN`:
- Schedule: Daily 9 AM ET
- Command: `python3 Skills/librarian-export/scripts/daily_export.py`
- Delivery: SMS notification to V on completion/error

### Change Detection

Only export skills with:
- Git changes since last export timestamp
- OR manual force flag
- OR first-time export

Store last export timestamp in `N5/data/last_export.json`

### Verification

After export:
1. Query zoputer endpoint to confirm receipt
2. Verify checksums match
3. Log success/failure to audit system
4. Update local export log

### SMS Notification

On success:
"[Zoffice] Daily export complete. 3 skills synced to zoputer: content-classifier, security-gate, audit-logger. All checksums verified."

On failure:
"[Zoffice] ⚠️ Export failed: content-classifier upload timeout. 2/3 skills synced. Check email for details."

## Files to Create
- `Skills/librarian-export/SKILL.md` - Export system guide
- `Skills/librarian-export/scripts/daily_export.py` - Main pipeline
- `Skills/librarian-export/scripts/bundle_skill.py` - Bundle creator
- `N5/data/last_export.json` - Export state

## Success Criteria
- [ ] Runs automatically at 9 AM ET daily
- [ ] Only exports changed content (efficient)
- [ ] V receives SMS confirmation within 5 minutes
- [ ] 100% checksum verification on successful exports
- [ ] Failed exports are retried 3x before alerting

## Notes
- Export is ONE-WAY: va → zoputer only
- Never sync back from zoputer (prevent contamination)
- Keep 30 days of export history for rollback
- Manual export possible: `python3 daily_export.py --force skill-name`
