# Drop 2.2: Content Classification System

## Objective
Build a scanner that categorizes all content in va's workspace into export tiers. This determines what can safely flow to zoputer and what must stay private.

## Requirements

### Tier Definitions

**Tier 0: Safe to Export**
- Skills/ (with API keys redacted)
- Prompts/ (generic patterns)
- Documents/System/ (architecture, setup guides)
- N5/docs/ (documentation, conventions)
- Examples without PII

**Tier 1: Review Required**
- Documents with personal examples that could be anonymized
- Architecture docs mentioning specific companies/people
- Skills with embedded personal workflows

**Tier 2: Never Export**
- Personal/ (all contents)
- Zo/ (identity space)
- Anything with .n5protected marker
- Configs with API keys, secrets
- Meeting transcripts, emails
- Deal intel, pipeline data
- Health data, biometrics
- Calendar, contacts

### Scanner Implementation

Build `Skills/content-classifier/scripts/scan.py`:

```bash
# Scan and classify
python3 Skills/content-classifier/scripts/scan.py classify \
  --path /home/workspace/Skills/content-classifier \
  --output json

# Generate export manifest
python3 Skills/content-classifier/scripts/scan.py manifest \
  --since "2026-02-01" \
  --output N5/builds/consulting-zoffice-stack/CONSULTING_MANIFEST.json

# Check specific path
python3 Skills/content-classifier/scripts/scan.py check \
  --path /home/workspace/Personal
```

### Classification Signals

**Automatic Tier 0:**
- Contains SKILL.md with proper structure
- No .n5protected in path
- No API keys in code (regex scan)
- No references to "Vrijen", "Careerspan", "va.zo.computer"

**Automatic Tier 2:**
- Path contains Personal/, Zo/, Inbox/, Datasets/
- Contains .n5protected file
- Contains "TODO(v)", "FIXME(v)", "NOTE(v)" markers
- Contains email addresses, phone numbers

**Tier 1 (Requires Review):**
- Documents/System/ files with personal examples
- Skills with hardcoded personal preferences
- Anything ambiguous

```json
    }
  ]
}
```
