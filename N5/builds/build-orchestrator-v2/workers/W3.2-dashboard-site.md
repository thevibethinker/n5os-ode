# Worker Brief: Build Dashboard Site

**Your Mission:** Create a simple dashboard site showing all active builds (Domino's pizza tracker style).

**Output:** `Sites/build-tracker/` (Zo Site)

---

## Context from Previous Waves

**W3.1 creates `build_status.py`** with:
- `list --json` outputs structured JSON for all builds
- `regenerate` writes to `Sites/build-tracker/data/builds.json`
- JSON schema includes: slug, title, status, type, created, workers (complete/total), progress_pct, is_stale, path

**W1.1 created system documentation** at `Documents/System/Build-Orchestrator-System.md` — reference for terminology and flow.

**Data contract:** Your site reads from `data/builds.json`. The schema is:
```json
{
  "generated_at": "2026-01-18T18:30:00Z",
  "builds": [
    {
      "slug": "calendar-v2",
      "title": "Calendar Integration v2", 
      "status": "active",
      "type": "code_build",
      "created": "2026-01-15",
      "workers": { "complete": 3, "total": 5 },
      "progress_pct": 60,
      "is_stale": false,
      "path": "N5/builds/calendar-v2"
    }
  ],
  "summary": {
    "total": 4,
    "active": 2,
    "complete": 2,
    "stale": 1
  }
}
```

## Design Requirements

### Layout

```
┌────────────────────────────────────────────────┐
│  🏗️ Build Tracker                              │
│  2 active · 1 stale · 15 complete              │
├────────────────────────────────────────────────┤
│                                                │
│  calendar-v2                        60%        │
│  Calendar Integration v2                       │
│  ██████████████████░░░░░░░░░░░░  3/5 workers   │
│  code_build · started Jan 15                   │
│                                                │
│  crm-refresh                ⚠️ STALE  33%      │
│  CRM Data Refresh                              │
│  ██████████░░░░░░░░░░░░░░░░░░░░  1/3 workers   │
│  code_build · started Jan 10 · no activity 8d │
│                                                │
├────────────────────────────────────────────────┤
│  Recent Completions                            │
│  ✓ auth-module (Jan 8) · ✓ docs-update (Jan 5)│
└────────────────────────────────────────────────┘
```

### Visual Elements

1. **Progress bars** — Color-coded:
   - Green: >75% complete
   - Yellow: 25-75% complete  
   - Red: <25% complete or stale

2. **Stale indicator** — ⚠️ badge + "no activity Xd"

3. **Build type badge** — Small pill (code_build, content, research, general)

4. **Quick actions** (optional):
   - Refresh button (re-reads builds.json)

### Technical Implementation

- Use Zo Sites (Hono + JSX)
- Read `data/builds.json` on each request
- Simple, clean CSS — no heavy frameworks
- Mobile-friendly

### Refresh Mechanism

Data refreshes when:
1. `build_status.py regenerate` is called (on orchestrator check-in)
2. Page reload

No live polling needed — static data is fine.
