# Worker Brief: init_build.py Upgrade

**Your Mission:** Upgrade `N5/scripts/init_build.py` to support v2 orchestration.

**Output:** Updated `N5/scripts/init_build.py`

---

## Context

The current init_build.py creates a basic folder with PLAN.md and STATUS.md. It needs to be upgraded to:
- Create the full v2 folder structure
- Support different build types
- Pre-screen for incomplete builds
- Generate meta.json
- Optionally scaffold worker brief stubs

---

## Required Changes

### 1. New Folder Structure

```
N5/builds/<slug>/
├── BUILD.md          # From BUILD_template.md
├── PLAN.md           # Existing (Architect's plan)
├── STATUS.md         # Existing
├── meta.json         # NEW - machine-readable state
├── workers/          # NEW - worker briefs folder
├── completions/      # NEW - completion reports folder
└── .n5protected      # Existing
```

### 2. New CLI Arguments

```bash
python3 N5/scripts/init_build.py <slug> \
  --title "Build Title" \
  --type code_build|content|research|general \
  --workers 5 \  # Optional: pre-generate N worker brief stubs
  --force \      # Existing: overwrite if exists
  --skip-prescreen  # Skip incomplete build check
```

### 3. Pre-Screen Logic

Before creating:
1. Check `N5/builds/*/meta.json` for `status: active`
2. If any have similar slug or title, warn:
   ```
   ⚠️  Found 2 incomplete builds:
      - calendar-v1 (3/5 workers complete)
      - calendar-refresh (active since 2026-01-10)
   Continue anyway? [y/N]
   ```
3. In non-interactive mode, just warn and continue (or fail with `--strict`)

### 4. meta.json Generation

Create initial meta.json:
```json
{
  "schema_version": "1.0",
  "slug": "<slug>",
  "title": "<title>",
  "created": "2026-01-18",
  "status": "draft",
  "completed_at": null,
  "type": "<type>",
  "orchestrator_convo_id": null,
  "workers": {
    "total": 0,
    "complete": 0,
    "in_progress": 0,
    "blocked": 0,
    "pending": 0
  },
  "waves": []
}
```

### 5. Worker Brief Stubs (Optional)

If `--workers N` is provided, generate N stub files:
- `workers/W1.1-task-1.md`
- `workers/W1.2-task-2.md`
- etc.

Use worker_brief_template.md with placeholders.

### 6. Output

Print:
```
✓ Build workspace created: N5/builds/<slug>/
  ├── BUILD.md      (orchestrator document)
  ├── PLAN.md       (fill with Architect)
  ├── STATUS.md     (progress tracking)
  ├── meta.json     (build state)
  ├── workers/      (worker briefs go here)
  ├── completions/  (worker reports go here)
  └── .n5protected  (deletion protection)

Build type: code_build
Pre-decided orchestrator title: [<slug>] ORCH: Orchestrator

Next: Route to Architect to create plan and worker briefs.
```
