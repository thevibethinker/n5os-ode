# Worker Brief: Rule Creation

**Your Mission:** Create the Zo rule that enforces correct build orchestrator behavior.

**Output:** Zo rule via `create_rule` tool

---

## Context

The previous rule (now deleted) was wrong — it told Zo to use `/zo/ask` API calls to spawn workers in parallel. That's not what V wants.

The correct pattern is:
- Build orchestrator creates **persistent documents** (worker briefs)
- V **manually opens new threads** and pastes the briefs
- Workers are conversations, not API calls
- State flows through the build folder, not API responses

---

## Rule Specification

**Condition:**
```
When build orchestrator pattern is invoked, or complex work needs distribution across threads
```

**Instruction:**
```
Build orchestration uses PERSISTENT DOCUMENTS, not API calls:

1. **Orchestrator = Document**: The orchestrator is a build folder (`N5/builds/<slug>/`) with BUILD.md, worker briefs, and completions. The conversation is just an interface to this folder.

2. **Workers = Documents V Pastes**: Generate self-contained worker briefs in `workers/`. V manually opens new threads and pastes these briefs. Do NOT use `/zo/ask` to spawn workers.

3. **Pre-Decide Titles**: Each worker brief includes a pre-decided thread title in format: `[<slug>] W<wave>.<seq>: <task>`

4. **Workers Don't Commit**: Workers close WITHOUT git commits. They write completion reports to `completions/<worker_id>.json`. Only the orchestrator commits at the end.

5. **Pre-Screen**: Before creating a new build, check for existing incomplete builds with similar scope (run `python3 N5/scripts/build_status.py list --incomplete`).

6. **State Flow**: V returns to orchestrator conversation (or opens new one and loads build). Orchestrator reads completions/ to understand progress and adapt remaining briefs.

Red flags to catch:
- About to use `/zo/ask` to spawn workers → STOP, generate documents instead
- Worker about to commit → STOP, write to completions/ only
- Creating build without checking incomplete builds → STOP, pre-screen first
```
