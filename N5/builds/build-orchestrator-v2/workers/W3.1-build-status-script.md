# Worker Brief: Build Status Script

**Your Mission:** Create `N5/scripts/build_status.py` for listing and querying all builds.

**Output:** `N5/scripts/build_status.py`

---

## Context from Previous Waves

**W2.2 created `update_build.py`** with these relevant details:
- `status <slug> --json` already outputs structured JSON for a single build
- The JSON schema includes: workers, waves, completed_workers, progress stats
- **Recommendation from W2.2:** "Reuse the status --json format rather than creating a separate schema"

**W1.2 created `meta_schema.json`** — the canonical schema for meta.json files.

**Key insight:** This script is the *global* view (all builds), while `update_build.py status` is the *single build* view. They should share output format conventions.

---

## What This Script Does

Global view across all builds:
- List all builds with status
- Filter by incomplete
- Generate JSON for dashboard site
- Detect stale builds

Distinct from `update_build.py` which operates on a single build.

---

## Required Commands

### 1. List All Builds

```bash
python3 N5/scripts/build_status.py list
```

Output:
```
Active Builds:
  ● calendar-v2      3/5 workers (60%)   code_build   2026-01-15
  ● crm-refresh      1/3 workers (33%)   code_build   2026-01-10 ⚠️ stale

Complete Builds (recent):
  ✓ auth-module      5/5 workers         code_build   2026-01-08
  ✓ docs-update      2/2 workers         content      2026-01-05

Total: 4 builds (2 active, 2 complete)
```

### 2. List Incomplete Only

```bash
python3 N5/scripts/build_status.py list --incomplete
```

Output: Same format, only shows active builds.

### 3. JSON Output (for dashboard)

```bash
python3 N5/scripts/build_status.py list --json
```

Output format should align with `update_build.py status --json` conventions:
```json
{
  "generated_at": "2026-01-18T18:30:00Z",
  "builds": [
    {
      "slug": "calendar-v2",
      "title": "Calendar Integration v2",
      "status": "active",
      "type": "code_build",
      "created": "2026-01-15",
      "workers": { "complete": 3, "total": 5 },
      "progress_pct": 60,
      "is_stale": false,
      "path": "N5/builds/calendar-v2"
    }
  ],
  "summary": {
    "total": 4,
    "active": 2,
    "complete": 2,
    "stale": 1
  }
}
```

### 4. Regenerate Dashboard

```bash
python3 N5/scripts/build_status.py regenerate
```

Behavior:
1. Generate JSON output
2. Write to `Sites/build-tracker/data/builds.json`
3. Print confirmation

### 5. Stale Detection

A build is stale if:
- Status is active
- No completion written in last 7 days
- No meta.json update in last 7 days

Flag with ⚠️ in list, `is_stale: true` in JSON.
