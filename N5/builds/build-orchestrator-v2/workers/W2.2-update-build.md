# Worker Brief: update_build.py Creation

**Your Mission:** Create `N5/scripts/update_build.py` for worker completion tracking and build status management.

**Output:** `N5/scripts/update_build.py`

---

## Context

This script is the bridge between workers and the orchestrator:
- Workers call it on close to report completion
- Orchestrator calls it to read status
- Dashboard calls it for JSON output

---

## Required Commands

### 1. Worker Completion

```bash
python3 N5/scripts/update_build.py complete <slug> <worker_id> \
  --status complete|partial|blocked \
  --summary "What was accomplished" \
  --artifacts "file1.py,file2.md" \
  --learnings "What was learned" \
  --concerns "Any concerns for orchestrator"
```

Behavior:
1. Write to `N5/builds/<slug>/completions/<worker_id>.json`:
   ```json
   {
     "worker_id": "W1.1",
     "status": "complete",
     "completed_at": "2026-01-18T18:30:00Z",
     "convo_id": "con_XXXXX",  # If available from env
     "summary": "...",
     "artifacts": ["file1.py", "file2.md"],
     "learnings": "...",
     "concerns": "..."
   }
   ```
2. Update `meta.json` worker counts
3. Print confirmation

### 2. Build Status

```bash
python3 N5/scripts/update_build.py status <slug>
```

Output:
```
Build: my-project
Status: active
Type: code_build
Created: 2026-01-18

Workers: 3/5 complete (60%)
  ✓ W1.1: System Documentation (complete)
  ✓ W1.2: Template Updates (complete)
  ○ W2.1: init_build.py (in_progress)
  ○ W2.2: update_build.py (pending)
  × W2.3: Close Conversation (blocked)

Waves:
  Wave 1: ✓ complete
  Wave 2: in_progress
```

### 3. Close Build

```bash
python3 N5/scripts/update_build.py close <slug>
```

Behavior:
1. Verify all workers complete (or `--force`)
2. Set `meta.json` status to "complete"
3. Set `completed_at` timestamp
4. Print summary

### 4. JSON Output (for dashboard)

```bash
python3 N5/scripts/update_build.py status <slug> --json
```

Returns meta.json content plus computed fields.
