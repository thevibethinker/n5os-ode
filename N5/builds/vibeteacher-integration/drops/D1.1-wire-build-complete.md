# D1.1: Wire build_complete Checkpoint into Finalize

## Objective
Add the `build_complete` teaching checkpoint activation to `finalize_build()` in pulse.py so that VibeTeacher generates a teaching report when builds complete.

## Context
The VibeTeacher system has 4 activation points configured:
- `interview_complete`
- `plan_review` ✅ (already wired in review_manager.py)
- `feedback`
- `build_complete` ❌ (NOT WIRED - this is the gap)

The `teaching_manager.py` has the `activate_at_checkpoint()` function ready, it just needs to be called from finalize.

## Files to Modify
- `/home/workspace/Skills/pulse/scripts/pulse.py` — add teaching activation in `finalize_build()`

## Implementation

In `finalize_build()`, after step 3 (harvest learnings) and before step 4 (save results), add:

```python
# 3.5. Activate VibeTeacher at build_complete checkpoint
print("[FINALIZE] Activating VibeTeacher...")
try:
    # Generate build summary for teaching context
    build_summary = await summarize_build(slug, meta)
    
    teaching_result = subprocess.run(
        ["python3", str(WORKSPACE / "N5" / "pulse" / "teaching" / "teaching_manager.py"),
         "activate",
         "--checkpoint", "build_complete",
         "--slug", slug,
         "--input", build_summary[:2000]],
        capture_output=True, text=True, cwd=str(WORKSPACE)
    )
    
    if teaching_result.returncode == 0:
        try:
            teaching_data = json.loads(teaching_result.stdout)
            if teaching_data.get("activated"):
                results["teaching_moment"] = teaching_data.get("teaching")
                print(f"[FINALIZE] ✅ Teaching moment captured: {teaching_data.get('teaching', {}).get('precise_term', 'unknown')}")
        except:
            pass
except Exception as e:
    print(f"[FINALIZE] Teaching activation error (non-blocking): {e}")
```
