# D1.3: Build Missing Scripts

## Mission

Create the two scripts referenced by Close Conversation that don't exist:
- `N5/scripts/update_build.py`
- `N5/scripts/build_worker_complete.py`

These are build-system utilities (not close-specific), so they stay in N5/scripts.

## Context

The Close Conversation prompt references these for v2 build orchestration:

```bash
# From the prompt:
python3 N5/scripts/update_build.py complete {build_slug} {worker_id} \
  --status "complete" \
  --summary "What was accomplished" \
  --artifacts "comma,separated,file,paths" \
  --learnings "Key insights" \
  --concerns "Issues for orchestrator"

python3 N5/scripts/update_build.py status {build_slug}
python3 N5/scripts/update_build.py close {build_slug}
```

## Scope

### update_build.py

CLI for build state management:

```bash
# Commands:
python3 N5/scripts/update_build.py status <slug>      # Show build status
python3 N5/scripts/update_build.py complete <slug> <drop_id> --status X --summary Y ...
python3 N5/scripts/update_build.py close <slug>       # Mark build closed
```

**Implementation:**
- Reads/writes `N5/builds/<slug>/meta.json`
- Writes completions to `N5/builds/<slug>/deposits/<drop_id>.json`
- Updates drop status in meta.json

### build_worker_complete.py

Legacy wrapper for backward compatibility:

```bash
python3 N5/scripts/build_worker_complete.py \
  --build-id <slug> \
  --worker-num <num> \
  --status complete \
  --summary "..."
```

**Implementation:**
- Thin wrapper that calls update_build.py
- Maps legacy `--worker-num` to drop_id format (W1.1, W1.2, etc.)
