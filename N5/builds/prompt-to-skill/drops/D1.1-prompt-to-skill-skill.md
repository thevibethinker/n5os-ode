# D1.1: Create prompt-to-skill Skill

## Mission

Create the `Skills/prompt-to-skill/` skill that documents and automates the prompt-to-skill conversion process.

## Context

We're systematizing the conversion of high-eligibility prompts (score ≥15) into Agent Skills format. This skill IS the process documentation and tooling.

The scoring heuristic from the catalog:
- +3 per script reference (N5/scripts, python3, bun)
- +1 per phase reference (Phase N, Step N)
- +2 per schema reference (JSON, YAML, schema)
- +3 per prompt reference (composability signal)
- +1 per file reference
- +1 per 5 headers (complexity)
- +1 per 3 code blocks (implementation detail)
- +2 if marked as tool

## Scope

### Files to Create

```
Skills/prompt-to-skill/
├── SKILL.md
├── scripts/
│   ├── assess.py
│   └── scaffold.py
└── assets/
    └── skill-template/
        └── SKILL.md.template
```

### SKILL.md Content

Document the systematic conversion process:
1. **Assessment** — Score prompt for eligibility
2. **Analysis** — Identify components (scripts, refs, assets, modes)
3. **Architecture** — Decide structure (usually full migration)
4. **Scaffold** — Generate skill folder
5. **Migration** — Move/create scripts, references, assets
6. **Documentation** — Write SKILL.md
7. **Validation** — Test, then expunge original

### assess.py

CLI that scores a prompt file:
```bash
python3 Skills/prompt-to-skill/scripts/assess.py "Prompts/Close Conversation.prompt.md"
# Output: Score: 23 (HIGH - recommend conversion)
# Components: 60 script refs, 22 phases, 13 schemas...
```

Use the scoring logic from the catalog analysis.

### scaffold.py

CLI that creates skill folder structure:
```bash
python3 Skills/prompt-to-skill/scripts/scaffold.py conversation-close --from "Prompts/Close Conversation.prompt.md"
# Creates: Skills/conversation-close/ with stub SKILL.md
```
