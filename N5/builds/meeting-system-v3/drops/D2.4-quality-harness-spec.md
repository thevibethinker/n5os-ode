# D2.4: Define Quality Harness Checks

## Objective
Specify all quality checks for the pipeline.

## Checks
1. **Pre-processing**: transcript length, format, encoding
2. **Identification**: participant confidence, calendar match score
3. **Block generation**: output length, format compliance, no hallucination markers
4. **Post-processing**: all requested blocks generated, no errors

## Tasks
1. Create `Skills/meeting-ingestion/references/quality-harness-checks.md`
2. Define pass/fail thresholds
3. Define retry logic
4. Define HITL escalation triggers

## Acceptance Criteria
- [ ] All checks documented
- [ ] Thresholds defined
- [ ] Retry logic specified
