# D0.4: Build Smart Block Selector

## Objective
Build an LLM-powered block selector that reads the transcript, consults V's priorities, and recommends which blocks to generate.

## Context
- V wants intelligent block selection, not just rule-based
- The selector should use the BLOCK_INDEX.yaml recipes as starting points
- It should analyze transcript content to activate conditional blocks
- It should factor in V's current priorities from priorities.yaml

## Input Files
- `file 'Prompts/Blocks/BLOCK_INDEX.yaml'` — Block index (from D0.3)
- `file 'Personal/config/priorities.yaml'` — V's priorities
- `file 'N5/builds/meeting-system-v3/PLAN.md'` — Block policy

## Tasks

### 1. Design Selector Logic

```
Input: transcript, meeting_type, participants
Output: { blocks_to_generate: [...], reasoning: {...} }

Steps:
1. Determine base recipe from meeting_type + participants
2. Call LLM to analyze transcript for conditional block triggers
3. Check for Zo Take Heed triggers (B07, B14)
4. Score each conditional block by relevance to V's priorities
5. Return final block list with reasoning
```

### 3. LLM Prompt for Selection

The selector should call /zo/ask with a prompt like:

```
Analyze this meeting transcript to determine which blocks should be generated.

TRANSCRIPT:
{transcript_excerpt}

MEETING TYPE: {external|internal}

CONDITIONAL BLOCKS AVAILABLE:
- B04 (Open Questions): Generate if unresolved questions exist
- B06 (Business Context): Generate if company-specific strategy discussed
- B07 (Warm Introductions): Generate if intro requests mentioned
- B10 (Relationship Trajectory): Generate if relationship discussed
...

V'S CURRENT PRIORITIES:
- Driving Careerspan recruiting revenue

For each conditional block, respond with:
- GENERATE: <reason>
- SKIP: <reason>

Also scan for these trigger phrases:
- "Zo, draft a blurb..." → activates B14
- "Zo, intro me to..." → activates B07
```

# Triggered: B14 (blurb requested)
# Total: 10 blocks
