# D1.1: Inventory Audit - Scan All Meeting Artifacts

## Objective
Comprehensively scan Inbox and root-level meeting folders to catalog all shapes, identify issues, and generate a migration plan.

## Context
From earlier debugging:
- 6 meetings in Inbox root as raw .md files (need staging)
- 67 meeting folders in Inbox (some with legacy manifest schema)
- 20 root-level dated meeting folders outside Week-of-* (need archiving)
- One folder has transcript.jsonl instead of .md (blocking stage.py)
- Legacy manifest uses different schema than v3

## Input Files
- `file 'Personal/Meetings/Inbox/'` — Current inbox state
- `file 'Personal/Meetings/'` — Root level (has Week-of-* and orphan folders)

## Tasks

### 1. Scan Inbox
For each item in Inbox:
- Is it a raw file or folder?
- If folder, does it have manifest.json?
- What's the manifest schema version?
- Does it have transcript.md, transcript.txt, or transcript.jsonl?
- What blocks exist (if any)?
- What's the current status?

### 2. Scan Root Level
For each dated folder at Personal/Meetings/ root:
- Extract date from folder name
- Determine target Week-of-* folder
- Check for collision with existing archived meeting
- Classify as internal or external (if determinable)

### 3. Identify Issues
Catalog:
- Folders with no transcript
- Folders with jsonl transcripts (need conversion)
- Folders with legacy manifest schema
- Orphaned block files
- Duplicate meetings

### 4. Generate Migration Plan
Create actionable plan:
- Which folders need manifest upgrade
- Which transcripts need format conversion
- Which folders need archiving (with target paths)
- Estimated collision count
