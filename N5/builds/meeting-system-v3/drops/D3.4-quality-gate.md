# D3.4: Build Quality Gate Module

## Objective
Validate meeting readiness before block generation.

## Tasks
1. Create `Skills/meeting-ingestion/scripts/quality_gate.py`
2. Implement checks from D2.4 spec
3. Compute quality score
4. Trigger HITL queue for failures
5. Update manifest with quality_gate section

## Gate Logic
- PASS if score >= threshold (e.g., 0.8)
- FAIL + HITL if participant identification < 0.5
- FAIL + RETRY if transcript too short
- PASS with WARNING if optional checks fail
