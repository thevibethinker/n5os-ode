# D3.3: Build CRM Enricher Module

## Objective
Enrich meetings with CRM data and classify internal/external.

## Tasks
1. Create `Skills/meeting-ingestion/scripts/crm_enricher.py`
2. Query N5/data/crm.db for participants
3. Classify meeting as internal if ALL participants have is_internal=true
4. Surface unidentified participants for HITL queue
5. Use LLM heuristics for participant name matching (fuzzy)

## Internal Classification Logic
- If ALL identified participants have is_internal=true → internal
- If ANY participant has is_internal=false or unknown → external
- If unable to identify ANY participant → HITL queue

## Acceptance Criteria
- [ ] CRM lookup works
- [ ] Internal/external classification correct
- [ ] Unidentified participants flagged for HITL
- [ ] Fuzzy name matching via LLM
