# D4.3: Block Quality Check (MANUAL DROP)

## Objective
Validate generated blocks meet quality standards.

## Why Manual
Quality checking is subjective. V wants to:
- Review check criteria
- Tune thresholds
- Approve what "good enough" means

## Tasks
1. Create `Skills/meeting-ingestion/scripts/block_quality_check.py`
2. For each generated block:
   - Check output length (not empty, not too short)
   - Check format compliance (starts with heading)
   - Check for hallucination markers
   - Score confidence
3. Flag low-confidence blocks for retry or V review
4. Update manifest with quality results

## Acceptance Criteria
- [ ] Quality checks implemented
- [ ] Scoring works
- [ ] Low-confidence flagged
- [ ] Manifest updated
