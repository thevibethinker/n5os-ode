# D0.1: Prompt Audit - Analyze All Block Prompts

## Objective
Audit all existing block prompts in `Prompts/Blocks/` to assess quality, identify redundancies, and recommend refinements.

## Context
- Current system has 27 blocks (17 external, 10 internal)
- V wants **refinement, not compression** — prompts should be more specific, extract only essential info
- Prompt length follows purpose (some shorter, some longer)
- B02 (Commitments) and B05 (Action Items) will be merged in D0.2

## Input Files
- `file 'Prompts/Blocks/'` — all Generate_B*.prompt.md files
- `file 'N5/builds/meeting-system-v3/PLAN.md'` — block policy and final list

## Tasks

### 1. Inventory All Prompts
For each prompt file, extract:
- Block code (B00, B01, etc.)
- Current word count
- Key extraction targets (what does it ask for?)
- Overlap with other blocks (if any)

### 2. Quality Assessment
Rate each prompt on:
- **Specificity** (1-5): Does it ask for precise information?
- **Actionability** (1-5): Will the output be directly useful?
- **Redundancy** (1-5): How much overlap with other blocks?
- **Brevity** (1-5): Is it as concise as possible while complete?

### 3. Generate Recommendations
For each block, recommend:
- **Keep as-is** — prompt is high quality
- **Refine** — needs tweaks (specify what)
- **Expand** — needs more detail to be specific enough
- **Compress** — too verbose, trim fat

### 4. Flag Merge Candidates
Beyond B02+B05 (already decided), identify any other blocks that substantially overlap.
