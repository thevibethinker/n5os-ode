# D2.2: Validation and regression checks

## Mission

Run validation and regression checks across all hotline datasets and updated query paths; produce machine-readable results and actionable findings.

## Context

Validation should include:
- row integrity checks (id/null/negative/reversed timestamps),
- zero-span timestamp anomaly counts before vs after,
- ET rollup sanity checks,
- canonical-source rollup verification.

Output should support close gating and future audits.

## Scenarios

S1: Integrity baseline check
Given updated ingestion/backfill are complete
When validation script executes
Then it emits per-dataset pass/fail booleans and counts
Verify output artifact is machine-readable JSON.

S2: Regression guard for timestamp bug
Given a simulated/real recent insert with non-zero duration
When post-fix anomaly query runs
Then row is not flagged as zero-span
Verify anomaly count for newly written rows is zero.

S3: Query consistency check
Given ET and UTC rollups over same window
When both are computed
Then ET rollup is used by default and documented
Verify dashboard query helper references ET rollup path.
