## Instructions

### Part 1: Quality Gate

Review each reasoning trace and flag issues:

| Check | What to Look For |
|-------|------------------|
| Evidence density | Are claims backed by specific citations? |
| Rubric adherence | Did the drop follow the evaluation rubric? |
| Source attribution | For D2.4, are sources ([RUBRIC], [CANDIDATE], etc.) explicit? |
| Thin reasoning | Does reasoning feel generic/boilerplate or substantive? |
| Consistency | Do panels contradict each other? |

Create `quality-flags.md` with any issues found. If no issues, note "All checks passed."

### Part 2: Assembly

Combine panel content into a single document structured for Gamma:

```markdown
# Candidate Profile: H.

## The Thesis
[Panel 1 content]

## Technical Fit
[Panel 2 content — table format]

## What Makes H. Different
[Panel 3 content]

## Why This Meeting Matters
[Panel 4 content]

---

### What This Report Assumes
- Data sources: [list]
- Known gaps: [list]
- Generated: [date]
- Powered by Careerspan
```

### Part 3: Design Integration

Reference D1.1-design-spec.md and add Gamma styling notes to the assembled content.
