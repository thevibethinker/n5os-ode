# D4.1: Autonomy Config

## Objective

Create the autonomy configuration that defines zoputer's decision-making thresholds and permissions.

## Scope

### Files (owns)
- `Documents/consulting/autonomy-config.yaml` - CREATE

### Responsibilities
- Define escalation thresholds (to va, to human)
- Define allowed vs forbidden self-edit paths
- Configure learning branch settings
- Make thresholds adjustable without code changes

### Must NOT Touch
- Learning logger implementation (D4.2's scope)
- GitHub Actions (D4.3's scope)

## MUST DO

1. **Create autonomy config:**
   ```yaml
   # Documents/consulting/autonomy-config.yaml
   version: 1.0
   autonomy_level: medium

   thresholds:
     escalate_to_va: 0.7      # Confidence below this → ask va
     escalate_to_human: 0.5   # Confidence below this → ask V
     auto_adapt: 0.9          # Confidence above this → do it

   allowed_self_edits:
     - Learnings/*
     - Runtime/*
     
   forbidden_self_edits:
     - Skills/*
     - Documents/System/*
     - anything with .n5protected

   learning_branch: zoputer/learnings
   require_pr_for_main: true
   ```

2. **Add validation logic:**
   - Create a simple validator script that checks if a path is allowed for zoputer edits
   - Script: `N5/scripts/check_autonomy_permission.py`
   
   ```bash
   python3 N5/scripts/check_autonomy_permission.py "Learnings/foo.md"  # → allowed
   python3 N5/scripts/check_autonomy_permission.py "Skills/bar/SKILL.md"  # → forbidden
   ```

3. **Document the config:**
   - Add comments explaining each threshold
   - Explain how zoputer should use these values

## MUST NOT DO

- Do NOT implement the learning logger (D4.2 does that)
- Do NOT create GitHub Actions (D4.3 does that)
- Do NOT modify existing skills or system files
