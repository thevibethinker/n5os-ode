# D2.2: Mentor Escalation Logic

## Objective

Create the logic that allows zoputer to escalate uncertain decisions to va for guidance, implementing the mentor-apprentice relationship.

## Scope

### Files (owns)
- `Skills/mentor-escalation/SKILL.md` - CREATE
- `Skills/mentor-escalation/scripts/escalate.py` - CREATE
- `Skills/mentor-escalation/scripts/va_client.py` - CREATE

### Responsibilities
- Build va_client.py (zoputer's equivalent of zoputer_client.py)
- Build escalation logic with confidence thresholds
- Integrate with audit system
- Define escalation request/response format

### Must NOT Touch
- `N5/scripts/zoputer_client.py` (va-side, already exists)
- Localization protocol (reference only)
- Human escalation (that's D3.x)

## MUST DO

1. **Create skill directory:**
   ```bash
   mkdir -p /home/workspace/Skills/mentor-escalation/scripts
   ```

2. **Create va_client.py:**
   
   This is the zoputer-side client for calling va. Mirror the structure of `N5/scripts/zoputer_client.py` but reversed:
   
   ```python
   # Key differences from zoputer_client.py:
   VA_API_URL = "https://api.zo.computer/zo/ask"
   # Uses VA_API_KEY environment variable
   # Logs to local audit as "zoputer-to-va" direction
   ```
   
   Commands:
   - `ping` - Test connectivity to va
   - `ask` - Send a question to va and get response
   - `escalate` - Structured escalation with context

3. **Create escalate.py:**
   
   Implements the decision framework from localization-protocol.md:
   
   ```python
   CONFIDENCE_THRESHOLDS = {
       "auto_decide": 0.7,      # >= 0.7: zoputer decides alone
       "ask_va": 0.5,           # 0.5-0.7: ask va
       "ask_v": 0.0             # < 0.5: escalate to human
   }
   ```
   
   Functions:
   - `assess_confidence(situation: str) -> float` - LLM-based confidence assessment
   - `should_escalate(confidence: float) -> str` - Returns "auto", "va", or "human"
   - `escalate_to_va(question: str, context: dict) -> str` - Calls va and returns guidance
   - `log_decision(decision: dict)` - Logs to audit system
   
   CLI:
   ```bash
   python3 escalate.py assess "Should I modify this workflow for client X?"
   python3 escalate.py ask-va "How should I handle this novel request?" --context '{"client": "X", "request": "..."}'
   ```

4. **Define escalation request format:**
   ```json
   {
     "type": "mentor_escalation",
     "from": "zoputer",
     "confidence": 0.6,
     "situation": "Client asked to remove review gate from workflow",
     "context": {
       "client": "example_client",
       "current_workflow": "...",
       "requested_change": "..."
     },
     "options_considered": [
       {"option": "A", "pros": "...", "cons": "..."},
       {"option": "B", "pros": "...", "cons": "..."}
     ],
     "question": "Which option should I recommend, or is there a better approach?"
   }
   ```

5. **Define escalation response format:**
   ```json
   {
     "type": "mentor_response",
     "from": "va",
     "recommendation": "Option A with modification",
     "rationale": "...",
     "precedent_set": true,
     "log_to_learnings": true,
     "learning_summary": "When clients ask to remove review gates, prefer X approach"
   }
   ```

6. **Integrate with audit system:**
   - Log every escalation request
   - Log every response
   - Include correlation_id for linking
