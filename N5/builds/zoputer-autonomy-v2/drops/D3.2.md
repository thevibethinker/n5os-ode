3. **Create notify.py:**
   
   Main function:
   ```python
   def notify_human(decision_id: str, immediate: bool = False) -> bool:
       """
       Send SMS notification to V about a pending decision.
       
       Args:
           decision_id: The pending decision to notify about
           immediate: Skip any batching, send now
       
       Returns:
           True if SMS sent successfully
       """
       decision = get_decision(decision_id)
       
       # Format SMS (must be < 160 chars ideally)
       message = format_sms_notification(decision)
       
       # Send via Zo's SMS capability
       # Note: This will use send_sms_to_user tool internally
       send_notification(message)
       
       # Log that notification was sent
       log_notification(decision_id)
   ```
   
   SMS format:
   ```
   🔔 Decision needed ({priority})
   {summary} (< 100 chars)
   
   Reply: n5 decision {short_id}
   ```
   
   Example:
   ```
   🔔 Decision needed (high)
   Zoputer: Should I remove review gate for ClientX workflow?
   
   Reply: n5 decision d7f3
   ```

4. **Create escalation_notifier.py** (orchestrator script):
   
   This script:
   - Checks pending_decisions for items needing notification
   - Sends initial notifications for new decisions
   - Sends reminders based on priority schedule
   - Can be called by scheduled agent or directly
   
   ```python
   def check_and_notify():
       """Check for decisions needing notification and send them."""
       pending = list_pending(status='pending')
       
       for decision in pending:
           if needs_initial_notification(decision):
               notify_human(decision['id'])
           elif needs_reminder(decision):
               notify_human(decision['id'], reminder=True)
   ```
   
   CLI:
   ```bash
   python3 N5/scripts/escalation_notifier.py check  # Check and send pending
   python3 N5/scripts/escalation_notifier.py send <decision_id>  # Force send
   python3 N5/scripts/escalation_notifier.py status  # Show notification status
   ```
