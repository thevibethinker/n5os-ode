# D1.3: Auto-Sync Scheduled Agent

## Objective

Create scheduled agents on both va and zoputer that automatically push/pull from the GitHub substrate.

## Scope

### Files (owns)
- Agent definition for va (via create_agent tool)
- Agent definition for zoputer (documented for manual creation)
- `Documents/consulting/auto-sync-agents.md` - CREATE

### Responsibilities
- Design va-side push schedule (daily 9 AM ET)
- Design zoputer-side pull schedule (daily 9:15 AM ET, after va push)
- Document manual agent creation steps for zoputer
- Include change detection (skip if nothing changed)

### Must NOT Touch
- The git-substrate-sync skill itself (that's D1.2)
- Existing scheduled agents
- zoputer directly (we document, V executes)

## MUST DO

1. **Create va-side push agent:**
   
   Use `create_agent` tool with:
   - **Instruction:** 
     ```
     Run the daily substrate sync to push Tier 0 skills to GitHub.
     
     1. Run: python3 Skills/git-substrate-sync/scripts/sync.py push
     2. If successful, no notification needed
     3. If failed, text V: "Substrate sync failed: [error summary]"
     4. Log result to audit system
     ```
   - **Schedule:** Daily at 9:00 AM ET (RRULE: `FREQ=DAILY;BYHOUR=14;BYMINUTE=0`)
   - Note: 14:00 UTC = 9:00 AM ET

2. **Document zoputer-side pull agent:**
   
   Create `Documents/consulting/auto-sync-agents.md` with:
   - Purpose of both agents
   - va agent details (for reference)
   - Step-by-step guide for V to create the zoputer agent:
     - Go to zoputer's [Scheduled Tasks](/?t=agents)
     - Create new agent with instruction:
       ```
       Pull latest content from the GitHub substrate.
       
       1. Run: python3 Skills/git-substrate-sync/scripts/pull.py
       2. If new content, log what was updated
       3. If failed, escalate to va: "Pull failed: [error]"
       ```
     - Schedule: Daily at 9:15 AM ET (15 min after va push)
     - RRULE: `FREQ=DAILY;BYHOUR=14;BYMINUTE=15`

3. **Include skip-if-unchanged logic in agent instructions:**
   
   The sync scripts should detect no changes and exit cleanly.
   Agent instruction should note: "If 'nothing to sync' message, complete silently."

4. **Document the sync window:**
   ```
   9:00 AM ET - va pushes to GitHub
   9:15 AM ET - zoputer pulls from GitHub
   
   This 15-minute gap ensures GitHub has processed the push
   before zoputer attempts to pull.
   ```

## MUST NOT DO

- Do NOT create the zoputer agent directly (V must do this manually on zoputer)
- Do NOT set up webhooks (scheduled pull is simpler and sufficient)
- Do NOT make the agents cornerstone (⇱) - they can be modified
