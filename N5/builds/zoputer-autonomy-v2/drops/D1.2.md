# D1.2: Git Sync Skill

## Objective

Create a skill that handles pushing content from va to the GitHub substrate and can be triggered by zoputer to pull updates.

## Scope

### Files (owns)
- `Skills/git-substrate-sync/SKILL.md` - CREATE
- `Skills/git-substrate-sync/scripts/sync.py` - CREATE
- `Skills/git-substrate-sync/scripts/pull.py` - CREATE (for zoputer-side)

### Responsibilities
- Build push logic (va → GitHub)
- Build pull logic (GitHub → zoputer)
- Integrate with content-classifier for Tier 0 filtering
- Update MANIFEST.json on each sync
- Handle git operations safely (clone, pull, push, conflict detection)

### Must NOT Touch
- `Skills/librarian-export/` (existing, different purpose)
- `Skills/consulting-api/` (existing, will call this skill)
- GitHub repo creation (that's D1.1)
- Scheduled agent setup (that's D1.3)

## MUST DO

1. **Create skill directory:**
   ```bash
   mkdir -p /home/workspace/Skills/git-substrate-sync/scripts
   ```

2. **Create SKILL.md** with:
   - Purpose: Sync content between va and zoputer via GitHub substrate
   - Usage examples for push and pull
   - Integration notes

3. **Create sync.py (va-side push):**
   
   Key functions:
   - `get_exportable_content()` - Uses content-classifier to get Tier 0 skills
   - `clone_or_pull_repo()` - Ensures local copy is current
   - `copy_skills_to_repo()` - Copies Tier 0 skills to repo clone
   - `commit_and_push()` - Commits changes with descriptive message
   - `update_manifest()` - Updates MANIFEST.json with sync timestamp
   
   CLI interface:
   ```bash
   python3 Skills/git-substrate-sync/scripts/sync.py push [--dry-run] [--skills skill1,skill2]
   python3 Skills/git-substrate-sync/scripts/sync.py status
   ```
   
   Important:
   - Clone to `/tmp/zoputer-substrate/` for operations
   - Use `git diff --stat` to show what changed
   - Always pull before push to avoid conflicts
   - Log to audit system

4. **Create pull.py (zoputer-side):**
   
   This script will be deployed TO zoputer. It:
   - Pulls latest from GitHub
   - Copies content to zoputer's workspace
   - Updates local MANIFEST.json
   
   CLI interface:
   ```bash
   python3 Skills/git-substrate-sync/scripts/pull.py [--dry-run]
   ```
   
   Note: This script needs to work standalone on zoputer (no va dependencies)

5. **Integrate with content-classifier:**
   ```python
   from pathlib import Path
   import sys
   sys.path.insert(0, "/home/workspace/Skills/content-classifier/scripts")
   from classifier import get_tier_0_skills
   ```

6. **Handle edge cases:**
   - Repo doesn't exist locally → clone
   - Repo exists but dirty → stash or abort
   - Push conflicts → abort and alert
   - Network failures → retry with backoff

## MUST NOT DO

- Do NOT include any secrets in the scripts
- Do NOT auto-resolve merge conflicts (abort and escalate)
- Do NOT sync Tier 1 or Tier 2 content
- Do NOT modify the content-classifier skill
