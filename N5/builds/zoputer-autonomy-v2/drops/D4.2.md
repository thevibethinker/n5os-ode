# D4.2: Learning Logger Skill

## Objective

Create a skill for zoputer to log learnings from client interactions, committing them to its learning branch for va review.

## Scope

### Files (owns)
- `Skills/learning-logger/SKILL.md` - CREATE
- `Skills/learning-logger/scripts/log_learning.py` - CREATE

### Responsibilities
- Provide CLI for zoputer to log learnings
- Commit learnings to zoputer/learnings branch
- Respect autonomy config for allowed paths
- Push to GitHub substrate

### Must NOT Touch
- Autonomy config (D4.1's scope, just read it)
- GitHub Actions workflow (D4.3's scope)
- Git sync skill (D1.2's scope, just use it)

## MUST DO

1. **Create skill directory:**
   ```bash
   mkdir -p /home/workspace/Skills/learning-logger/scripts
   ```

2. **Create SKILL.md:**
   - Purpose: Log learnings from zoputer's client interactions
   - Usage: Called when zoputer discovers something worth remembering
   - Integration: git-sync skill, autonomy config

3. **Create log_learning.py:**
   
   ```python
   # log_learning.py
   def log_learning(learning: str, source: str, category: str):
       """Commit a learning to zoputer's branch."""
       timestamp = datetime.now().strftime("%Y%m%d_%H%M%S")
       filename = f"Learnings/{category}/{timestamp}_{slugify(learning[:30])}.md"
       
       content = f"""---
   created: {date.today()}
   source: {source}
   category: {category}
   ---

   # Learning

   {learning}
   """
       
       # Check autonomy permission first
       if not check_autonomy_permission(filename):
           raise PermissionError(f"Cannot write to {filename}")
       
       # Write file
       Path(filename).parent.mkdir(parents=True, exist_ok=True)
       Path(filename).write_text(content)
       
       # Commit and push to learning branch
       git_commit(filename, f"Learning: {learning[:50]}")
       git_push("zoputer/learnings")
   ```

4. **CLI interface:**
   ```bash
   python3 Skills/learning-logger/scripts/log_learning.py \
     --learning "Client X prefers async communication over meetings" \
     --source "con_abc123" \
     --category "client-preferences"
   ```

5. **Categories to support:**
   - `client-preferences` - How clients like to work
   - `domain-knowledge` - Industry/technical learnings
   - `workflow-improvements` - Better ways to do things
   - `mistakes-to-avoid` - What not to do
