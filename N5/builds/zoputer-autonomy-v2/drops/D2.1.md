Create documentation for V to set up the reverse API key (VA_API_KEY on zoputer) so zoputer can call va.

- Include verification steps

### Must NOT Touch
- Actual API keys (V handles this)
- zoputer's filesystem directly
- Existing setup guides

## MUST DO

1. **Create comprehensive guide** at `Documents/consulting/va-api-setup-guide.md`:

   Structure:
   ```markdown
   # VA API Setup Guide (Reverse Bridge)
   
   ## Overview
   This enables zoputer to call va for guidance (the reverse of the existing bridge).
   
   ## Step 1: Create API Key on va
   1. Go to va.zo.computer
   2. Navigate to Settings > Developers
   3. Create new API key named "ZOPUTER_CALLBACK"
   4. Copy the key immediately
   
   ## Step 2: Add Key to zoputer
   1. Go to zoputer.zo.computer
   2. Navigate to Settings > Developers > Secrets
   3. Add secret: VA_API_KEY = [paste key]
   
   ## Step 3: Verify
   On zoputer, run: python3 N5/scripts/va_client.py ping
   Expected: "va responded in X.XXs"
   
   ## Security Notes
   - This key allows zoputer to send prompts to va
   - va's responses are logged to both audit DBs
   - Key should be rotated quarterly
   ```

2. **Include the communication model diagram:**
   ```
   va.zo.computer                    zoputer.zo.computer
        │                                   │
        │ ←── VA_API_KEY (zoputer has) ──── │
        │                                   │
        │ ──── ZOPUTER_API_KEY (va has) ──→ │
        │                                   │
   ```

3. **Document when zoputer should call va:**
   - Confidence < 0.7 on a decision
   - Novel situation without precedent
   - Client request that might change core patterns
   - Any structural changes (per localization protocol)

4. **Include troubleshooting section:**
   - Key not working: Check for trailing spaces
   - 401 errors: Verify key is correct
   - Timeout: Check va is running
