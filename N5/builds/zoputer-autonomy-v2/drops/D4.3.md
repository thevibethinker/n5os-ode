# D4.3: GitHub Actions + PR Review

## Objective

Create GitHub Actions workflow to auto-create PRs from zoputer's learning branch, and a va-side script to review these PRs.

## Scope

### Files (owns)
- GitHub workflow file (in zoputer-substrate repo)
- `N5/scripts/review_learning_pr.py` - CREATE

### Responsibilities
- Auto-create PR when zoputer pushes to learning branch
- Provide va-side script to review and merge/comment on PRs
- Notify V if review is uncertain

### Must NOT Touch
- Learning logger (D4.2's scope)
- Autonomy config (D4.1's scope)
- Git sync skill (D1.2's scope)

## MUST DO

1. **Create GitHub Actions workflow:**
   
   This needs to be pushed to the zoputer-substrate repo at `.github/workflows/learning-pr.yaml`:
   
   ```yaml
   name: Auto-PR for Learnings
   on:
     push:
       branches: [zoputer/learnings]
       paths: [Learnings/**]

   jobs:
     create-pr:
       runs-on: ubuntu-latest
       steps:
         - uses: actions/checkout@v4
         - name: Create PR
           uses: peter-evans/create-pull-request@v5
           with:
             branch: zoputer/learnings
             base: main
             title: "[Zoputer Learning] New insights from client interactions"
             body: |
               Auto-generated PR from zoputer's learning branch.
               
               ## Review Checklist
               - [ ] Learnings are accurate and useful
               - [ ] No sensitive client information exposed
               - [ ] Categories are appropriate
               
               Review before merging.
   ```

2. **Create PR review script (va-side):**
   
   `N5/scripts/review_learning_pr.py`:
   
   ```python
   def review_learning_prs():
       """Check for open PRs from zoputer and review them."""
       # Get open PRs from zoputer/learnings branch
       prs = get_open_prs(base='main', head='zoputer/learnings')
       
       for pr in prs:
           diff = get_pr_diff(pr['number'])
           
           # Assess quality
           assessment = assess_learning_quality(diff)
           
           if assessment['confidence'] > 0.8:
               if assessment['action'] == 'approve':
                   approve_and_merge(pr['number'])
               else:
                   request_changes(pr['number'], assessment['feedback'])
           else:
               # Uncertain - notify V
               notify_v_for_review(pr)
   ```

3. **CLI interface:**
   ```bash
   python3 N5/scripts/review_learning_pr.py check   # List open PRs
   python3 N5/scripts/review_learning_pr.py review  # Auto-review all
   python3 N5/scripts/review_learning_pr.py merge <pr_number>  # Manual merge
   ```

4. **Scheduled check (optional):**
   - Consider adding a scheduled agent to run PR review weekly
   - Or integrate into existing maintenance routines

## MUST NOT DO

- Do NOT modify the learning logger (D4.2's scope)
- Do NOT change autonomy config (D4.1's scope)
- Do NOT auto-merge without confidence threshold
