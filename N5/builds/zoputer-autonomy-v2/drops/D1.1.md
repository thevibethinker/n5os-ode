# D1.1: GitHub Repo & Branch Protection

## Objective

Create the GitHub repository that serves as the shared substrate between va and zoputer, with proper branch protection for the content distribution model.

## Scope

### Files (owns)
- Repository: `vrijenattawar/zoputer-substrate` (to be created)
- `README.md` in repo
- `.github/CODEOWNERS`
- Branch protection rules configuration

### Responsibilities
- Create private GitHub repo under vrijenattawar
- Set up branch structure (main, zoputer/learnings)
- Configure branch protection on main
- Create initial folder structure matching archetype
- Document the repo purpose in README

### Must NOT Touch
- Any files in /home/workspace (this is GitHub-only work)
- Existing Skills/ content (that's for the sync skill)
- zoputer.zo.computer (that's D2.x work)

## MUST DO

1. **Create the repository:**
   ```bash
   gh repo create vrijenattawar/zoputer-substrate --private --description "Shared substrate for zoputer archetype Zo"
   ```

2. **Clone locally to conversation workspace:**
   ```bash
   cd /home/.z/workspaces/$(pwd | xargs basename)
   git clone https://github.com/vrijenattawar/zoputer-substrate.git
   cd zoputer-substrate
   ```

3. **Create initial structure:**
   ```
   zoputer-substrate/
   ├── README.md
   ├── .github/
   │   └── CODEOWNERS
   ├── Skills/
   │   └── .gitkeep
   ├── Prompts/
   │   └── .gitkeep
   ├── Documents/
   │   └── System/
   │       └── .gitkeep
   └── MANIFEST.json
   ```

4. **Create README.md:**
   - Purpose: Shared substrate for zoputer archetype
   - Content model: va pushes to main, zoputer PRs to zoputer/learnings
   - Branch protection explanation
   - Do NOT include sensitive info

5. **Create CODEOWNERS:**
   ```
   # va owns main branch content
   * @vrijenattawar
   ```

6. **Create MANIFEST.json:**
   ```json
   {
     "schema_version": "1.0",
     "archetype": "zoputer",
     "source": "va.zo.computer",
     "content_model": {
       "main": "va-controlled, protected",
       "zoputer/learnings": "zoputer can push, requires PR to main"
     },
     "last_sync": null
   }
   ```

7. **Push initial commit:**
   ```bash
   git add .
   git commit -m "Initial substrate structure"
   git push origin main
   ```

8. **Create zoputer/learnings branch:**
   ```bash
   git checkout -b zoputer/learnings
   git push origin zoputer/learnings
   git checkout main
   ```

9. **Set branch protection on main:**
   ```bash
   gh api repos/vrijenattawar/zoputer-substrate/branches/main/protection \
     -X PUT \
     -H "Accept: application/vnd.github+json" \
     -f required_pull_request_reviews='{"required_approving_review_count":1}' \
     -f enforce_admins=false \
     -f required_status_checks=null \
     -f restrictions=null
   ```

## MUST NOT DO

- Do NOT add any actual skills yet (that's D1.2's job)
- Do NOT configure webhooks (not needed for this architecture)
- Do NOT make the repo public
- Do NOT add API keys or secrets to the repo
