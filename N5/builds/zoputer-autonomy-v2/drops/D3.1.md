# D3.1: Pending Decisions Store

## Objective

Create a persistent store for decisions that need human (V) input, accessible from any conversation thread.

## Scope

### Files (owns)
- `N5/data/pending_decisions.db` - CREATE (SQLite)
- `N5/scripts/pending_decisions.py` - CREATE
- `N5/config/pending_decisions_schema.sql` - CREATE

### Responsibilities
- Design schema for pending decisions
- Build CRUD operations
- Enable cross-conversation access
- Include expiration and priority logic

### Must NOT Touch
- `conversations.db` (reference only)
- Audit system (separate concern)
- SMS sending (D3.2's scope)

## MUST DO

1. **Design the schema** (`N5/config/pending_decisions_schema.sql`):
   ```sql
   CREATE TABLE IF NOT EXISTS pending_decisions (
       id TEXT PRIMARY KEY,
       created_at TEXT NOT NULL,
       expires_at TEXT,
       priority TEXT DEFAULT 'normal',  -- low, normal, high, critical
       status TEXT DEFAULT 'pending',   -- pending, resolved, expired, cancelled
       
       -- Origin info
       origin TEXT NOT NULL,            -- 'va' or 'zoputer'
       origin_conversation TEXT,        -- conversation ID where it originated
       
       -- Decision content
       summary TEXT NOT NULL,           -- Brief for SMS (< 160 chars)
       full_context TEXT NOT NULL,      -- JSON blob with all details
       options TEXT,                    -- JSON array of options if applicable
       
       -- Resolution
       resolved_at TEXT,
       resolved_by TEXT,                -- conversation ID that resolved it
       resolution TEXT,                 -- The decision made
       resolution_notes TEXT
   );
   
   CREATE INDEX idx_pending_status ON pending_decisions(status);
   CREATE INDEX idx_pending_priority ON pending_decisions(priority, created_at);
   ```

2. **Create pending_decisions.py:**
   
   Functions:
   ```python
   def create_decision(
       origin: str,
       origin_conversation: str,
       summary: str,
       full_context: dict,
       options: list = None,
       priority: str = "normal",
       expires_in_hours: int = 24
   ) -> str:
       """Create a new pending decision. Returns decision_id."""
   
   def get_decision(decision_id: str) -> dict:
       """Get a single decision by ID."""
   
   def list_pending(priority: str = None, origin: str = None) -> list:
       """List all pending decisions, optionally filtered."""
   
   def resolve_decision(
       decision_id: str,
       resolution: str,
       resolved_by: str,
       notes: str = None
   ) -> bool:
       """Mark a decision as resolved."""
   
   def get_context_for_sms(decision_id: str) -> dict:
       """Get minimal context for SMS notification."""
   
   def expire_old_decisions() -> int:
       """Mark expired decisions. Returns count expired."""
   ```
   
   CLI:
   ```bash
   python3 N5/scripts/pending_decisions.py create --origin va --summary "..." --context '{...}'
   python3 N5/scripts/pending_decisions.py list [--pending] [--origin va|zoputer]
   python3 N5/scripts/pending_decisions.py get <decision_id>
   python3 N5/scripts/pending_decisions.py resolve <decision_id> --resolution "..." --by <convo_id>
   python3 N5/scripts/pending_decisions.py expire
   ```

3. **Full context structure:**
   ```json
   {
     "question": "The core question requiring decision",
     "background": "Why this decision is needed",
     "options": [
       {"id": "A", "description": "...", "pros": [...], "cons": [...]},
       {"id": "B", "description": "...", "pros": [...], "cons": [...]}
     ],
     "recommendation": "What va/zoputer recommends, if any",
     "recommendation_confidence": 0.6,
     "deadline": "2026-02-08T12:00:00Z or null",
     "related_files": ["path/to/relevant/file.md"],
     "escalation_chain": [
       {"from": "zoputer", "to": "va", "timestamp": "..."},
       {"from": "va", "to": "human", "timestamp": "..."}
     ]
   }
   ```

4. **Cross-conversation access:**
   
   Any conversation can:
   - Query pending decisions: `pending_decisions.py list`
   - Resolve a decision: `pending_decisions.py resolve <id> --by <current_convo_id>`
   
   The `resolved_by` field creates the audit trail.

5. **Priority escalation:**
   - `low`: 48 hour expiry, no SMS
   - `normal`: 24 hour expiry, single SMS
   - `high`: 12 hour expiry, SMS + reminder at 6h
   - `critical`: 4 hour expiry, SMS + reminder at 2h + 1h
