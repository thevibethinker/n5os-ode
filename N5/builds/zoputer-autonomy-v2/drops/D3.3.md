# D3.3: Cross-Thread Resolution Handler

## Objective

Enable V to resolve pending decisions from ANY conversation thread, with the resolution routed back to the originating context.

## Scope

### Files (owns)
- `N5/scripts/resolve_decision.py` - CREATE
- Rule for "n5 decision" SMS command - CREATE via create_rule

### Responsibilities
- Parse V's decision response
- Update pending_decisions store
- Route resolution back to origin (va or zoputer)
- Handle the "n5 decision <id>" SMS command pattern

### Must NOT Touch
- `pending_decisions.py` (call it, don't modify)
- `escalation_notifier.py` (separate concern)
- Existing rules

## MUST DO

1. **Create resolve_decision.py:**
   
   Main functions:
   ```python
   def resolve_from_sms(message: str, current_convo_id: str) -> dict:
       """
       Parse an SMS response and resolve the decision.
       
       Expected formats:
       - "n5 decision d7f3" - View decision details
       - "n5 decision d7f3 approve" - Quick resolution
       - "n5 decision d7f3 A" - Choose option A
       - "n5 decision d7f3 reject" - Quick rejection
       
       Returns:
           {"success": bool, "decision": dict, "action": str, "message": str}
       """
   
   def show_decision_context(decision_id: str) -> str:
       """
       Format the full decision context for display.
       
       Returns a markdown-formatted string with:
       - Question
       - Background
       - Options (if any)
       - Recommendation
       - How to respond
       """
   
   def apply_resolution(
       decision_id: str,
       resolution: str,
       resolved_by_convo: str,
       notes: str = None
   ) -> dict:
       """
       Apply a resolution and notify the origin.
       
       1. Update pending_decisions store
       2. If origin is zoputer: call zoputer with resolution
       3. If origin is va: update internal state
       4. Log to audit
       
       Returns:
           {"success": bool, "routed_to": str, "message": str}
       """
   
   def route_to_origin(decision: dict, resolution: str) -> bool:
       """
       Send the resolution back to the originating Zo.
       
       If origin == "zoputer":
           Use zoputer_client.py to send resolution
       If origin == "va":
           Just update local state
       """
   ```
   
   CLI:
   ```bash
   python3 N5/scripts/resolve_decision.py show <decision_id>
   python3 N5/scripts/resolve_decision.py resolve <decision_id> --choice "A" --notes "..."
   python3 N5/scripts/resolve_decision.py route <decision_id>  # Re-route if failed
   ```

2. **Create conditional rule for SMS command:**
   
   Use create_rule tool:
   - Condition: "When I send an SMS containing 'n5 decision'"
   - Rule: "Parse the decision command and handle it:
     - 'n5 decision <id>' alone: Show full context
     - 'n5 decision <id> <choice>': Apply resolution
     Run: python3 N5/scripts/resolve_decision.py from-sms '<message>'"

3. **Resolution routing to zoputer:**
   
   When routing back to zoputer:
   ```python
   prompt = f"""
   A decision you escalated has been resolved by V.
   
   Decision ID: {decision_id}
   Original Question: {decision['summary']}
   Resolution: {resolution}
   Notes: {notes or 'None'}
   
   Please:
   1. Acknowledge receipt
   2. Apply this guidance to the pending situation
   3. Log this as a learning if applicable
   """
   
   zoputer_client.call_zoputer(prompt, correlation_id=decision_id)
   ```

4. **Handle decision viewing:**
   
   When V just texts "n5 decision d7f3" without a choice:
   ```
   📋 Decision d7f3
   
   **Question:** Should I remove review gate for ClientX?
   
   **Background:** Client says it slows them down...
   
   **Options:**
   A) Remove the gate (faster, less safe)
   B) Keep gate, add fast-track option
   C) Customize gate for their workflow
   
   **Recommendation:** B (confidence: 0.6)
   
   **Reply with:**
   - "n5 decision d7f3 A" to choose option A
   - "n5 decision d7f3 B" to choose option B
   - "n5 decision d7f3 custom: <your instruction>"
   ```

5. **Handle edge cases:**
   - Decision already resolved: "This decision was resolved on [date]"
   - Decision expired: "This decision expired. Create new if still needed"
   - Invalid ID: "Decision not found. Current pending: [list short IDs]"
   - Ambiguous response: Ask for clarification

## MUST NOT DO

- Do NOT make decisions autonomously
