# D2.3: VA Mentor Handler

## Objective

Create the va-side handler that receives escalation requests from zoputer and responds with guidance.

## Scope

### Files (owns)
- `Skills/mentor-handler/SKILL.md` - CREATE
- `Skills/mentor-handler/scripts/handle_escalation.py` - CREATE

### Responsibilities
- Parse incoming escalation requests from zoputer
- Generate thoughtful mentor responses
- Log precedents for future reference
- Optionally flag for human review if va is also uncertain

### Must NOT Touch
- `Skills/mentor-escalation/` (zoputer-side, D2.2's scope)
- Human escalation logic (D3.x)
- Existing va rules or behaviors

## MUST DO

1. **Create skill directory:**
   ```bash
   mkdir -p /home/workspace/Skills/mentor-handler/scripts
   ```

2. **Create SKILL.md:**
   - Purpose: Handle escalation requests from zoputer
   - When this runs: Called by va when receiving mentor_escalation type requests
   - Integration: Works with audit-system, may trigger human escalation

3. **Create handle_escalation.py:**
   
   Main function flow:
   ```python
   def handle_escalation(request: dict) -> dict:
       """
       Process an escalation request from zoputer.
       
       1. Validate request format
       2. Check if we have relevant precedent
       3. Assess our own confidence
       4. If confident: provide guidance
       5. If uncertain: escalate to human (D3.x integration point)
       6. Log the interaction
       7. Return response
       """
   ```
   
   Key functions:
   - `validate_request(request: dict) -> bool`
   - `check_precedent(situation: str) -> Optional[dict]` - Check learnings DB
   - `assess_mentor_confidence(request: dict) -> float` - va's confidence
   - `generate_guidance(request: dict, precedent: Optional[dict]) -> dict`
   - `log_precedent(request: dict, response: dict)` - For future reference
   
   CLI (for testing):
   ```bash
   python3 handle_escalation.py --request '{"type": "mentor_escalation", ...}'
   python3 handle_escalation.py --test  # Run with sample request
   ```

4. **Precedent storage:**
   
   Store precedents in `N5/data/mentor_precedents.json`:
   ```json
   {
     "precedents": [
       {
         "id": "prec_001",
         "created": "2026-02-07T12:00:00Z",
         "situation_pattern": "client requests removing review gate",
         "guidance_given": "...",
         "outcome": "successful",
         "reusable": true
       }
     ]
   }
   ```
   
   When handling new requests, fuzzy-match against existing patterns.

5. **Integration point for human escalation:**
   
   If va's confidence is also low (< 0.5), the handler should:
   ```python
   if mentor_confidence < 0.5:
       # Flag for human escalation (D3.x will implement the actual mechanism)
       return {
           "type": "mentor_response",
           "recommendation": None,
           "needs_human": True,
           "human_escalation_reason": "Both zoputer and va uncertain",
           "packaged_context": {...}  # For D3.x to use
       }
   ```

6. **Response quality guidelines:**
   - Always explain rationale
   - Reference precedent if applicable
   - Indicate if this sets new precedent
   - Suggest what zoputer should learn from this

## MUST NOT DO

- Do NOT implement the full human escalation (D3.x does that)
- Do NOT modify zoputer's code
- Do NOT make autonomous decisions that should go to V
