# Script Frontmatter Standard

## Purpose

Python scripts in `N5/scripts/` use YAML frontmatter embedded in their docstrings to enable categorization and discovery.

## Format

```python
#!/usr/bin/env python3
"""Short description of what the script does.

---
product: careerspan
version: 1.0
created: 2026-01-20
updated: 2026-01-20
requires_confirm: true
audit_log: N5/logs/careerspan_audit.jsonl
---

Usage examples:

  python3 N5/scripts/example.py --help
  
"""
```

## Fields

### Required Fields

| Field | Type | Description |
|-------|------|-------------|
| `product` | string | Product affiliation: `careerspan`, `zo`, `n5`, `standalone` |
| `version` | string | Script version (semver style) |
| `created` | date | Creation date (YYYY-MM-DD) |

### Optional Fields

| Field | Type | Description |
|-------|------|-------------|
| `updated` | date | Last update date |
| `requires_confirm` | boolean | Whether script requires `--confirm` for mutations |
| `audit_log` | path | Path to audit log file if script logs operations |
| `tags` | list | Additional categorization tags |

## Product Values

From `N5/config/frontmatter_taxonomy.json`:

| Value | Description |
|-------|-------------|
| `careerspan` | Careerspan product API integrations |
| `zo` | Zo platform integrations |
| `n5` | Internal N5 system scripts |
| `standalone` | General utilities, no product affiliation |

To find all Careerspan scripts:
```bash

```

To list scripts by product:
```bash
grep -h "^product:" N5/scripts/*.py | sort | uniq -c
```
