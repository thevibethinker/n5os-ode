## Context

We already have an atomic script pattern at N5/scripts/set_employer_password.py.

We need a new atomic script that lets a founder:
- enable/disable scanning for an employer
- add credits
- configure org-restricted scanning

API doc (already validated):
- Endpoint: `POST /etc/manage_employer_scan_access`
- Auth: `Authorization: Bearer <founder_token>` (token from env var `FOUNDER_AUTH_TOKEN`)
- Base URL: `https://the-apply-ai--dossier-ai-all-main-fastapi-app.modal.run`

## CLI Design (Required)

Positional:
- `email` (employer email)

Flags:
- `--enable` / `--disable` (mutually exclusive; maps to `scanning_enabled: true/false`)
- `--credits N` (int; maps to `credits_to_add`)
- `--org-scanning-enabled` / `--org-scanning-disabled` (mutually exclusive)
- `--allowed-orgs <one or more values>` (maps to `allowed_organizations`)
- `--base-url` (default to the known base URL)
- `--dry-run` (never call API)
- `--confirm` (required to call API)

Validation:
- If none of `--enable/--disable/--credits/--org-scanning-*/--allowed-orgs` provided → error
- If `--credits` provided and < 0 → error
- If `--allowed-orgs` provided but `--org-scanning-enabled` NOT provided → allow (API may still accept); but print warning explaining likely intent.
