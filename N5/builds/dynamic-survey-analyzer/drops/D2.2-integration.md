# D2.2: Integration Worker

## Mission

You are the **Integration Worker** for the Dynamic Survey Analyzer build. Your job is to wire up the automated pipeline so surveys trigger analysis automatically, and expose the system via SMS commands.

**This worker runs AFTER synthesis (D2.1) is reviewed and approved.**

## Inputs

1. **D2.1 deposit**: Verified synthesis artifacts (dashboard, analysis.md, scripts)
2. **Existing webhook**: `Personal/Integrations/fillout/app.py`
3. **SMS command system**: N5's `sms_deal_handler.py` pattern

## Scope

### Files You Own
- `Personal/Integrations/fillout/app.py` - UPDATE (add auto-trigger)
- `N5/scripts/sms_survey_handler.py` - CREATE
- `Skills/dynamic-survey-analyzer/scripts/run_analysis.py` - CREATE (orchestrator)

### Responsibilities
- Wire webhook to trigger analysis on new form detection
- Create SMS command interface (`n5 survey <command>`)
- Set up the 30-day monitoring agent
- Test end-to-end flow

### Must NOT Touch
- `generate_dashboard.py` (D2.1 owns)
- `fillout_client.py` (D1.2 owns)
- `analysis.md` content (D2.1 owns)
- Level Upper prompts (D1.3 owns)

## MUST DO

### 1. Create `run_analysis.py` Orchestrator

Single-command entry point that runs the full pipeline:

```python
#!/usr/bin/env python3
"""
Dynamic Survey Analyzer - Full Pipeline Orchestrator

Usage:
    python3 run_analysis.py <form_id> [--account personal|careerspan] [--skip-level-upper]
    
Runs:
1. Fetches form structure and submissions
2. Generates interpretation framework (D1.1 logic)
3. Computes quantitative analysis (D1.2 logic)
4. Runs Level Upper analysis (D1.3 logic) unless --skip-level-upper
5. Synthesizes findings and generates dashboard (D2.1 logic)
6. Outputs to Datasets/survey-analyses/<form_id>/
"""
```

### 2. Create `sms_survey_handler.py`

SMS command interface following existing `sms_deal_handler.py` pattern:

```bash
# Commands to support:
n5 survey analyze <form_id>      # Run full analysis
n5 survey status <form_id>       # Check if analysis exists, when last updated
n5 survey list                   # List analyzed surveys
n5 survey refresh <form_id>      # Re-run analysis with latest data
```

### 3. Update Webhook Auto-Trigger

Enhance `Personal/Integrations/fillout/app.py`:
- On NEW_FORM_DETECTED: Call `/zo/ask` with analysis prompt
- On submission for tracked form: Check if threshold reached for re-analysis
- Add configurable threshold (default: 5 new submissions triggers refresh)

### 4. Create Monitoring Agent

Set up scheduled agent that runs daily for 30 days:
- Check all tracked forms for new submissions
- If new submissions since last analysis, queue refresh
- Text V summary: "[SURVEY] 3 forms updated, 12 new responses total"

Agent RRULE: `FREQ=DAILY;INTERVAL=1;COUNT=30`

### 5. Test End-to-End

1. Verify SMS commands work
2. Verify webhook triggers analysis
3. Verify dashboard regenerates correctly
4. Document any issues in handoff notes

## MUST NOT DO

- Do not modify analysis content or dashboard visuals
- Do not change fillout_client.py interface
- Do not create new scheduled agents without documenting them
