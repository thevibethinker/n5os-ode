# D1.2: Quantitative Analysis Worker

## Mission

You are the **Quantitative Analysis Worker** for the Dynamic Survey Analyzer build. Your job is to create the data infrastructure (API client, webhook enhancement) and perform core statistical analysis on survey responses.

## Inputs

1. **Fillout API access** via environment variables:
   - `FILLOUT_SECRET_PERSONAL`
   - `FILLOUT_SECRET_CAREERSPAN`

2. **Existing webhook code** at `N5/Integrations/fillout/app.py`

3. **Interpretation framework** from D1.1 (via build ledger or parallel execution)

## Scope

### Files You Own
- `Skills/dynamic-survey-analyzer/scripts/fillout_client.py` - CREATE
- `N5/Integrations/fillout/app.py` - UPDATE (add new form detection)
- `N5/Integrations/fillout/known_forms.json` - CREATE (state tracking)

### Responsibilities
- Multi-account Fillout API client
- New form detection in webhook
- Frequency distributions for all questions
- Cross-tabulation for key question pairs
- Statistical summaries (counts, percentages, averages for scales)

### Must NOT Touch
- `SKILL.md` (D1.1 owns this)
- Dashboard HTML generation (D2.1 owns this)
- Creative/divergent analysis (D1.3 owns this)

## MUST DO

1. **Create `fillout_client.py`** with CLI interface:
   ```bash
   # List all forms from both accounts
   python3 fillout_client.py --list-forms
   
   # Get form structure
   python3 fillout_client.py --form-structure <formId>
   
   # Get all submissions
   python3 fillout_client.py --submissions <formId>
   
   # Auto-detect account for a formId
   python3 fillout_client.py --detect-account <formId>
   ```

2. **Update webhook** (`N5/Integrations/fillout/app.py`):
   - Load/save `known_forms.json` (set of seen formIds)
   - On POST: check if formId is in known_forms
   - If new: add to known_forms, log "NEW_FORM_DETECTED" event
   - Continue logging submission as before

3. **Create analysis output** as JSON:
   ```json
   {
     "form_id": "...",
     "analysis_timestamp": "...",
     "total_submissions": N,
     "question_summaries": {
       "<question_id>": {
         "question_text": "...",
         "question_type": "multiple_choice|text|scale|...",
         "response_count": N,
         "distribution": { "option_a": 5, "option_b": 3 },
         "percentage": { "option_a": 62.5, "option_b": 37.5 }
       }
     },
     "cross_tabs": [
       {
         "q1": "...",
         "q2": "...",
         "matrix": { ... }
       }
     ]
   }
   ```

4. **Write analysis to deposit** for D1.3 and D2.1 to consume

## MUST NOT DO

- Do not interpret what the numbers mean (D1.1's hypothesis + D1.3's insights)
- Do not generate HTML/visualizations (D2.1's job)
- Do not modify SKILL.md

## Technical Notes

- Use `requests` library for API calls
- Both API keys use same base URL: `https://api.fillout.com/v1/api/`
- Forms endpoint: `GET /forms`
- Form structure: `GET /forms/<formId>`
- Submissions: `GET /forms/<formId>/submissions`
- The webhook already logs to JSONL at `N5/Integrations/fillout/events/`
