# W2.3 — Documentation + operator workflows

## Goal
Update documentation and define operator workflows so Content Library is unambiguous and consistently usable.

## Wave-1 Learnings (must incorporate)
- **Normalization standard (B):** store *clean original text* + *short summary in frontmatter*.
- **Extraction:** prefer companion HTML → `trafilatura` → clean text. Fallback to heuristic markdown stripping.
- **Social/X:** trafilatura may fail on JS-rendered pages; use markdown fallback + light stripping.
- **Context preservation:** never overwrite curated fields; automation only fills missing.
- **Calendly:** links should carry usage context (audience/priority/use_when), and sync should create DB records.

## Context
- System guide: `Documents/System/guides/content-library-system.md`
- CLI: `N5/scripts/content_library.py`, `N5/scripts/content_ingest.py`
- Good example file: `Knowledge/content-library/links/calendly/*.md`

## Tasks
1. Define canonical workflows (step-by-step commands + expected artifacts):
   - Add a **link** (structured, minimal content)
   - Add an **article** (save_webpage → clean original text + summary)
   - Add a **profile/resource/reference** (structured frontmatter + short about)
   - Add **media** (store file + companion markdown metadata)

2. Define Calendly usage workflow:
   - Where to store Calendly links (files + DB)
   - How to set curated context:
     - `audience`
     - `priority`
     - `use_when`
   - How merge-safe behavior works (defaults fill only when missing)

3. Document normalization behavior:
   - what “clean original text” means (remove nav/footer/chrome)
   - when summary is generated / when omitted
   - how to re-normalize an existing file (if supported)

4. Update the guide accordingly:
   - correct role framing: Content Library = first port of call for shareable assets
   - fix any outdated notes about auto-ingest (`--type article` etc.)
