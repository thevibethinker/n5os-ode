# W3.1 — Schema Migration + API Bug Fix (IMPLEMENTATION)

## Goal
Execute the schema migration and fix the critical `content_library.py` add() bug. This is foundational work that W3.2 and W3.3 depend on.

## Context from Prior Waves
- **W1.2** designed schema additions: `subtype`, `summary`, `managed_fields` columns + indexes
- **W2.1** identified critical bug: `content_library.py` add() uses `type` column but actual DB has `content_type`
- **W2.2** requires `id_redirects` table for migration

## Tasks

### 1. Fix content_library.py add() method (CRITICAL)
**File:** `N5/scripts/content_library.py`

The add() method currently has:
```python
INSERT INTO items (id, type, title, ...)
```

But actual DB schema has `content_type` not `type`. Fix:
- Change `type` → `content_type` in INSERT statement
- Add `subtype` parameter (optional, default None)
- Add `summary` parameter (optional, default None)
- Keep backward compat: `type` param still accepted, mapped to `content_type`

### 2. Run schema migration
**Create:** `N5/scripts/migrations/content_library_v5_schema.py`

Execute these ALTER TABLE statements (idempotent):
```sql
-- Add new columns (ignore if exists)
ALTER TABLE items ADD COLUMN subtype TEXT;
ALTER TABLE items ADD COLUMN summary TEXT;
ALTER TABLE items ADD COLUMN managed_fields TEXT;
ALTER TABLE items ADD COLUMN external_id TEXT;

-- Add indexes
CREATE INDEX IF NOT EXISTS idx_items_subtype ON items(subtype);
CREATE INDEX IF NOT EXISTS idx_items_type_subtype ON items(content_type, subtype);
CREATE INDEX IF NOT EXISTS idx_items_external_id ON items(external_id);

-- Create id_redirects table for migration
CREATE TABLE IF NOT EXISTS id_redirects (
    old_id TEXT PRIMARY KEY,
    new_id TEXT NOT NULL,
    migrated_at TEXT NOT NULL DEFAULT (datetime('now'))
);
```

### 3. Update ContentItem dataclass
**File:** `N5/scripts/content_library.py`

Add fields to ContentItem:
- `subtype: Optional[str] = None`
- `summary: Optional[str] = None`
- `managed_fields: Optional[str] = None`
- `external_id: Optional[str] = None`

Update `_row_to_item()` to map these from DB row.

### 4. Add search by subtype
**File:** `N5/scripts/content_library.py`

Update `search()` method to accept `subtype` parameter:
```python
def search(self, query=None, item_type=None, content_type=None, subtype=None, ...):
```
