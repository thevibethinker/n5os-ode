# W1.3 — Ingest v5 (clean text + summary; no raw scrape)

## Goal
Redesign ingest so that web-derived inputs become:
- **clean original text** (main content only)
- **short summary in frontmatter**
- no UI/nav/boilerplate “raw scrape” stored

## Context
- Current `content_ingest.py` reads a file and stores entire contents.
- V wants the “original text” preserved, but cleaned.
- We can use the HTML saved by `save_webpage`/`read_webpage` when available.

## Tasks
1. Propose (and optionally implement) a normalization pipeline:
   - Inputs: (a) markdown from `save_webpage`, (b) HTML file, (c) raw md (fallback)
   - Output: canonical markdown (frontmatter + clean text body)
   - Summary generation: 1–3 sentences (LLM allowed) placed in frontmatter.
2. Define *how* we classify ingest modes:
   - `article`: keep full article text
   - `link/profile/resource/reference`: **do not** store full scrape; store structured metadata + (optional) short “about” text.
3. Define deterministic boilerplate stripping strategy:
   - Prefer readability/trafilatura or similar
   - Ensure reproducibility and cost awareness
4. Identify tests:
   - Golden fixtures: known inputs → expected output
   - Ensure no “Skip to content”/nav remains

## Deliverables
- A proposed function-level design (interfaces, functions to add).
- If you implement: ensure it’s behind a flag or only affects new ingest flows.
