# Workers README — Content Library v5

## How to run a worker thread
1. Open a new Zo chat thread.
2. Set the thread title exactly to the `thread_title` in the worker brief frontmatter.
3. Paste the worker brief contents into the thread.
4. Worker executes and writes completion JSON to:
   - `N5/builds/content-library-v5/completions/<worker_id>.json`

## Rules
- Workers do **not** commit.
- If a worker changes code locally for validation, they must still only report outcomes + diffs and leave commit to orchestrator.

## Wave Rules
- Run Wave 1 first (W1.1–W1.4).
- Return here with all completions; orchestrator updates briefs for Wave 2.
