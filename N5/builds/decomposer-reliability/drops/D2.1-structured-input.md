# Drop D2.1: Structured Input Mode

## Objective

Add `--input-json` flag to decompose.py that accepts pre-structured data from the Careerspan endpoint. When structured data is provided, copy verbatim fields directly instead of extracting them via LLM.

## Context

The upstream Careerspan endpoint will eventually provide cleaner structured data. When that data arrives, we should:
- Copy `our_take` blocks verbatim (no LLM extraction needed)
- Use provided story IDs directly (no hallucination risk)
- Still validate that everything matches the source document

This is preparation for the cleaner data pipeline.

### 2. Create structured input processing path

```python
def process_structured_input(json_path: str, doc_content: str, output_dir: Path) -> tuple[list, list]:
    """
    Process pre-structured JSON input.
    
    Expected JSON structure:
    {
        "candidate": {...},
        "skills": [
            {
                "skill_name": "...",
                "category": "...",
                "rating": "...",
                "our_take": "...",  # COPY VERBATIM
                "story_ids": ["..."],
                ...
            }
        ],
        "stories": {
            "STORY_ID": {
                "title": "...",
                "content": "..."
            }
        }
    }
    
    Returns: (skills_list, validation_failures)
    """
    with open(json_path) as f:
        data = json.load(f)
    
    skills = []
    failures = []
    
    for skill_data in data.get('skills', []):
        # COPY verbatim fields directly - NO LLM
        skill = {
            'skill_name': skill_data['skill_name'],
            'category': skill_data.get('category'),
            'rating': skill_data.get('rating'),
            'our_take': skill_data.get('our_take'),  # VERBATIM
            'support': [],
            # ... other fields
        }
        
        # Build support array from story_ids
        for story_id in skill_data.get('story_ids', []):
            skill['support'].append({
                'source': story_id,
                'type': skill_data.get('evidence_type', 'Direct'),
                'score': None,
                'rating': None
            })
        
        # STILL VERIFY against source doc
        if skill['our_take'] and not fuzzy_substring_match(skill['our_take'], doc_content)[0]:
            failures.append({
                'skill_name': skill['skill_name'],
                'reason': 'our_take from JSON not found in source doc',
                'extracted_text': skill['our_take'][:200]
            })
        else:
            skills.append(skill)
    
    return skills, failures
```

  "$schema": "http://json-schema.org/draft-07/schema#",

      "type": "object",

      "type": "array",
      "items": {
        "type": "object",
        "required": ["skill_name", "our_take"],

  }
}

## Anti-Patterns to Avoid

- Do NOT run LLM extraction when structured input is provided
- Do NOT skip validation just because input is "trusted"
- Do NOT modify our_take text in any way (trim, normalize, etc.)
