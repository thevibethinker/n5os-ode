# Drop D1.1: Provenance Verification Layer

## Objective

Create `verify.py` — a script that validates decomposer outputs by checking that every `our_take` field exists as a substring in the source document.

## Context

The careerspan-decomposer is hallucinating — mixing up story details, paraphrasing "Our Take" sections that should be verbatim, and assigning wrong story IDs. The root cause is that nothing currently verifies extracted text against the source.

## Deliverables


## Success Criteria

1. `verify.py hardik-flowfuse --source <doc>` runs and reports verification status
2. Fuzzy match threshold of 0.95 catches paraphrasing but allows whitespace differences
3. Exit code reflects pass/fail status
4. `validate.py --source` integrates verification

## Anti-Patterns to Avoid

- Do NOT use exact string matching (will fail on whitespace differences)
- Do NOT lower threshold below 0.90 (defeats purpose)
- Do NOT skip story ID verification (these are hallucinated too)
