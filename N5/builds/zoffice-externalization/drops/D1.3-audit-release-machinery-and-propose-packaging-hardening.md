# Drop Brief: Audit release machinery and propose packaging hardening

**Mission:** Audit the existing Zoffice release path and determine exactly what must be hardened or updated so that the next output can be a reviewable external release folder rather than an ambiguous archive.

**Output:** deposit with implementation recommendations for D2.1 and, if helpful, a small artifact note under `N5/builds/zoffice-externalization/artifacts/package-readiness-checklist.md` outline suggestions.

---

## Context

`Zoffice/scripts/create_release_bundle.py` already creates release bundles and prior release folders exist for `v2.0.0-rc1` and `v2.0.0-rc2`. However, the build goal now is more specific: create a **reviewable external package** with integrated transfer capability and a coherent deployment story. This drop owns the audit of the release machinery and what needs to change before we generate the next version folder.

## MUST DO
- Read the current release script and inspect prior release folder contents
- Identify strengths, gaps, and any ambiguity in the current release path
- Recommend what should change in Bootloader/Manifest/release script/package docs before D3.1 creates the next version folder
- Propose the likely next version tag progression after rc2

## MUST NOT DO
- Do not implement code changes in this drop
- Do not create the new release folder yet
- Do not ignore prior rc1/rc2 outputs when making recommendations
