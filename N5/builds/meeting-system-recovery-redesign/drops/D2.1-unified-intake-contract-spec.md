# Drop Brief: Unified Intake Contract Spec

**Mission:** Define the canonical landing contract that normalizes Fireflies, Fathom, and Pocket into one ready-to-process meeting/reflection intake format.

**Output:** `N5/builds/meeting-system-recovery-redesign/artifacts/intake-contract.md`

---

## Context

Historical data will be recovered from the zip, but future resilience depends on one canonical intake contract. Every source should either land already normalized in `Personal/Meetings/Inbox/` or be transformed immediately upon landing into the same ready-to-process representation. Pocket is a special case: default to reflection unless initial evaluation indicates meeting signals such as multiple speakers.

## Technical Concepts

| Concept | V's Level | Why It Matters Here |
|---------|-----------|---------------------|
| Canonical contract | Intermediate | Multiple tools can feed one pipeline if they agree on the landing shape |
| Source adapters | Intermediate | Fireflies/Fathom/Pocket can differ upstream while still converging downstream |
| Ready-to-process state | Intermediate | The inbox should contain normalized assets, not raw ambiguity |
