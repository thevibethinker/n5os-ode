# D1.1: Core Import Script

## Context

You are building a skill importer for N5 OS. The skills.sh ecosystem (agentskills.io spec) has 100+ skills hosted on GitHub that can be imported into the N5 `Skills/` directory with frontmatter adaptation.

## Scope

### Files You Own
- `Skills/skills-importer/scripts/import_skill.py` - CREATE

### Must Not Touch
- `Skills/skills-importer/SKILL.md` (D1.2 owns this)
- `Skills/skills-importer/references/` (D1.2 owns this)
- Any other existing skills

## MUST DO

1. **Create the directory structure:**
   ```
   Skills/skills-importer/
   └── scripts/
       └── import_skill.py
   ```

2. **Implement CLI with these commands:**
   ```bash
   # Import a single skill
   python3 Skills/skills-importer/scripts/import_skill.py anthropics/skills/frontend-design
   
   # List skills in a repo
   python3 Skills/skills-importer/scripts/import_skill.py vercel-labs/agent-skills --list
   
   # Import all skills from a repo
   python3 Skills/skills-importer/scripts/import_skill.py anthropics/skills --all
   
   # Dry run (show what would happen)
   python3 Skills/skills-importer/scripts/import_skill.py obra/superpowers/brainstorming --dry-run
   
   # Force overwrite existing
   python3 Skills/skills-importer/scripts/import_skill.py anthropics/skills/frontend-design --force
   
   # Custom destination name
   python3 Skills/skills-importer/scripts/import_skill.py anthropics/skills/frontend-design --dest my-frontend
   ```

3. **GitHub fetching logic:**
   - Parse source: `owner/repo/skill-name` or `owner/repo`
   - Detect repo structure:
     - Some repos have skills at root: `owner/repo/SKILL.md`
     - Some have `skills/` subdir: `owner/repo/skills/skill-name/SKILL.md`
   - Use raw.githubusercontent.com (no auth needed):
     ```
     https://raw.githubusercontent.com/{owner}/{repo}/main/{path}
     ```
   - Try `main` branch first, fall back to `master`
   - Fetch SKILL.md first, then supporting directories (scripts/, references/, assets/)

4. **Frontmatter transformation:**
   ```python
   # INPUT fields (skills.sh)
   {
       "name": "skill-name",
       "description": "...",
       "license": "...",  # optional
       "compatibility": "...",  # optional
       "metadata": {...}  # optional
   }
   
   # OUTPUT fields (N5)
   {
       "name": "skill-name",  # preserve
       "description": "...",  # preserve
       "compatibility": "Imported from skills.sh",  # set if not present
       "metadata": {
           "author": "{owner} (imported)",  # from GitHub owner
           "source": "{owner}/{repo}/{skill-name}",  # full reference
           "imported_at": "2026-02-07"  # import date
       },
       "created": "2026-02-07",  # today
       "last_edited": "2026-02-07",  # today
       "version": "1.0",  # reset to 1.0
       "provenance": "skills-importer"  # attribution
   }
   ```

5. **Validation checks:**
   - SKILL.md must exist at source
   - `name` field must be present in frontmatter
   - Destination must not exist (unless `--force`)
   - Warn if scripts/ directory not found (skill may be documentation-only)

6. **Installation:**
   - Create `Skills/{name}/` directory
   - Write transformed SKILL.md
   - Copy scripts/, references/, assets/ if they exist
   - Print success message with path

7. **Use Python with these libraries only:**
   - `requests` (for HTTP)
   - `pyyaml` (for frontmatter parsing)
   - `argparse` (for CLI)
   - `pathlib` (for paths)
   - Standard library (json, os, sys, datetime, re)

## MUST NOT DO

- Do NOT create SKILL.md documentation (D1.2 handles that)
- Do NOT create references/ directory (D1.2 handles that)
- Do NOT modify any existing skills
- Do NOT use GitHub API (use raw.githubusercontent.com instead)
- Do NOT require authentication
