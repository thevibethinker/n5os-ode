# B21: Key Quotes & Strategic Questions

## Key Quotes

**"What you effectively want to do is you first have to create a way for meetings to be generated and processed... Then you have to create a layer on top of that that takes those meeting blocks... And that is so that you have a nice store... It's an actual fucking database, if that makes sense."**  
— Vrijen Attawar  
*Context:* Mapping how the meeting ingestion pipeline, modular block processing, and a content library database all link together to turn transcripts into actionable knowledge.

**"The quality of this is so incredibly high because it not only relies on memory because it is agentic, it's able to... you actually get more bang for your buck."**  
— Vrijen Attawar  
*Context:* Comparing Zo’s agentic orchestration—selection of files, compaction, and decision-making—to one-off consumer LLM experiences like ChatGPT.

**"The key problem for me is that while I do repeat a good amount of stuff in each call, also in each call something new comes up and I forget which things are new... What I really, really want is for something else... to parse through... Is this a new concept... tag it and add it as a new unit of learning."**  
— David Speigel  
*Context:* Articulating the need for the system to detect novel insights, explain how they differ from past lessons, and turn them into structured learning blocks.

**"Conversation End is a function that will clean up after itself... it'll summarize what was done, it'll summarize the workers, it will commit to Git and it'll identify what artifacts were created... those are the kind of quality-of-life improvements that I firmly believe are lacking in Zo."**  
— Vrijen Attawar  
*Context:* Explaining why Conversation End and the Build Orchestrator are essential parts of the workflow: they keep sessions persistent, organized, and auditable.

**"I built out as promised for fucking ages. I finally built out a repo with most of the functionality... Let me just send you the GitHub."**  
— Vrijen Attawar  
*Context:* Signaling delivery of the promised automation framework as a tangible milestone before unpacking the remaining scoping work.

## Strategic Questions Raised

**"How is this different from doing it in cloud code?"**  
— David Speigel  
*Significance:* Forced clarification on Zo’s persistence, file-system advantages, and the value-added ergonomics that distinguish the platform from Claude’s transient sessions.

**"What are the high level things as I'm going forward that I should be doing? Like closing conversations or having it do things—just give me the ongoing things I should be thinking about and doing in an instruction document."**  
— David Speigel  
*Significance:* Highlights the need for an explicit “playbook” for habitual behaviors (e.g., closing conversations, running bootloader/personalize) so that Zo can scale from tactical coaching into a predictable operating rhythm.

**"What would be the container of the information you extract?"**  
— Vrijen Attawar  
*Significance:* Prompted modelers to define the unit of learning (slides, decks, modules) before automating transcript parsing, ensuring extracted insights align to downstream artifacts.
