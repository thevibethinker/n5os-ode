# D1.3: Processor Fix

## Objective
Update `Skills/meeting-ingestion/scripts/processor.py` to load and use the canonical prompt files from `Prompts/Blocks/` instead of inline one-liner definitions.

## Scope

### Current broken implementation:
```python
BLOCK_DEFINITIONS = {
    "B01_DETAILED_RECAP": "Comprehensive meeting summary covering all topics...",
    # One-liner descriptions
}

def generate_block(transcript_text, block_code, meeting_context):
    block_description = BLOCK_DEFINITIONS.get(block_code, "...")
    prompt = f"""Generate the {block_code} intelligence block...
## Block Definition
{block_code}: {block_description}
...
"""
```

### Required fix:

1. Add function to load prompt files:
```python
def load_prompt_file(block_code: str) -> Optional[str]:
    """Load canonical prompt from Prompts/Blocks/."""
    prompt_path = Path(f"/home/workspace/Prompts/Blocks/Generate_{block_code}.prompt.md")
    if prompt_path.exists():
        return prompt_path.read_text()
    return None
```

2. Update `generate_block()` to use loaded prompts:
```python
def generate_block(transcript_text, block_code, meeting_context):
    # Try to load canonical prompt
    prompt_template = load_prompt_file(block_code)
    
    if prompt_template:
        # Inject transcript and context into the prompt template
        prompt = f"""{prompt_template}

## Transcript to Analyze
{transcript_text[:30000]}

## Meeting Context
- Date: {meeting_context.get('date')}
- Participants: {', '.join(meeting_context.get('participants', []))}
"""
    else:
        # Fallback to inline definition
        block_description = BLOCK_DEFINITIONS.get(block_code, "Meeting intelligence block")
        prompt = f"""... (existing fallback logic)"""
```
