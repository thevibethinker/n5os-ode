# D1.1: Meeting Block Quality Scan

## Objective
Scan all meetings from the past week (Jan 19-26, 2026) and assess the quality of their intelligence blocks.

## Scope

### Files to scan:
- All meeting folders in `Personal/Meetings/Week-of-2026-01-19/`
- All meeting folders in `Personal/Meetings/Inbox/`
- Optionally `Personal/Meetings/Week-of-2026-01-12/` for comparison

### Quality signals to check per block:
1. **YAML frontmatter present?** — Look for `---` at start
2. **Has `provenance` field?** — Traces back to conversation
3. **Has `block_id` and `block_name`?** — Proper metadata
4. **Has expected section headers?** — Per block type spec
5. **Content depth** — Word count, presence of tables/quotes
6. **Starts with markdown fence?** — Bad sign (```markdown at top = raw LLM output)

### Good block example (from Anna Bao meeting):
```yaml
---
created: 2026-01-23
provenance: con_MGF42FmSjo20ltKq
block_id: B01
block_name: DETAILED_RECAP
---
# B01: Detailed Recap
## Chronological Summary
...
```

### Bad block example (from RBV meeting):
```markdown
# B01: Meeting Intelligence Block
## Meeting Overview
...
```
(No frontmatter, generic structure, starts with fence)

## MUST DO
- Scan all meetings in the specified folders
- For each meeting, check each block file (B01*.md, B05*.md, etc.)
- Assign a quality score: `good`, `degraded`, `unknown`
- Output a JSON summary

## MUST NOT DO
- Do not regenerate any blocks
- Do not modify any meeting files
- Do not delete anything
