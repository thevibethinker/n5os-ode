# D1.1: Data Layer Consolidation

## Context
Building Performance Intelligence Dashboard v2. This drop creates the unified data access layer that all subsequent drops will use.

## Scope

### Files to Create
- `N5/scripts/performance_data.py` — Unified data access module

### Files to Read (reference only)
- `Personal/Health/workouts.db` — Fitbit data (resting HR, activity, sleep, workouts)
- `N5/task_system/tasks.db` — Task registry
- `N5/data/content_library.db` — Content library items
- `N5/data/journal.db` — Journal entries and bio snapshots

### Must NOT Touch
- `N5/scripts/performance_dashboard.py` — Will be rewritten in D1.4
- Any database schemas (read-only access)

## MUST DO

1. **Create `performance_data.py`** with these functions:

```python
def get_weekly_resting_hr(days=7) -> dict:
    """Returns: {avg, min, max, trend, daily_values, data_points}"""

def get_weekly_activity(days=7) -> dict:
    """Returns: {steps_avg, steps_total, active_min_avg, workout_count, daily_values, data_points}"""

def get_weekly_sleep(days=7) -> dict:
    """Returns: {avg_hours, min_hours, max_hours, daily_values, data_points}"""

def get_weekly_tasks(days=7) -> dict:
    """Returns: {completed, added, pending_total, completion_rate, net_change}"""

def get_weekly_content(days=7) -> dict:
    """Returns: {items_added, by_type: {article: N, link: N, ...}, sources: [...]}"""

def get_weekly_mood(days=7) -> dict:
    """Returns: {entries: [...], predominant_mood, data_points}"""

def get_data_freshness() -> dict:
    """Returns: {source_name: {last_update, days_stale, status: 'ok'|'stale'|'missing'}}"""
```

2. **Trend calculation**: Compare current period to previous period
   - `trend` should be: "up", "down", or "stable" (±5% = stable)

3. **Graceful handling**: If a data source is empty or missing, return sensible defaults with `data_points: 0`

4. **Add CLI test mode**: `python3 performance_data.py --test` prints sample output from all functions

## MUST NOT DO

- Do NOT modify any database schemas
- Do NOT create new database tables
- Do NOT touch the existing `performance_dashboard.py` yet
- Do NOT integrate calendar or weather (those are D1.2 and D1.3)

## Expected Output

1. File created: `N5/scripts/performance_data.py`
2. All 7 functions implemented and tested
3. CLI test passes: `python3 N5/scripts/performance_data.py --test`

## Acceptance Criteria

- [ ] `performance_data.py` exists and is syntactically valid
- [ ] `--test` flag runs without errors
- [ ] Each function returns a dict with documented keys
- [ ] Empty data sources don't crash (return `data_points: 0`)
- [ ] Trend calculation works (up/down/stable)

## Database Schemas (Reference)

**workouts.db:**
- `daily_resting_hr`: date, resting_hr
- `daily_activity_summary`: date, steps, active_minutes, calories
- `daily_sleep`: date, minutes_asleep, sleep_score
- `workouts`: date, workout_type, duration_min, hr_avg

**tasks.db:**
- `tasks`: id, title, status, created_at, completed_at, domain_id, priority_bucket

**content_library.db:**
- `items`: id, title, type, source, created_at

**journal.db:**
- `bio_snapshots`: created_at, time_period, mood, resting_hr, sleep_minutes
