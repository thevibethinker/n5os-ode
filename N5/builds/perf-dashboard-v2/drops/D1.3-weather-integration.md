# D1.3: Weather Integration

## Context
Building Performance Intelligence Dashboard v2. This drop adds weather data from Open-Meteo API (free, no key required).

## Scope

### Files to Create
- `N5/scripts/weather_data.py` — Weather API client

### Files to Modify
- `N5/scripts/performance_data.py` — Add `get_weekly_weather()` function

### Must NOT Touch
- `N5/scripts/performance_dashboard.py` — Will be rewritten in D1.4
- Calendar or other integrations

2. **Open-Meteo API**:
   - Endpoint: `https://archive-api.open-meteo.com/v1/archive` (for historical data)
   - Or: `https://api.open-meteo.com/v1/forecast` with `past_days` parameter
   - Parameters: `latitude`, `longitude`, `daily=temperature_2m_max,temperature_2m_min,precipitation_sum,weathercode`
   - Temperature units: Request Fahrenheit with `temperature_unit=fahrenheit`
   - No API key required

3. **Add to `performance_data.py`**:


4. **Error handling**: If API fails, return empty data with `data_points: 0` and `error: "API unavailable"`

## Open-Meteo API Example

```bash
curl "https://api.open-meteo.com/v1/forecast?latitude=40.71&longitude=-74.01&daily=temperature_2m_max,temperature_2m_min,precipitation_sum,weathercode&temperature_unit=fahrenheit&past_days=7&timezone=America/New_York"
```

Response structure:
```json
{
  "daily": {
    "time": ["2026-01-19", "2026-01-20", ...],
    "temperature_2m_max": [45.2, 42.1, ...],
    "temperature_2m_min": [32.1, 30.5, ...],
    "precipitation_sum": [0.0, 0.12, ...],
    "weathercode": [3, 61, ...]
  }
}
```
