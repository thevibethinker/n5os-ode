# Drop Brief: Future-State Canonical System Map

**Mission:** Produce the canonical future-state N5 system map, including target file/folder structure and the placement of kernel, services, zoffice, apps, data, runtime, and records.

## Context

This Drop converts the foundational architecture into a concrete canonical system map. It should define where things live, why they live there, and how current N5 subsystems map into the target structure.

It should be elegant enough to serve as the north-star model, but migration-aware enough to be used incrementally.

## Scenarios

S1: Canonical architecture clarity
  Given: The system needs a clean target model
  When: The canonical map is produced
  Then: It clearly places kernel, services, zoffice, apps, interfaces, data, runtime, and records
  Verify: LLM: Check that each major layer has an explicit path and responsibility

S2: Current-to-future mapping
  Given: Existing systems already occupy many paths
  When: The map is written
  Then: It includes how major current subsystems map into the future structure
  Verify: LLM: Check that at least 8 major current subsystems are mapped into future placement

S3: Migration-aware elegance
  Given: The future-state should be clean without being naïve
  When: The map is reviewed
  Then: It is elegant as a north star but not dependent on immediate physical reorganization
  Verify: LLM: Check that the map is compatible with staged migration rather than requiring a big bang move
