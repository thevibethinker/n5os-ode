## MUST DO

### 1. Create Voice-Optimized Format

Every document should follow voice-friendly structure:
- Short paragraphs (2-3 sentences max)
- Clear headings for scanning
- "Trigger phrases" — what callers actually say
- Actionable next steps
- Max 200 words per knowledge chunk

### 2. Document the Three Levels

**Level 1: Conversation Engineering**
- Delay the Draft tactic
- Clarification Gates
- Adversarial Probing
- Threshold Rubrics
- Offensive/Defensive Modes

**Level 2: Environment Engineering**
- Personalization instructions
- Memory & preferences
- Personas (Teacher, Strategist, Writer, etc.)
- Tool access patterns
- Cognitive guardrails

**Level 3: Pipeline Engineering**
- Import/adapt building blocks
- Bring data in (webhooks, APIs, syncs)
- Let it act (scheduled tasks, triggers)
- Cross-tool orchestration

### 3. Create Assessment Subroutine

4 optional questions to determine caller level:

**Q1:** "When you start a new AI conversation, do you typically:
- Jump straight to asking for deliverables (Level 1 early)
- Spend time setting context first, then request (Level 1 advanced)
- Use saved prompts or templates (Level 2)
- Have systems that trigger without you (Level 3)"

**Q2:** "How do you handle AI responses that feel off?
- Start a new chat (Level 1 early)
- Ask clarifying questions in the same thread (Level 1 advanced)
- Refer to saved preferences or rubrics (Level 2)
- Check automated validation outputs (Level 3)"

**Q3:** "What's your relationship with 'organization'?
- I organize files and notes manually (Level 1)
- I have systems that organize for me (Level 2)
- Organization happens automatically through flows (Level 3)"

**Q4:** "Think about your last 3 repetitive tasks. How were they handled?
- I did them manually (Level 1)
- I used AI to help but still drove (Level 2)
- They ran without my involvement (Level 3)"

Scoring: Mostly A = Level 1, Mostly B = Level 2, Mostly C = Level 3

### 4. Document V's Patterns

From the source material, extract:
- Build pattern: Need → Decompose → Build blocks → Connect
- "What are the blocks?" methodology
- Cross-conversation workflow splitting
- Semantic hunger awareness
- Pools vs. flows design principle

### 5. Create Use Case Inspiration

Reference V's actual systems:
