## CRITICAL SECURITY CONSTRAINTS

**THIS WEBHOOK MUST BE READ-ONLY:**
1. NO Zo API calls (`ZO_CLIENT_IDENTITY_TOKEN` - DO NOT USE)
2. NO calendar access (`checkAvailability`, `createCalendarEvent` - DO NOT INCLUDE)
3. NO external data retrieval
4. NO execution of workflows
5. NO personal caller data storage (only anonymous patterns)

**ALLOWED:**
- Read local knowledge base files (`Knowledge/zo-hotline/`)
- Log anonymous call patterns (topic, level assessed, escalation requested)
- Generate responses from the system prompt
- Offer escalation (collect contact for V callback)

## Webhook Endpoints

### POST /assistant-request
Returns the Guide assistant config with:
- System prompt from `prompts/system-prompt.md`
- Voice settings
- Tool definitions (assessment, recommendations, concepts, logging - NO calendar)

### POST /tool-calls
Handle these tools ONLY:
1. `assessCallerLevel` - Run 4-question diagnostic, return score and level
2. `getRecommendations` - Return level-appropriate tactics from knowledge base
3. `explainConcept` - Return framework explanations from knowledge base
4. `requestEscalation` - Log escalation request (caller contact for V)

### POST /end-of-call-report
Log anonymous metrics:
- Call duration
- Topics discussed (no PII)
- Level assessed (if applicable)
- Escalation requested (boolean)

## Implementation Notes

1. Use Bun.serve() for HTTP server
2. Port from VAPI_HOTLINE_PORT env (default 4243)
3. Read knowledge files synchronously at startup
4. Log to DuckDB at Datasets/zo-hotline-calls/data.duckdb

**Tool Implementation Pattern:**

```typescript
if (toolName === "assessCallerLevel") {
  // Questions are embedded in prompt - tool just records answer
  const score = calculateScore(params.answers);
  const level = score < 1.5 ? 1 : score < 2.5 ? 2 : 3;
  return { score, level, interpretation: getInterpretation(level) };
}

if (toolName === "getRecommendations") {
  // Read from knowledge base, return level-appropriate content
  const content = readKnowledgeFile(`Knowledge/zo-hotline/quick-wins-by-level.md`);
  return { recommendations: extractForLevel(content, params.level) };
}

if (toolName === "explainConcept") {
  // Map concepts to knowledge files
  const conceptMap = {
    "meta-os": "00-meta-os-overview.md",
    "delay-the-draft": "10-level-1-conversation/delay-the-draft.md",
    // etc
  };
  return { explanation: readKnowledgeFile(conceptMap[params.concept]) };
}

if (toolName === "requestEscalation") {
  // Log request for V follow-up
  logEscalation({ name: params.name, contact: params.contact, reason: params.reason });
  return { success: true, message: "V will reach out within 24 hours." };
}
```
