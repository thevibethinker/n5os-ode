# D1.3: Push Corrected Repo to GitHub + Clean Inbox

## Context

After D1.2, the local export has:
- Full 36+ commit history
- New Skills/ content from GitHub
- Merged README.md and install.sh

Now we need to push this corrected version to GitHub (force push to restore history) and clean up stale Inbox artifacts.

## Scope

### MUST DO
1. Create backup branch on GitHub before force push:
   ```bash
   git push origin main:backup-before-history-restore-2026-01-26
   ```
2. Force push local to GitHub with lease (safety):
   ```bash
   git push --force-with-lease origin main
   ```
3. Verify GitHub now has 37+ commits
4. Delete stale Inbox artifacts:
   - `/home/workspace/Inbox/20251103-*_n5os-lite*`
   - `/home/workspace/Inbox/20251105-*_n5os-lite*`
   - `/home/workspace/Inbox/20251028-*_n5os_core_spec*.md`
5. List any other n5os-related files in Inbox for audit

### MUST NOT DO
- Do NOT modify the local export content
- Do NOT touch the backup from D1.1
- Do NOT delete non-n5os files from Inbox

### Files You Own
- REMOTE: GitHub repo (force push)
- DELETE: `Inbox/20251103-*_n5os-lite*`
- DELETE: `Inbox/20251105-*_n5os-lite*`
- DELETE: `Inbox/20251028-*_n5os_core_spec*.md`

## ⚠️ Trap Door Warning

Force push is irreversible once done. The backup branch provides rollback capability.

## Strategy

```bash
cd /home/workspace/N5/export/n5os-ode

# 1. Verify we're in correct state
git log --oneline | head -5
git status --porcelain  # Should be empty

# 2. Create backup branch on GitHub
git push origin main:backup-before-history-restore-2026-01-26

# 3. Force push with lease
git push --force-with-lease origin main

# 4. Verify GitHub
git fetch origin
git log origin/main --oneline | wc -l  # Should be 37+

# 5. Clean Inbox
cd /home/workspace
rm -f Inbox/20251103-*_n5os-lite*
rm -f Inbox/20251105-*_n5os-lite*
rm -f Inbox/20251028-*_n5os_core_spec*.md

# 6. Audit remaining n5os files in Inbox
ls Inbox/*n5os* 2>/dev/null || echo "No remaining n5os files"
```
