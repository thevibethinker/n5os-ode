## Objective

Develop **R05 (Product Idea)** — the block that extracts user problems, solution sketches, MVP scope, and validation approaches for Careerspan features.

## R05 Domain: Product Thinking (Careerspan-Specific)

### What R05 Sees
- User problems observed or hypothesized
- Solution sketches and feature ideas
- Build vs buy analysis
- MVP scope thinking
- Validation approach ideas
- User workflow insights
- Product-market fit signals

### What R05 Ignores
- Business model innovation → R08
- Market dynamics → R04
- Strategic fit decisions → R03
- Personal reactions → R01

## Analysis Dimensions for R05

1. **Problem Clarity:**
   - Clear, validated problem
   - Hypothesized problem
   - Solution looking for problem

2. **User Segment:**
   - Candidates
   - Employers/Recruiters
   - Career coaches
   - Platform (internal)

3. **Effort Estimate:**
   - Quick win (days)
   - Medium build (weeks)
   - Major feature (months)

4. **Validation Status:**
   - Validated (user feedback)
   - Partially validated
   - Untested assumption

5. **Priority Signal:**
   - Urgent (blocking growth)
   - Important (enables growth)
   - Nice to have
