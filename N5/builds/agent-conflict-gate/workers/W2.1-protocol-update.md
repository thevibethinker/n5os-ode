# W2.1: Protocol Update, Rule, and Cleanup

## Objective
Wire the rule/safety layer around the newly implemented script, document the gate in the scheduling protocols, and execute the immediate cleanup actions flagged by the audit.

## Scope
- Files:
  - `N5/prefs/system/safety.md` (UPDATE rule + logging requirement)
  - `N5/prefs/operations/scheduled-task-protocol.md` (UPDATE sections for the gate and cleanup process)
  - `N5/logs/agent_conflict_gate.log` (CREATE append-only log)
  - `N5/notes/agent-conflict-cleanup.md` (CREATE to record duplicates/stale agents and proposed actions)
- Responsibilities:
  - Write the conditional safety rule that runs the script before `create_agent`/reactivation, surfaces conflicts, and requires explicit approval or `--force` overrides
  - Ensure the rule logs results (timestamp, agents, reason, approval status) to `N5/logs/agent_conflict_gate.log`
  - Document the gate (steps, approvals, cleanup responsibilities) in the scheduled-task protocol, including the weekly audit checkpoint
  - Capture cleanup actions (Skill Evolution duplicates, stale agents) and next steps in the cleanup notes file, flagged for V approval before disabling agents

## Must Do
- Safety rule must reference the script output schema and clearly describe how to handle conflicts (block, approve, log)
- Protocol text must mention the gate, where to find the script, and the cleanup expectations for flagged entries
- Log file entries must be append-only with ISO timestamps, conflict summaries, and override notices
- Cleanup notes must list agent IDs, conflict type, recommended action (disable/deactivate) + owner or next step

## Must Not Do
- Do not delete or disable agents without explicit confirmation recorded in cleanup notes
- Do not modify unrelated sections of the safety or protocol documents
- Do not use placeholder text in logs or notes (each entry should list actual IDs and reasons)

## Expected Output
1. `safety.md` updated with the rule, logging requirement, and override instructions
2. `scheduled-task-protocol.md` updated with the gate workflow, approval checklist, and weekly audit step
3. `N5/logs/agent_conflict_gate.log` created with sample entries for current duplicates/stale agents
4. `N5/notes/agent-conflict-cleanup.md` summarizing the duplicates/stale items and next steps (awaiting V approval)
