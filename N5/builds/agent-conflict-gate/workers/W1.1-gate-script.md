# W1.1: Gate Script

## Objective
Implement `N5/scripts/agent_conflict_gate.py` with the heuristics, CLI surface, and output contract described in the plan.

## Scope
- Files:
  - `N5/scripts/agent_conflict_gate.py` (CREATE)
- Responsibilities:
  - Inventory existing agents via `list_agents`
  - Implement the six conflict heuristics from the plan (title similarity, API combos, shared commands, delivery duplicates, stale `next_run`, time-slot density)
  - Offer CLI flags like `--dry-run`, `--force`, `--summary`, and `--check-log`
  - Emit machine-readable output (e.g., JSON table) plus optional human summary so downstream rule can parse it
  - Exit codes: `0` = clean, `1` = conflicts detected, `2` = fatal error

## Must Do
- Keep dependencies limited to Python stdlib + existing Zo helper modules (no network calls)
- Provide structured output (agent IDs, conflict type, evidence, recommended action) and document the schema
- Allow configuration of check weights or enabling/disabling heuristics via command-line options
- Include inline comments explaining each heuristic logic for future maintainers

## Must Not Do
- Do not mutate scheduled agent definitions or states
- Do not hardcode data about specific agents (IDs should remain dynamic)
- Do not rely on external MR files that may not exist (only use `list_agents` data)

## Expected Output
1. Working `agent_conflict_gate.py` with documented CLI usage (README section or docstring)
2. Structured output documented so the rule can read conflict_type, agent_id list, and confidence
3. Minimal unit-style tests (script invocations with simulated payloads) described in comments or README
