## 🎯 Root Cause (Confirmed)

**The recipe didn't tell the AI to execute the command.**

### What Was Wrong

file `Recipes/Close Conversation.md` contained:
- Bash command in code block
- Description of what the command does
- Parameters explained
- But **NO explicit instruction for AI to execute it**

### What Happened

1. User invokes `/close-conversation`
2. AI reads recipe markdown
3. AI sees documentation about a script
4. AI generates manual summary (interpreting as reference doc)
5. **Script never runs** ❌

### What SHOULD Happen

1. User invokes `/close-conversation`
2. AI reads recipe markdown
3. AI sees **explicit instruction to execute command**
4. AI runs `python3 N5/scripts/n5_conversation_end.py --auto --convo-id <ID>`
5. **Full 11-phase workflow executes** ✅

---

## ✅ The Fix

Added explicit AI execution instruction to recipe:

```markdown
## Execution

**When this recipe is invoked, you (the AI) MUST execute the following command:**

\`\`\`bash
python3 /home/workspace/N5/scripts/n5_conversation_end.py \\
  --auto \\
  --convo-id <current_conversation_id>
\`\`\`

**Parameter substitution:**
- Replace `<current_conversation_id>` with the actual conversation ID from SESSION_STATE.md or context

**Your task:** Run this command using `run_bash_command`. Do not just describe what would happen - actually execute it.
```

**Why this works:**
- Uses imperative language ("MUST execute")
- Addresses AI directly ("you (the AI)")
- Clarifies tool to use (`run_bash_command`)
- Prevents documentation interpretation ("Do not just describe")
- Matches recipe-execution-guide.md pattern
