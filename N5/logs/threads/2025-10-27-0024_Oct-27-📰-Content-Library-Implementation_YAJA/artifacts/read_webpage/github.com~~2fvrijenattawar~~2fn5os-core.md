N5 OS Core Bootstrap
====================

**Minimal, high-confidence base layer for AI knowledge work**

---

What This Is
------------

This is the **core foundation** of N5 OS—the minimum viable system to bootstrap AI-enabled knowledge work on your Zo instance.

**Philosophy**: Start small, prove it works, expand gradually.

---

What's Included
---------------

### 1. Architectural Knowledge

* **Zero-Touch Manifesto** — Core philosophy and principles
* **Architectural Principles** — System design patterns
* Foundation for building AI-native workflows

### 2. Preferences System

* **25 essential prefs** — Core system configuration
* System prefs (command triggering, safety, state)
* Operations prefs (protocols, workflows)
* NOT the full 105 files—just what you need to start

### 3. List Management System

* **POLICY.md** — How lists work
* JSONL templates for structured tracking
* Foundation for ideas, contacts, opportunities
* Zero-Touch pattern: capture → route → review

### 4. Essential Scripts (4 only)

* `n5_index_rebuild.py` — Rebuild system index
* `n5_git_check.py` — Git safety verification
* `n5_safety.py` — Safety checks
* `session_state_manager.py` — Session state tracking

### 5. Supporting Infrastructure

* **16 schemas** — JSON validation
* **4 command docs** — Documentation for scripts
* **.gitignore** — Security

**Total**: ~60 files, <500 KB

---

What's NOT Included (Yet)
-------------------------

These come in Phase 2+ after core is proven:

* ❌ Meeting workflows (Phase 2)
* ❌ 250+ additional scripts (Phase 2-3)
* ❌ Communication templates (Phase 3)
* ❌ Full knowledge base (Phase 2)
* ❌ Command registry (Phase 2—only after scripts proven)

---

Installation
------------

### Quick Install

```
bash bootstrap.sh
```

### Manual Install

```
# 1. Copy core components
cp -r core/* /home/workspace/

# 2. Copy scripts
cp scripts/* /home/workspace/N5/scripts/

# 3. Initialize lists
mkdir -p /home/workspace/Lists
cp core/lists/*.template /home/workspace/Lists/

# 4. Test
python3 /home/workspace/N5/scripts/n5_index_rebuild.py --dry-run
```

---

After Installation
------------------

### 1. Read the Philosophy

```
cat docs/zero_touch_manifesto.md
```

### 2. Understand the Architecture

```
cat core/knowledge/architectural_principles.md
```

### 3. Review Prefs

```
cat core/prefs/prefs.md
```

### 4. Test Core Scripts

```
# Dry run safety check
python3 N5/scripts/n5_git_check.py --dry-run

# Rebuild index
python3 N5/scripts/n5_index_rebuild.py
```

### 5. Initialize Lists

```
# Copy templates
cp Lists/*.template Lists/ideas.jsonl
cp Lists/*.template Lists/must-contact.jsonl

# Start tracking
echo '{"tag": "ideas", "title": "Test N5 OS core", "description": "Verify system works", "date_added": "'$(date -Iseconds)'"}' >> Lists/ideas.jsonl
```

---

Design Philosophy
-----------------

This package follows **Zero-Touch principles**:

1. **Minimal Touch** — Start with smallest viable system
2. **Flow Over Pools** — Information flows, doesn't pool
3. **Maintenance First** — System maintains itself
4. **AIR Pattern** — Assess → Intervene → Review
5. **SSOT** — Single source of truth for each data type

See `docs/zero_touch_manifesto.md` for full philosophy.

---

Next Steps (Phase 2)
--------------------

After proving core works:

1. **Add meeting workflows** — Process transcripts
2. **Expand scripts** — Add specialized utilities
3. **Build command registry** — Natural language triggers
4. **Add knowledge modules** — Glossary, bio, references

But first: **prove the core works**.
