# Voice Pattern Library (Recomposable)

Purpose: A modular index of small, reusable techniques (not fixed templates) that can be combined to fit any channel.

How this differs from templates:
- Patterns are atomic moves (hook types, specificity devices, cadence shifts, proof styles, CTA forms)
- Each pattern documents: effect, when to use, how to execute, anti-patterns, and one authentic example
- Patterns are tagged so the system can compose 2–5 moves per piece of writing

Use with:
- Core voice → `file 'N5/prefs/communication/voice.md'`
- Social voice → `file 'N5/prefs/communication/social-media-voice.md'`
- Style guides → `file 'N5/prefs/communication/style-guides/README.md'`
- Transformation method → `file 'N5/prefs/communication/voice-transformation-system.md'`

Structure of a pattern:
- Name
- Effect (what the reader feels/does)
- When to use
- How (steps)
- Example (authentic)
- Anti-patterns / pitfalls
- Tags

See: `file 'N5/prefs/communication/patterns/patterns-index.md'`
