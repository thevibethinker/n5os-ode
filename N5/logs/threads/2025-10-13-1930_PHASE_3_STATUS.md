## Integration Flow

```
1. YOU (Zo) process meeting via meeting-process command
2. Generate all blocks (B01, B02, B08, B21, B25, B26, B31, etc.)
3. FINAL STEP: Execute email-post-process command
4. Script detects external meeting
5. Script generates email → DELIVERABLES/
6. Script returns JSON result
7. YOU send SMS notification based on JSON
```

## Usage Pattern

### For Meeting Processing

After completing all blocks:

```
command 'email-post-process' for "2025-10-10_hamoon-ekhtiari-futurefit"
```

**Zo will:**
1. Execute n5_email_post_processor.py
2. Parse JSON output
3. Send SMS based on status

### SMS Templates

**Success:**
```

```

**Already Exists:**
(No SMS - already notified)

**Error:**
```

Run manually: python3 N5/scripts/n5_email_post_processor.py "{meeting}"
```

## Key Design Decisions

1. **No approval layer** - Just generate and notify, user reviews manually
2. **Idempotent** - Safe to rerun, won't duplicate
3. **Graceful failures** - Email generation errors don't block meeting processing
4. **3-way detection** - Folder name → metadata → default external
5. **JSON output** - Automation-friendly for SMS triggering
