# B09: Reference Data & Resources

## Objective

Capture links, documents, tools, data sources, and reference materials mentioned or shared during the meeting.

## What to Capture

**URLs & Links:**
- Websites mentioned
- Documentation referenced
- Tools or platforms discussed
- Shared resources

**Documents:**
- Files mentioned
- Reports or studies referenced
- Templates or examples cited
- Presentations or decks

**Data Sources:**
- Databases mentioned
- APIs or integrations
- Research sources
- Industry reports

**Tools & Platforms:**
- Software mentioned
- Services discussed
- Technologies referenced

**People & Experts:**
- Subject matter experts to contact
- Authors or researchers cited
- Consultants or vendors mentioned

**Other Resources:**
- Books or articles
- Case studies
- Examples or precedents
- Industry standards

## Output Format

Organize by category:

### Links & URLs
- **[Title/Description]**: [URL]
  - Context: [Why this was mentioned, how it's relevant]

### Documents & Files
- **[Document Name]**
  - Type: [Report, deck, template, etc.]
  - Location: [Where to find it]
  - Relevance: [Why it matters]

### Tools & Platforms
- **[Tool Name]**
  - Purpose: [What it does]
  - Context: [Why discussed]
  - Link: [URL if available]

### People & Experts
- **[Name - Title/Company]**
  - Expertise: [What they know]
  - Why Relevant: [Connection to discussion]
  - Contact: [How to reach]

### Research & Data
- **[Source Name]**
  - Type: [Study, report, database, etc.]
  - Key Findings: [If discussed]
  - Access: [How to get it]

## Generation Guidelines

1. Capture ALL references mentioned:
   - Even if not directly shared
   - Include what was said about them
   - Note where to find them

2. Add context:
   - Why was this mentioned?
   - How is it relevant?
   - What should be done with it?

3. Make actionable:
   - Include URLs when possible
   - Note how to access resources
   - Flag if something needs to be obtained

4. Organize logically:
   - Group similar items
   - Prioritize by relevance
   - Make easy to reference later

5. Flag follow-ups:
   - Resources to obtain
   - Links to retrieve
   - People to contact

## Edge Cases

**Vague References:**
If someone says "that article I mentioned" without details:
"**Article on [topic]** (details to be provided by [person])"

**Internal Resources:**
For company-internal resources, note location and access requirements

**Broken/Missing Links:**
If URL mentioned but not shared, note: "(URL to be provided)"

## Quality Checklist

Before finalizing, check:
- [ ] All URLs and links included
- [ ] Context provided for each resource
- [ ] Access information noted
- [ ] Organized by logical categories
- [ ] People and experts captured with contact info
- [ ] Follow-up actions flagged
- [ ] Easy to reference and use later
