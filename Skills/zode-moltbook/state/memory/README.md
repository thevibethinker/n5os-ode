# Zøde Semantic Memory

This directory is Zøde's living knowledge base — a structured understanding of Moltbook that accumulates across sessions and informs every engagement decision.

## Structure

| File | Purpose | Update Cadence |
|------|---------|----------------|
| `community.md` | Cultural norms, unwritten rules, what works/doesn't | After each deep scan |
| `agents.jsonl` | Agent profiles with Zøde-specific assessments | After influence scans + interactions |
| `narratives.jsonl` | What narrative patterns resonate, with evidence | After each distillation |
| `submolts.md` | Submolt landscape — what each is about, activity level | Weekly |
| `opportunities.jsonl` | Active engagement opportunities with context | Each heartbeat |
| `lessons.jsonl` | What Zøde has learned from its own posts/comments | After each distillation |
| `landscape.md` | High-level landscape analysis (refreshed periodically) | After major scans |

## Design Principles

1. **Understanding over metrics** — This isn't analytics. It's knowledge.
2. **Queryable** — Any session should be able to load relevant context fast.
3. **Decaying** — Old observations get lower confidence over time.
4. **Narrative-level** — Per V's directive: tune narratives, not word choices.
5. **Human-readable** — V should be able to open any file and understand it.
