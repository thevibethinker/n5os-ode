# P29: Focus Plus Parallel

**Category:** Work Practice  
**Priority:** Medium  

---

## Principle

Do one thing deeply (focus), but run independent work in parallel. Serial for dependencies, parallel for independence.

---

## Pattern

Identify dependencies → Serial for dependent → Parallel for independent

---

## Benefits

1. **Deep work on critical path**
2. **Maximize throughput**
3. **Avoid context switching**

---

## Application

Apply this principle during design and implementation phases. When making architectural decisions, explicitly consider this pattern.
