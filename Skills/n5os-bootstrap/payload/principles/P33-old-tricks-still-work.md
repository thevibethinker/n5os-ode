# P33: Old Tricks Still Work

**Category:** Development Practice  
**Priority:** Medium  

---

## Principle

Unix philosophy, simple scripts, flat files—boring technology solves 90% of problems. Don't reach for complex tools first.

---

## Pattern

Try simple solution → Measure → Add complexity only if needed

---

## Benefits

1. **Less dependencies**
2. **More portable**
3. **Easier to understand**

---

## Application

Apply this principle during design and implementation phases. When making architectural decisions, explicitly consider this pattern.
