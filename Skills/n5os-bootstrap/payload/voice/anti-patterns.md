# Voice Anti-Patterns

**Purpose:** Common language patterns to avoid when representing the user externally  
**Source:** Extracted from N5 voice learning database  
**Updated:** Automatically curated for N5OS bootstrap

---

## Corporate/Generic Patterns

**Avoid these overused business phrases:**

- "Hi [Name]" — Generic greeting template
- "wanted to reach out because" — Explaining obvious action
- "Would you be open to" — Tentative, high-friction framing  
- "Best, [Name]" — Formal email closing
- "really enjoying it" — Generic enthusiasm
- "interesting synergies" — Corporate buzzword
- "quick call sometime" — Vague scheduling

**Better:** Direct, specific openings that lead with value

---

## Positioning Anti-Patterns

**Weak positioning language:**

- "There's real partnership potential here" — Stating obvious
- "helping [Company] build out their [thing]" — Positions as subordinate
- "we're in conversations with several other top providers" — Vague, unconvincing
- "Are you free for a 15-minute call" — Focuses on recipient's schedule

**Better:** Lead with concrete value and specific outcomes

---

## Content Structure Anti-Patterns

**Poor information organization:**

- Long messages with polite padding that dilute impact
- Separate paragraphs for related credentials/role information
- Burying value in the middle of generic outreach
- Bullet-point lists that interrupt reading flow

**Better:** Front-load value, consolidate related information

---

## Teaching/Educational Anti-Patterns

**Generic educational framing:**

- "I'll show you" / "I'll teach you" — Solo teaching frame without context
- Vague terms like "fundamentals," "crash course," generic descriptions
- Plain technical descriptions that don't immediately click
- "Link in comments" — Indirect references requiring user hunt
- Formal labels like "workshop," "crash course"
- Failing to recognize partners, community, or contributors

**Better:** Collaborative learning, specific outcomes, direct access

---

## Technical Communication Anti-Patterns

**Weak technical descriptions:**

- Generic technical terms without context
- Explaining what instead of why it matters
- Unattributed claims without supporting evidence
- Vague references that require interpretation

**Better:** Context-rich explanations that connect to user value

---

## Email/Outreach Anti-Patterns

**Common outreach mistakes:**

- Starting with generic context ("Quick context:")
- Multiple asks in one message
- Unclear value proposition
- Timeline pressure without justification
- Generic sign-offs that add no value

**Better:** Single clear ask with obvious mutual benefit

---

## Meta Anti-Patterns

**Self-referential mistakes:**

- "I had to reach out" — Commenting on your own action
- Explaining why you're messaging instead of leading with value
- Apologizing for messaging when providing genuine value
- Over-qualifying statements that undermine confidence

**Better:** Assume your message has value and act accordingly

---

## Usage Guidelines

1. **Before writing:** Scan this list for patterns in your draft
2. **During editing:** Replace weak patterns with direct alternatives  
3. **Global application:** These patterns apply across all content types
4. **Context matters:** Some patterns acceptable in specific contexts (internal notes, casual communication)
