# Founder Input → Weights Mapping (v0.1)

Goal: derive weights organically from founder input and role context.

Process:
1) Founder intake: top risks, must-win bets, product phase, team complement.
2) Map to criteria: translate risks → criteria emphasis (e.g., "need speed" → higher w_speed).
3) Simulate: run hypothetical candidates through rubric to test separation.
4) Calibrate: adjust weights until separation reflects founder priorities without collapsing variance.
5) Freeze v1: document weights + rationale; revisit after 3–5 hires or quarterly.

Output artifact: rubric.<role>.<stage>.<version>.json with weights + rationale notes.
