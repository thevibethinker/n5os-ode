## Story: Launching AI Agents at Zo

**Context:** When Holden joined Zo as a PM in mid-2024, the product had powerful AI capabilities but users struggled with the "what do I even ask it to do?" problem. Users would try Zo, get excited, but then abandon it because they couldn't figure out how to automate their repetitive tasks.

The hypothesis: If we let users schedule AI tasks to run automatically (agents), it would move Zo from "cool demo" to "daily habit."

**Challenge:** This was a 0→1 product launch with no clear template. Questions everywhere:
- What should the UX be? (Cron syntax? Natural language? Calendar interface?)
- How much technical detail should we expose? (Zo users range from non-technical to developers)
- What safety rails do we need? (Agents run unsupervised - what could go wrong?)
- How do we price it? (Should agents be a premium feature or core to the product?)

**Approach:**

1. **User Research (Week 1-2)**
   - Conducted 25 interviews with power users who were already trying to "hack" recurring tasks
   - Found pattern: People wanted to schedule recurring research/summaries, not complex automations
   - Key insight: "I don't want to write code, I just want to say 'check X every morning and email me a summary'"

2. **Rapid Prototyping (Week 3-4)**
   - Holden built a rough prototype in Notion + Zapier to validate the UX concept
   - Shared with 10 beta users and watched them interact with it
   - Learned: Natural language scheduling ("every weekday at 9am") was intuitive, but users wanted to see a calendar view to build confidence

3. **Scoping with Engineering (Week 5)**
   - Worked with engineers to define v1 scope: text-based scheduling, no calendar UI yet
   - Trade-off decision: Ship without calendar view to validate core hypothesis faster
   - Added safety features: email confirmation before first run, ability to pause agents easily

4. **Iterative Launch (Week 6-12)**
   - Soft launch to 50 beta users, gathered feedback daily
   - Discovered unexpected use case: People were using agents for personal tasks (meal planning, birthday reminders) not just work
   - Added features iteratively: preview mode, test runs, simple scheduling templates
   - Expanded to all users in week 10

**Outcome:**
- **12K+ agents deployed in first 90 days** (way above internal projections of 5K)
- **35% user adoption rate** (35% of active users created at least one agent)
- **NPS of 72** for the agent feature specifically
- **40% increase in 30-day retention** - agents turned Zo from "occasional tool" to "daily habit"

**What Holden Learned:**
- **Prototype to clarify requirements:** The Notion prototype took 2 days and saved 3 weeks of building the wrong thing
- **Launch narrow, expand based on real usage:** If they'd waited to build calendar UI, they would have delayed learning that personal use cases mattered
- **Safety features earn trust:** Email confirmations felt like "friction" internally but users said it made them confident to try agents
- **User research ≠ user requests:** Users asked for "cron syntax" and "advanced scheduling" but actually needed natural language and templates

**Skills Demonstrated:**
- 0→1 product development (ambiguity → launched product)
- User research driving decisions (25 interviews → clear insights)
- Cross-functional collaboration (worked with eng, design, support to ship)
- Iterative development (soft launch → feedback → expand)
- User advocacy (prioritized trust-building over feature richness)
