# Real Community Use Cases

Actual things real Zo users have built and shared publicly. These are not hypothetical — each one was posted by a real person.

## 1. DocSend Replacement (Ben Guo, Zo co-founder)
Built a custom file-sharing tool in 10 minutes. Serves files, verifies email with confirmation link, sends download link, styled how he wants. His quote: "I'm never going back to DocSend. SaaS is dead."
**Zo features used:** Zo Sites, email sending, file system
**Complexity:** Medium — multiple steps but straightforward
**Role:** Startup founder / investor relations

## 2. Read-It-Later App (Stack Snacks, YouTube creator)
Saves articles with full plain-text access. Works like Pocket or Instapaper but lives on his Zo with full file access. Built live on camera in under 10 minutes.
**Zo features used:** Zo Sites, file storage, web scraping
**Complexity:** Low-medium
**Role:** Content creator / developer

## 3. YouTube-to-Blog Content Repurposer (Stack Snacks)
Custom Zo Skill that takes YouTube videos and converts them into blog posts and LinkedIn content. Built it as a reusable skill so it runs on demand.
**Zo features used:** Skills, integrations, content generation
**Complexity:** Medium
**Role:** Content creator

## 4. Scheduled Text Reminders (Stack Snacks)
Doctor's appointments, to-dos, personal reminders — all sent via text at scheduled times. No app needed.
**Zo features used:** Scheduled agents, SMS
**Complexity:** Low — this is a great first project
**Role:** Anyone

## 5. Custom Slack AI Assistant (Reddit user, r/ZoComputerClub)
Built a custom AI assistant that lives in Slack and works within their specific work and life flow. Iterating on it over time as needs change.
**Zo features used:** Webhooks, integrations, file system, persistent memory
**Complexity:** High — ongoing iteration
**Role:** Knowledge worker

## 6. AI Coding Side Projects (Reddit user, r/vibecoding)
Uses Zo as go-to tool for AI coding side projects and personal automations. Also uses it for research and generating feature plans.
**Zo features used:** Terminal, file system, AI models, Sites
**Complexity:** Medium-high
**Role:** Developer

## 7. Markdown Blog Auto-Publisher (Stack Snacks)
Simple blog that auto-pulls posts from a markdown folder. Write a post, drop it in the folder, it's live.
**Zo features used:** Zo Sites, file system
**Complexity:** Low
**Role:** Writer / blogger

## 8. HPC Cluster Connection for Research (Dr. Yixian Zheng, Carnegie Institution)
Connected Zo to institutional high-performance computing clusters to train on large datasets using open-source models like Llama. Student use case for scientific research.

**Complexity:** High
**Role:** Scientist / researcher

## 9. Hello Kitty RPG Game (Tiff, X user)
Onboarded their deskmate from Claude to Zo. By end of day, deskmate was building a Hello Kitty RPG game. Showed how cross-project work is natural on Zo versus Claude's context silos.
**Zo features used:** Sites, AI coding, file system
**Complexity:** Medium
**Role:** Creative / hobbyist

## 10. Personal DevRel Setup (Andrea, GitHub DevRel)
Built custom tools for managing their two primary DevRel projects. Considering open-sourcing the setup. Uses Zo as an always-on development environment.
**Zo features used:** Skills, file system, integrations
**Complexity:** Medium-high
**Role:** Developer Relations engineer

## 11. Knowledge Management + OpenCode Server (Dev Agrawal, PowerSync)
Uses Zo as a 24/7 personal sandbox with all data, research, and projects. Combines with OpenCode server for custom workflows and Codex auth.

## 12. Multi-Project Personal Server (ben guo comparison post)
Runs multiple projects, automations, image generation, all on one Zo. Contrasted with Replit charging $20/mo per deployed project on a single 0.5vcpu VM.

**Role:** Developer / founder
