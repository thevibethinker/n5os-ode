# How ATS Systems Work: A Primer

**Comprehensive guide to ATS parsing, scoring, and optimization**

---

## 1. Resume Bullet Points

### What ATS Systems Look For

- **Keyword Relevance & Synonyms:** Beyond core terms, NLP recognizes synonyms (e.g., "budget oversight" ≈ "financial planning")
- **Standard Formatting:** Simple bullet symbols (• or -), avoid tables/columns
- **Quantifiable Impact:** Metrics like "Increased sales by 30%" with action verbs ("led," "optimized," "designed")

### Rewarded Features
- Action-oriented language (strong verbs)
- Quantifiable achievements (specific metrics)
- Clean, simple formatting

### Penalized Features
- Vague statements without quantified results
- Keyword stuffing without context
- Complex or non-standard formatting

---

## 2. Executive Summaries

### ATS Evaluation Criteria

- **Targeted Keywords:** Industry-specific terms from job description
- **Conciseness:** 2-3 sentences, punchy and focused
- **Relevance:** Alignment with job requirements

### Do's
- Customize keywords for each application
- Highlight relevant achievements with quantifiable outcomes
- Use strong action verbs

### Don'ts
- Generic phrases ("results-driven professional")
- Keyword overstuffing
- Copy-pasting job description

---

## 3. Education Section

### What Increases Score
- Clear degree and field of study (exact terminology)
- Institution name and location
- Consistent date format (MM/YYYY or Month YYYY)
- Academic honors if applicable
- Relevant coursework for recent graduates

### What Decreases Score
- Non-standard abbreviations
- Inconsistent date formatting
- Information in headers/footers (often ignored)
- Outdated details (high school when you have college degree)

---

## 4. Skills Section

### Best Practices
- **Match job description exactly** — use their terminology
- **Prioritize** — most important skills first
- **Categorize** — "Technical Skills" vs "Soft Skills"
- **Simple format** — bullet list or comma-separated

### Do's
- Be precise and relevant
- Group similar skills
- Tailor for each application
- Include proficiency levels if valuable

### Don'ts
- Keyword stuffing
- Irrelevant skills
- Vague/generic terms
- Non-standard formatting
- Repetition without purpose

---

## 5. Keyword Matching Mechanics

### How It Works

| Type | Description |
|------|-------------|
| **Exact Matching** | Rigid systems require exact phrases |
| **Contextual Matching** | Advanced NLP understands related terms |
| **Diminishing Returns** | 2-3 occurrences optimal; more looks forced |

### Synonym Handling
- Older systems: exact matches only
- Modern systems: semantic analysis recognizes synonyms
- **Best practice:** Include both exact term AND synonym naturally

---

## 6. ATS System Differences

### Advanced Systems (Greenhouse, Workday)
- NLP techniques
- Understand context and synonyms
- Weight quantified results and action verbs

### Basic Systems
- Simple keyword counting
- Exact language matching required
- Format sensitivity higher

**Implication:** Standard formatting ensures compatibility across all systems.

---

## 7. Grammatical Principles

### What Matters
- **Spelling consistency** — errors cause mismatches
- **Punctuation** — used as delimiters for parsing
- **Sentence structure** — concise (1-2 lines) with clear subject-verb-object
- **Verb tense** — present for current role, past for previous

---

## 8. General Best Practices

1. **File Format:** .docx or text-based PDF (not image-based)
2. **Standard Headings:** "Work Experience," "Education," "Skills"
3. **Tailoring:** Customize for each application with exact keywords
4. **Simple Layout:** No graphics, columns, or tables

---

## Summary: What Wins

| Element | Winning Approach |
|---------|-----------------|
| Bullet points | Action verbs + metrics + clean format |
| Summary | Targeted keywords, concise, unique value |
| Education | Exact terminology, consistent dates |
| Skills | Matched to job, prioritized, categorized |
| Overall | Tailored, standard format, error-free |

**The goal:** Pass ATS filters AND make a strong impression on human reviewers who follow.
