## H-PROD-001: AI Coaching Creates Richer Profiles Than Resumes

**Stakeholders:** `[job_seekers]` `[employers]`

**Statement:**
AI-mediated coaching can extract candidate context (motivations, values, fit factors) that traditional resumes miss, leading to better matching outcomes.

**Why it matters:**
- If true: Core product differentiation, defensible moat
- If false: We're just another ATS/job board
- Product focus: How much to invest in coaching vs. other features

**How to test:**
- Measure: Profile completeness (coaching vs. no coaching)
- Track: Employer feedback on candidate quality
- Compare: Interview-to-placement rate for rich profiles vs. thin profiles
- Timeline: 100+ candidates through system

**Status:** `TESTING` (Core feature exists, gathering quality data)

**Evidence:**

*Supporting:*

- [AWAITING] Employer feedback on candidate quality
- [AWAITING] Placement rate data

## H-PROD-002: Virtual Recruiter Model Delivers Hiring Manager Value

**Stakeholders:** `[employers]`

**Statement:**
The "virtual recruiter" positioning (vs. job board or ATS) resonates with employers because it promises managed service feel with tech efficiency.

**Why it matters:**
- If true: Market as "recruiter alternative" not "platform"
- If false: Reposition as self-service platform
- Pricing: Impacts whether we can charge placement fees vs. subscription

**How to test:**
- Track: Employer response to "virtual recruiter" positioning
- Measure: Willingness to pay placement fees (recruiter model) vs. flat fees
- Compare: Engagement with "white glove" elements vs. self-service
- Timeline: 10+ employer conversations

**Status:** `TESTING` (Positioning confirmed in use, value validation pending)

## H-PROD-003: Warm Intros Beat Direct Applications

**Stakeholders:** `[job_seekers]` `[employers]`

**Statement:**
Careerspan "vouching" for candidates (warm intro model) increases interview rates compared to cold applications because it adds trust layer.

**Why it matters:**
- If true: Warm intro is core product feature, not nice-to-have
- If false: Can simplify to direct application flow
- Positioning: Do we emphasize curation/quality or volume?

**How to test:**

- Compare: Benchmark against industry interview rates
- Timeline: 50+ placements to analyze

**Status:** `PROPOSED` (Feature exists, no data yet)

**Evidence:**

- None yet (awaiting placement data)

## H-PROD-004: Code-Based Access Works for Community Partnerships

**Stakeholders:** `[community_partners]` `[job_seekers]`

**Statement:**
Custom access codes (e.g., REMOTELYGOOD) are sufficient technical infrastructure for community partnerships - no need for complex white-label or API initially.

**Why it matters:**
- If true: Fast partnership deployment, lower eng cost
- If false: Need to build white-label platform or API
- Product roadmap: When to invest in community platform features

**How to test:**
- Track: Activation rate (can users find and use codes easily?)
- Gather: Community leader feedback (is this too manual/clunky?)
- Measure: Friction points in rollout process
- Timeline: 3-5 community partnerships

**Status:** `TESTING` (REMOTELYGOOD deployed, awaiting data)

**Evidence:**

*Supporting:*
- **[Sep 29, 2025]** Remotely Good: Code deployed (100-200 licenses), Theresa accepted this approach without pushing for more sophisticated integration. (Meeting: 46a9c7) [PENDING REVIEW]

*Contradicting:*
- None yet (may emerge if activation is low or community leaders complain)

## H-PROD-005: Job Seekers Will Engage with Multi-Step Coaching

**Stakeholders:** `[job_seekers]`

**Statement:**
Job seekers will complete AI coaching process (multiple interactions, profile building) rather than abandoning for simpler one-click apply experiences.

**Why it matters:**
- If true: Can build depth into coaching without losing users
- If false: Need to streamline to bare minimum

- Compare: Completion rate vs. industry benchmarks
- Gather: User feedback on experience
- Timeline: 200+ users through funnel
