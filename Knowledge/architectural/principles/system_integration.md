# Principle 27: System Integration Patterns

**Category**: Design  
**Priority**: High  
**Related**: P20 (Modular Design), P30 (Platform Orchestration), ZD6 (Gestalt Evaluation)

---

## Statement

Optimize for system-wide effectiveness, not individual component excellence. Evaluate changes by their impact on total cognitive load and end-to-end flow, not isolated metrics.

---

## Rationale

From Zero-Doc Principle 6: The question isn't "Is Notion better than Obsidian?" The question is "Does my information flow create the context and maintain the state I need to do my best work?"

**Component optimization**: Make one tool perfect  
**System optimization**: Make all tools work together perfectly

You can have the best tools in every category and still have a terrible system if they don't integrate well.

---

## Core Questions for System Evaluation

Before adding/changing any component:

1. **Flow impact**: Does this improve flow or create a new pool?
2. **Cognitive load**: Does this reduce total load or shift it elsewhere?
3. **Integration cost**: What's the cost of connecting this to existing components?
4. **Redundancy check**: Does this duplicate capability that exists elsewhere?
5. **Exit strategy**: Can we remove this later without breaking the system?

**Rule**: Changes must improve system-wide metrics, not just local ones.

---

## Key Insights

1. **Interfaces matter more than features**: The connection between tools is more important than what each tool does.

2. **Simplicity is system property**: A simple system might use complex tools, or complex system might use simple tools. Simplicity is about total cognitive load.

3. **Optimization paradox**: Optimizing one component often makes system worse. (e.g., "perfect" note-taking in Notion that never flows to action)

4. **Integration is ongoing**: Not one-time. As components evolve, integration needs maintenance.

---

## Related Principles

- **P20 (Modular Design)**: Modules must integrate, not just exist independently
- **P30 (Platform Orchestration)**: Integration via orchestrator, not point-to-point
- **P2 (SSOT)**: Integration enables single source, not duplication
